# Microsoft Teams

> "Оставь надежду, всяк сюда входящий".

Обновлено: 2026-01-21

Статус: поддерживаются текст и вложения в личных сообщениях (DM); для отправки файлов в каналах/группах требуется `sharePointSiteId` и разрешения Graph (см. [Отправка файлов в групповых чатах](#отправка-файлов-в-групповых-чатах)). Опросы отправляются через Adaptive Cards. Действия с сообщениями предоставляют явный параметр `upload-file` для отправки файлов в первую очередь.

## Встроенный плагин

Microsoft Teams поставляется как встроенный плагин в текущих релизах OpenClaw, поэтому отдельная установка в стандартной сборке не требуется.

Если вы используете более старую сборку или пользовательскую установку, в которой нет встроенного Teams, установите его вручную:

```bash
openclaw plugins install @openclaw/msteams
```

Локальная проверка (при работе из репозитория git):

```bash
openclaw plugins install ./path/to/local/msteams-plugin
```

Подробности: [Плагины](/tools/plugin)

## Быстрая настройка (для начинающих)

1. Убедитесь, что плагин Microsoft Teams доступен.
   - В текущих упакованных релизах OpenClaw он уже включён.
   - В более старых или пользовательских установках его можно добавить вручную с помощью команд выше.
2. Создайте **Azure Bot** (ID приложения + секрет клиента + ID арендатора).
3. Настройте OpenClaw с этими учётными данными.
4. Сделайте `/api/messages` (по умолчанию порт 3978) доступным через публичный URL или туннель.
5. Установите пакет приложения Teams и запустите шлюз.

Минимальная конфигурация:

```json5
{
  channels: {
    msteams: {
      enabled: true,
      appId: "<APP_ID>",
      appPassword: "<APP_PASSWORD>",
      tenantId: "<TENANT_ID>",
      webhook: { port: 3978, path: "/api/messages" },
    },
  },
}
```

Примечание: групповые чаты по умолчанию заблокированы (`channels.msteams.groupPolicy: "allowlist"`). Чтобы разрешить ответы в группах, установите `channels.msteams.groupAllowFrom` (или используйте `groupPolicy: "open"`, чтобы разрешить любым участникам, с ограничением по упоминаниям).

## Цели

- Общение с OpenClaw через личные сообщения (DM), групповые чаты или каналы в Teams.
- Обеспечение детерминированной маршрутизации: ответы всегда возвращаются в тот канал, откуда они поступили.
- Поведение канала по умолчанию — безопасное (требуются упоминания, если не настроено иначе).

## Запись конфигурации

По умолчанию Microsoft Teams разрешено записывать обновления конфигурации, инициированные через `/config set|unset` (требуется `commands.config: true`).

Отключите с помощью:

```json5
{
  channels: { msteams: { configWrites: false } },
}
```

## Контроль доступа (DM и группы)

**Доступ к DM**

- По умолчанию: `channels.msteams.dmPolicy = "pairing"`. Неизвестные отправители игнорируются до одобрения.
- `channels.msteams.allowFrom` должен использовать стабильные идентификаторы объектов AAD.
- UPN/отображаемые имена могут изменяться; прямое сопоставление по умолчанию отключено и включается только с `channels.msteams.dangerouslyAllowNameMatching: true`.
- Мастер может разрешить имена в идентификаторы через Microsoft Graph, если учётные данные это позволяют.

**Доступ к группам**

- По умолчанию: `channels.msteams.groupPolicy = "allowlist"` (заблокировано, если не добавлено `groupAllowFrom`). Используйте `channels.defaults.groupPolicy`, чтобы переопределить значение по умолчанию, если оно не задано.
- `channels.msteams.groupAllowFrom` контролирует, какие отправители могут инициировать действия в групповых чатах/каналах (возвращается к `channels.msteams.allowFrom`).
- Установите `groupPolicy: "open"`, чтобы разрешить любым участникам (по-прежнему с ограничением по упоминаниям по умолчанию).
- Чтобы запретить **все каналы**, установите `channels.msteams.groupPolicy: "disabled"`.

Пример:

```json5
{
  channels: {
    msteams: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["user@org.com"],
    },
  },
}
```

**Teams и список разрешённых каналов**

- Ограничьте ответы в группах/каналах, перечислив команды и каналы в `channels.msteams.teams`.
- Ключи должны использовать стабильные идентификаторы команд и идентификаторы разговоров в каналах.
- Когда `groupPolicy="allowlist"` и список разрешённых команд присутствует, принимаются только указанные команды/каналы (с ограничением по упоминаниям).
- Мастер настройки принимает записи `Team/Channel` и сохраняет их за вас.
- При запуске OpenClaw преобразует имена команд/каналов и пользователей из списка разрешённых в идентификаторы (если разрешения Graph это позволяют) и записывает сопоставление; неразрешённые имена команд/каналов сохраняются как введённые, но по умолчанию игнорируются для маршрутизации, если не включено `channels.msteams.dangerouslyAllowNameMatching: true`.

Пример:

```json5
{
  channels: {
    msteams: {
      groupPolicy: "allowlist",
      teams: {
        "My Team": {
          channels: {
            General: { requireMention: true },
          },
        },
      },
    },
  },
}
```

## Как это работает

1. Убедитесь, что плагин Microsoft Teams доступен.
   - В текущих упакованных релизах OpenClaw он уже включён.
   - В более старых или пользовательских установках его можно добавить вручную с помощью команд выше.
2. Создайте **Azure Bot** (ID приложения + секрет + ID арендатора).
3. Создайте **пакет приложения Teams**, который ссылается на бота и включает разрешения RSC, указанные ниже.
4. Загрузите/установите приложение Teams в команду (или в личное пространство для DM).
5. Настройте `msteams` в `~/.openclaw/openclaw.json` (или через переменные окружения) и запустите шлюз.
6. Шлюз прослушивает трафик вебхуков Bot Framework на `/api/messages` по умолчанию.

## Настройка Azure Bot (предварительные требования)

Перед настройкой OpenClaw необходимо создать ресурс Azure Bot.

### Шаг 1: Создание Azure Bot

1. Перейдите на страницу [Создание Azure Bot](https://portal.azure.com/#create/Microsoft.AzureBot).
2. Заполните вкладку **Basics**:

   | Поле | Значение |
   | --- | --- |
   | **Bot handle** | Имя вашего бота, например, `openclaw-msteams` (должно быть уникальным) |
   | **Subscription** | Выберите подписку Azure |
   | **Resource group** | Создайте новую или используйте существующую |
   | **Pricing tier** | **Free** для разработки/тестирования |
   | **Type of App** | **Single Tenant** (рекомендуется — см. примечание ниже) |
   | **Creation type** | **Create new Microsoft App ID** |

> **Уведомление об устаревании:** создание новых мультитенантных ботов устарело после 2025-07-31. Для новых ботов используйте **Single Tenant**.

3. Нажмите **Review + create** → **Create** (ожидайте ~1–2 минуты).

### Шаг 2: Получение учётных данных

1. Перейдите к ресурсу Azure Bot → **Configuration**.
2. Скопируйте **Microsoft App ID** — это ваш `appId`.
3. Нажмите **Manage Password** → перейдите к регистрации приложения.
4. В разделе **Certificates & secrets** → **New client secret** → скопируйте **Value** — это ваш `appPassword`.
5. Перейдите в **Overview** → скопируйте **Directory (tenant) ID** — это ваш `tenantId`.

### Шаг 3: Настройка конечной точки обмена сообщениями

1. В Azure Bot → **Configuration**.
2. Установите **Messaging endpoint** на ваш URL вебхука:
   - Для продакшена: `https://your-domain.com/api/messages`.
   - Для локальной разработки: используйте туннель (см. [Локальная разработка (туннелирование)](#локальная-разработка-туннелирование) ниже).

### Шаг 4: Включение канала Teams

1. В Azure Bot → **Channels**.
2. Нажмите **Microsoft Teams** → Configure → Save.
3. Примите условия обслуживания.
