# Zalo Personal (неофициальный)

Статус: экспериментальный. Эта интеграция автоматизирует работу с **личным аккаунтом Zalo** через нативный `zca-js` внутри OpenClaw.

> **Предупреждение:** это неофициальная интеграция, которая может привести к приостановке действия аккаунта или его блокировке. Используйте на свой страх и риск.

## Входящий в комплект плагин

Zalo Personal поставляется как входящий в комплект плагин в текущих версиях OpenClaw, поэтому для обычных упакованных сборок отдельная установка не требуется.

Если вы используете более старую сборку или пользовательскую установку, в которой отсутствует Zalo Personal, установите его вручную:

- Установка через CLI: `openclaw plugins install @openclaw/zalouser`
- Или из локальной копии исходного кода: `openclaw plugins install ./path/to/local/zalouser-plugin`
- Подробности: [Плагины](/tools/plugin)

Внешний бинарный файл `zca`/`openzca` CLI не требуется.

## Быстрая настройка (для начинающих)

1. Убедитесь, что плагин Zalo Personal доступен.
   - В текущих упакованных версиях OpenClaw он уже включён.
   - В более старых или пользовательских установках его можно добавить вручную с помощью команд, приведённых выше.
2. Вход в систему (по QR-коду на машине с Gateway):
   - `openclaw channels login --channel zalouser`
   - Отсканируйте QR-код с помощью мобильного приложения Zalo.
3. Включите канал:

```json5
{
  channels: {
    zalouser: {
      enabled: true,
      dmPolicy: "pairing",
    },
  },
}
```

4. Перезапустите Gateway (или завершите настройку).
5. Доступ к личным сообщениям по умолчанию осуществляется через сопряжение; подтвердите код сопряжения при первом контакте.

## Что это такое

- Работает полностью внутри процесса через `zca-js`.
- Использует нативные прослушиватели событий для получения входящих сообщений.
- Отправляет ответы напрямую через JS API (текст/медиа/ссылка).
- Предназначен для сценариев использования "личного аккаунта", когда Zalo Bot API недоступен.

## Именование

Идентификатор канала — `zalouser`, чтобы чётко указать, что он автоматизирует **личный аккаунт пользователя Zalo** (неофициальный). Мы оставляем `zalo` для потенциальной будущей официальной интеграции с API Zalo.

## Поиск идентификаторов (каталог)

Используйте CLI каталога, чтобы найти участников/группы и их идентификаторы:

```bash
openclaw directory self --channel zalouser
openclaw directory peers list --channel zalouser --query "name"
openclaw directory groups list --channel zalouser --query "work"
```

## Ограничения

- Исходящий текст разбивается на фрагменты размером около 2000 символов (ограничения клиента Zalo).
- Потоковая передача по умолчанию заблокирована.

## Контроль доступа (личные сообщения)

`channels.zalouser.dmPolicy` поддерживает следующие значения: `pairing | allowlist | open | disabled` (по умолчанию: `pairing`).

`channels.zalouser.allowFrom` принимает идентификаторы пользователей или их имена. Во время настройки имена преобразуются в идентификаторы с помощью поиска контактов, выполняемого плагином внутри процесса.

Подтвердите через:

- `openclaw pairing list zalouser`
- `openclaw pairing approve zalouser <code>`

## Доступ к группам (опционально)

- По умолчанию: `channels.zalouser.groupPolicy = "open"` (группы разрешены). Чтобы переопределить значение по умолчанию, если оно не задано, используйте `channels.defaults.groupPolicy`.
- Ограничьте доступ списком разрешённых с помощью:
  - `channels.zalouser.groupPolicy = "allowlist"`
  - `channels.zalouser.groups` (ключи должны быть стабильными идентификаторами групп; имена преобразуются в идентификаторы при запуске, если это возможно)
  - `channels.zalouser.groupAllowFrom` (контролирует, какие отправители в разрешённых группах могут активировать бота)
- Запретите доступ ко всем группам: `channels.zalouser.groupPolicy = "disabled"`.
- Мастер настройки может запросить список разрешённых групп.
- При запуске OpenClaw преобразует имена групп/пользователей в списках разрешённых в идентификаторы и записывает сопоставление в журнал.
- Сопоставление списка разрешённых групп по умолчанию выполняется только по идентификаторам. Неразрешённые имена игнорируются при аутентификации, если не включён параметр `channels.zalouser.dangerouslyAllowNameMatching: true`.
- `channels.zalouser.dangerouslyAllowNameMatching: true` — это режим совместимости для экстренных случаев, который повторно включает сопоставление изменчивых имён групп.
- Если `groupAllowFrom` не задан, во время выполнения для проверки отправителей в группах используется `allowFrom`.
- Проверки отправителей применяются как к обычным сообщениям в группах, так и к командным (например, `/new`, `/reset`).

Пример:

```json5
{
  channels: {
    zalouser: {
      groupPolicy: "allowlist",
      groupAllowFrom: ["1471383327500481391"],
      groups: {
        "123456789": { allow: true },
        "Work Chat": { allow: true },
      },
    },
  },
}
```

### Контроль упоминаний в группах

- `channels.zalouser.groups.<group>.requireMention` определяет, требуются ли упоминания для ответов в группах.
- Порядок разрешения: точный идентификатор/имя группы → нормализованный слаг группы → `*` → значение по умолчанию (`true`).
- Это применяется как к группам из списка разрешённых, так и к режиму открытых групп.
- Цитирование сообщения бота считается неявным упоминанием для активации группы.
- Авторизованные командные запросы (например, `/new`) могут обходить контроль упоминаний.
- Если сообщение в группе пропущено из-за требования упоминания, OpenClaw сохраняет его как ожидающую историю группы и включает в следующее обработанное сообщение группы.
- Лимит истории группы по умолчанию определяется параметром `messages.groupChat.historyLimit` (запасное значение — `50`). Вы можете переопределить его для каждой учётной записи с помощью `channels.zalouser.historyLimit`.

Пример:

```json5
{
  channels: {
    zalouser: {
      groupPolicy: "allowlist",
      groups: {
        "*": { allow: true, requireMention: true },
        "Work Chat": { allow: true, requireMention: false },
      },
    },
  },
}
```

## Несколько учётных записей

Учётные записи сопоставляются с профилями `zalouser` в состоянии OpenClaw. Пример:

```json5
{
  channels: {
    zalouser: {
      enabled: true,
      defaultAccount: "default",
      accounts: {
        work: { enabled: true, profile: "work" },
      },
    },
  },
}
```

## Индикация набора, реакции и подтверждения доставки

- OpenClaw отправляет событие индикации набора перед отправкой ответа (по возможности).
- Действие реакции на сообщение `react` поддерживается для `zalouser` в действиях канала.
  - Используйте `remove: true`, чтобы удалить определённый эмодзи-реакцию из сообщения.
  - Семантика реакций: [Реакции](/tools/reactions)
- Для входящих сообщений, содержащих метаданные события, OpenClaw отправляет подтверждения доставки и прочтения (по возможности).

## Устранение неполадок

**Вход в систему не сохраняется:**

- `openclaw channels status --probe`
- Повторный вход: `openclaw channels logout --channel zalouser && openclaw channels login --channel zalouser`

**Имя из списка разрешённых/группы не было преобразовано:**

- Используйте числовые идентификаторы в `allowFrom`/`groupAllowFrom`/`groups` или точные имена друзей/групп.

**Обновление с предыдущей настройки на основе CLI:**

- Удалите все предположения о внешнем процессе `zca`.
- Теперь канал работает полностью в OpenClaw без внешних бинарных файлов CLI.
