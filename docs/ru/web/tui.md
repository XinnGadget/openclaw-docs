# TUI (Терминальный пользовательский интерфейс, Terminal UI)

## Быстрый старт

1. Запустите Gateway.

```bash
openclaw gateway
```

2. Откройте TUI.

```bash
openclaw tui
```

3. Введите сообщение и нажмите Enter.

Удаленный Gateway:

```bash
openclaw tui --url ws://<host>:<port> --token <gateway-token>
```

Используйте `--password`, если в вашем Gateway используется аутентификация по паролю.

## Что вы видите

- Заголовок: URL подключения, текущий агент, текущая сессия.
- Журнал чата: сообщения пользователя, ответы ассистента, системные уведомления, карточки инструментов.
- Строка состояния: состояние подключения/выполнения (подключение, выполнение, потоковая передача, ожидание, ошибка).
- Нижний колонтитул: состояние подключения + агент + сессия + модель + think/fast/verbose/reasoning + количество токенов + доставка.
- Поле ввода: текстовый редактор с автодополнением.

## Концептуальная модель: агенты + сессии

- Агенты — это уникальные идентификаторы (например, `main`, `research`). Gateway предоставляет список агентов.
- Сессии принадлежат текущему агенту.
- Ключи сессий хранятся в формате `agent:<agentId>:<sessionKey>`.
  - Если вы введёте `/session main`, TUI развернёт это до `agent:<currentAgent>:main`.
  - Если вы введёте `/session agent:other:main`, вы явно переключитесь на эту сессию агента.
- Область действия сессии:
  - `per-sender` (по умолчанию): у каждого агента много сессий.
  - `global`: TUI всегда использует сессию `global` (список выбора может быть пустым).
- Текущий агент и сессия всегда отображаются в нижнем колонтитуле.

## Отправка и доставка

- Сообщения отправляются в Gateway; доставка провайдерам по умолчанию отключена.
- Включите доставку:
  - `/deliver on`
  - или через панель настроек
  - или запустите с `openclaw tui --deliver`

## Списки выбора и наложения

- Список моделей: отображает доступные модели и позволяет задать переопределение для сессии.
- Список агентов: позволяет выбрать другого агента.
- Список сессий: отображает только сессии для текущего агента.
- Настройки: включение/выключение доставки, расширения вывода инструментов и отображения размышлений.

## Сочетания клавиш

- Enter: отправить сообщение
- Esc: прервать активное выполнение
- Ctrl+C: очистить ввод (нажмите дважды для выхода)
- Ctrl+D: выход
- Ctrl+L: список моделей
- Ctrl+G: список агентов
- Ctrl+P: список сессий
- Ctrl+O: переключение расширения вывода инструментов
- Ctrl+T: переключение отображения размышлений (перезагружает историю)

## Команды со слешем

Основные:

- `/help`
- `/status`
- `/agent <id>` (или `/agents`)
- `/session <key>` (или `/sessions`)
- `/model <provider/model>` (или `/models`)

Управление сессией:

- `/think <off|minimal|low|medium|high>`
- `/fast <status|on|off>`
- `/verbose <on|full|off>`
- `/reasoning <on|off|stream>`
- `/usage <off|tokens|full>`
- `/elevated <on|off|ask|full>` (псевдоним: `/elev`)
- `/activation <mention|always>`
- `/deliver <on|off>`

Жизненный цикл сессии:

- `/new` или `/reset` (сбросить сессию)
- `/abort` (прервать активное выполнение)
- `/settings`
- `/exit`

Другие команды Gateway со слешем (например, `/context`) перенаправляются в Gateway и отображаются как системный вывод. См. [Команды со слешем](/tools/slash-commands).

## Локальные команды оболочки

- Добавьте префикс `!` к строке, чтобы выполнить локальную команду оболочки на хосте TUI.
- TUI запрашивает разрешение на локальное выполнение один раз за сессию; отказ оставляет `!` отключённым для сессии.
- Команды выполняются в новой неинтерактивной оболочке в рабочем каталоге TUI (без сохранения `cd`/окружения).
- Локальные команды оболочки получают `OPENCLAW_SHELL=tui-local` в своём окружении.
- Одиночный `!` отправляется как обычное сообщение; начальные пробелы не запускают локальное выполнение.

## Вывод инструментов

- Вызовы инструментов отображаются в виде карточек с аргументами и результатами.
- Ctrl+O переключает между свёрнутым и развёрнутым видами.
- Во время выполнения инструментов частичные обновления поступают в ту же карточку.

## Цвета терминала

- TUI оставляет текст тела ответа ассистента в цвете переднего плана вашего терминала по умолчанию, чтобы текст был читаем как на тёмном, так и на светлом фоне.
- Если ваш терминал использует светлый фон и автоматическое определение неверно, установите `OPENCLAW_THEME=light` перед запуском `openclaw tui`.
- Чтобы принудительно использовать исходную тёмную палитру, установите `OPENCLAW_THEME=dark`.

## История и потоковая передача

- При подключении TUI загружает последнюю историю (по умолчанию 200 сообщений).
- Потоковые ответы обновляются на месте до завершения.
- TUI также отслеживает события инструментов агента для более подробных карточек инструментов.

## Детали подключения

- TUI регистрируется в Gateway как `mode: "tui"`.
- При переподключении отображается системное сообщение; пробелы в событиях отображаются в журнале.

## Параметры

- `--url <url>`: URL WebSocket Gateway (по умолчанию берётся из конфигурации или `ws://127.0.0.1:<port>`)
- `--token <token>`: токен Gateway (если требуется)
- `--password <password>`: пароль Gateway (если требуется)
- `--session <key>`: ключ сессии (по умолчанию: `main`, или `global`, если область действия глобальная)
- `--deliver`: доставка ответов ассистента провайдеру (по умолчанию отключена)
- `--thinking <level>`: переопределение уровня размышлений для отправки
- `--message <text>`: отправка начального сообщения после подключения
- `--timeout-ms <ms>`: тайм-аут агента в мс (по умолчанию `agents.defaults.timeoutSeconds`)
- `--history-limit <n>`: количество записей истории для загрузки (по умолчанию `200`)

Примечание: при указании `--url` TUI не использует конфигурацию или учётные данные из окружения.
Передайте `--token` или `--password` явно. Отсутствие явных учётных данных приведёт к ошибке.

## Устранение неполадок

Нет вывода после отправки сообщения:

- Выполните `/status` в TUI, чтобы убедиться, что Gateway подключён и находится в состоянии ожидания/выполнения.
- Проверьте журналы Gateway: `openclaw logs --follow`.
- Убедитесь, что агент может выполняться: `openclaw status` и `openclaw models status`.
- Если вы ожидаете сообщения в канале чата, включите доставку (`/deliver on` или `--deliver`).

## Устранение неполадок подключения

- `disconnected`: убедитесь, что Gateway запущен и ваши параметры `--url/--token/--password` указаны верно.
- Нет агентов в списке выбора: проверьте `openclaw agents list` и вашу конфигурацию маршрутизации.
- Пустой список сессий: возможно, вы находитесь в глобальной области действия или у вас ещё нет сессий.
