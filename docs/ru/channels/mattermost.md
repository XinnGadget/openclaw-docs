# Mattermost

Статус: плагин в комплекте (токен бота + события WebSocket). Поддерживаются каналы, группы и личные сообщения (DM).
Mattermost — платформа для обмена сообщениями в команде с возможностью самостоятельного размещения; подробности о продукте и загрузки доступны на официальном сайте [mattermost.com](https://mattermost.com).

## Плагин в комплекте

В текущих версиях OpenClaw Mattermost поставляется как плагин в комплекте, поэтому для стандартных сборок отдельная установка не требуется.

Если вы используете более старую сборку или пользовательскую установку, в которой Mattermost отсутствует, установите его вручную:

Установка через CLI (реестр npm):

```bash
openclaw plugins install @openclaw/mattermost
```

Локальная проверка (при работе из репозитория git):

```bash
openclaw plugins install ./path/to/local/mattermost-plugin
```

Подробности: [Плагины](/tools/plugin)

## Быстрая настройка

1. Убедитесь, что плагин Mattermost доступен.
   - В текущих упакованных версиях OpenClaw он уже включён.
   - В более старых или пользовательских установках его можно добавить вручную с помощью команд выше.
2. Создайте учётную запись бота Mattermost и скопируйте **токен бота**.
3. Скопируйте **базовый URL** Mattermost (например, `https://chat.example.com`).
4. Настройте OpenClaw и запустите шлюз.

Минимальная конфигурация:

```json5
{
  channels: {
    mattermost: {
      enabled: true,
      botToken: "mm-token",
      baseUrl: "https://chat.example.com",
      dmPolicy: "pairing",
    },
  },
}
```

## Встроенные команды с косой чертой

Встроенные команды с косой чертой активируются по желанию. При включении OpenClaw регистрирует команды `oc_*` через API Mattermost и получает обратные POST-запросы на HTTP-сервере шлюза.

```json5
{
  channels: {
    mattermost: {
      commands: {
        native: true,
        nativeSkills: true,
        callbackPath: "/api/channels/mattermost/command",
        // Используйте, если Mattermost не может напрямую обратиться к шлюзу (обратный прокси/публичный URL).
        callbackUrl: "https://gateway.example.com/api/channels/mattermost/command",
      },
    },
  },
}
```

Примечания:

- `native: "auto"` по умолчанию отключён для Mattermost. Установите `native: true`, чтобы включить.
- Если `callbackUrl` не указан, OpenClaw формирует его на основе хоста/порта шлюза + `callbackPath`.
- Для настроек с несколькими учётными записями параметр `commands` можно задать на верхнем уровне или в `channels.mattermost.accounts.<id>.commands` (значения для учётных записей переопределяют поля верхнего уровня).
- Обратные вызовы команд проверяются с помощью токенов для каждой команды, возвращаемых Mattermost при регистрации команд `oc_*` OpenClaw.
- Обратные вызовы команд завершаются с ошибкой, если регистрация не удалась, запуск был неполным или токен обратного вызова не соответствует ни одной из зарегистрированных команд.
- Требование доступности: конечная точка обратного вызова должна быть доступна из сервера Mattermost.
  - Не устанавливайте `callbackUrl` как `localhost`, если Mattermost не работает на том же хосте/в том же сетевом пространстве имён, что и OpenClaw.
  - Не устанавливайте `callbackUrl` как базовый URL Mattermost, если этот URL не перенаправляет `/api/channels/mattermost/command` на OpenClaw через обратный прокси.
  - Быстрая проверка: `curl https://<gateway-host>/api/channels/mattermost/command`; GET должен возвращать `405 Method Not Allowed` от OpenClaw, а не `404`.
- Требование к списку разрешённых исходящих подключений Mattermost:
  - Если ваши обратные вызовы нацелены на частные/tailnet/внутренние адреса, установите в Mattermost `ServiceSettings.AllowedUntrustedInternalConnections` так, чтобы включить хост/домен обратного вызова.
  - Используйте записи хоста/домена, а не полные URL.
    - Правильно: `gateway.tailnet-name.ts.net`
    - Неправильно: `https://gateway.tailnet-name.ts.net`

## Переменные окружения (учётная запись по умолчанию)

Установите их на хосте шлюза, если предпочитаете использовать переменные окружения:

- `MATTERMOST_BOT_TOKEN=...`
- `MATTERMOST_URL=https://chat.example.com`

Переменные окружения применяются только к учётной записи **по умолчанию** (`default`). Для других учётных записей необходимо использовать значения конфигурации.

## Режимы чата

Mattermost автоматически отвечает на личные сообщения. Поведение в каналах контролируется параметром `chatmode`:

- `oncall` (по умолчанию): отвечать только при упоминании с `@` в каналах.
- `onmessage`: отвечать на каждое сообщение в канале.
- `onchar`: отвечать, когда сообщение начинается с префикса-триггера.

Пример конфигурации:

```json5
{
  channels: {
    mattermost: {
      chatmode: "onchar",
      oncharPrefixes: [">", "!"],
    },
  },
}
```

Примечания:

- В режиме `onchar` также отвечают на явные упоминания с `@`.
- Параметр `channels.mattermost.requireMention` учитывается для устаревших конфигураций, но предпочтительнее использовать `chatmode`.

## Потоки и сессии

Используйте `channels.mattermost.replyToMode`, чтобы контролировать, остаются ли ответы в каналах и группах в основном канале или запускают поток под инициирующим сообщением.

- `off` (по умолчанию): отвечать в потоке только в том случае, если входящее сообщение уже находится в потоке.
- `first`: для сообщений верхнего уровня в канале/группе запускать поток под этим сообщением и направлять разговор в сессию, ограниченную потоком.
- `all`: в настоящее время поведение аналогично `first` для Mattermost.
- Личные сообщения игнорируют эту настройку и остаются вне потоков.

Пример конфигурации:

```json5
{
  channels: {
    mattermost: {
      replyToMode: "all",
    },
  },
}
```

Примечания:

- Сессии, ограниченные потоком, используют идентификатор инициирующего сообщения в качестве корня потока.
- `first` и `all` в настоящее время эквивалентны, поскольку после того, как в Mattermost появляется корень потока, последующие фрагменты и медиафайлы продолжают поступать в тот же поток.

## Контроль доступа (личные сообщения)

- По умолчанию: `channels.mattermost.dmPolicy = "pairing"` (неизвестные отправители получают код для сопряжения).
- Утверждение через:
  - `openclaw pairing list mattermost`
  - `openclaw pairing approve mattermost <CODE>`
- Открытые личные сообщения: `channels.mattermost.dmPolicy="open"` плюс `channels.mattermost.allowFrom=["*"]`.

## Каналы (группы)

- По умолчанию: `channels.mattermost.groupPolicy = "allowlist"` (с ограничением по упоминаниям).
- Разрешите отправителей с помощью `channels.mattermost.groupAllowFrom` (рекомендуется использовать идентификаторы пользователей).
- Переопределения упоминаний для отдельных каналов находятся в `channels.mattermost.groups.<channelId>.requireMention` или `channels.mattermost.groups["*"].requireMention` для значения по умолчанию.
- Сопоставление `@username` можно изменить, и оно включено только при `channels.mattermost.dangerouslyAllowNameMatching: true`.
- Открытые каналы: `channels.mattermost.groupPolicy="open"` (с ограничением по упоминаниям).
- Примечание о времени выполнения: если `channels.mattermost` полностью отсутствует, во время выполнения используется `groupPolicy="allowlist"` для проверок групп (даже если `channels.defaults.groupPolicy` задан).

Пример:

```json5
{
  channels: {
    mattermost: {
      groupPolicy: "open",
      groups: {
        "*": { requireMention: true },
        "team-channel-id": { requireMention: false },
      },
    },
  },
}
```

## Цели для исходящей доставки

Используйте следующие форматы целей с `openclaw message send` или cron/вебхуками:

- `channel:<id>` для канала
- `user:<id>` для личного сообщения
- `@username` для личного сообщения (разрешается через API Mattermost)

Простые непрозрачные идентификаторы (например, `64ifufp...`)
