# Поиск с Gemini

OpenClaw поддерживает модели Gemini со встроенной
[опорой на Google Search](https://ai.google.dev/gemini-api/docs/grounding),
которая возвращает синтезированные ИИ ответы, подкреплённые актуальными результатами поиска Google с цитатами.

## Получение API-ключа

<Steps>
  <Step title="Создать ключ">
    Перейдите на [Google AI Studio](https://aistudio.google.com/apikey) и создайте API-ключ.
  </Step>
  <Step title="Сохранить ключ">
    Установите `GEMINI_API_KEY` в окружении Gateway либо настройте через:

    ```bash
    openclaw configure --section web
    ```

  </Step>
</Steps>

## Настройка

```json5
{
  plugins: {
    entries: {
      google: {
        config: {
          webSearch: {
            apiKey: "AIza...", // необязательно, если задан GEMINI_API_KEY
            model: "gemini-2.5-flash", // по умолчанию
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        provider: "gemini",
      },
    },
  },
}
```

**Альтернатива через окружение:** задайте `GEMINI_API_KEY` в окружении Gateway.
При установке шлюза поместите его в `~/.openclaw/.env`.

## Как это работает

В отличие от традиционных поисковых провайдеров, которые возвращают список ссылок и фрагментов, Gemini использует опору на Google Search для формирования синтезированных ИИ ответов с встроенными цитатами. Результаты включают как синтезированный ответ, так и URL-адреса источников.

- URL-адреса цитат из опоры Gemini автоматически преобразуются из перенаправляющих URL Google в прямые URL.
- Разрешение перенаправлений использует путь защиты от SSRF (HEAD + проверка перенаправлений + валидация http/https) перед возвратом конечного URL цитаты.
- Разрешение перенаправлений использует строгие значения по умолчанию для защиты от SSRF, поэтому перенаправления на частные/внутренние цели блокируются.

## Поддерживаемые параметры

Поиск с Gemini поддерживает параметр `query`.

Параметр `count` принимается для совместимости с общим `web_search`, но опора Gemini по-прежнему возвращает один синтезированный ответ с цитатами, а не список из N результатов.

Специфичные для провайдера фильтры, такие как `country`, `language`, `freshness` и `domain_filter`, не поддерживаются.

## Выбор модели

Модель по умолчанию — `gemini-2.5-flash` (быстрая и экономичная). Любую модель Gemini, поддерживающую опору (grounding), можно использовать через `plugins.entries.google.config.webSearch.model`.
