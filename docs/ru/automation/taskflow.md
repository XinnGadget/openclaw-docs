# Task Flow

Task Flow — это субстрат оркестрации потоков, который располагается над [фоновыми задачами](/automation/tasks). Он управляет устойчивыми многоэтапными потоками со своим собственным состоянием, отслеживанием версий и семантикой синхронизации, при этом отдельные задачи остаются единицами автономной работы.

## Когда использовать Task Flow

Используйте Task Flow, когда работа включает несколько последовательных или разветвляющихся этапов и вам необходимо устойчивое отслеживание прогресса после перезапуска шлюза. Для одиночных фоновых операций достаточно обычной [задачи](/automation/tasks).

| Сценарий | Что использовать |
| --- | --- |
| Одиночная фоновая задача | Обычная задача |
| Многоэтапный конвейер (A, затем B, затем C) | Task Flow (управляемый) |
| Наблюдение за задачами, созданными извне | Task Flow (зеркальный) |
| Одноразовое напоминание | Cron-задача |

## Режимы синхронизации

### Управляемый режим (Managed mode)

Task Flow полностью управляет жизненным циклом. Он создаёт задачи в качестве этапов потока, доводит их до завершения и автоматически продвигает состояние потока.

Пример: поток еженедельного отчёта, который (1) собирает данные, (2) формирует отчёт и (3) доставляет его. Task Flow создаёт каждый этап в виде фоновой задачи, ожидает завершения, затем переходит к следующему этапу.

```
Поток: weekly-report
  Этап 1: gather-data     → задача создана → выполнена успешно
  Этап 2: generate-report → задача создана → выполнена успешно
  Этап 3: deliver         → задача создана → выполняется
```

### Зеркальный режим (Mirrored mode)

Task Flow наблюдает за задачами, созданными извне, и поддерживает синхронизацию состояния потока, не беря на себя управление созданием задач. Это полезно, когда задачи инициируются cron-задачами, командами CLI или из других источников, и вам нужен единый взгляд на их прогресс в рамках потока.

Пример: три независимые cron-задачи, которые вместе формируют процедуру "утренних операций". Зеркальный поток отслеживает их совокупный прогресс, не контролируя, когда и как они выполняются.

## Устойчивое состояние и отслеживание версий

Каждый поток сохраняет своё собственное состояние и отслеживает версии, благодаря чему прогресс сохраняется после перезапуска шлюза. Отслеживание версий позволяет выявлять конфликты, когда несколько источников пытаются одновременно продвинуть один и тот же поток.

## Поведение при отмене

Команда `openclaw tasks flow cancel` устанавливает устойчивое намерение отменить поток. Активные задачи в рамках потока отменяются, и новые этапы не запускаются. Намерение об отмене сохраняется после перезапусков, поэтому отменённый поток остаётся отменённым, даже если шлюз перезапустится до завершения всех дочерних задач.

## Команды CLI

```bash
# Вывести список активных и недавних потоков
openclaw tasks flow list

# Показать детали конкретного потока
openclaw tasks flow show <lookup>

# Отменить выполняющийся поток и его активные задачи
openclaw tasks flow cancel <lookup>
```

| Команда | Описание |
| --- | --- |
| `openclaw tasks flow list` | Показывает отслеживаемые потоки с их статусом и режимом синхронизации |
| `openclaw tasks flow show <id>` | Позволяет изучить один поток по его идентификатору или ключу поиска |
| `openclaw tasks flow cancel <id>` | Отменяет выполняющийся поток и его активные задачи |

## Как потоки связаны с задачами

Потоки координируют задачи, а не заменяют их. Один поток может управлять несколькими фоновыми задачами на протяжении своего жизненного цикла. Используйте `openclaw tasks`, чтобы изучить отдельные записи о задачах, и `openclaw tasks flow`, чтобы изучить координирующий поток.
