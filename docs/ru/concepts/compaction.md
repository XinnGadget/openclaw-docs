# Компакция

У каждой модели есть окно контекста — максимальное количество токенов, которое она может обработать. Когда диалог приближается к этому лимиту, OpenClaw **компактирует** старые сообщения, преобразуя их в сводку, чтобы диалог можно было продолжить.

## Как это работает

1. Старые фрагменты диалога суммируются в компактную запись.
2. Сводка сохраняется в транскрипте сессии.
3. Недавние сообщения остаются без изменений.

Когда OpenClaw разделяет историю на фрагменты для компакции, он сохраняет вызовы инструментов ассистента вместе с соответствующими записями `toolResult`. Если точка разделения попадает внутрь блока инструмента, OpenClaw перемещает границу так, чтобы пара осталась вместе, а текущая несосуммированная часть была сохранена.

Полная история диалога сохраняется на диске. Компакция изменяет только то, что модель видит на следующем шаге.

## Автокомпакция

Автокомпакция включена по умолчанию. Она запускается, когда сессия приближается к лимиту контекста или когда модель возвращает ошибку переполнения контекста (в этом случае OpenClaw выполняет компакцию и повторяет запрос). Типичные признаки переполнения включают:
- `request_too_large`;
- `context length exceeded`;
- `input exceeds the maximum number of tokens`;
- `input token count exceeds the maximum number of input tokens`;
- `input is too long for the model`;
- `ollama error: context length exceeded`.

<Info>
Перед компакцией OpenClaw автоматически напоминает агенту сохранить важные заметки в файлы [памяти](/concepts/memory). Это позволяет избежать потери контекста.
</Info>

Для настройки поведения компакции (режим, целевое количество токенов и т. д.) используйте параметр `agents.defaults.compaction` в файле `openclaw.json`. По умолчанию при суммировании компакции сохраняются непрозрачные идентификаторы (`identifierPolicy: "strict"`). Вы можете отключить это с помощью `identifierPolicy: "off"` или задать собственный текст с помощью `identifierPolicy: "custom"` и `identifierInstructions`.

При необходимости можно указать другую модель для суммирования при компакции через `agents.defaults.compaction.model`. Это полезно, если ваша основная модель — локальная или небольшая, а вы хотите, чтобы сводки компакции создавались более мощной моделью. Переопределение принимает любую строку `provider/model-id`:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "openrouter/anthropic/claude-sonnet-4-6"
      }
    }
  }
}
```

Это также работает с локальными моделями, например со второй моделью Ollama, предназначенной для суммирования, или с доработанной моделью для компакции:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "model": "ollama/llama3.1:8b"
      }
    }
  }
}
```

Если параметр не задан, для компакции используется основная модель агента.

## Подключаемые провайдеры компакции

Плагины могут зарегистрировать пользовательский провайдер компакции через `registerCompactionProvider()` в API плагина. Когда провайдер зарегистрирован и настроен, OpenClaw делегирует суммирование ему вместо встроенного конвейера LLM.

Чтобы использовать зарегистрированный провайдер, укажите его идентификатор в конфигурации:

```json
{
  "agents": {
    "defaults": {
      "compaction": {
        "provider": "my-provider"
      }
    }
  }
}
```

Задание параметра `provider` автоматически устанавливает `mode: "safeguard"`. Провайдеры получают те же инструкции по компакции и политику сохранения идентификаторов, что и встроенный механизм. OpenClaw по-прежнему сохраняет контекст последних и разделённых шагов после вывода провайдера. Если провайдер выдаёт ошибку или возвращает пустой результат, OpenClaw возвращается к встроенному суммированию LLM.

## Автокомпакция (включена по умолчанию)

Когда сессия приближается к окну контекста модели или превышает его, OpenClaw запускает автокомпакцию и может повторить исходный запрос с использованием компактного контекста.

Вы увидите:
- `🧹 Автокомпакция завершена` в подробном режиме;
- в `/status` — `🧹 Компакции: <количество>`.

Перед компакцией OpenClaw может выполнить **тихую очистку памяти**, чтобы сохранить надёжные заметки на диске. Подробнее см. в разделе [Память](/concepts/memory) и в настройках.

## Ручная компакция

Введите `/compact` в любом чате, чтобы принудительно выполнить компакцию. Добавьте инструкции, чтобы направить суммирование:

```
/compact Сосредоточься на решениях по проектированию API
```

## Использование другой модели

По умолчанию для компакции используется основная модель вашего агента. Вы можете использовать более мощную модель для получения лучших сводок:

```json5
{
  agents: {
    defaults: {
      compaction: {
        model: "openrouter/anthropic/claude-sonnet-4-6",
      },
    },
  },
}
```

## Уведомление о начале компакции

По умолчанию компакция выполняется незаметно. Чтобы отображать краткое уведомление при начале компакции, включите `notifyUser`:

```json5
{
  agents: {
    defaults: {
      compaction: {
        notifyUser: true,
      },
    },
  },
}
```

При включении пользователь увидит короткое сообщение (например, "Выполняется компакция контекста…") в начале каждого запуска компакции.

## Компакция и обрезка

| | Компакция | Обрезка |
| --- | --- | --- |
| **Что делает** | Суммирует старые фрагменты диалога | Удаляет старые результаты инструментов |
| **Сохраняется?** | Да (в транскрипте сессии) | Нет (только в памяти, для каждого запроса) |
| **Область действия** | Весь диалог | Только результаты инструментов |

[Обрезка сессии](/concepts/session-pruning) — это более лёгкий механизм, который удаляет выходные данные инструментов без суммирования.

## Устранение неполадок

**Компакция выполняется слишком часто?** Возможно, окно контекста модели невелико или выходные данные инструментов занимают много места. Попробуйте включить [обрезку сессии](/concepts/session-pruning).

**После компакции контекст кажется устаревшим?** Используйте `/compact Сосредоточься на <теме>`, чтобы направить суммирование, или включите [очистку памяти](/concepts/memory), чтобы заметки сохранились.

**Нужен чистый лист?** Команда `/new` начинает новую сессию без компакции.

Для расширенной настройки (резервные токены, сохранение идентификаторов, пользовательские механизмы контекста, серверная компакция OpenAI) см. раздел [Глубокое погружение в управление сессиями](/reference/session-management-compaction).
