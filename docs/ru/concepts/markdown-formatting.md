# Форматирование Markdown

OpenClaw форматирует исходящий Markdown, преобразуя его в общее промежуточное представление (IR) перед генерацией вывода, специфичного для канала. IR сохраняет исходный текст в неизменном виде, при этом содержит диапазоны стилей и ссылок, что позволяет обеспечить согласованность разбиения на чанки и рендеринга в разных каналах.

## Цели

- **Согласованность:** один этап парсинга, несколько рендереров.
- **Безопасное разбиение на чанки:** разделение текста до рендеринга, чтобы встроенное форматирование никогда не разрывалось между чанками.
- **Адаптация под канал:** сопоставление одного и того же IR с mrkdwn в Slack, HTML в Telegram и диапазонами стилей в Signal без повторного парсинга Markdown.

## Конвейер

1. **Парсинг Markdown → IR**
   - IR представляет собой обычный текст плюс диапазоны стилей (жирный/курсив/зачёркнутый/код/спойлер) и диапазоны ссылок.
   - Смещение задаётся в единицах кода UTF-16, чтобы диапазоны стилей в Signal соответствовали его API.
   - Таблицы парсятся только в том случае, если канал поддерживает преобразование таблиц.
2. **Разбиение IR на чанки (сначала форматирование)**
   - Разбиение на чанки выполняется для текста IR до рендеринга.
   - Встроенное форматирование не разрывается между чанками; диапазоны разбиваются по чанкам.
3. **Рендеринг для каждого канала**
   - **Slack:** токены mrkdwn (жирный/курсив/зачёркнутый/код), ссылки в формате `<url|label>`.
   - **Telegram:** HTML-теги (`<b>`, `<i>`, `<s>`, `<code>`, `<pre><code>`, `<a href>`).
   - **Signal:** обычный текст + диапазоны `text-style`; ссылки преобразуются в `label (url)`, если label отличается от URL.

## Пример IR

Входной Markdown:

```markdown
Hello **world** — see [docs](https://docs.openclaw.ai).
```

IR (схематично):

```json
{
  "text": "Hello world — see docs.",
  "styles": [{ "start": 6, "end": 11, "style": "bold" }],
  "links": [{ "start": 19, "end": 23, "href": "https://docs.openclaw.ai" }]
}
```

## Где используется

- Адаптеры исходящих каналов Slack, Telegram и Signal выполняют рендеринг на основе IR.
- Другие каналы (WhatsApp, iMessage, Microsoft Teams, Discord) по-прежнему используют обычный текст или собственные правила форматирования; преобразование таблиц Markdown применяется до разбиения на чанки, если оно включено.

## Обработка таблиц

Таблицы Markdown не поддерживаются единообразно в разных чат-клиентах. Используйте `markdown.tables` для управления преобразованием таблиц для каждого канала (и для каждой учётной записи).

- `code`: отображение таблиц в виде блоков кода (по умолчанию для большинства каналов).
- `bullets`: преобразование каждой строки в маркированный список (по умолчанию для Signal и WhatsApp).
- `off`: отключение парсинга и преобразования таблиц; необработанный текст таблицы передаётся без изменений.

Ключи конфигурации:

```yaml
channels:
  discord:
    markdown:
      tables: code
    accounts:
      work:
        markdown:
          tables: off
```

## Правила разбиения на чанки

- Ограничения на размер чанков задаются адаптерами каналов/конфигурацией и применяются к тексту IR.
- Блоки кода (code fences) сохраняются как единый блок с завершающей новой строкой, чтобы каналы корректно их отображали.
- Префиксы списков и префиксы цитат являются частью текста IR, поэтому разбиение на чанки не происходит посередине префикса.
- Встроенные стили (жирный/курсив/зачёркнутый/встроенный код/спойлер) никогда не разрываются между чанками; рендерер возобновляет стили внутри каждого чанка.

Если вам нужна дополнительная информация о поведении разбиения на чанки в разных каналах, см. [Streaming + chunking](/concepts/streaming).

## Политика обработки ссылок

- **Slack:** `[label](url)` → `<url|label>`; голые URL остаются без изменений. Автоматическое создание ссылок (autolink) отключено во время парсинга, чтобы избежать двойного связывания.
- **Telegram:** `[label](url)` → `<a href="url">label</a>` (режим парсинга HTML).
- **Signal:** `[label](url)` → `label (url)`, если label не совпадает с URL.

## Спойлеры

Маркеры спойлеров (`||spoiler||`) парсятся только для Signal, где они сопоставляются с диапазонами стилей SPOILER. В других каналах они обрабатываются как обычный текст.

## Как добавить или обновить форматер канала

1. **Один раз выполнить парсинг:** использовать общий помощник `markdownToIR(...)` с параметрами, подходящими для канала (autolink, стиль заголовков, префикс цитат).
2. **Рендеринг:** реализовать рендерер с `renderMarkdownWithMarkers(...)` и картой маркеров стилей (или диапазонами стилей Signal).
3. **Разбиение на чанки:** вызвать `chunkMarkdownIR(...)` перед рендерингом; выполнить рендеринг каждого чанка.
4. **Настройка адаптера:** обновить адаптер исходящего канала, чтобы использовать новый разбиватель на чанки и рендерер.
5. **Тестирование:** добавить или обновить тесты форматирования и тест исходящей доставки, если канал использует разбиение на чанки.

## Распространённые подводные камни

- Токены в угловых скобках в Slack (`<@U123>`, `<#C123>`, `<https://...>`) должны сохраняться; безопасно экранируйте сырой HTML.
- В Telegram HTML требуется экранировать текст вне тегов, чтобы избежать некорректной разметки.
- Диапазоны стилей в Signal зависят от смещений в UTF-16; не используйте смещения в кодовых точках.
- Сохраняйте завершающие новые строки для блоков кода в ограждении, чтобы закрывающие маркеры находились на отдельной строке.
