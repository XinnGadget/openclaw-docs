# Сокращение сессий (Session Pruning)

Сокращение сессий (session pruning) удаляет **старые результаты работы инструментов** из контекста перед каждым вызовом LLM. Это позволяет сократить раздувание контекста из-за накопленных выходных данных инструментов (результатов выполнения, чтения файлов, результатов поиска), не затрагивая при этом обычный текст диалога.

<Info>
Сокращение выполняется только в оперативной памяти — оно не изменяет транскрипт сессии на диске.
Ваша полная история всегда сохраняется.
</Info>

## Почему это важно

В ходе длительных сессий накапливаются выходные данные инструментов, которые увеличивают размер окна контекста. Это ведёт к росту затрат и может привести к более раннему запуску [компакции](/concepts/compaction), чем это необходимо.

Сокращение особенно полезно для **кэширования промтов в Anthropic**. После истечения времени жизни кэша (TTL) следующий запрос заново кэширует полный промт. Сокращение уменьшает размер записываемого в кэш данных, напрямую снижая затраты.

## Как это работает

1. Дождаться истечения TTL кэша (по умолчанию — 5 минут).
2. Найти старые результаты работы инструментов для обычного сокращения (текст диалога остаётся нетронутым).
3. **Мягкое усечение (soft-trim)** слишком больших результатов — сохранить начало и конец, вставить `...`.
4. **Жёсткая очистка (hard-clear)** остальных данных — заменить их заполнителем.
5. Сбросить TTL, чтобы последующие запросы использовали обновлённый кэш.

## Очистка устаревших изображений

OpenClaw также выполняет отдельную идемпотентную очистку для старых сессий, в которых в истории сохранялись необработанные блоки изображений.

- Сохраняются **3 самых последних завершённых хода** байт в байт, чтобы префиксы кэша промтов для недавних продолжений оставались стабильными.
- Более старые уже обработанные блоки изображений в истории `user` или `toolResult` могут быть заменены на `[image data removed — already processed by model]`.
- Это действие отдельно от обычного сокращения по TTL кэша. Оно предназначено для того, чтобы повторные полезные нагрузки с изображениями не нарушали работу кэша промтов на последующих ходах.

## Умные значения по умолчанию

OpenClaw автоматически включает сокращение для профилей Anthropic:

| Тип профиля | Сокращение включено | Пульс (heartbeat) |
| --- | --- | --- |
| Anthropic OAuth/аутентификация по токену (включая повторное использование Claude CLI) | Да | 1 час |
| API-ключ | Да | 30 минут |

Если вы задаёте явные значения, OpenClaw их не переопределяет.

## Включение и отключение

По умолчанию сокращение отключено для провайдеров, не относящихся к Anthropic. Чтобы включить:

```json5
{
  agents: {
    defaults: {
      contextPruning: { mode: "cache-ttl", ttl: "5m" },
    },
  },
}
```

Чтобы отключить: задайте `mode: "off"`.

## Сокращение и компакция

| | Сокращение (Pruning) | Компакция (Compaction) |
| --- | --- | --- |
| **Что делает** | Удаляет результаты работы инструментов | Суммирует диалог |
| **Сохраняется ли?** | Нет (для каждого запроса) | Да (в транскрипте) |
| **Область действия** | Только результаты работы инструментов | Весь диалог |

Эти механизмы дополняют друг друга: сокращение поддерживает компактность выходных данных инструментов между циклами компакции.
