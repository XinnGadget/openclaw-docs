# Автоматизация и задачи

OpenClaw выполняет работу в фоновом режиме с помощью задач, запланированных заданий, хуков событий и постоянных инструкций. Эта страница поможет вам выбрать подходящий механизм и понять, как они сочетаются друг с другом.

## Краткое руководство по выбору

```mermaid
flowchart TD
    START([Что вам нужно?]) --> Q1{Запланировать работу?}
    START --> Q2{Отслеживать отсоединённую работу?}
    START --> Q3{Оркестрировать многоэтапные потоки?}
    START --> Q4{Реагировать на события жизненного цикла?}
    START --> Q5{Дать агенту постоянные инструкции?}

    Q1 -->|Да| Q1a{Точное время или гибкое?}
    Q1a -->|Точное| CRON["Запланированные задачи (Cron)"]
    Q1a -->|Гибкое| HEARTBEAT[Heartbeat]

    Q2 -->|Да| TASKS[Фоновые задачи]
    Q3 -->|Да| FLOW[Поток задач (Task Flow)]
    Q4 -->|Да| HOOKS[Хуки]
    Q5 -->|Да| SO[Постоянные инструкции (Standing Orders)]
```

| Сценарий использования | Рекомендуемый механизм | Обоснование |
| --------------------------------------- | ---------------------- | ------------------------------------------------ |
| Отправлять ежедневный отчёт ровно в 9:00 | Запланированные задачи (Cron) | Точное время, изолированное выполнение |
| Напомнить мне через 20 минут | Запланированные задачи (Cron) | Однократное выполнение с точным временем (`--at`) |
| Проводить глубокий анализ раз в неделю | Запланированные задачи (Cron) | Автономная задача, можно использовать другую модель |
| Проверять почту каждые 30 минут | Heartbeat | Пакетная обработка вместе с другими проверками, учёт контекста |
| Отслеживать календарь на предмет предстоящих событий | Heartbeat | Естественная пригодность для периодического мониторинга |
| Проверять статус субагента или выполнения ACP | Фоновые задачи | Журнал задач отслеживает всю отсоединённую работу |
| Проводить аудит того, что и когда выполнялось | Фоновые задачи | `openclaw tasks list` и `openclaw tasks audit` |
| Провести многоэтапное исследование, затем составить резюме | Поток задач (Task Flow) | Надёжная оркестрация с отслеживанием версий |
| Запускать скрипт при сбросе сессии | Хуки | Управление событиями, срабатывание при событиях жизненного цикла |
| Выполнять код при каждом вызове инструмента | Хуки | Хуки могут фильтровать по типу события |
| Всегда проверять соответствие требованиям перед ответом | Постоянные инструкции (Standing Orders) | Автоматически внедряются в каждую сессию |

### Запланированные задачи (Cron) против Heartbeat

| Параметр | Запланированные задачи (Cron) | Heartbeat |
| --------------- | ----------------------------------- | ------------------------------------- |
| Время выполнения | Точное (выражения cron, однократное выполнение) | Приблизительное (по умолчанию каждые 30 минут) |
| Контекст сессии | Свежий (изолированный) или общий | Полный контекст основной сессии |
| Записи о задачах | Всегда создаются | Никогда не создаются |
| Доставка | Канал, вебхук или тихо | Встроенно в основную сессию |
| Лучше всего подходит для | Отчётов, напоминаний, фоновых заданий | Проверки почты, календаря, уведомлений |

Используйте запланированные задачи (Cron), когда вам нужно точное время выполнения или изолированное выполнение. Используйте Heartbeat, когда работа выигрывает от полного контекста сессии и приблизительное время выполнения допустимо.

## Основные понятия

### Запланированные задачи (cron)

Cron — это встроенный в шлюз планировщик для точного времени выполнения. Он сохраняет задания, пробуждает агента в нужное время и может доставлять результаты в чат-канал или на конечную точку вебхука. Поддерживает однократные напоминания, повторяющиеся выражения и входящие триггеры вебхука.

См. [Запланированные задачи](/automation/cron-jobs).

### Задачи

Журнал фоновых задач отслеживает всю отсоединённую работу: выполнения ACP, создание субагентов, изолированные выполнения cron и операции CLI. Задачи — это записи, а не планировщики. Используйте `openclaw tasks list` и `openclaw tasks audit` для их проверки.

См. [Фоновые задачи](/automation/tasks).

### Поток задач (Task Flow)

Поток задач (Task Flow) — это субстрат для оркестрации потоков над фоновыми задачами. Он управляет надёжными многоэтапными потоками с управляемыми и зеркальными режимами синхронизации, отслеживанием версий и командами `openclaw tasks flow list|show|cancel` для проверки.

См. [Поток задач (Task Flow)](/automation/taskflow).

### Постоянные инструкции (Standing Orders)

Постоянные инструкции предоставляют агенту постоянные полномочия для выполнения определённых программ. Они хранятся в файлах рабочей области (обычно в `AGENTS.md`) и внедряются в каждую сессию. Сочетайте с cron для обеспечения выполнения по времени.

См. [Постоянные инструкции (Standing Orders)](/automation/standing-orders).

### Хуки (Hooks)

Хуки — это скрипты, управляемые событиями, которые запускаются при событиях жизненного цикла агента (`/new`, `/reset`, `/stop`), сжатии сессии, запуске шлюза, потоке сообщений и вызовах инструментов. Хуки автоматически обнаруживаются в каталогах и могут управляться с помощью `openclaw hooks`.

См. [Хуки (Hooks)](/automation/hooks).

### Heartbeat

Heartbeat — это периодический ход основной сессии (по умолчанию каждые 30 минут). Он объединяет несколько проверок (почта, календарь, уведомления) в один ход агента с полным контекстом сессии. Ходы Heartbeat не создают записей о задачах. Используйте `HEARTBEAT.md` для небольшого чек-листа или блок `tasks:`, если вы хотите выполнять периодические проверки только по необходимости внутри самого Heartbeat. Пустые файлы Heartbeat пропускаются как `empty-heartbeat-file`; режим задач только по необходимости пропускается как `no-tasks-due`.

См. [Heartbeat](/gateway/heartbeat).

## Как они работают вместе

- **Cron** обрабатывает точные расписания (ежедневные отчёты, еженедельные обзоры) и однократные напоминания. Все выполнения cron создают записи о задачах.
- **Heartbeat** обрабатывает рутинный мониторинг (почта, календарь, уведомления) в одном пакетном ходе каждые 30 минут.
- **Хуки** реагируют на конкретные события (вызовы инструментов, сброс сессии, сжатие) с помощью пользовательских скриптов.
- **Постоянные инструкции** предоставляют агенту постоянный контекст и границы полномочий.
- **Поток задач (Task Flow)** координирует многоэтапные потоки над отдельными задачами.
- **Задачи** автоматически отслеживают всю отсоединённую работу, чтобы вы могли её проверять и проводить аудит.
