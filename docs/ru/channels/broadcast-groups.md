# Группы рассылки

**Статус:** Экспериментальная  
**Версия:** Добавлена в 2026.1.9

## Обзор

Группы рассылки позволяют нескольким агентам одновременно обрабатывать одно и то же сообщение и отвечать на него. Это даёт возможность создавать специализированные команды агентов, которые работают вместе в одной группе WhatsApp или в личных сообщениях — используя один номер телефона.

Текущая область применения: **только WhatsApp** (веб-канал).

Группы рассылки обрабатываются после списков разрешённых каналов и правил активации групп. В группах WhatsApp это означает, что рассылка происходит, когда OpenClaw обычно отвечает (например, при упоминании, в зависимости от настроек вашей группы).

## Варианты использования

### 1. Специализированные команды агентов

Разверните несколько агентов с узконаправленными обязанностями:

```
Группа: "Команда разработки"
Агенты:
  - CodeReviewer (проверяет фрагменты кода)
  - DocumentationBot (генерирует документацию)
  - SecurityAuditor (проверяет на уязвимости)
  - TestGenerator (предлагает тестовые сценарии)
```

Каждый агент обрабатывает одно и то же сообщение и предоставляет свою специализированную точку зрения.

### 2. Поддержка нескольких языков

```
Группа: "Международная поддержка"
Агенты:
  - Agent_EN (отвечает на английском)
  - Agent_DE (отвечает на немецком)
  - Agent_ES (отвечает на испанском)
```

### 3. Рабочие процессы обеспечения качества

```
Группа: "Поддержка клиентов"
Агенты:
  - SupportAgent (даёт ответ)
  - QAAgent (проверяет качество, отвечает только при обнаружении проблем)
```

### 4. Автоматизация задач

```
Группа: "Управление проектами"
Агенты:
  - TaskTracker (обновляет базу данных задач)
  - TimeLogger (фиксирует затраченное время)
  - ReportGenerator (создаёт сводки)
```

## Настройка

### Базовая настройка

Добавьте раздел верхнего уровня `broadcast` (рядом с `bindings`). Ключи — идентификаторы участников WhatsApp:

- групповые чаты: JID группы (например, `120363403215116621@g.us`);
- личные сообщения: номер телефона в формате E.164 (например, `+15551234567`).

```json
{
  "broadcast": {
    "120363403215116621@g.us": ["alfred", "baerbel", "assistant3"]
  }
}
```

**Результат:** когда OpenClaw должен ответить в этом чате, он запустит всех трёх агентов.

### Стратегия обработки

Управляйте тем, как агенты обрабатывают сообщения:

#### Параллельно (по умолчанию)

Все агенты обрабатывают сообщения одновременно:

```json
{
  "broadcast": {
    "strategy": "parallel",
    "120363403215116621@g.us": ["alfred", "baerbel"]
  }
}
```

#### Последовательно

Агенты обрабатывают сообщения по порядку (каждый ждёт завершения предыдущего):

```json
{
  "broadcast": {
    "strategy": "sequential",
    "120363403215116621@g.us": ["alfred", "baerbel"]
  }
}
```

### Полный пример

```json
{
  "agents": {
    "list": [
      {
        "id": "code-reviewer",
        "name": "Code Reviewer",
        "workspace": "/path/to/code-reviewer",
        "sandbox": { "mode": "all" }
      },
      {
        "id": "security-auditor",
        "name": "Security Auditor",
        "workspace": "/path/to/security-auditor",
        "sandbox": { "mode": "all" }
      },
      {
        "id": "docs-generator",
        "name": "Documentation Generator",
        "workspace": "/path/to/docs-generator",
        "sandbox": { "mode": "all" }
      }
    ]
  },
  "broadcast": {
    "strategy": "parallel",
    "120363403215116621@g.us": ["code-reviewer", "security-auditor", "docs-generator"],
    "120363424282127706@g.us": ["support-en", "support-de"],
    "+15555550123": ["assistant", "logger"]
  }
}
```

## Как это работает

### Поток сообщений

1. **Входящее сообщение** поступает в группу WhatsApp.
2. **Проверка на рассылку:** система проверяет, есть ли идентификатор участника в `broadcast`.
3. **Если идентификатор есть в списке рассылки:**
   - все указанные агенты обрабатывают сообщение;
   - у каждого агента есть свой ключ сессии и изолированный контекст;
   - агенты обрабатывают сообщение параллельно (по умолчанию) или последовательно.
4. **Если идентификатора нет в списке рассылки:**
   - применяется обычная маршрутизация (первое подходящее binding).

Примечание: группы рассылки не обходят списки разрешённых каналов или правила активации групп (упоминания, команды и т. д.). Они лишь меняют то, **какие агенты запускаются**, когда сообщение подходит для обработки.

### Изоляция сессий

Каждый агент в группе рассылки имеет полностью отдельные:

- **Ключи сессии** (`agent:alfred:whatsapp:group:120363...` против `agent:baerbel:whatsapp:group:120363...`);
- **Историю переписки** (агент не видит сообщения других агентов);
- **Рабочее пространство** (отдельные песочницы, если они настроены);
- **Доступ к инструментам** (разные списки разрешений/запретов);
- **Память/контекст** (отдельные IDENTITY.md, SOUL.md и т. д.);
- **Буфер контекста группы** (последние сообщения группы, используемые для контекста) — общий для всех агентов в рамках одного участника, поэтому все агенты в группе рассылки видят один и тот же контекст при запуске.

Это позволяет каждому агенту иметь:

- разные "личности";
- разный доступ к инструментам (например, только для чтения против чтения и записи);
- разные модели (например, opus против sonnet);
- разные установленные навыки.

### Пример: изолированные сессии

В группе `120363403215116621@g.us` с агентами `["alfred", "baerbel"]`:

**Контекст Альфреда:**

```
Session: agent:alfred:whatsapp:group:120363403215116621@g.us
History: [сообщение пользователя, предыдущие ответы Альфреда]
Workspace: /Users/user/openclaw-alfred/
Tools: read, write, exec
```

**Контекст Бэрбель:**

```
Session: agent:baerbel:whatsapp:group:120363403215116621@g.us
History: [сообщение пользователя, предыдущие ответы Бэрбель]
Workspace: /Users/user/openclaw-baerbel/
Tools: read only
```

## Лучшие практики

### 1. Сохраняйте узкую специализацию агентов

Проектируйте каждого агента с одной чёткой обязанностью:

```json
{
  "broadcast": {
    "DEV_GROUP": ["formatter", "linter", "tester"]
  }
}
```

✅ **Хорошо:** у каждого агента одна задача  
❌ **Плохо:** один универсальный агент "dev-helper"

### 2. Используйте описательные имена

Сделайте так, чтобы было понятно, чем занимается каждый агент:

```json
{
  "agents": {
    "security-scanner": { "name": "Security Scanner" },
    "code-formatter": { "name": "Code Formatter" },
    "test-generator": { "name": "Test Generator" }
  }
}
```

### 3. Настройте разный доступ к инструментам

Предоставляйте агентам только те инструменты, которые им нужны:

```json
{
  "agents": {
    "reviewer": {
      "tools": { "allow": ["read", "exec"] } // Только для чтения
    },
    "fixer": {
      "tools": { "allow": ["read", "write", "edit", "exec"] } // Чтение и запись
    }
  }
}
