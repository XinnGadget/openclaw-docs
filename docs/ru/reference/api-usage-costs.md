# Использование API и затраты

В этом документе перечислены **функции, которые могут использовать ключи API**, а также указано, где отображаются соответствующие затраты. Основное внимание уделено функциям OpenClaw, которые могут приводить к использованию ресурсов провайдера или вызовам платных API.

## Где отображаются затраты (чат + CLI)

**Снимок затрат за сеанс**

- Команда `/status` показывает текущую модель сеанса, использование контекста и токены последнего ответа.
- Если модель использует **аутентификацию по ключу API**, команда `/status` также отображает **ориентировочную стоимость** последнего ответа.
- Если метаданные активного сеанса скудны, команда `/status` может восстановить счётчики токенов/кэша и метку активной модели среды выполнения из последней записи об использовании транскрипции. Существующие ненулевые значения в активном сеансе имеют приоритет, а суммарные значения транскрипции, соответствующие размеру промпта, могут быть использованы, если сохранённые суммы отсутствуют или меньше.

**Футер с затратами за сообщение**

- Команда `/usage full` добавляет футер с информацией об использовании к каждому ответу, включая **ориентировочную стоимость** (только для ключей API).
- Команда `/usage tokens` показывает только токены; при использовании OAuth/токенов по подписке и потоков CLI стоимость в долларах скрывается.
- Примечание для Gemini CLI: когда CLI возвращает вывод в формате JSON, OpenClaw считывает данные об использовании из `stats`, нормализует `stats.cached` в `cacheRead` и при необходимости выводит входные токены из `stats.input_tokens - stats.cached`.

Примечание для Anthropic: сотрудники Anthropic сообщили, что использование CLI Claude в стиле OpenClaw снова разрешено, поэтому OpenClaw считает повторное использование CLI Claude и `claude -p` допустимым для этой интеграции, если Anthropic не опубликует новую политику. При этом Anthropic по-прежнему не предоставляет оценку стоимости за сообщение в долларах, которую OpenClaw мог бы отобразить в `/usage full`.

**Окна использования в CLI (квоты провайдера)**

- Команды `openclaw status --usage` и `openclaw channels list` показывают **окна использования провайдера** (снимки квот, а не затраты за сообщение).
- Человекочитаемый вывод нормализуется до формата `X% осталось` для всех провайдеров.
- Провайдеры, поддерживающие окна использования: Anthropic, GitHub Copilot, Gemini CLI, OpenAI Codex, MiniMax, Xiaomi и z.ai.
- Примечание для MiniMax: необработанные поля `usage_percent` / `usagePercent` означают оставшуюся квоту, поэтому OpenClaw инвертирует их перед отображением. Поля, основанные на подсчёте, имеют приоритет при их наличии. Если провайдер возвращает `model_remains`, OpenClaw отдаёт предпочтение записи о чат-модели, при необходимости выводит метку окна из временных меток и включает название модели в метку плана.
- Аутентификация использования для этих окон квот осуществляется через хуки, специфичные для провайдера, если они доступны; в противном случае OpenClaw использует учётные данные OAuth/ключа API из профилей аутентификации, переменных окружения или конфигурации.

Подробнее см. в разделе [Использование токенов и затраты](/reference/token-use).

## Как обнаруживаются ключи

OpenClaw может получать учётные данные из следующих источников:

- **Профили аутентификации** (для каждого агента, хранятся в `auth-profiles.json`).
- **Переменные окружения** (например, `OPENAI_API_KEY`, `BRAVE_API_KEY`, `FIRECRAWL_API_KEY`).
- **Конфигурация** (`models.providers.*.apiKey`, `plugins.entries.*.config.webSearch.apiKey`, `plugins.entries.firecrawl.config.webFetch.apiKey`, `memorySearch.*`, `talk.providers.*.apiKey`).
- **Навыки** (`skills.entries.<name>.apiKey`), которые могут экспортировать ключи в окружение процесса навыка.

## Функции, которые могут расходовать ключи

### 1) Ответы основной модели (чат + инструменты)

Каждый ответ или вызов инструмента использует **текущего провайдера модели** (OpenAI, Anthropic и т. д.). Это основной источник использования и затрат.

Сюда также входят провайдеры с подпиской, которые выставляют счета вне локального интерфейса OpenClaw, например **OpenAI Codex**, **Alibaba Cloud Model Studio Coding Plan**, **MiniMax Coding Plan**, **Z.AI / GLM Coding Plan** и путь входа в Claude через OpenClaw от Anthropic с включённой опцией **Extra Usage**.

См. [Модели](/providers/models) для конфигурации цен и [Использование токенов и затраты](/reference/token-use) для отображения информации.

### 2) Понимание медиа (аудио/изображение/видео)

Входящие медиафайлы могут быть суммированы/транскрибированы до выполнения ответа. Для этого используются API модели/провайдера.

- Аудио: OpenAI / Groq / Deepgram / Google / Mistral.
- Изображения: OpenAI / OpenRouter / Anthropic / Google / MiniMax / Moonshot / Qwen / Z.AI.
- Видео: Google / Qwen / Moonshot.

См. [Понимание медиа](/nodes/media-understanding).

### 3) Генерация изображений и видео

Общие возможности генерации также могут расходовать ключи провайдера:

- Генерация изображений: OpenAI / Google / fal / MiniMax.
- Генерация видео: Qwen.

При генерации изображений может быть выбран провайдер по умолчанию с аутентификацией, если `agents.defaults.imageGenerationModel` не задан. Для генерации видео в настоящее время требуется явное указание `agents.defaults.videoGenerationModel`, например `qwen/wan2.6-t2v`.

См. [Генерация изображений](/tools/image-generation), [Qwen Cloud](/providers/qwen) и [Модели](/concepts/models).

### 4) Встраивания памяти + семантический поиск

Семантический поиск в памяти использует **API встраиваний**, если настроен для удалённых провайдеров:

- `memorySearch.provider = "openai"` → встраивания OpenAI.
- `memorySearch.provider = "gemini"` → встраивания Gemini.
- `memorySearch.provider = "voyage"` → встраивания Voyage.
- `memorySearch.provider = "mistral"` → встраивания Mistral.
- `memorySearch.provider = "ollama"` → встраивания Ollama (локальные/саморазмещённые; обычно без выставления счетов за API).
- Опциональный переход на удалённого провайдера, если локальные встраивания не сработали.

Вы можете оставить всё локальным, установив `memorySearch.provider = "local"` (без использования API).

См. [Память](/concepts/memory).

### 5) Инструмент веб-поиска

`web_search` может приводить к затратам в зависимости от вашего провайдера:

- **Brave Search API**: `BRAVE_API_KEY` или `plugins.entries.brave.config.webSearch.apiKey`.
- **Exa**: `EXA_API_KEY` или `plugins.entries.exa.config.webSearch.apiKey`.
- **Firecrawl**: `FIRECRAWL_API_KEY` или `plugins.entries.firecrawl.config.webSearch.apiKey`.
- **Gemini (Google Search)**: `GEMINI_API_KEY` или `plugins.entries.google.config.webSearch.apiKey`.
- **Grok (xAI)**: `XAI_API_KEY` или `plugins.entries.xai.config.webSearch.apiKey`.
- **Kimi (Moonshot)**: `KIMI_API_KEY`, `MOONSHOT_API_KEY` или `plugins.entries.moonshot.config.webSearch.apiKey`.
- **MiniMax Search**: `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY`, `MINIMAX_API_KEY` или `plugins.entries.minimax.config.webSearch.apiKey`.
- **Ollama Web Search**: по умолчанию не требует ключа, но требует доступного хоста Ollama плюс `ollama signin`; также может повторно использовать обычную аутентификацию bearer для провайдера Ollama, если хост этого требует.
- **Perplexity Search API**: `PERPLEXITY_API_KEY`, `OPENROUTER_API_KEY` или `plugins.entries.perplexity.config.webSearch.apiKey`.
- **Tavily**: `TAVILY_API_KEY` или `plugins.entries.tavily.config.webSearch.apiKey`.
- **DuckDuckGo**: запасной вариант без ключа (без выставления счетов за API, но неофициально и на основе HTML).
