# Подпись для macOS (отладочные сборки)

Данное приложение обычно собирается с помощью скрипта [`scripts/package-mac-app.sh`](https://github.com/openclaw/openclaw/blob/main/scripts/package-mac-app.sh), который сейчас:

- задаёт стабильный идентификатор отладочного бандла: `ai.openclaw.mac.debug`;
- записывает Info.plist с этим идентификатором бандла (можно переопределить через `BUNDLE_ID=...`);
- вызывает скрипт [`scripts/codesign-mac-app.sh`](https://github.com/openclaw/openclaw/blob/main/scripts/codesign-mac-app.sh) для подписи основного бинарного файла и бандла приложения, чтобы macOS воспринимала каждую пересборку как один и тот же подписанный бандл и сохраняла разрешения TCC (уведомления, доступность, запись экрана, микрофон, речь). Для стабильных разрешений используйте реальный идентификатор подписи; ad‑hoc — по желанию и ненадёжен (см. [разрешения macOS](/platforms/mac/permissions));
- по умолчанию использует `CODESIGN_TIMESTAMP=auto` — это включает доверенные временные метки для подписей Developer ID. Установите `CODESIGN_TIMESTAMP=off`, чтобы пропустить добавление временной метки (для отладочных сборок в офлайн‑режиме);
- внедряет метаданные сборки в Info.plist: `OpenClawBuildTimestamp` (UTC) и `OpenClawGitCommit` (короткий хеш), чтобы в разделе "О программе" отображались данные о сборке, Git и канале отладки/релиза;
- **по умолчанию используется Node 24**: скрипт запускает сборки TS и сборку Control UI. Node 22 LTS (в настоящее время `22.14+`) по‑прежнему поддерживается для обеспечения совместимости;
- считывает `SIGN_IDENTITY` из окружения. Добавьте `export SIGN_IDENTITY="Apple Development: Ваше Имя (TEAMID)"` (или сертификат Developer ID Application) в ваш shell rc, чтобы всегда подписывать с использованием вашего сертификата. Для ad‑hoc подписи требуется явное согласие через `ALLOW_ADHOC_SIGNING=1` или `SIGN_IDENTITY="-"` (не рекомендуется для тестирования разрешений);
- выполняет проверку Team ID после подписи и выдаёт ошибку, если какой‑либо Mach‑O внутри бандла приложения подписан другим Team ID. Установите `SKIP_TEAM_ID_CHECK=1`, чтобы обойти проверку.

## Использование

```bash
# из корня репозитория
scripts/package-mac-app.sh               # автоматически выбирает идентификатор; выдаёт ошибку, если не найден
SIGN_IDENTITY="Developer ID Application: Ваше Имя" scripts/package-mac-app.sh   # реальный сертификат
ALLOW_ADHOC_SIGNING=1 scripts/package-mac-app.sh    # ad‑hoc (разрешения не сохранятся)
SIGN_IDENTITY="-" scripts/package-mac-app.sh        # явная ad‑hoc подпись (с теми же ограничениями)
DISABLE_LIBRARY_VALIDATION=1 scripts/package-mac-app.sh   # только для разработки — обходной путь для несоответствия Team ID в Sparkle
```

### Примечание о ad‑hoc подписи

При подписи с `SIGN_IDENTITY="-"` (ad‑hoc) скрипт автоматически отключает **Hardened Runtime** (`--options runtime`). Это необходимо для предотвращения сбоев, когда приложение пытается загрузить встроенные фреймворки (например, Sparkle), которые не имеют того же Team ID. Ad‑hoc подписи также нарушают сохранение разрешений TCC; см. [разрешения macOS](/platforms/mac/permissions) для шагов по восстановлению.

## Метаданные сборки для раздела "О программе"

`package-mac-app.sh` добавляет в бандл следующие данные:

- `OpenClawBuildTimestamp`: ISO8601 UTC на момент упаковки;
- `OpenClawGitCommit`: короткий хеш Git (или `unknown`, если недоступен).

Раздел "О программе" считывает эти ключи для отображения версии, даты сборки, хеша Git и информации о том, является ли сборка отладочной (через `#if DEBUG`). Запустите упаковщик, чтобы обновить эти значения после внесения изменений в код.

## Почему это необходимо

Разрешения TCC привязаны к идентификатору бандла _и_ подписи кода. Незаписанные отладочные сборки с изменяющимися UUID приводили к тому, что macOS забывала предоставленные разрешения после каждой пересборки. Подпись бинарных файлов (по умолчанию — ad‑hoc) и сохранение фиксированного идентификатора/пути бандла (`dist/OpenClaw.app`) позволяет сохранять разрешения между сборками, что соответствует подходу VibeTunnel.
