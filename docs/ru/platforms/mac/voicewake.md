# Пробуждение по голосу и push-to-talk

## Режимы

- **Режим пробуждения по ключевому слову** (по умолчанию): постоянно активный распознаватель речи ожидает триггерные токены (`swabbleTriggerWords`). При совпадении начинается запись, отображается оверлей с частичным текстом, после паузы отправка выполняется автоматически.
- **Push-to-talk (удержание правой клавиши Option)**: удерживайте правую клавишу Option, чтобы немедленно начать запись — триггер не требуется. Оверлей отображается во время удержания; при отпускании клавиша завершает запись и отправляет её после небольшой задержки, чтобы вы могли скорректировать текст.

## Поведение во время выполнения (пробуждение по ключевому слову)

- Распознаватель речи находится в `VoiceWakeRuntime`.
- Триггер срабатывает только при наличии **значимой паузы** между ключевым словом и следующим словом (интервал около 0,55 с). Оверлей/звуковой сигнал может появиться во время паузы, ещё до начала команды.
- Интервалы тишины: 2,0 с при непрерывной речи, 5,0 с, если было услышано только ключевое слово.
- Жёсткое ограничение по времени: 120 с для предотвращения бесконтрольного продолжения сессии.
- Дебаунсинг между сессиями: 350 мс.
- Оверлей управляется через `VoiceWakeOverlayController` с использованием цветов для зафиксированных и временных данных.
- После отправки распознаватель корректно перезапускается, чтобы ожидать следующий триггер.

## Инварианты жизненного цикла

- Если функция пробуждения по голосу включена и разрешения предоставлены, распознаватель ключевых слов должен слушать (за исключением явной записи в режиме push-to-talk).
- Видимость оверлея (в том числе ручное закрытие с помощью кнопки X) ни в коем случае не должна препятствовать возобновлению работы распознавателя.

## Режим сбоя с "зависшим" оверлеем (предыдущий)

Ранее, если оверлей "зависал" в видимом состоянии и вы закрывали его вручную, функция пробуждения по голосу могла казаться "неработоспособной", поскольку попытка перезапуска во время выполнения могла быть заблокирована из‑за видимости оверлея, а последующий перезапуск не планировался.

Усовершенствования:

- Перезапуск во время выполнения больше не блокируется видимостью оверлея.
- Завершение закрытия оверлея запускает `VoiceWakeRuntime.refresh(...)` через `VoiceSessionCoordinator`, поэтому ручное закрытие с помощью кнопки X всегда возобновляет прослушивание.

## Особенности push-to-talk

- Обнаружение горячей клавиши использует глобальный монитор `.flagsChanged` для **правой клавиши Option** (`keyCode 61` + `.option`). Мы только отслеживаем события (не перехватываем их).
- Конвейер записи находится в `VoicePushToTalk`: немедленно запускает распознавание речи, передаёт частичные данные в оверлей и вызывает `VoiceWakeForwarder` при отпускании клавиши.
- При запуске push-to-talk режим пробуждения по ключевому слову приостанавливается, чтобы избежать конфликта аудиопотоков; он автоматически возобновляется после отпускания клавиши.
- Разрешения: требуются разрешения на использование микрофона и распознавания речи; для отслеживания событий требуется одобрение "Специальные возможности / Мониторинг ввода".
- Внешние клавиатуры: некоторые из них могут не предоставлять ожидаемую функциональность правой клавиши Option — предложите альтернативную горячую клавишу, если пользователи сообщают о проблемах.

## Настройки, доступные пользователю

- Переключатель **Voice Wake**: включает режим пробуждения по ключевому слову.
- **Удерживайте Cmd+Fn для разговора**: включает монитор push-to-talk. Отключён в macOS < 26.
- Выбор языка и микрофона, индикатор уровня сигнала в реальном времени, таблица ключевых слов, тестер (только локально; не выполняет пересылку).
- Выбор микрофона сохраняет последний выбор, если устройство отключается, отображает подсказку о разъединении и временно переключается на системный стандарт до возвращения устройства.
- **Звуки**: звуковые сигналы при обнаружении триггера и при отправке; по умолчанию используется системный звук macOS "Glass". Вы можете выбрать любой файл, загружаемый через `NSSound` (например, MP3/WAV/AIFF) для каждого события или выбрать **No Sound** ("Без звука").

## Поведение при пересылке

- Когда функция Voice Wake включена, транскрипты пересылаются активному шлюзу/агенту (используется тот же режим локального и удалённого взаимодействия, что и в остальной части приложения для macOS).
- Ответы доставляются **последнему использованному основному провайдеру** (WhatsApp/Telegram/Discord/WebChat). Если доставка завершается ошибкой, ошибка регистрируется, а сессия остаётся видимой через WebChat/журналы сессий.

## Полезная нагрузка при пересылке

- `VoiceWakeForwarder.prefixedTranscript(_:)` добавляет подсказку о машине перед отправкой. Используется как для режима пробуждения по ключевому слову, так и для push-to-talk.

## Быстрая проверка

- Включите push-to-talk, удерживайте Cmd+Fn, говорите, отпустите клавишу: оверлей должен отобразить частичные данные, а затем отправить их.
- Во время удержания значки "ушей" в строке меню должны оставаться увеличенными (используется `triggerVoiceEars(ttl:nil)`); они уменьшаются после отпускания клавиши.
