# Уровни мышления (директивы /think)

## Что это делает

- Встроенная директива в любом входящем теле сообщения: `/t <уровень>`, `/think:<уровень>` или `/thinking <уровень>`.
- Уровни (псевдонимы): `off | minimal | low | medium | high | xhigh | adaptive`:
  - minimal → "think";
  - low → "think hard";
  - medium → "think harder";
  - high → "ultrathink" (максимальный бюджет);
  - xhigh → "ultrathink+" (только для моделей GPT-5.2 и Codex);
  - adaptive → управляемый провайдером адаптивный бюджет рассуждений (поддерживается для семейства моделей Anthropic Claude 4.6);
  - `x-high`, `x_high`, `extra-high`, `extra high` и `extra_high` сопоставляются с `xhigh`;
  - `highest`, `max` сопоставляются с `high`.
- Примечания для провайдеров:
  - Модели Anthropic Claude 4.6 по умолчанию используют уровень `adaptive`, если явно не задан другой уровень мышления.
  - MiniMax (`minimax/*`) на пути потоковой передачи, совместимом с Anthropic, по умолчанию использует `thinking: { type: "disabled" }`, если вы явно не задаёте уровень мышления в параметрах модели или запроса. Это позволяет избежать утечки дельт `reasoning_content` из-за нестандартного формата потока Anthropic в MiniMax.
  - Z.AI (`zai/*`) поддерживает только бинарное мышление (`on`/`off`). Любой уровень, отличный от `off`, трактуется как `on` (сопоставляется с `low`).
  - Moonshot (`moonshot/*`) сопоставляет `/think off` с `thinking: { type: "disabled" }`, а любой уровень, отличный от `off`, — с `thinking: { type: "enabled" }`. Когда мышление включено, Moonshot принимает только `tool_choice` `auto|none`; OpenClaw нормализует несовместимые значения до `auto`.

## Порядок разрешения

1. Встроенная директива в сообщении (применяется только к этому сообщению).
2. Переопределение для сессии (задаётся отправкой сообщения, содержащего только директиву).
3. Значение по умолчанию для агента (`agents.list[].thinkingDefault` в конфигурации).
4. Глобальное значение по умолчанию (`agents.defaults.thinkingDefault` в конфигурации).
5. Резервный вариант: `adaptive` для моделей Anthropic Claude 4.6, `low` для других моделей с поддержкой рассуждений, `off` в остальных случаях.

## Установка значения по умолчанию для сессии

- Отправьте сообщение, которое содержит **только** директиву (допускается наличие пробелов), например `/think:medium` или `/t high`.
- Это значение сохраняется для текущей сессии (по умолчанию — для каждого отправителя); сбрасывается командой `/think:off` или при сбросе неактивной сессии.
- Отправляется подтверждающее сообщение (`Уровень мышления установлен на высокий.` / `Мышление отключено.`). Если уровень задан некорректно (например, `/thinking big`), команда отклоняется с подсказкой, а состояние сессии остаётся неизменным.
- Отправьте `/think` (или `/think:`) без аргументов, чтобы узнать текущий уровень мышления.

## Применение агентом

- **Embedded Pi**: разрешённый уровень передаётся во время выполнения агента Pi в процессе.

## Быстрый режим (/fast)

- Уровни: `on|off`.
- Сообщение, содержащее только директиву, переключает переопределение быстрого режима для сессии и возвращает ответ `Быстрый режим включён.` / `Быстрый режим отключён.`.
- Отправьте `/fast` (или `/fast status`) без указания режима, чтобы узнать текущее действующее состояние быстрого режима.
- OpenClaw разрешает быстрый режим в следующем порядке:
  1. Встроенная директива или сообщение, содержащее только `/fast on|off`.
  2. Переопределение для сессии.
  3. Значение по умолчанию для агента (`agents.list[].fastModeDefault`).
  4. Конфигурация для модели: `agents.defaults.models["<провайдер>/<модель>"].params.fastMode`.
  5. Резервный вариант: `off`.
- Для `openai/*` быстрый режим сопоставляется с приоритетной обработкой OpenAI путём отправки `service_tier=priority` в поддерживаемых запросах Responses.
- Для `openai-codex/*` быстрый режим отправляет тот же флаг `service_tier=priority` в Codex Responses. OpenClaw использует единый переключатель `/fast` для обоих путей аутентификации.
- Для прямых публичных запросов `anthropic/*`, включая трафик с OAuth-аутентификацией, отправляемый на `api.anthropic.com`, быстрый режим сопоставляется с уровнями обслуживания Anthropic: `/fast on` задаёт `service_tier=auto`, `/fast off` задаёт `service_tier=standard_only`.
- Для `minimax/*` на пути, совместимом с Anthropic, `/fast on` (или `params.fastMode: true`) преобразует `MiniMax-M2.7` в `MiniMax-M2.7-highspeed`.
- Явные параметры модели Anthropic `serviceTier` / `service_tier` переопределяют значение по умолчанию для быстрого режима, если оба параметра заданы. OpenClaw по-прежнему пропускает внедрение уровня обслуживания Anthropic для базовых URL-адресов прокси, не относящихся к Anthropic.

## Подробные директивы (/verbose или /v)

- Уровни: `on` (минимальный) | `full` | `off` (по умолчанию).
- Сообщение, содержащее только директиву, переключает подробный режим для сессии и возвращает ответ `Подробное логирование включено.` / `Подробное логирование отключено.`; при указании недопустимого уровня возвращается подсказка без изменения состояния.
- `/verbose off` сохраняет явное переопределение для сессии; его можно сбросить через интерфейс сессий, выбрав `наследовать`.
- Встроенная директива влияет только на это сообщение; в остальных случаях применяются значения по умолчанию для сессии/глобальные.
- Отправьте `/verbose` (или `/verbose:`) без аргументов, чтобы узнать текущий уровень подробного вывода.
- Если подробный режим включён, агенты, выдающие структурированные результаты инструментов (Pi и другие JSON-агенты), отправляют каждый вызов инструмента в виде отдельного сообщения только с метаданными, с префиксом `<эмодзи> <имя_инструмента>: <аргумент>`, если доступно (путь/команда). Эти сводки по инструментам отправляются сразу после запуска каждого инструмента (отдельные блоки), а не в виде потоковых дельт.
- Сводки о сбоях инструментов остаются видимыми в обычном режиме, но суффиксы с подробными сведениями об ошибках скрыты, если подробный режим не установлен в `on` или `full`.
- Если подробный режим установлен в `full`, выходные данные инструментов также пересылаются после завершения (отдельный блок, усечённый до безопасной длины). Если вы переключаете `/verbose on|full|off` во время выполнения задачи, последующие блоки с инструментами учитывают новую настройку.

## Видимость рассуждений (/reasoning)

- Уровни: `on|off|stream`.
- Сообщение, содержащее только директиву, переключает отображение блоков рассуждений в ответах.
- При включении рассуждения отправляются в виде **отдельного сообщения** с префиксом `Рассуждения:`.
- `stream` (только для Telegram): передаёт рассуждения в черновик сообщения Telegram во время генерации ответа, затем отправляет окончательный ответ без рассуждений.
- Псевдоним: `/reason`.
- Отправьте `/reasoning` (или `/reasoning:`) без аргументов, чтобы узнать текущий уровень видимости рассуждений.
- Порядок разрешения: встроенная директива, затем переопределение для сессии, затем значение по умолчанию для агента (`agents.list[].reasoningDefault`), затем резервный вариант (`off`).

## Пульсации (heartbeats)

- Тело запроса пульсации — это настроенная подсказка пульсации (по умолчанию: `Прочитайте HEARTBEAT.md, если он существует (контекст рабочей области). Строго следуйте ему. Не выводите предположения и не повторяйте старые задачи из предыдущих чатов. Если ничего не требует внимания, ответьте HEARTBEAT_OK.`). Встроенные директивы в сообщении пульсации применяются как обычно (но избегайте изменения значений по умолчанию для сессии из сообщений пульсации).
- По умолчанию доставка пульсации осуществляется только с окончательным пакетом данных. Чтобы также отправлять отдельное сообщение `Рассуждения:` (если доступно), установите `agents.defaults.heartbeat.includeReasoning: true` либо для агента — `agents.list[].heartbeat.includeReasoning: true`.

## Веб-интерфейс чата

- Селектор уровня мышления в веб-чате отражает сохранённый уровень сессии из хранилища/конфигурации сессии при загрузке страницы.
- Выбор другого уровня немедленно записывает переопределение для сессии через `sessions.patch`; он не ждёт следующей отправки и не является разовым переопределением `thinkingOnce`.
- Первый вариант всегда — `По умолчанию (<разрешённый уровень>)`, где разрешённый уровень по умолчанию берётся из активной модели сессии: `adaptive` для Claude 4.6 на Anthropic/Bedrock, `low` для других моделей с поддержкой рассуждений, `off` в остальных случаях.
- Селектор учитывает провайдера:
  - большинство провайдеров отображают `off | minimal | low | medium | high | adaptive`;
  - Z.AI отображает бинарные значения `off | on`.
- `/think:<уровень>` по-прежнему работает и обновляет тот же сохранённый уровень сессии, поэтому директивы чата и селектор остаются синхронизированными.
