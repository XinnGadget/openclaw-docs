# Системный промпт

OpenClaw формирует индивидуальный системный промпт для каждого запуска агента. Этот промпт **принадлежит OpenClaw** и не использует стандартный промпт pi-coding-agent.

Промпт формируется OpenClaw и внедряется в каждый запуск агента.

Плагины провайдера могут вносить рекомендации для промптов с учётом кэша, не заменяя при этом полный промпт, принадлежащий OpenClaw. Среда выполнения провайдера может:

- заменить небольшой набор именованных основных разделов (`interaction_style`, `tool_call_style`, `execution_bias`);
- внедрить **стабильный префикс** над границей кэша промптов;
- внедрить **динамический суффикс** под границей кэша промптов.

Используйте вклады, принадлежащие провайдеру, для настройки под конкретное семейство моделей. Сохраняйте устаревшую мутацию промптов `before_prompt_build` для обеспечения совместимости или внесения действительно глобальных изменений в промпт — это не относится к обычному поведению провайдера.

## Структура

Промпт намеренно компактен и использует фиксированные разделы:

- **Инструменты (Tooling)**: напоминание об источнике достоверной информации о структурированных инструментах, а также рекомендации по использованию инструментов во время выполнения.
- **Безопасность (Safety)**: краткое напоминание о необходимости избегать поведения, направленного на получение дополнительных полномочий, или обхода надзора.
- **Навыки (Skills)** (если доступны): указание модели, как загружать инструкции по навыкам по требованию.
- **Самообновление OpenClaw (OpenClaw Self-Update)**: как безопасно проверять конфигурацию с помощью `config.schema.lookup`, вносить изменения в конфигурацию с помощью `config.patch`, заменять полную конфигурацию с помощью `config.apply` и запускать `update.run` только по явному запросу пользователя. Инструмент `gateway`, доступный только владельцу, также отказывается перезаписывать `tools.exec.ask` / `tools.exec.security`, включая устаревшие псевдонимы `tools.bash.*`, которые нормализуются до этих защищённых путей выполнения.
- **Рабочая область (Workspace)**: рабочий каталог (`agents.defaults.workspace`).
- **Документация (Documentation)**: локальный путь к документации OpenClaw (репозиторий или npm-пакет) и указания, когда её следует читать.
- **Файлы рабочей области (внедряются) (Workspace Files (injected))**: указание на то, что ниже включены файлы начальной загрузки.
- **Песочница (Sandbox)** (если включена): указание на изолированную среду выполнения, пути в песочнице и наличие возможности выполнения с повышенными привилегиями.
- **Текущая дата и время (Current Date & Time)**: локальное время пользователя, часовой пояс и формат времени.
- **Теги ответов (Reply Tags)**: необязательный синтаксис тегов ответов для поддерживаемых провайдеров.
- **Пульсы (Heartbeats)**: промпт для пульса и поведение подтверждения, когда пульсация включена для агента по умолчанию.
- **Среда выполнения (Runtime)**: хост, ОС, узел, модель, корень репозитория (если обнаружен), уровень мышления (одна строка).
- **Рассуждения (Reasoning)**: текущий уровень видимости + подсказка по переключателю `/reasoning`.

Раздел "Инструменты" также содержит рекомендации по работе в течение длительного времени:

- используйте cron для последующих действий в будущем (`check back later`, напоминания, повторяющаяся работа) вместо циклов `exec` с ожиданием, приёмов с `yieldMs` или повторного опроса `process`;
- используйте `exec` / `process` только для команд, которые запускаются сейчас и продолжают выполняться в фоновом режиме;
- если включено автоматическое пробуждение по завершении, запустите команду один раз и полагайтесь на механизм пробуждения на основе push-уведомлений, когда команда выдаст результат или завершится с ошибкой;
- используйте `process` для журналов, статуса, ввода или вмешательства, если вам нужно проверить выполняющуюся команду;
- если задача объёмная, предпочтительнее использовать `sessions_spawn`; завершение работы суб-агента основано на push-уведомлениях и автоматически сообщает об этом инициатору запроса;
- не опрашивайте `subagents list` / `sessions_list` в цикле только для ожидания завершения.

Если включён экспериментальный инструмент `update_plan`, раздел "Инструменты" также указывает модели использовать его только для нетривиальной многоэтапной работы, сохранять ровно один шаг `in_progress` и избегать повторения всего плана после каждого обновления.

Ограничения безопасности в системном промпте носят рекомендательный характер. Они направляют поведение модели, но не обеспечивают принудительное соблюдение политики. Для жёсткого обеспечения соблюдения используйте политику инструментов, утверждения выполнения, песочницу и списки разрешённых каналов; операторы могут отключить их по своему усмотрению.

На каналах с нативными карточками/кнопками утверждения среда выполнения теперь указывает агенту в первую очередь полагаться на этот нативный интерфейс утверждения. Команда `/approve` должна включаться только в том случае, если результат инструмента указывает на отсутствие возможности утверждения в чате или если ручное утверждение — единственный путь.

## Режимы промптов

OpenClaw может формировать более компактные системные промпты для суб-агентов. Среда выполнения задаёт `promptMode` для каждого запуска (это не пользовательская конфигурация):

- `full` (по умолчанию): включает все перечисленные выше разделы.
- `minimal`: используется для суб-агентов; исключает разделы **Навыки (Skills)**, **Воспоминание о памяти (Memory Recall)**, **Самообновление OpenClaw (OpenClaw Self-Update)**, **Псевдонимы моделей (Model Aliases)**, **Идентификация пользователя (User Identity)**, **Теги ответов (Reply Tags)**, **Обмен сообщениями (Messaging)**, **Тихие ответы (Silent Replies)** и **Пульсы (Heartbeats)**. Разделы "Инструменты", "Безопасность", "Рабочая область", "Песочница", "Текущая дата и время" (если известно), "Среда выполнения" и внедренный контекст остаются доступными.
- `none`: возвращает только базовую строку идентификации.

При `promptMode=minimal` дополнительные внедряемые промпты помечаются как **Контекст суб-агента (Subagent Context)** вместо **Контекста группового чата (Group Chat Context)**.

## Внедрение начальной загрузки рабочей области

Файлы начальной загрузки обрезаются и добавляются в раздел **Контекст проекта (Project Context)**, чтобы модель видела контекст идентификации и профиля без необходимости явного чтения:

- `AGENTS.md`;
- `SOUL.md`;
- `TOOLS.md`;
- `IDENTITY.md`;
- `USER.md`;
- `HEARTBEAT.md`;
- `BOOTSTRAP.md` (только в совершенно новых рабочих областях);
- `MEMORY.md`, если присутствует, в противном случае — `memory.md` (в нижнем регистре в качестве запасного варианта).

Все эти файлы **внедряются в окно контекста** на каждом шаге, если не применяется шлюз для конкретного файла. `HEARTBEAT.md` исключается при обычных запусках, если пульсация отключена для агента по умолчанию или `agents.defaults.heartbeat.includeSystemPromptSection` имеет значение `false`. Старайтесь делать внедряемые файлы краткими — особенно `MEMORY.md`, который со временем может увеличиваться и приводить к неожиданно высокому использованию контекста и более частой компактификации.

> **Примечание:** ежедневные файлы `memory/*.md` **не** внедряются автоматически. Они доступны по требованию с помощью инструментов `memory_search` и `memory_get`, поэтому они не учитываются в окне контекста, если модель явно не читает их.

Большие файлы обрезаются с маркером. Максимальный размер файла задаётся параметром `agents.defaults.bootstrapMaxChars` (по умолчанию: 20 000). Общий объём внедряемого содержимого начальной загрузки по всем файлам ограничен параметром `agents.defaults.bootstrapTotalMaxChars` (по умолчанию: 150 000). Для отсутствующих файлов внедряется короткий маркер об отсутствии файла. При обрезке OpenClaw может внедрить блок предупреждения в "Контекст проекта"; это можно настроить с помощью `agents.defaults.bootstrapPromptTruncationWarning` (`off`, `once`, `always`; по умолчанию: `once`).

Сессии суб-агентов внедряют только `AGENTS.md` и `TOOLS.md` (другие файлы начальной загрузки отфильтровываются, чтобы сохранить небольшой контекст суб-агента).

Внутренние хуки могут перехватить этот шаг через `agent:bootstrap`, чтобы изменить или заменить внедряемые файлы начальной загрузки (например, заменить `SOUL.md` на альтернативную личность).

Если вы хотите, чтобы агент звучал менее шаблонно, начните с [Руководства по личности в SOUL.md](/concepts/soul).

Чтобы проверить, какой вклад вносит каждый внедряемый файл (исходный и внедряемый, обрезка, а также накладные расходы на схему инструментов), используйте `/context list` или `/context detail`. См. [Контекст](/concepts/context).

## Обработка времени

Системный промпт включает специальный раздел **Текущая дата и время (Current Date & Time)**, если известен часовой пояс пользователя. Чтобы сохранить стабильность кэша промптов, теперь в нём указывается только **часовой пояс** (без динамических часов или формата времени).
