# Песочница против политики инструментов против повышенных прав

В OpenClaw есть три связанных (но разных) механизма управления:

1. **Песочница** (`agents.defaults.sandbox.*` / `agents.list[].sandbox.*`) определяет, **где выполняются инструменты** (в Docker или на хосте).
2. **Политика инструментов** (`tools.*`, `tools.sandbox.tools.*`, `agents.list[].tools.*`) определяет, **какие инструменты доступны/разрешены**.
3. **Повышенные права** (`tools.elevated.*`, `agents.list[].tools.elevated.*`) — это **механизм выхода только для exec**, позволяющий выполнять действия вне песочницы, когда вы находитесь в песочнице (`gateway` по умолчанию или `node`, если цель exec настроена на `node`).

## Быстрая отладка

Используйте инспектор, чтобы увидеть, что на самом деле делает OpenClaw:

```bash
openclaw sandbox explain
openclaw sandbox explain --session agent:main:main
openclaw sandbox explain --agent work
openclaw sandbox explain --json
```

Он выводит:

- действующий режим/область действия/доступ к рабочей области песочницы;
- находится ли сессия в данный момент в песочнице (основная vs неосновная);
- действующие правила разрешения/запрета инструментов в песочнице (и откуда они взяты — из настроек агента, глобальных настроек или настроек по умолчанию);
- ворота с повышенными правами и пути к ключам для исправления.

## Песочница: где выполняются инструменты

Управление песочницей осуществляется через `agents.defaults.sandbox.mode`:

- `"off"`: всё выполняется на хосте.
- `"non-main"`: в песочнице находятся только неосновные сессии (часто становится неожиданностью для групп/каналов).
- `"all"`: всё находится в песочнице.

См. [Песочница](/gateway/sandboxing) для полной матрицы (область действия, монтирование рабочей области, образы).

### Примонтированные тома (быстрая проверка безопасности)

- `docker.binds` **проникает** в файловую систему песочницы: всё, что вы монтируете, будет видно внутри контейнера с заданным вами режимом (`:ro` или `:rw`).
- По умолчанию используется режим чтения и записи, если вы не указали режим; для исходных кодов/секретов предпочтительнее `:ro`.
- `scope: "shared"` игнорирует привязки для отдельных агентов (применяются только глобальные привязки).
- OpenClaw дважды проверяет источники привязок: сначала по нормализованному пути к источнику, затем после разрешения через самый глубокий существующий предок. Попытки выхода через родительские символические ссылки не обходят проверки заблокированных путей или разрешённых корневых каталогов.
- Несуществующие конечные пути также проверяются безопасно. Если `/workspace/alias-out/new-file` через родительскую символическую ссылку ведёт к заблокированному пути или находится за пределами настроенных разрешённых корневых каталогов, привязка будет отклонена.
- Привязка `/var/run/docker.sock` фактически передаёт управление хостом песочнице; делайте это только осознанно.
- Доступ к рабочей области (`workspaceAccess: "ro"`/`"rw"`) не зависит от режимов привязки.

## Политика инструментов: какие инструменты существуют/могут быть вызваны

Имеют значение два уровня:

- **Профиль инструмента**: `tools.profile` и `agents.list[].tools.profile` (базовый белый список);
- **Профиль инструмента провайдера**: `tools.byProvider[provider].profile` и `agents.list[].tools.byProvider[provider].profile`;
- **Глобальная политика инструментов/политика для отдельного агента**: `tools.allow`/`tools.deny` и `agents.list[].tools.allow`/`agents.list[].tools.deny`;
- **Политика инструментов провайдера**: `tools.byProvider[provider].allow/deny` и `agents.list[].tools.byProvider[provider].allow/deny`;
- **Политика инструментов песочницы** (применяется только в песочнице): `tools.sandbox.tools.allow`/`tools.sandbox.tools.deny` и `agents.list[].tools.sandbox.tools.*`.

Основные правила:

- `deny` всегда имеет приоритет.
- Если `allow` не пуст, всё остальное считается заблокированным.
- Политика инструментов — это жёсткое ограничение: `/exec` не может переопределить запрещённый инструмент `exec`.
- `/exec` только изменяет настройки по умолчанию для сессии для авторизованных отправителей; он не предоставляет доступ к инструментам.
- Ключи инструментов провайдера принимают либо `provider` (например, `google-antigravity`), либо `provider/model` (например, `openai/gpt-5.4`).

### Группы инструментов (сокращения)

Политики инструментов (глобальные, для агента, для песочницы) поддерживают записи `group:*`, которые расширяются до нескольких инструментов:

```json5
{
  tools: {
    sandbox: {
      tools: {
        allow: ["group:runtime", "group:fs", "group:sessions", "group:memory"],
      },
    },
  },
}
```

Доступные группы:

- `group:runtime`: `exec`, `process`, `code_execution` (`bash` принимается как псевдоним для `exec`);
- `group:fs`: `read`, `write`, `edit`, `apply_patch`;
- `group:sessions`: `sessions_list`, `sessions_history`, `sessions_send`, `sessions_spawn`, `sessions_yield`, `subagents`, `session_status`;
- `group:memory`: `memory_search`, `memory_get`;
- `group:web`: `web_search`, `x_search`, `web_fetch`;
- `group:ui`: `browser`, `canvas`;
- `group:automation`: `cron`, `gateway`;
- `group:messaging`: `message`;
- `group:nodes`: `nodes`;
- `group:agents`: `agents_list`;
- `group:media`: `image`, `image_generate`, `video_generate`, `tts`;
- `group:openclaw`: все встроенные инструменты OpenClaw (исключая плагины провайдера).

## Повышенные права: только для exec "выполнить на хосте"

Повышенные права **не** предоставляют дополнительные инструменты; они влияют только на `exec`.

- Если вы находитесь в песочнице, `/elevated on` (или `exec` с `elevated: true`) выполняется вне песочницы (могут по-прежнему применяться утверждения).
- Используйте `/elevated full`, чтобы пропустить утверждения для exec в рамках сессии.
- Если вы уже работаете напрямую, повышенные права фактически не действуют (всё ещё ограничены воротами).
- Повышенные права **не** привязаны к навыкам и **не** переопределяют правила разрешения/запрета инструментов.
- Повышенные права не предоставляют произвольные переопределения между хостами из `host=auto`; они следуют обычным правилам цели exec и сохраняют `node` только в том случае, если настроенная/сессионная цель уже является `node`.
- `/exec` отделён от повышенных прав. Он только корректирует настройки exec для сессии для авторизованных отправителей.

Ворота:

- Включение: `tools.elevated.enabled` (и, опционально, `agents.list[].tools.elevated.enabled`);
- Белые списки отправителей: `tools.elevated.allowFrom.<provider>` (и, опционально, `agents.list[].tools.elevated.allowFrom.<provider>`).

См. [Режим с повышенными правами](/tools/elevated).

## Распространённые способы устранения "изоляции в песочнице"

### "Инструмент X заблокирован политикой инструментов песочницы"

Ключи для исправления (выберите один):

- Отключить песочницу: `agents.defaults.sandbox.mode=off` (или для отдельного агента `agents.list[].sandbox.mode=off`);
- Разрешить инструмент в песочнице:
  - удалить его из `tools.sandbox.tools.deny` (или для отдельного агента `agents.list[].tools.sandbox.tools.deny`);
  - или добавить его в `tools.sandbox.tools.allow` (или в список разрешений для отдельного агента).

### "Я думал, что это основная сессия, почему она в песочнице?"

В режиме `"non-main"` ключи групп/каналов **не** являются основными. Используйте ключ основной сессии (отображается командой `sandbox explain`) или переключитесь в режим `"off"`.
