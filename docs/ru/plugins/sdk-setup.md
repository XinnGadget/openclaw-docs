# Настройка и конфигурация плагина

Справочник по упаковке плагина (`package.json` метаданные), манифестам (`openclaw.plugin.json`), точкам входа для настройки и схемам конфигурации.

<Tip>
  **Ищете пошаговое руководство?** В руководствах по работе описаны вопросы упаковки в контексте:
  [Плагины каналов](/plugins/sdk-channel-plugins#step-1-package-and-manifest) и
  [Плагины провайдеров](/plugins/sdk-provider-plugins#step-1-package-and-manifest).
</Tip>

## Метаданные пакета

В вашем `package.json` должно быть поле `openclaw`, которое сообщает системе плагинов, что предоставляет ваш плагин:

**Плагин канала:**

```json
{
  "name": "@myorg/openclaw-my-channel",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./index.ts"],
    "setupEntry": "./setup-entry.ts",
    "channel": {
      "id": "my-channel",
      "label": "My Channel",
      "blurb": "Краткое описание канала."
    }
  }
}
```

**Плагин провайдера / базовая конфигурация для публикации в ClawHub:**

```json openclaw-clawhub-package.json
{
  "name": "@myorg/openclaw-my-plugin",
  "version": "1.0.0",
  "type": "module",
  "openclaw": {
    "extensions": ["./index.ts"],
    "compat": {
      "pluginApi": ">=2026.3.24-beta.2",
      "minGatewayVersion": "2026.3.24-beta.2"
    },
    "build": {
      "openclawVersion": "2026.3.24-beta.2",
      "pluginSdkVersion": "2026.3.24-beta.2"
    }
  }
}
```

Если вы публикуете плагин во внешнем ClawHub, поля `compat` и `build` обязательны. Стандартные фрагменты для публикации находятся в `docs/snippets/plugin-publish/`.

### Поля `openclaw`

| Поле | Тип | Описание |
| ------------ | ---------- | ------------------------------------------------------------------------------------------------------ |
| `extensions` | `string[]` | Файлы точек входа (относительно корня пакета) |
| `setupEntry` | `string` | Лёгкая точка входа только для настройки (необязательно) |
| `channel` | `object` | Метаданные каталога каналов для настройки, выбора, быстрого старта и отображения состояния |
| `providers` | `string[]` | Идентификаторы провайдеров, зарегистрированные этим плагином |
| `install` | `object` | Подсказки по установке: `npmSpec`, `localPath`, `defaultChoice`, `minHostVersion`, `allowInvalidConfigRecovery` |
| `startup` | `object` | Флаги поведения при запуске |

### `openclaw.channel`

`openclaw.channel` — это простые метаданные пакета для обнаружения каналов и поверхностей настройки до загрузки во время выполнения.

| Поле | Тип | Значение |
| -------------------------------------- | ---------- | ----------------------------------------------------------------------------- |
| `id` | `string` | Канонический идентификатор канала. |
| `label` | `string` | Основной ярлык канала. |
| `selectionLabel` | `string` | Ярлык для выбора/настройки, если он должен отличаться от `label`. |
| `detailLabel` | `string` | Вторичный ярлык для более подробных каталогов каналов и поверхностей состояния. |
| `docsPath` | `string` | Путь к документации для ссылок по настройке и выбору. |
| `docsLabel` | `string` | Переопределённый ярлык, используемый для ссылок на документацию, если он должен отличаться от идентификатора канала. |
| `blurb` | `string` | Краткое описание для ознакомления/каталога. |
| `order` | `number` | Порядок сортировки в каталогах каналов. |
| `aliases` | `string[]` | Дополнительные псевдонимы для выбора канала. |
| `preferOver` | `string[]` | Идентификаторы плагинов/каналов с более низким приоритетом, которые этот канал должен превосходить. |
| `systemImage` | `string` | Необязательное имя иконки/системного изображения для каталогов UI каналов. |
| `selectionDocsPrefix` | `string` | Префикс текста перед ссылками на документацию на поверхностях выбора. |
| `selectionDocsOmitLabel` | `boolean` | Отображать путь к документации напрямую вместо помеченной ссылки на документацию в тексте выбора. |
| `selectionExtras` | `string[]` | Дополнительные короткие строки, добавляемые в текст выбора. |
| `markdownCapable` | `boolean` | Помечает канал как поддерживающий Markdown для решений по форматированию исходящих данных. |
| `exposure` | `object` | Управление видимостью канала для настройки, настроенных списков и поверхностей документации. |
| `quickstartAllowFrom` | `boolean` | Включить этот канал в стандартный поток настройки быстрого старта `allowFrom`. |
| `forceAccountBinding` | `boolean` | Требовать явного связывания учётной записи, даже если существует только одна учётная запись. |
| `preferSessionLookupForAnnounceTarget` | `boolean` | Предпочитать поиск сессии при разрешении целей объявления для этого канала. |

Пример:

```json
{
  "openclaw": {
    "channel": {
      "id": "my-channel",
      "label": "My Channel",
      "selectionLabel": "My Channel (self-hosted)",
      "detailLabel": "My Channel Bot",
      "docsPath": "/channels/my-channel",
      "docsLabel": "my-channel",
      "blurb": "Интеграция чата на основе вебхуков с самостоятельным размещением.",
      "order": 80,
      "aliases": ["mc"],
      "preferOver": ["my-channel-legacy"],
      "selectionDocsPrefix": "Guide:",
      "selectionExtras": ["Markdown"],
      "markdownCapable": true,
      "exposure": {
        "configured": true,
        "setup": true,
        "docs": true
      },
      "quickstartAllowFrom": true
    }
  }
}
```

`exposure` поддерживает:

- `configured`: включать канал в настроенные поверхности/поверхности со списком состояний;
- `setup`: включать канал в интерактивные средства выбора настройки/конфигурации;
- `docs`: помечать канал как общедоступный на поверхностях документации/навигации.

`showConfigured` и `showInSetup` по-прежнему поддерживаются как устаревшие псевдонимы. Предпочитайте `exposure`.

### `openclaw.install`

`openclaw.install` — это метаданные пакета, а не метаданные манифеста.

| Поле | Тип | Значение |
| ---------------------------- | -------------------- | -------------------------------------------------------------------------------- |
| `npmSpec` | `string` | Каноническая спецификация npm для потоков установки/обновления. |
| `localPath` | `string` | Локальный путь разработки или установки в комплекте. |
| `defaultChoice` | `"npm"` \| `"local"` | Предпочтительный источник установки, когда доступны оба варианта. |
| `minHostVersion` | `string` | Минимальная поддерживаемая версия OpenClaw в формате `>=x.y.z`. |
| `allowInvalidConfigRecovery` | `boolean` | Позволяет потокам переустановки плагинов в комплекте восстанавливать работу после определённых сбоев из-за устаревшей конфигурации. |

Если задано `minHostVersion`, установка и загрузка реестра манифестов применяют это требование. Хосты более старых версий пропускают плагин; недопустимые строки версий отклоняются.

`allowInvalidConfigRecovery` не является общим обходным путём для сломанных конфигураций. Он предназначен только для узкого восстановления плагинов в комплекте, чтобы переустановка/настройка могла исправить известные остатки обновления, такие как отсутствующий путь к плагину в комплекте или устаревшая запись `channels.<id>` для того же плагина. Если конфигурация сломана по несвязанным причинам, установка по-прежнему завершается с ошибкой, и оператору предлагается запустить `openclaw doctor --fix`.
