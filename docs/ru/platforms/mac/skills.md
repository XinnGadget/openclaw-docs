# Навыки (macOS)

Приложение macOS предоставляет навыки OpenClaw через шлюз; локальный разбор навыков не выполняется.

## Источник данных

- `skills.status` (шлюз) возвращает все навыки, а также информацию о соответствии требованиям и отсутствующих требованиях (включая блокирование по белому списку для встроенных навыков).
- Требования определяются на основе `metadata.openclaw.requires` в каждом файле `SKILL.md`.

## Действия по установке

- `metadata.openclaw.install` определяет варианты установки (brew/node/go/uv).
- Приложение вызывает `skills.install` для запуска установщиков на хосте шлюза.
- Встроенные обнаружения опасного кода (`critical`) по умолчанию блокируют `skills.install`; при обнаружении подозрительного кода выводится только предупреждение. Возможность переопределения опасного кода существует в запросе к шлюзу, но по умолчанию поток приложения остаётся в режиме "отказ при ошибке".
- Если все варианты установки указаны как `download`, шлюз предоставляет все варианты загрузки.
- В противном случае шлюз выбирает один предпочтительный установщик, используя текущие настройки установки и двоичные файлы хоста: сначала Homebrew, если включён `skills.install.preferBrew` и присутствует `brew`, затем `uv`, затем настроенный менеджер узлов из `skills.install.nodeManager`, а затем альтернативные варианты, такие как `go` или `download`.
- Метки установки Node отражают настроенный менеджер узлов, включая `yarn`.

## Переменные окружения / API-ключи

- Приложение хранит ключи в `~/.openclaw/openclaw.json` в разделе `skills.entries.<skillKey>`.
- `skills.update` обновляет значения `enabled`, `apiKey` и `env`.

## Удаленный режим

- Установка и обновление конфигурации выполняются на хосте шлюза (а не на локальном Mac).
