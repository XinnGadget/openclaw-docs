# Поддержка изображений и медиа (2025-12-05)

Канал WhatsApp работает через **Baileys Web**. В этом документе описаны текущие правила обработки медиа для отправки, шлюза и ответов агентов.

## Цели

- Отправлять медиа с необязательными подписями через `openclaw message send --media`.
- Разрешить автоответам из веб-входящих включать медиа вместе с текстом.
- Сохранять разумные и предсказуемые лимиты для каждого типа данных.

## Интерфейс командной строки (CLI)

- `openclaw message send --media <путь-или-url> [--message <подпись>]`
  - `--media` — необязательный параметр; подпись может быть пустой при отправке только медиа.
  - `--dry-run` выводит сформированный payload; `--json` выдаёт `{ channel, to, messageId, mediaUrl, caption }`.

## Поведение канала WhatsApp Web

- Входные данные: локальный путь к файлу **или** URL по протоколу HTTP(S).
- Процесс: загрузка в Buffer, определение типа медиа и формирование корректного payload:
  - **Изображения:** изменение размера и перекодировка в JPEG (максимальная сторона — 2048 пикселей) с учётом лимита `channels.whatsapp.mediaMaxMb` (по умолчанию: 50 МБ).
  - **Аудио/голосовые сообщения/видео:** передача без изменений до 16 МБ; аудио отправляется как голосовое сообщение (`ptt: true`).
  - **Документы:** любые другие файлы до 100 МБ, при наличии сохраняется имя файла.
- Воспроизведение в стиле GIF в WhatsApp: отправка MP4 с `gifPlayback: true` (в CLI: `--gif-playback`), чтобы мобильные клиенты воспроизводили видео в цикле inline.
- Определение MIME-типа: приоритет отдаётся магическим байтам, затем заголовкам, затем расширению файла.
- Подпись берётся из `--message` или `reply.text`; пустая подпись допускается.
- Логирование: в недетализированном режиме отображаются `↩️`/`✅`; в детализированном — размер и исходный путь/URL.

## Конвейер автоответов

- `getReplyFromConfig` возвращает `{ text?, mediaUrl?, mediaUrls? }`.
- Если присутствуют медиа, веб-отправитель разрешает локальные пути или URL с использованием того же конвейера, что и `openclaw message send`.
- Несколько медиаотправлений отправляются последовательно, если они предоставлены.

## Входящие медиа в команды (Pi)

- Когда входящие веб-сообщения содержат медиа, OpenClaw загружает их во временный файл и предоставляет переменные для шаблонизации:
  - `{{MediaUrl}}` — псевдо-URL для входящих медиа.
  - `{{MediaPath}}` — локальный временный путь, записанный перед выполнением команды.
- Если включена песочница Docker для сеанса, входящие медиа копируются в рабочее пространство песочницы, а `MediaPath`/`MediaUrl` переписываются в относительный путь, например `media/inbound/<имя_файла>`.
- Анализ медиа (если настроен через `tools.media.*` или общие `tools.media.models`) выполняется до шаблонизации и может вставлять блоки `[Image]`, `[Audio]` и `[Video]` в `Body`.
  - Для аудио устанавливается `{{Transcript}}`, и транскрипция используется для парсинга команд, чтобы команды со слешем продолжали работать.
  - Описания видео и изображений сохраняют текст подписи для парсинга команд.
  - Если активная основная модель для изображений изначально поддерживает зрение, OpenClaw пропускает блок сводки `[Image]` и передаёт исходную картинку модели.
- По умолчанию обрабатывается только первое подходящее вложение (изображение/аудио/видео); чтобы обработать несколько вложений, установите `tools.media.<cap>.attachments`.

## Лимиты и ошибки

**Лимиты на исходящую отправку (отправка через WhatsApp Web)**

- Изображения: до `channels.whatsapp.mediaMaxMb` (по умолчанию: 50 МБ) после перекодировки.
- Аудио/голосовые сообщения/видео: лимит 16 МБ; документы: лимит 100 МБ.
- Слишком большие или нечитаемые медиа → чёткая ошибка в логах, ответ пропускается.

**Лимиты на анализ медиа (транскрипция/описание)**

- Изображение (по умолчанию): 10 МБ (`tools.media.image.maxBytes`).
- Аудио (по умолчанию): 20 МБ (`tools.media.audio.maxBytes`).
- Видео (по умолчанию): 50 МБ (`tools.media.video.maxBytes`).
- Слишком большие медиа пропускаются при анализе, но ответы всё равно отправляются с исходным телом.

## Примечания для тестов

- Проверить потоки отправки и ответов для случаев с изображениями, аудио и документами.
- Убедиться в корректности перекодировки изображений (ограничение по размеру) и установки флага голосового сообщения для аудио.
- Убедиться, что ответы с несколькими медиа отправляются последовательно.
