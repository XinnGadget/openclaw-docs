# Использование токенов и затраты

OpenClaw отслеживает **токены**, а не символы. Токены зависят от модели, но в большинстве моделей в стиле OpenAI на английский текст в среднем приходится около 4 символов на токен.

## Как формируется системный промпт

OpenClaw формирует собственный системный промпт при каждом запуске. В него входят:

- Список инструментов + краткие описания
- Список навыков (только метаданные; инструкции загружаются по требованию с помощью `read`)
- Инструкции по самообновлению
- Файлы рабочей области и начальной загрузки (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md` — при создании, а также `MEMORY.md` — если присутствует, или `memory.md` в качестве запасного варианта в нижнем регистре). Большие файлы усекаются параметром `agents.defaults.bootstrapMaxChars` (по умолчанию: 20 000), а общий объём внедряемых данных начальной загрузки ограничен параметром `agents.defaults.bootstrapTotalMaxChars` (по умолчанию: 150 000). Файлы `memory/*.md` загружаются по требованию с помощью инструментов работы с памятью и не внедряются автоматически.
- Время (UTC + часовой пояс пользователя)
- Теги ответов + поведение при отправке сигналов активности (heartbeat)
- Метаданные среды выполнения (хост/ОС/модель/режим мышления)

Полный разбор см. в разделе [Системный промпт](/concepts/system-prompt).

## Что учитывается в окне контекста

Всё, что получает модель, учитывается в пределах лимита контекста:

- Системный промпт (все разделы, перечисленные выше)
- История диалога (сообщения пользователя и ассистента)
- Вызовы инструментов и результаты их работы
- Вложения/транскрипты (изображения, аудио, файлы)
- Сводные данные о сжатии и артефакты обрезки
- Обёртки провайдера или заголовки безопасности (не видны, но учитываются)

Для изображений OpenClaw уменьшает размер полезной нагрузки транскриптов/изображений инструментов перед вызовами провайдера. Используйте параметр `agents.defaults.imageMaxDimensionPx` (по умолчанию: `1200`), чтобы настроить это:

- Более низкие значения обычно снижают использование токенов для обработки изображений и размер полезной нагрузки.
- Более высокие значения сохраняют больше визуальных деталей для скриншотов с большим количеством текста или элементов интерфейса (OCR/UI).

Для практического разбора (по каждому внедряемому файлу, инструментам, навыкам и размеру системного промпта) используйте команды `/context list` или `/context detail`. См. раздел [Контекст](/concepts/context).

## Как посмотреть текущее использование токенов

Используйте следующие команды в чате:

- `/status` → **карточка состояния с эмодзи**, содержащая информацию о модели сессии, использовании контекста, количестве токенов на вход/выход в последнем ответе и **ориентировочной стоимости** (только для API-ключа).
- `/usage off|tokens|full` → добавляет **нижний колонтитул с данными об использовании для каждого ответа** к каждому ответу.
  - Сохраняется в рамках сессии (хранится как `responseUsage`).
  - При авторизации через OAuth **стоимость не отображается** (только токены).
- `/usage cost` → показывает локальную сводку затрат на основе журналов сессии OpenClaw.

Другие интерфейсы:

- **TUI/Web TUI:** поддерживаются команды `/status` и `/usage`.
- **CLI:** команды `openclaw status --usage` и `openclaw channels list` показывают нормализованные окна квот провайдера (`X% осталось`, без указания затрат для каждого ответа).
  Текущие провайдеры с окнами использования: Anthropic, GitHub Copilot, Gemini CLI, OpenAI Codex, MiniMax, Xiaomi и z.ai.

Интерфейсы использования нормализуют распространённые псевдонимы полей, специфичные для провайдера, перед отображением. Для трафика ответов семейства OpenAI это включает как `input_tokens` / `output_tokens`, так и `prompt_tokens` / `completion_tokens`, поэтому имена полей, специфичные для транспорта, не влияют на `/status`, `/usage` или сводки сессии.

JSON-данные об использовании в Gemini CLI также нормализуются: текст ответа берётся из `response`, а `stats.cached` сопоставляется с `cacheRead`, при этом `stats.input_tokens - stats.cached` используется, когда в CLI отсутствует явное поле `stats.input`.

Для нативного трафика ответов семейства OpenAI псевдонимы использования WebSocket/SSE нормализуются аналогичным образом, а итоговые значения возвращаются к нормализованным входным и выходным данным, когда `total_tokens` отсутствует или равен `0`.

Когда текущий снимок сессии неполный, `/status` и `session_status` также могут восстановить счётчики токенов/кэша и метку активной модели среды выполнения из самого последнего журнала использования транскриптов. Существующие ненулевые текущие значения имеют приоритет над резервными значениями из транскриптов, а более крупные итоговые значения, ориентированные на промпт, могут иметь приоритет, если сохранённые итоги отсутствуют или меньше.

Авторизация использования для окон квот провайдера осуществляется через специфические для провайдера хуки, если они доступны; в противном случае OpenClaw использует учётные данные OAuth/API-ключа из профилей авторизации, переменных окружения или конфигурации.

## Оценка затрат (когда отображается)

Затраты оцениваются на основе конфигурации цен вашей модели:

```
models.providers.<provider>.models[].cost
```

Это **доллары США за 1 млн токенов** для `input`, `output`, `cacheRead` и `cacheWrite`. Если цены отсутствуют, OpenClaw отображает только токены. Токены OAuth никогда не показывают стоимость в долларах.

## TTL кэша и влияние обрезки

Кэширование промпта провайдера применяется только в пределах окна TTL кэша. OpenClaw может при необходимости выполнять **обрезку по TTL кэша**: он обрезает сессию после истечения TTL кэша, затем сбрасывает окно кэша, чтобы последующие запросы могли повторно использовать свежезакешированный контекст вместо повторного кэширования всей истории. Это снижает затраты на запись в кэш, когда сессия простаивает после истечения TTL.

Настройте это в разделе [Конфигурация шлюза](/gateway/configuration) и ознакомьтесь с деталями поведения в разделе [Обрезка сессии](/concepts/session-pruning).

Сигнал активности (heartbeat) может поддерживать кэш **в активном состоянии** во время периодов простоя. Если TTL кэша вашей модели составляет `1 час`, установка интервала heartbeat чуть меньше этого значения (например, `55 минут`) может помочь избежать повторного кэширования полного промпта, снижая затраты на запись в кэш.

В настройках с несколькими агентами вы можете использовать одну общую конфигурацию модели и настраивать поведение кэша для каждого агента с помощью `agents.list[].params.cacheRetention`.

Полное руководство по настройкам см. в разделе [Кэширование промпта](/reference/prompt-caching).

Для ценообразования API Anthropic чтение из кэша значительно дешевле, чем входные токены, а запись в кэш тарифицируется с более высоким коэффициентом. Ознакомьтесь с актуальными тарифами и коэффициентами TTL для кэширования промпта в документации Anthropic:
[https://docs.anthropic.com/docs/build-with-claude/prompt-caching](https://docs.anthropic.com/docs/build-with-claude/prompt-caching)

### Пример: поддержание активности кэша на 1 час с помощью heartbeat

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long"
    heartbeat:
      every: "55m"
```

### Пример: смешанный трафик с индивидуальной стратегией кэширования для каждого агента

```yaml
agents:
  defaults:
    model:
      primary: "anthropic/claude-opus-4-6"
    models:
      "anthropic/claude-opus-4-6":
        params:
          cacheRetention: "long" # базовый уровень по умолчанию для большинства агентов
  list:
    - id: "research"
      default: true
      heartbeat:
        every: "55m" # поддержание длительного кэша для глубоких сессий
    - id: "alerts"
      params:
        cacheRetention: "none" # избегание записи в кэш для уведомлений с всплесками активности
```

`agents.list[].params` накладывается на `params` выбранной модели, поэтому вы можете переопределить только `cacheRetention`, сохранив остальные параметры модели по умолчанию без изменений.
