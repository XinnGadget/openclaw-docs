# ClawHub

ClawHub — это публичный реестр для **навыков и плагинов OpenClaw**.

- Используйте встроенные команды `openclaw` для поиска, установки и обновления навыков, а также для установки плагинов из ClawHub.
- Используйте отдельный CLI `clawhub`, когда требуется аутентификация в реестре, публикация, удаление, восстановление или синхронизация рабочих процессов.

Сайт: [clawhub.ai](https://clawhub.ai)

## Встроенные потоки OpenClaw

Навыки:

```bash
openclaw skills search "calendar"
openclaw skills install <skill-slug>
openclaw skills update --all
```

Плагины:

```bash
openclaw plugins install clawhub:<package>
openclaw plugins update --all
```

Спецификации плагинов, совместимые с npm, также проверяются в ClawHub перед npm:

```bash
openclaw plugins install openclaw-codex-app-server
```

Встроенные команды `openclaw` устанавливают компоненты в активную рабочую область и сохраняют метаданные источника, чтобы последующие вызовы `update` могли оставаться в ClawHub.

При установке плагинов проверяется совместимость объявленных `pluginApi` и `minGatewayVersion` до начала установки архива. Это позволяет несовместимым хостам завершать работу на раннем этапе, а не устанавливать пакет частично.

Команда `openclaw plugins install clawhub:...` принимает только устанавливаемые семейства плагинов. Если пакет в ClawHub на самом деле является навыком, OpenClaw остановит процесс и направит вас к команде `openclaw skills install <slug>`.

## Что такое ClawHub

- Публичный реестр для навыков и плагинов OpenClaw.
- Хранилище версий наборов навыков и метаданных.
- Платформа для поиска, использования тегов и сигналов использования.

## Как это работает

1. Пользователь публикует набор навыков (файлы + метаданные).
2. ClawHub сохраняет набор, анализирует метаданные и присваивает версию.
3. Реестр индексирует навык для поиска и обнаружения.
4. Пользователи просматривают, загружают и устанавливают навыки в OpenClaw.

## Что вы можете делать

- Публиковать новые навыки и новые версии существующих навыков.
- Находить навыки по имени, тегам или через поиск.
- Загружать наборы навыков и просматривать их файлы.
- Сообщать о навыках, которые являются вредными или небезопасными.
- Если вы модератор — скрывать, показывать, удалять или блокировать.

## Для кого это предназначено (подходит для начинающих)

Если вы хотите добавить новые возможности в свой агент OpenClaw, ClawHub — это самый простой способ найти и установить навыки. Вам не нужно знать, как работает бэкенд. Вы можете:

- Искать навыки с помощью обычного языка.
- Установить навык в свою рабочую область.
- Обновлять навыки позже с помощью одной команды.
- Создавать резервные копии своих навыков, публикуя их.

## Быстрый старт (для нетехнических пользователей)

1. Найдите то, что вам нужно:
   - `openclaw skills search "calendar"`
2. Установите навык:
   - `openclaw skills install <skill-slug>`
3. Запустите новую сессию OpenClaw, чтобы она учла новый навык.
4. Если вы хотите публиковать материалы или управлять аутентификацией в реестре, установите также отдельный CLI `clawhub`.

## Установка CLI ClawHub

Он нужен только для рабочих процессов с аутентификацией в реестре, таких как публикация/синхронизация:

```bash
npm i -g clawhub
```

```bash
pnpm add -g clawhub
```

## Как это вписывается в OpenClaw

Встроенная команда `openclaw skills install` устанавливает компоненты в каталог `skills/` активной рабочей области. Команда `openclaw plugins install clawhub:...` регистрирует обычную управляемую установку плагина плюс метаданные источника ClawHub для обновлений.

Анонимные установки плагинов ClawHub также завершаются с ошибкой для частных пакетов. Каналы сообщества или другие неофициальные каналы всё ещё могут устанавливать пакеты, но OpenClaw выдаёт предупреждение, чтобы операторы могли проверить источник и верификацию перед их включением.

Отдельный CLI `clawhub` также устанавливает навыки в `./skills` в текущей рабочей директории. Если рабочая область OpenClaw настроена, `clawhub` использует её, если вы не переопределите параметр `--workdir` (или `CLAWHUB_WORKDIR`). OpenClaw загружает навыки рабочей области из `<workspace>/skills` и учтёт их в **следующей** сессии. Если вы уже используете `~/.openclaw/skills` или встроенные навыки, навыки рабочей области имеют приоритет.

Для получения более подробной информации о том, как навыки загружаются, делятся и контролируются, см. [Skills](/tools/skills).

## Обзор системы навыков

Навык — это версионированный набор файлов, который учит OpenClaw выполнять определённую задачу. Каждая публикация создаёт новую версию, а реестр хранит историю версий, чтобы пользователи могли отслеживать изменения.

Типичный навык включает:

- Файл `SKILL.md` с основным описанием и инструкцией по использованию.
- Дополнительные конфигурации, скрипты или вспомогательные файлы, используемые навыком.
- Метаданные, такие как теги, краткое описание и требования к установке.

ClawHub использует метаданные для обеспечения поиска и безопасного предоставления возможностей навыков. Реестр также отслеживает сигналы использования (например, звёзды и загрузки), чтобы улучшить ранжирование и видимость.

## Возможности сервиса (функции)

- **Публичный просмотр** навыков и их содержимого `SKILL.md`.
- **Поиск**, основанный на эмбеддингах (векторный поиск), а не только на ключевых словах.
- **Версионирование** с использованием semver, журналов изменений и тегов (включая `latest`).
- **Загрузки** в виде ZIP-архива для каждой версии.
- **Звёзды и комментарии** для обратной связи от сообщества.
- **Механизмы модерации** для утверждений и аудитов.
- **API, удобный для CLI**, для автоматизации и написания скриптов.

## Безопасность и модерация

ClawHub по умолчанию открыт. Любой может загружать навыки, но для публикации учётная запись GitHub должна существовать не менее одной недели. Это помогает замедлить злоупотребления, не блокируя при этом легитимных участников.

Сообщение о нарушениях и модерация:

- Любой авторизованный пользователь может сообщить о навыке.
- Причины сообщений обязательны и записываются.
- Каждый пользователь может иметь до 20 активных сообщений одновременно.
- Навыки с более чем 3 уникальными сообщениями автоматически скрываются по умолчанию.
- Модераторы могут просматривать скрытые навыки, показывать их, удалять или блокировать пользователей.
- Злоупотребление функцией сообщений может привести к блокировке учётной записи.

Хотите стать модератором? Обратитесь в Discord OpenClaw и свяжитесь с модератором или сопровождающим.

## Команды и параметры CLI

Глобальные параметры (применяются ко всем командам):

- `--workdir <dir>`: Рабочая директория (по умолчанию: текущая директория; используется рабочая область OpenClaw).
- `--dir <dir>`: Директория навыков относительно рабочей директории (по умолчанию: `skills`).
- `--site <url>`: Базовый URL сайта (вход через браузер).
- `--registry <url>`: Базовый URL API реестра.
- `--no-input`: Отключить подсказки (неинтерактивный режим).
- `-V, --cli-version`: Вывести версию CLI.

Аутентификация:

- `clawhub login` (поток через браузер) или `clawhub login --token <token>`
- `clawhub logout`
- `clawhub whoami`

Параметры:

- `--token <token>`: Вставить токен API.
- `--label <label>`: Метка, сохраняемая для токенов входа через браузер (по умолчанию: `CLI token`).
- `--no-browser`: Не открывать браузер (требуется `--token`).

Поиск:

- `clawhub search "query"`
- `--limit <n>`: Максимальное количество результатов.

Установка:

- `clawhub install <slug>`
- `--version <version>`: Установить конкретную версию.
- `--force`: Перезаписать, если папка уже существует.

Обновление:

- `clawhub update <slug>`
- `clawhub update --all`
- `--version <version>`: Обновить до конкретной версии (только для одного slug).
- `--force`: Перезаписать, если локальные файлы не соответствуют ни одной опубликованной версии.

Список:

- `clawhub list` (читает `.clawhub/lock.json`)

Публикация навыков:

- `clawhub skill publish <path>`
- `--slug <slug>`: Slug навыка.
- `--name <name>`: Отображаемое имя.
- `--version <version>`: Версия в формате semver.
- `--changelog <text>`: Текст журнала изменений (может быть пустым).
- `--tags <tags>`: Теги, разделённые запятыми (по умолчанию: `latest`).
