# Руководство по настройке

Это полное руководство по команде `openclaw onboard`.
Для общего обзора см. [Настройка (CLI)](/start/wizard).

## Детали процесса (локальный режим)

<Steps>
  <Step title="Обнаружение существующей конфигурации">
    - Если файл `~/.openclaw/openclaw.json` существует, выберите **Сохранить / Изменить / Сбросить**.
    - Повторный запуск настройки **не** удаляет данные, если вы явно не выбрали **Сбросить** (или не передали флаг `--reset`).
    - В CLI флаг `--reset` по умолчанию очищает `config+creds+sessions`; используйте `--reset-scope full`, чтобы также удалить рабочее пространство.
    - Если конфигурация некорректна или содержит устаревшие ключи, мастер останавливается и просит запустить `openclaw doctor` перед продолжением.
    - При сбросе используется `trash` (не `rm`) и предлагаются следующие области:
      - Только конфигурация
      - Конфигурация + учётные данные + сессии
      - Полный сброс (также удаляет рабочее пространство)
  </Step>
  <Step title="Модель/Аутентификация">
    - **Ключ API Anthropic**: использует `ANTHROPIC_API_KEY`, если он есть, или запрашивает ключ, затем сохраняет его для использования демоном.
    - **Ключ API Anthropic**: предпочтительный выбор помощника Anthropic при настройке.
    - **Setup-token для Anthropic**: по-прежнему доступен при настройке, хотя OpenClaw теперь предпочитает повторное использование Claude CLI, если это возможно.
    - **Подписка OpenAI Code (Codex) (Codex CLI)**: если файл `~/.codex/auth.json` существует, настройка может повторно использовать его. Повторно используемые учётные данные Codex CLI остаются под управлением Codex CLI; по истечении срока действия OpenClaw сначала повторно считывает этот источник и, если провайдер может обновить данные, записывает обновлённые учётные данные обратно в хранилище Codex, не беря управление на себя.
    - **Подписка OpenAI Code (Codex) (OAuth)**: поток через браузер; вставьте `code#state`.
      - Устанавливает `agents.defaults.model` в `openai-codex/gpt-5.4`, если модель не задана или равна `openai/*`.
    - **Ключ API OpenAI**: использует `OPENAI_API_KEY`, если он есть, или запрашивает ключ, затем сохраняет его в профилях аутентификации.
      - Устанавливает `agents.defaults.model` в `openai/gpt-5.4`, если модель не задана, равна `openai/*` или `openai-codex/*`.
    - **Ключ API xAI (Grok)**: запрашивает `XAI_API_KEY` и настраивает xAI в качестве провайдера модели.
    - **OpenCode**: запрашивает `OPENCODE_API_KEY` (или `OPENCODE_ZEN_API_KEY`, получите его на https://opencode.ai/auth) и позволяет выбрать каталог Zen или Go.
    - **Ollama**: запрашивает базовый URL Ollama, предлагает режимы **Cloud + Local** или **Local**, обнаруживает доступные модели и автоматически загружает выбранную локальную модель при необходимости.
    - Подробнее: [Ollama](/providers/ollama)
    - **Ключ API**: сохраняет ключ для вас.
    - **Vercel AI Gateway (прокси для нескольких моделей)**: запрашивает `AI_GATEWAY_API_KEY`.
    - Подробнее: [Vercel AI Gateway](/providers/vercel-ai-gateway)
    - **Cloudflare AI Gateway**: запрашивает ID аккаунта, ID шлюза и `CLOUDFLARE_AI_GATEWAY_API_KEY`.
    - Подробнее: [Cloudflare AI Gateway](/providers/cloudflare-ai-gateway)
    - **MiniMax**: конфигурация записывается автоматически; по умолчанию используется `MiniMax-M2.7`.
      Настройка ключа API использует `minimax/...`, а настройка OAuth — `minimax-portal/...`.
    - Подробнее: [MiniMax](/providers/minimax)
    - **StepFun**: конфигурация записывается автоматически для StepFun Standard или Step Plan на китайских или глобальных конечных точках.
    - В настоящее время Step Standard включает `step-3.5-flash`, а Step Plan также включает `step-3.5-flash-2603`.
    - Подробнее: [StepFun](/providers/stepfun)
    - **Synthetic (совместимый с Anthropic)**: запрашивает `SYNTHETIC_API_KEY`.
    - Подробнее: [Synthetic](/providers/synthetic)
    - **Moonshot (Kimi K2)**: конфигурация записывается автоматически.
    - **Kimi Coding**: конфигурация записывается автоматически.
    - Подробнее: [Moonshot AI (Kimi + Kimi Coding)](/providers/moonshot)
    - **Пропустить**: аутентификация пока не настроена.
    - Выберите модель по умолчанию из обнаруженных вариантов (или введите провайдера/модель вручную). Для наилучшего качества и снижения риска внедрения подсказок выберите самую мощную модель последнего поколения, доступную в вашем стеке провайдеров.
    - Настройка выполняет проверку модели и предупреждает, если сконфигурированная модель неизвестна или отсутствует аутентификация.
    - Режим хранения ключа API по умолчанию — значения профиля аутентификации в виде обычного текста. Используйте `--secret-input-mode ref`, чтобы вместо этого хранить ссылки, поддерживаемые окружением (например, `keyRef: { source: "env", provider: "default", id: "OPENAI_API_KEY" }`).
    - Профили аутентификации хранятся в `~/.openclaw/agents/<agentId>/agent/auth-profiles.json` (ключи API + OAuth). Файл `~/.openclaw/credentials/oauth.json` предназначен только для импорта устаревших данных.
    - Подробнее: [/concepts/oauth](/concepts/oauth)
    <Note>
    Совет для headless/серверных систем: выполните OAuth на машине с браузером, затем скопируйте `auth-profiles.json` этого агента (например, `~/.openclaw/agents/<agentId>/agent/auth-profiles.json` или соответствующий путь `$OPENCLAW_STATE_DIR/...`) на хост шлюза. Файл `credentials/oauth.json` предназначен только для импорта устаревших данных.
    </Note>
  </Step>
  <Step title="Рабочее пространство">
    - По умолчанию `~/.openclaw/workspace` (можно настроить).
    - Заполняет файлы рабочего пространства, необходимые для ритуала начальной загрузки агента.
    - Полное описание структуры рабочего пространства и руководство по резервному копированию: [Рабочее пространство агента](/concepts/agent-workspace)
  </Step>
  <Step title="Шлюз">
    - Порт, привязка, режим аутентификации, доступность через Tailscale.
    - Рекомендация по аутентификации: сохраняйте режим **Token** даже для loopback, чтобы локальные клиенты WS проходили аутентификацию.
    - В режиме токена интерактивная настройка предлагает:
      - **Сгенерировать/сохранить токен в виде обычного текста** (по умолчанию)
      - **Использовать SecretRef** (по желанию)
      - Quickstart повторно использует существующие SecretRef `gateway.auth.token` для провайдеров `env`, `file` и `exec` для начальной загрузки пробника/панели управления.
      - Если этот SecretRef настроен, но не может быть разрешён, настройка завершается с ошибкой на раннем этапе с чётким сообщением о способе исправления, вместо того чтобы молча ухудшать аутентификацию во время выполнения.
    - В режиме пароля интерактивная настройка также поддерживает хранение в виде обычного текста или SecretRef.
    - Путь к SecretRef токена шлюза в неинтерактивном режиме: `--gateway-token-ref-env <ENV_VAR>`.
      - Требует непустой переменной окружения в процессе настройки.
      - Нельзя комбинировать с `--gateway-token`.
    - Отключайте аутентификацию только в том случае, если вы полностью доверяете каждому локальному процессу.
    - Привязки, отличные от loopback, по-прежнему требуют аутентификации.
  </Step>
  <Step title="Каналы">
    - [WhatsApp](/channels/whatsapp): необязательный вход через QR-код.
    - [Telegram](/channels/telegram): токен бота.
    - [Discord](/channels/discord): токен бота.
    - [Google Chat](/channels/googlechat): JSON учётной записи службы + аудитория вебхука.
    - [Mattermost](/channels/mattermost) (плагин): токен бота + базовый URL.
    - [Signal](/channels/signal): необязательная установка `signal-cli` + настройка учётной записи.
    - [BlueBubbles](/channels/bluebubbles): **рекомендуется для iMessage**; URL сервера + пароль + вебхук.
    - [iMessage](/channels/imessage): устаревший путь CLI `imsg` + доступ к БД.
    - Безопасность DM: по умолчанию используется сопряжение. Первое DM отправляет код; подтвердите через `openclaw pairing approve <channel> <code>` или используйте
