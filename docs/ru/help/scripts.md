# Скрипты

Каталог `scripts/` содержит вспомогательные скрипты для локальных рабочих процессов и операционных задач.
Используйте их, если задача явно связана с конкретным скриптом; в остальных случаях предпочтительнее использовать CLI.

## Соглашения

- Скрипты являются **необязательными**, если они не упомянуты в документации или чек-листах релизов.
- Предпочитайте интерфейсы CLI, если они существуют (пример: для мониторинга аутентификации используется команда `openclaw models status --check`).
- Предполагайте, что скрипты привязаны к конкретному хосту; ознакомьтесь с ними перед запуском на новом компьютере.

## Скрипты мониторинга аутентификации

Мониторинг аутентификации описан в разделе [Аутентификация](/gateway/authentication). Скрипты в каталоге `scripts/` — это дополнительные необязательные инструменты для рабочих процессов в systemd/Termux на телефонах.

## Помощник для чтения данных из GitHub

Используйте `scripts/gh-read`, если хотите, чтобы утилита `gh` применяла токен установки GitHub App для вызовов чтения с областью действия репозитория, при этом оставив обычную утилиту `gh` с вашим личным логином для операций записи.

Требуемые переменные окружения:

- `OPENCLAW_GH_READ_APP_ID`
- `OPENCLAW_GH_READ_PRIVATE_KEY_FILE`

Необязательные переменные окружения:

- `OPENCLAW_GH_READ_INSTALLATION_ID` — если вы хотите пропустить поиск установки на основе репозитория
- `OPENCLAW_GH_READ_PERMISSIONS` — переопределение подмножества прав на чтение в виде списка, разделённого запятыми

Порядок разрешения репозитория:

- `gh ... -R owner/repo`
- `GH_REPO`
- `git remote origin`

Примеры:

- `scripts/gh-read pr view 123`
- `scripts/gh-read run list -R openclaw/openclaw`
- `scripts/gh-read api repos/openclaw/openclaw/pulls/123`

## При добавлении скриптов

- Следите за тем, чтобы скрипты были сфокусированными и документированными.
- Добавьте краткую запись в соответствующий документ (или создайте его, если его нет).
