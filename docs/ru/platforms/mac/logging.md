# Логирование (macOS)

## Ротируемый файл журнала диагностики (панель отладки)

OpenClaw направляет логи приложения macOS через swift-log (по умолчанию используется унифицированное логирование) и может записывать локальный ротируемый файл журнала на диск, когда требуется надёжное сохранение данных.

- Уровень детализации: **Панель отладки → Логи → Логирование приложения → Уровень детализации**
- Включение: **Панель отладки → Логи → Логирование приложения → "Записывать ротируемый журнал диагностики (JSONL)"**
- Расположение: `~/Library/Logs/OpenClaw/diagnostics.jsonl` (автоматически ротируется; старые файлы получают суффиксы `.1`, `.2` и т. д.)
- Очистка: **Панель отладки → Логи → Логирование приложения → "Очистить"**

Примечания:

- По умолчанию эта функция **отключена**. Включайте её только во время активной отладки.
- Относитесь к файлу как к конфиденциальному; не распространяйте его без предварительной проверки.

## Унифицированное логирование приватных данных в macOS

Унифицированное логирование скрывает большую часть полезных данных, если подсистема не включена в режим `privacy -off`. Согласно статье Питера о "хитростях конфиденциальности логирования в macOS" ([logging privacy shenanigans](https://steipete.me/posts/2025/logging-privacy-shenanigans), 2025), это регулируется plist-файлом в каталоге `/Library/Preferences/Logging/Subsystems/`, где ключом выступает имя подсистемы. Только новые записи логов учитывают флаг, поэтому включите его до воспроизведения проблемы.

## Включение для OpenClaw (`ai.openclaw`)

- Сначала запишите plist-файл во временный файл, затем установите его атомарно с правами root:

```bash
cat <<'EOF' >/tmp/ai.openclaw.plist
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE plist PUBLIC "-//Apple//DTD PLIST 1.0//EN" "http://www.apple.com/DTDs/PropertyList-1.0.dtd">
<plist version="1.0">
<dict>
    <key>DEFAULT-OPTIONS</key>
    <dict>
        <key>Enable-Private-Data</key>
        <true/>
    </dict>
</dict>
</plist>
EOF
sudo install -m 644 -o root -g wheel /tmp/ai.openclaw.plist /Library/Preferences/Logging/Subsystems/ai.openclaw.plist
```

- Перезагрузка не требуется; logd быстро обнаруживает файл, но только новые строки логов будут включать приватные полезные данные.
- Просмотрите более подробные выходные данные с помощью существующего помощника, например: `./scripts/clawlog.sh --category WebChat --last 5m`.

## Отключение после отладки

- Удалите переопределение: `sudo rm /Library/Preferences/Logging/Subsystems/ai.openclaw.plist`.
- При необходимости выполните `sudo log config --reload`, чтобы заставить logd немедленно сбросить переопределение.
- Помните, что в данных могут содержаться номера телефонов и тексты сообщений; оставляйте plist-файл только на время, пока вам действительно нужны дополнительные детали.
