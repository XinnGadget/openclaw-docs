# Vydra

Встроенный плагин Vydra добавляет следующие возможности:

- генерацию изображений через `vydra/grok-imagine`;
- генерацию видео через `vydra/veo3` и `vydra/kling`;
- синтез речи через маршрут TTS (text-to-speech) Vydra на базе ElevenLabs.

OpenClaw использует один и тот же ключ `VYDRA_API_KEY` для всех трёх функций.

## Важный базовый URL

Используйте `https://www.vydra.ai/api/v1`.

Основной хост Vydra (`https://vydra.ai/api/v1`) в настоящее время перенаправляет на `www`. Некоторые HTTP-клиенты удаляют заголовок `Authorization` при таком межхостовом перенаправлении, из-за чего действительный API-ключ может привести к ошибочному сообщению об ошибке аутентификации. Встроенный плагин напрямую использует базовый URL с `www`, чтобы избежать этой проблемы.

## Настройка

Интерактивная настройка:

```bash
openclaw onboard --auth-choice vydra-api-key
```

Или задайте переменную окружения напрямую:

```bash
export VYDRA_API_KEY="vydra_live_..."
```

## Генерация изображений

Модель изображения по умолчанию:

- `vydra/grok-imagine`.

Задайте её в качестве поставщика изображений по умолчанию:

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: {
        primary: "vydra/grok-imagine",
      },
    },
  },
}
```

В настоящее время встроенная поддержка ограничена преобразованием текста в изображение. Хостируемые маршруты редактирования Vydra ожидают URL-адреса удалённых изображений, а OpenClaw пока не добавляет в встроенный плагин специальный мост для загрузки файлов в Vydra.

См. [Генерация изображений](/tools/image-generation) для ознакомления с общим поведением инструмента.

## Генерация видео

Зарегистрированные модели видео:

- `vydra/veo3` для преобразования текста в видео;
- `vydra/kling` для преобразования изображения в видео.

Задайте Vydra в качестве поставщика видео по умолчанию:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "vydra/veo3",
      },
    },
  },
}
```

Примечания:

- `vydra/veo3` встроен только для преобразования текста в видео.
- `vydra/kling` в настоящее время требует указания URL-адреса удалённого изображения. Загрузка локальных файлов сразу отклоняется.
- Текущий HTTP-маршрут `kling` в Vydra непоследователен в отношении того, требуется ли `image_url` или `video_url`; встроенный провайдер передаёт один и тот же URL удалённого изображения в оба поля.
- Встроенный плагин придерживается консервативного подхода и не передаёт недокументированные параметры настройки, такие как соотношение сторон, разрешение, водяной знак или генерируемый аудиоряд.

Актуальное покрытие провайдера в режиме реального времени:

```bash
OPENCLAW_LIVE_TEST=1 \
OPENCLAW_LIVE_VYDRA_VIDEO=1 \
pnpm test:live -- extensions/vydra/vydra.live.test.ts
```

Встроенный файл Vydra в режиме реального времени теперь охватывает:

- `vydra/veo3` — преобразование текста в видео;
- `vydra/kling` — преобразование изображения в видео с использованием URL-адреса удалённого изображения.

При необходимости переопределите фикстуру удалённого изображения:

```bash
export OPENCLAW_LIVE_VYDRA_KLING_IMAGE_URL="https://example.com/reference.png"
```

См. [Генерация видео](/tools/video-generation) для ознакомления с общим поведением инструмента.

## Синтез речи

Задайте Vydra в качестве поставщика речи:

```json5
{
  messages: {
    tts: {
      provider: "vydra",
      providers: {
        vydra: {
          apiKey: "${VYDRA_API_KEY}",
          voiceId: "21m00Tcm4TlvDq8ikWAM",
        },
      },
    },
  },
}
```

Значения по умолчанию:

- модель: `elevenlabs/tts`;
- идентификатор голоса: `21m00Tcm4TlvDq8ikWAM`.

Встроенный плагин в настоящее время предоставляет один проверенный голос по умолчанию и возвращает аудиофайлы в формате MP3.
