# Создание навыков

Навыки учат агента, как и когда использовать инструменты. Каждый навык — это каталог, содержащий файл `SKILL.md` с YAML-заголовком (frontmatter) и инструкциями в формате Markdown.

О том, как загружаются и приоритизируются навыки, см. [Навыки](/tools/skills).

## Создайте свой первый навык

<Steps>
  <Step title="Создайте каталог для навыка">
    Навыки хранятся в вашей рабочей области. Создайте новую папку:

    ```bash
    mkdir -p ~/.openclaw/workspace/skills/hello-world
    ```

  </Step>

  <Step title="Напишите SKILL.md">
    Создайте файл `SKILL.md` внутри этого каталога. Заголовок (frontmatter) определяет метаданные, а тело в формате Markdown содержит инструкции для агента.

    ```markdown
    ---
    name: hello_world
    description: Простой навык, который приветствует пользователя.
    ---

    # Навык "Hello World"

    Когда пользователь запрашивает приветствие, используйте инструмент `echo`, чтобы вывести сообщение "Hello from your custom skill!".
    ```

  </Step>

  <Step title="Добавьте инструменты (необязательно)">
    Вы можете определить схемы пользовательских инструментов в заголовке (frontmatter) или указать агенту использовать существующие системные инструменты (например, `exec` или `browser`). Навыки также могут поставляться внутри плагинов вместе с инструментами, которые они описывают.

  </Step>

  <Step title="Загрузите навык">
    Запустите новый сеанс, чтобы OpenClaw обнаружил навык:

    ```bash
    # Из чата
    /new

    # Или перезапустите шлюз
    openclaw gateway restart
    ```

    Проверьте, что навык загружен:

    ```bash
    openclaw skills list
    ```

  </Step>

  <Step title="Протестируйте его">
    Отправьте сообщение, которое должно активировать навык:

    ```bash
    openclaw agent --message "give me a greeting"
    ```

    Или просто пообщайтесь с агентом и попросите его поздороваться.

  </Step>
</Steps>

## Справочник по метаданным навыка

Заголовок YAML (frontmatter) поддерживает следующие поля:

| Поле | Обязательно | Описание |
| --- | --- | --- |
| `name` | Да | Уникальный идентификатор (в формате snake_case) |
| `description` | Да | Краткое описание (одна строка), отображаемое для агента |
| `metadata.openclaw.os` | Нет | Фильтр по ОС (`["darwin"]`, `["linux"]` и т. д.) |
| `metadata.openclaw.requires.bins` | Нет | Необходимые исполняемые файлы в PATH |
| `metadata.openclaw.requires.config` | Нет | Необходимые ключи конфигурации |

## Лучшие практики

- **Будьте лаконичны** — указывайте модели, _что_ нужно сделать, а не как быть ИИ;
- **Безопасность прежде всего** — если ваш навык использует `exec`, убедитесь, что в промтах нет возможности внедрения произвольных команд из ненадёжных входных данных;
- **Тестируйте локально** — используйте `openclaw agent --message "..."` для тестирования перед публикацией;
- **Используйте ClawHub** — просматривайте навыки и делитесь ими на [ClawHub](https://clawhub.ai).

## Где хранятся навыки

| Расположение | Приоритет | Область действия |
| --- | --- | --- |
| `\<workspace\>/skills/` | Наивысший | Для конкретного агента |
| `\<workspace\>/.agents/skills/` | Высокий | Для агента в рабочей области |
| `~/.agents/skills/` | Средний | Общий профиль агента |
| `~/.openclaw/skills/` | Средний | Общий (для всех агентов) |
| Встроенные (поставляются с OpenClaw) | Низкий | Глобальный |
| `skills.load.extraDirs` | Наинизший | Пользовательские общие папки |
