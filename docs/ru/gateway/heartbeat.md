# Heartbeat (шлюз)

> **Heartbeat или Cron?** См. раздел [Автоматизация и задачи](/automation), чтобы узнать, когда использовать каждый из них.

Heartbeat запускает **периодические циклы агента** в основной сессии, чтобы модель могла обратить внимание на всё, что требует внимания, не засыпая вас сообщениями.

Heartbeat — это запланированный цикл в основной сессии, он **не** создаёт записи [фоновых задач](/automation/tasks).
Записи задач предназначены для автономной работы (запуски ACP, субагенты, изолированные задания cron).

Устранение неполадок: [Запланированные задачи](/automation/cron-jobs#troubleshooting)

## Быстрый старт (для начинающих)

1. Оставьте сердцебиение включённым (по умолчанию — `30m`, или `1h` для OAuth/токен-аутентификации Anthropic, включая повторное использование Claude CLI) или задайте свою частоту.
2. Создайте небольшой чек-лист `HEARTBEAT.md` или блок `tasks:` в рабочей области агента (необязательно, но рекомендуется).
3. Определите, куда должны направляться сообщения сердцебиения (`target: "none"` — значение по умолчанию; установите `target: "last"`, чтобы направлять сообщения последнему контакту).
4. Опционально: включите передачу рассуждений сердцебиения для прозрачности.
5. Опционально: используйте облегчённый контекст начальной загрузки, если для запусков сердцебиения требуется только `HEARTBEAT.md`.
6. Опционально: включите изолированные сессии, чтобы не отправлять полную историю переписки при каждом сердцебиении.
7. Опционально: ограничьте сердцебиение активными часами (по местному времени).

Пример конфигурации:

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last", // явная доставка последнему контакту (по умолчанию — "none")
        directPolicy: "allow", // по умолчанию: разрешена доставка в прямые сообщения/DM; установите "block", чтобы запретить
        lightContext: true, // опционально: вставлять только HEARTBEAT.md из файлов начальной загрузки
        isolatedSession: true, // опционально: новая сессия при каждом запуске (без истории переписки)
        // activeHours: { start: "08:00", end: "24:00" },
        // includeReasoning: true, // опционально: также отправлять отдельное сообщение Reasoning:
      },
    },
  },
}
```

## Значения по умолчанию

- Интервал: `30m` (или `1h`, если обнаружена аутентификация через OAuth/токен Anthropic, включая повторное использование Claude CLI). Задайте `agents.defaults.heartbeat.every` или `agents.list[].heartbeat.every` для конкретного агента; используйте `0m`, чтобы отключить.
- Тело промпта (настраивается через `agents.defaults.heartbeat.prompt`):
  `Прочитайте HEARTBEAT.md, если он существует (контекст рабочей области). Строго следуйте ему. Не делайте выводов и не повторяйте старые задачи из предыдущих чатов. Если ничего не требует внимания, ответьте HEARTBEAT_OK.`
- Промпт сердцебиения отправляется **дословно** как сообщение пользователя. Системный промпт включает раздел "Heartbeat" только тогда, когда сердцебиение включено для агента по умолчанию, и запуск помечен внутри системы.
- Когда сердцебиение отключено с помощью `0m`, обычные запуски также исключают `HEARTBEAT.md` из контекста начальной загрузки, чтобы модель не видела инструкции, предназначенные только для сердцебиения.
- Активные часы (`heartbeat.activeHours`) проверяются в настроенном часовом поясе.
  Вне этого интервала сердцебиение пропускается до следующего запуска внутри интервала.

## Для чего предназначен промпт сердцебиения

Стандартный промпт намеренно универсален:

- **Фоновые задачи**: фраза "Рассмотрите невыполненные задачи" побуждает агента проверить последующие действия (входящие сообщения, календарь, напоминания, поставленные в очередь задачи) и обратить внимание на срочные вопросы.
- **Проверка состояния человека**: фраза "Иногда проверяйте состояние вашего человека в течение дня" побуждает агента время от времени отправлять лёгкое сообщение "Вам что-нибудь нужно?", но избегает спама ночью благодаря использованию настроенного местного часового пояса (см. [/concepts/timezone](/concepts/timezone)).

Heartbeat может реагировать на завершённые [фоновые задачи](/automation/tasks), но сам запуск сердцебиения не создаёт запись задачи.

Если вы хотите, чтобы сердцебиение выполняло что-то очень конкретное (например, "проверить статистику Gmail PubSub" или "проверить работоспособность шлюза"), задайте `agents.defaults.heartbeat.prompt` (или `agents.list[].heartbeat.prompt`) с пользовательским текстом (отправляется дословно).

## Контракт на ответ

- Если ничего не требует внимания, ответьте **`HEARTBEAT_OK`**.
- Во время запусков сердцебиения OpenClaw обрабатывает `HEARTBEAT_OK` как подтверждение, если оно находится **в начале или в конце** ответа. Токены удаляются, а ответ отбрасывается, если оставшийся текст содержит **≤ `ackMaxChars`** (по умолчанию: 300).
- Если `HEARTBEAT_OK` находится **в середине** ответа, он не обрабатывается особым образом.
- Для оповещений **не** включайте `HEARTBEAT_OK`; верните только текст оповещения.

Вне сердцебиения случайное `HEARTBEAT_OK` в начале/конце сообщения удаляется и регистрируется; сообщение, состоящее только из `HEARTBEAT_OK`, отбрасывается.

## Конфигурация

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m", // по умолчанию: 30m (0m отключает)
        model: "anthropic/claude-opus-4-6",
        includeReasoning: false, // по умолчанию: false (отправлять отдельное сообщение Reasoning:, если доступно)
        lightContext: false, // по умолчанию: false; true сохраняет только HEARTBEAT.md из файлов начальной загрузки рабочей области
        isolatedSession: false, // по умолчанию: false; true запускает каждое сердцебиение в новой сессии (без истории переписки)
        target: "last", // по умолчанию: none | варианты: last | none | <идентификатор канала> (основной или плагин, например, "bluebubbles")
        to: "+15551234567", // опциональный переопределение для конкретного канала
        accountId: "ops-bot", // опциональный идентификатор канала для нескольких учётных записей
        prompt: "Прочитайте HEARTBEAT.md, если он существует (контекст рабочей области). Строго следуйте ему. Не делайте выводов и не повторяйте старые задачи из предыдущих чатов. Если ничего не требует внимания, ответьте HEARTBEAT_OK.",
        ackMaxChars: 300, // максимальное количество символов после HEARTBEAT_OK
      },
    },
  },
}
```

### Область действия и приоритет

- `agents.defaults.heartbeat` задаёт глобальное поведение сердцебиения.
- `agents.list[].heartbeat` переопределяет глобальные настройки; если у какого-либо агента есть блок `heartbeat`, **только эти агенты** выполняют сердцебиение.
- `channels.defaults.heartbeat` задаёт настройки видимости по умолчанию для всех каналов.
- `channels.<channel>.heartbeat` переопределяет настройки по умолчанию для канала.
- `channels.<channel>.accounts.<id>.heartbeat` (каналы с несколькими учётными записями) переопределяет настройки для конкретного канала.

### Сердцебиение для отдельных агентов

Если какой-либо элемент `agents.list[]` содержит блок `heartbeat`, **только эти агенты** выполняют сердцебиение. Блок для конкретного агента переопределяет `agents.defaults.heartbeat` (так что вы можете задать общие настройки по умолчанию и переопределить их для каждого агента).

Пример: два агента, только второй агент выполняет сердцебиение.

```json5
{
  agents: {
    defaults: {
      heartbeat: {
        every: "30m",
        target: "last", // явная доставка последнему контакту (по умолчанию — "none")
      },
    },
    list: [
      { id: "main", default: true },
      {
        id: "ops",
        heartbeat: {
          every: "1h",
          target: "whatsapp",
          to: "+15551234567",
          prompt: "Прочитайте HEARTBEAT.md, если он существует (контекст рабочей области). Строго следуйте ему. Не делайте выводов и не повторяйте старые задачи из предыдущих чатов. Если ничего не требует внимания, ответьте HEARTBEAT_OK.",
        },
      },
    ],
  },
}
```

### Пример настройки активных часов

Ограничьте сердцебиение рабочими часами в определённом часовом поясе:

```json5
{
  agents: {
    defaults: {
