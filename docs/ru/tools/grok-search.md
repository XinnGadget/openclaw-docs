# Поиск с Grok

OpenClaw поддерживает Grok в качестве провайдера `web_search`, используя ответы xAI с привязкой к веб-данным для формирования синтезированных ИИ-ответов на основе актуальных результатов поиска с цитатами.

Тот же `XAI_API_KEY` можно использовать для встроенного инструмента `x_search` — поиска постов в X (ранее Twitter). Если сохранить ключ в `plugins.entries.xai.config.webSearch.apiKey`, OpenClaw будет повторно использовать его в качестве запасного варианта и для встроенного провайдера модели xAI.

Для получения метрик на уровне поста в X (таких как репосты, ответы, закладки или просмотры) предпочтительно использовать `x_search` с точным URL-адресом поста или ID статуса вместо широкого поискового запроса.

## Подключение и настройка

Если вы выбираете **Grok** во время:

- `openclaw onboard`
- `openclaw configure --section web`

OpenClaw может отобразить отдельный последующий шаг для включения `x_search` с тем же `XAI_API_KEY`. Этот последующий шаг:

- появляется только после того, как вы выбрали Grok для `web_search`
- не является отдельным вариантом выбора провайдера веб-поиска верхнего уровня
- при желании позволяет настроить модель `x_search` в рамках того же процесса

Если вы пропустите этот шаг, вы сможете включить или изменить `x_search` позже в конфигурации.

## Получение API-ключа

<Steps>
  <Step title="Создание ключа">
    Получите API-ключ на [xAI](https://console.x.ai/).
  </Step>
  <Step title="Сохранение ключа">
    Установите `XAI_API_KEY` в окружении Gateway либо настройте с помощью команды:

    ```bash
    openclaw configure --section web
    ```

  </Step>
</Steps>

## Конфигурация

```json5
{
  plugins: {
    entries: {
      xai: {
        config: {
          webSearch: {
            apiKey: "xai-...", // необязательно, если задан XAI_API_KEY
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        provider: "grok",
      },
    },
  },
}
```

**Альтернатива с использованием окружения:** задайте `XAI_API_KEY` в окружении Gateway. При установке шлюза поместите его в `~/.openclaw/.env`.

## Принцип работы

Grok использует ответы xAI с привязкой к веб-данным для синтеза ответов с встроенными цитатами — аналогично подходу Gemini с привязкой к поиску Google.

## Поддерживаемые параметры

Поиск с Grok поддерживает параметр `query`.

Параметр `count` принимается для обеспечения совместимости с общим `web_search`, но Grok по-прежнему возвращает один синтезированный ответ с цитатами, а не список из N результатов.

Специфичные для провайдера фильтры в настоящее время не поддерживаются.
