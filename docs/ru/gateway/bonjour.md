# Обнаружение через Bonjour / mDNS

OpenClaw использует Bonjour (mDNS / DNS-SD) для обнаружения активного шлюза (WebSocket-конечной точки).
Просмотр в многоадресной рассылке `local.` — это **удобство только для локальной сети (LAN)**. Для обнаружения в разных сетях тот же маяк можно опубликовать через настроенную доменную зону DNS-SD для широкой области. Обнаружение по-прежнему осуществляется по принципу "наилучших усилий" и **не заменяет** подключение через SSH или Tailnet.

## Bonjour для широкой области (одноадресный DNS-SD) через Tailscale

Если узел и шлюз находятся в разных сетях, многоадресный mDNS не сможет пересечь границу. Вы можете сохранить тот же пользовательский интерфейс обнаружения, переключившись на **одноадресный DNS-SD** ("Bonjour для широкой области") через Tailscale.

Основные шаги:

1. Запустите DNS-сервер на хосте шлюза (доступном через Tailnet).
2. Опубликуйте записи DNS-SD для `_openclaw-gw._tcp` в выделенной зоне (например: `openclaw.internal.`).
3. Настройте **раздельный DNS** в Tailscale, чтобы выбранный домен разрешался через этот DNS-сервер для клиентов (включая iOS).

OpenClaw поддерживает любой домен обнаружения; `openclaw.internal.` — это лишь пример.
Узлы на iOS/Android просматривают как `local.`, так и настроенную доменную зону для широкой области.

### Конфигурация шлюза (рекомендуется)

```json5
{
  gateway: { bind: "tailnet" }, // только tailnet (рекомендуется)
  discovery: { wideArea: { enabled: true } }, // включает публикацию DNS-SD для широкой области
}
```

### Однократная настройка DNS-сервера (хост шлюза)

```bash
openclaw dns setup --apply
```

Это установит CoreDNS и настроит его следующим образом:

- прослушивать порт 53 только на интерфейсах Tailscale шлюза;
- обслуживать выбранный домен (например, `openclaw.internal.`) из `~/.openclaw/dns/<domain>.db`.

Проверьте с машины, подключённой к tailnet:

```bash
dns-sd -B _openclaw-gw._tcp openclaw.internal.
dig @<TAILNET_IPV4> -p 53 _openclaw-gw._tcp.openclaw.internal PTR +short
```

### Настройки DNS в Tailscale

В консоли администратора Tailscale:

- добавьте сервер имён, указывающий на tailnet-IP шлюза (UDP/TCP 53);
- настройте раздельный DNS, чтобы ваш домен обнаружения использовал этот сервер имён.

Как только клиенты примут tailnet DNS, узлы на iOS и CLI-обнаружение смогут просматривать `_openclaw-gw._tcp` в вашем домене обнаружения без многоадресной рассылки.

### Безопасность прослушивателя шлюза (рекомендуется)

Порт WS шлюза (по умолчанию `18789`) по умолчанию привязан к loopback. Для доступа через LAN/tailnet явно укажите привязку и оставьте аутентификацию включённой.

Для настроек только для tailnet:

- установите `gateway.bind: "tailnet"` в `~/.openclaw/openclaw.json`;
- перезапустите шлюз (или перезапустите приложение в строке меню macOS).

## Что публикует объявления

Только шлюз публикует объявления для `_openclaw-gw._tcp`.

## Типы служб

- `_openclaw-gw._tcp` — маяк транспортного шлюза (используется узлами на macOS/iOS/Android).

## Ключи TXT (несекретные подсказки)

Шлюз публикует небольшие несекретные подсказки, чтобы упростить работу пользовательского интерфейса:

- `role=gateway`;
- `displayName=<удобное имя>`;
- `lanHost=<имя_хоста>.local`;
- `gatewayPort=<порт>` (Gateway WS + HTTP);
- `gatewayTls=1` (только если включён TLS);
- `gatewayTlsSha256=<sha256>` (только если включён TLS и доступен отпечаток);
- `canvasPort=<порт>` (только если включён хост canvas; в настоящее время совпадает с `gatewayPort`);
- `transport=gateway`;
- `tailnetDns=<magicdns>` (дополнительная подсказка, если доступен Tailnet);
- `sshPort=<порт>` (только в полном режиме mDNS; в DNS-SD для широкой области может отсутствовать);
- `cliPath=<путь>` (только в полном режиме mDNS; в DNS-SD для широкой области всё ещё записывается как подсказка для удалённой установки).

Примечания по безопасности:

- TXT-записи Bonjour/mDNS **не аутентифицируются**. Клиенты не должны рассматривать TXT как авторитетный источник маршрутизации.
- Клиенты должны маршрутизировать, используя разрешённую конечную точку службы (SRV + A/AAAA). Рассматривайте `lanHost`, `tailnetDns`, `gatewayPort` и `gatewayTlsSha256` только как подсказки.
- Автоматическое нацеливание на SSH также должно использовать разрешённый хост службы, а не только подсказки из TXT.
- Прикрепление TLS никогда не должно позволять рекламируемому `gatewayTlsSha256` переопределять ранее сохранённый пин.
- Узлы на iOS/Android должны рассматривать прямое подключение на основе обнаружения как **только TLS** и требовать явного подтверждения пользователя перед тем, как доверять отпечатку при первом подключении.

## Отладка на macOS

Полезные встроенные инструменты:

- Просмотр экземпляров:

  ```bash
  dns-sd -B _openclaw-gw._tcp local.
  ```

- Разрешение одного экземпляра (замените `<instance>`):

  ```bash
  dns-sd -L "<instance>" _openclaw-gw._tcp local.
  ```

Если просмотр работает, но разрешение не удаётся, обычно это связано с политикой LAN или проблемой с резолвером mDNS.

## Отладка в журналах шлюза

Шлюз записывает циклический файл журнала (выводится при запуске как `gateway log file: ...`). Ищите строки `bonjour:`, особенно:

- `bonjour: advertise failed ...`;
- `bonjour: ... name conflict resolved` / `hostname conflict resolved`;
- `bonjour: watchdog detected non-announced service ...`.

## Отладка на узле iOS

Узел iOS использует `NWBrowser` для обнаружения `_openclaw-gw._tcp`.

Чтобы захватить журналы:

- Настройки → Шлюз → Дополнительно → **Журналы отладки обнаружения**
- Настройки → Шлюз → Дополнительно → **Журналы обнаружения** → воспроизведите проблему → **Копировать**

Журнал включает переходы состояний браузера и изменения набора результатов.

## Типичные сбои

- **Bonjour не работает между сетями**: используйте Tailnet или SSH.
- **Многоадресная рассылка заблокирована**: некоторые сети Wi-Fi отключают mDNS.
- **Режим сна / смена интерфейса**: macOS может временно удалять результаты mDNS; повторите попытку.
- **Просмотр работает, но разрешение не удаётся**: используйте простые имена машин (избегайте эмодзи и знаков препинания), затем перезапустите шлюз. Имя экземпляра службы зависит от имени хоста, поэтому слишком сложные имена могут сбивать с толку некоторые резолверы.

## Экранированные имена экземпляров (`\032`)

Bonjour/DNS-SD часто экранирует байты в именах экземпляров служб в виде десятичных последовательностей `\DDD` (например, пробелы становятся `\032`).

- Это нормально на уровне протокола.
- Пользовательские интерфейсы должны декодировать для отображения (iOS использует `BonjourEscapes.decode`).

## Отключение / конфигурация

- `OPENCLAW_DISABLE_BONJOUR=1` отключает публикацию объявлений (устаревшее: `OPENCLAW_DISABLE_BONJOUR`).
- `gateway.bind` в `~/.openclaw/openclaw.json` управляет режимом привязки шлюза.
- `OPENCLAW_SSH_PORT` переопределяет порт SSH, когда рекламируется `sshPort` (устаревшее: `OPENCLAW_SSH_PORT`).
- `OPENCLAW_TAILNET_DNS` публикует подсказку MagicDNS в TXT (устаревшее: `OPENCLAW_TAILNET_DNS`).
- `OPENCLAW_CLI_PATH` переопределяет рекламируемый путь CLI (устаревшее: `OPENCLAW_CLI_PATH`).
