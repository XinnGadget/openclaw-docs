# QQ Bot

QQ Bot подключается к OpenClaw через официальный API QQ Bot (шлюз WebSocket). Плагин поддерживает личные чаты C2C, упоминания в групповых чатах (@messages) и сообщения в каналах гильдии с мультимедийным контентом (изображения, голосовые сообщения, видео, файлы).

Статус: встроенный плагин. Поддерживаются личные сообщения, групповые чаты, каналы гильдии и медиа. Реакции и потоки не поддерживаются.

## Встроенный плагин

В текущих версиях OpenClaw плагин QQ Bot включён по умолчанию, поэтому в обычных сборках не требуется отдельный шаг `openclaw plugins install`.

## Настройка

1. Перейдите на [QQ Open Platform](https://q.qq.com/) и отсканируйте QR-код с помощью приложения QQ на телефоне, чтобы зарегистрироваться или войти в систему.
2. Нажмите **Create Bot**, чтобы создать нового бота QQ.
3. Найдите **AppID** и **AppSecret** на странице настроек бота и скопируйте их.

> AppSecret не хранится в виде обычного текста — если вы покинете страницу, не сохранив его, вам придётся сгенерировать новый.

4. Добавьте канал:

```bash
openclaw channels add --channel qqbot --token "AppID:AppSecret"
```

5. Перезапустите шлюз.

Пути интерактивной настройки:

```bash
openclaw channels add
openclaw configure --section channels
```

## Конфигурация

Минимальная конфигурация:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecret: "YOUR_APP_SECRET",
    },
  },
}
```

Переменные окружения для учётной записи по умолчанию:

- `QQBOT_APP_ID`
- `QQBOT_CLIENT_SECRET`

AppSecret, хранящийся в файле:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "YOUR_APP_ID",
      clientSecretFile: "/path/to/qqbot-secret.txt",
    },
  },
}
```

Примечания:

- Резервное использование переменных окружения применяется только к учётной записи QQ Bot по умолчанию.
- Команда `openclaw channels add --channel qqbot --token-file ...` задаёт только AppSecret; AppID должен быть уже указан в конфигурации или в переменной `QQBOT_APP_ID`.
- Параметр `clientSecret` также принимает ввод в формате SecretRef, а не только строку в виде обычного текста.

### Настройка нескольких учётных записей

Запуск нескольких ботов QQ в рамках одного экземпляра OpenClaw:

```json5
{
  channels: {
    qqbot: {
      enabled: true,
      appId: "111111111",
      clientSecret: "secret-of-bot-1",
      accounts: {
        bot2: {
          enabled: true,
          appId: "222222222",
          clientSecret: "secret-of-bot-2",
        },
      },
    },
  },
}
```

Каждая учётная запись устанавливает собственное соединение WebSocket и поддерживает независимый кэш токенов (изолированный по `appId`).

Добавление второго бота через CLI:

```bash
openclaw channels add --channel qqbot --account bot2 --token "222222222:secret-of-bot-2"
```

### Голосовые функции (STT / TTS)

STT и TTS поддерживают двухуровневую конфигурацию с резервным переключением приоритетов:

| Настройка | Специфичная для плагина | Резервная в рамках фреймворка |
| ------- | -------------------- | ----------------------------- |
| STT | `channels.qqbot.stt` | `tools.media.audio.models[0]` |
| TTS | `channels.qqbot.tts` | `messages.tts` |

```json5
{
  channels: {
    qqbot: {
      stt: {
        provider: "your-provider",
        model: "your-stt-model",
      },
      tts: {
        provider: "your-provider",
        model: "your-tts-model",
        voice: "your-voice",
      },
    },
  },
}
```

Установите `enabled: false` для отключения любой из функций.

Поведение при загрузке/перекодировании исходящего аудио также можно настроить с помощью `channels.qqbot.audioFormatPolicy`:

- `sttDirectFormats`
- `uploadDirectFormats`
- `transcodeEnabled`

## Целевые форматы

| Формат | Описание |
| -------------------------- | ------------------ |
| `qqbot:c2c:OPENID` | Личный чат (C2C) |
| `qqbot:group:GROUP_OPENID` | Групповой чат |
| `qqbot:channel:CHANNEL_ID` | Канал гильдии |

> У каждого бота свой набор пользовательских OpenID. OpenID, полученный ботом A, **нельзя** использовать для отправки сообщений через бота B.

## Команды со слешем

Встроенные команды, перехватываемые до очереди ИИ:

| Команда | Описание |
| -------------- | ------------------------------------ |
| `/bot-ping` | Тест задержки |
| `/bot-version` | Показать версию фреймворка OpenClaw |
| `/bot-help` | Вывести список всех команд |
| `/bot-upgrade` | Показать ссылку на руководство по обновлению QQBot |
| `/bot-logs` | Экспортировать последние логи шлюза в файл |

Добавьте `?` к любой команде, чтобы получить справку по её использованию (например, `/bot-upgrade ?`).

## Устранение неполадок

- **Бот отвечает "улетел на Марс":** учётные данные не настроены или шлюз не запущен.
- **Нет входящих сообщений:** проверьте правильность `appId` и `clientSecret`, а также убедитесь, что бот включён на QQ Open Platform.
- **Настройка с `--token-file` по-прежнему показывает, что конфигурация не выполнена:** параметр `--token-file` задаёт только AppSecret. Вам всё ещё нужно указать `appId` в конфигурации или в переменной `QQBOT_APP_ID`.
- **Проактивные сообщения не доставляются:** QQ может перехватывать сообщения, инициированные ботом, если пользователь давно не взаимодействовал с ним.
- **Голос не транскрибируется:** убедитесь, что STT настроен и провайдер доступен.
