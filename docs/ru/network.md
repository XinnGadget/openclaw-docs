# Сетевой хаб

Этот хаб содержит основные документы о том, как OpenClaw подключает, сопрягает и обеспечивает безопасность устройств в localhost, LAN и tailnet.

## Базовая модель

Большинство операций проходят через шлюз (`openclaw gateway`) — единый долго работающий процесс, который управляет подключениями каналов и плоскостью управления WebSocket.

- **Приоритет loopback**: WebSocket шлюза по умолчанию использует `ws://127.0.0.1:18789`.
  Для привязок, не использующих loopback, требуется действительный путь аутентификации шлюза: аутентификация по токену/паролю с общим секретом или корректно настроенная привязка `trusted-proxy` без loopback.
- **Рекомендуется использовать один шлюз на хост**. Для изоляции запустите несколько шлюзов с изолированными профилями и портами ([Несколько шлюзов](/gateway/multiple-gateways)).
- **Хост Canvas** обслуживается на том же порту, что и шлюз (`/__openclaw__/canvas/`, `/__openclaw__/a2ui/`), и защищён аутентификацией шлюза при привязке за пределами loopback.
- **Удалённый доступ** обычно осуществляется через SSH-туннель или VPN Tailscale ([Удалённый доступ](/gateway/remote)).

Локальное доверие:

- Прямые подключения через loopback могут автоматически утверждаться для сопряжения, чтобы обеспечить удобство работы на одном хосте.
- В OpenClaw также предусмотрен узкий путь самоподключения для бэкенда/контейнера для доверенных потоков с общим секретом.
- Клиенты tailnet и LAN, включая привязки tailnet на одном хосте, по-прежнему требуют явного утверждения сопряжения.
