# Разрешения macOS (TCC)

Разрешения в macOS неустойчивы. TCC связывает предоставление разрешения с подписью кода приложения, идентификатором пакета и путём к файлу на диске. Если что‑либо из этого изменяется, macOS рассматривает приложение как новое и может отменить или скрыть запросы разрешений.

## Требования для стабильных разрешений

- Один и тот же путь: запускайте приложение из фиксированного местоположения (для OpenClaw — `dist/OpenClaw.app`).
- Один и тот же идентификатор пакета: изменение идентификатора пакета создаёт новый идентификатор разрешений.
- Подписанное приложение: сборки без подписи или с временной (ad-hoc) подписью не сохраняют разрешения.
- Стабильная подпись: используйте настоящий сертификат Apple Development или Developer ID, чтобы подпись оставалась неизменной при пересборке.

Временные (ad-hoc) подписи генерируют новый идентификатор при каждой сборке. macOS забывает предыдущие разрешения, и запросы могут полностью исчезнуть, пока устаревшие записи не будут удалены.

## Чек-лист восстановления при исчезновении запросов

1. Закройте приложение.
2. Удалите запись о приложении в "Системные настройки" → "Конфиденциальность и безопасность".
3. Перезапустите приложение из того же пути и повторно предоставьте разрешения.
4. Если запрос по‑прежнему не появляется, сбросьте записи TCC с помощью `tccutil` и попробуйте снова.
5. Некоторые разрешения появляются только после полной перезагрузки macOS.

Примеры сброса (замените идентификатор пакета при необходимости):

```bash
sudo tccutil reset Accessibility ai.openclaw.mac
sudo tccutil reset ScreenCapture ai.openclaw.mac
sudo tccutil reset AppleEvents
```

## Разрешения для файлов и папок (Desktop/Documents/Downloads)

macOS также может ограничивать доступ к папкам Desktop, Documents и Downloads для терминальных/фоновых процессов. Если чтение файлов или вывод списка директорий зависает, предоставьте доступ тому же контексту процесса, который выполняет файловые операции (например, Terminal/iTerm, приложение, запущенное через LaunchAgent, или процесс SSH).

Обходной путь: перемещайте файлы в рабочую область OpenClaw (`~/.openclaw/workspace`), если хотите избежать предоставления разрешений для каждой папки.

Если вы тестируете разрешения, всегда подписывайте приложение настоящим сертификатом. Временные (ad-hoc) сборки допустимы только для быстрых локальных запусков, где разрешения не имеют значения.
