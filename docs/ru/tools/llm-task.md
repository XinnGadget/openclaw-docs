# Задача для LLM

`llm-task` — это **опциональный плагин-инструмент**, который выполняет задачу для LLM с выводом только в формате JSON и возвращает структурированные данные (при необходимости — проверенные на соответствие JSON Schema).

Это оптимально для систем управления рабочими процессами вроде Lobster: вы можете добавить один шаг для LLM без написания кастомного кода OpenClaw для каждого рабочего процесса.

## Включение плагина

1. Включите плагин:

```json
{
  "plugins": {
    "entries": {
      "llm-task": { "enabled": true }
    }
  }
}
```

2. Добавьте инструмент в белый список (он зарегистрирован с параметром `optional: true`):

```json
{
  "agents": {
    "list": [
      {
        "id": "main",
        "tools": { "allow": ["llm-task"] }
      }
    ]
  }
}
```

## Конфигурация (опционально)

```json
{
  "plugins": {
    "entries": {
      "llm-task": {
        "enabled": true,
        "config": {
          "defaultProvider": "openai-codex",
          "defaultModel": "gpt-5.4",
          "defaultAuthProfileId": "main",
          "allowedModels": ["openai-codex/gpt-5.4"],
          "maxTokens": 800,
          "timeoutMs": 30000
        }
      }
    }
  }
}
```

`allowedModels` — белый список строк формата `provider/model`. Если параметр задан, все запросы, не входящие в список, отклоняются.

## Параметры инструмента

- `prompt` (строка, обязательно);
- `input` (любой тип, опционально);
- `schema` (объект, опциональный JSON Schema);
- `provider` (строка, опционально);
- `model` (строка, опционально);
- `thinking` (строка, опционально);
- `authProfileId` (строка, опционально);
- `temperature` (число, опционально);
- `maxTokens` (число, опционально);
- `timeoutMs` (число, опционально).

Параметр `thinking` поддерживает стандартные пресеты рассуждений OpenClaw, такие как `low` или `medium`.

## Вывод

Возвращает файл `details.json`, содержащий обработанный JSON (при наличии параметра `schema` выполняется проверка на соответствие схеме).

## Пример: шаг рабочего процесса Lobster

```lobster
openclaw.invoke --tool llm-task --action json --args-json '{
  "prompt": "На основе входящего письма верни намерение и черновик.",
  "thinking": "low",
  "input": {
    "subject": "Hello",
    "body": "Can you help?"
  },
  "schema": {
    "type": "object",
    "properties": {
      "intent": { "type": "string" },
      "draft": { "type": "string" }
    },
    "required": ["intent", "draft"],
    "additionalProperties": false
  }
}'
```

## Примечания по безопасности

- Инструмент работает **только с JSON** и даёт модели указание выводить исключительно JSON (без блоков кода, без комментариев).
- В рамках этого запуска модели не предоставляются никакие инструменты.
- Относитесь к выводу как к ненадёжному, если вы не проверили его с помощью `schema`.
- Размещайте этапы утверждения перед любыми шагами с побочными эффектами (send, post, exec).
