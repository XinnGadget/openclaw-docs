# Agent Send

Команда `openclaw agent` запускает один шаг агента из командной строки — без необходимости получать входящее чат-сообщение. Используйте её для скриптов рабочих процессов, тестирования и программной отправки данных.

## Быстрый старт

<Steps>
  <Step title="Запустить простой шаг агента">
    ```bash
    openclaw agent --message "What is the weather today?"
    ```

    Это отправляет сообщение через Gateway и выводит ответ.

  </Step>

  <Step title="Указать конкретного агента или сессию">
    ```bash
    # Указать конкретного агента
    openclaw agent --agent ops --message "Summarize logs"

    # Указать номер телефона (получает ключ сессии)
    openclaw agent --to +15555550123 --message "Status update"

    # Использовать существующую сессию
    openclaw agent --session-id abc123 --message "Continue the task"
    ```

  </Step>

  <Step title="Отправить ответ в канал">
    ```bash
    # Отправить в WhatsApp (канал по умолчанию)
    openclaw agent --to +15555550123 --message "Report ready" --deliver

    # Отправить в Slack
    openclaw agent --agent ops --message "Generate report" \
      --deliver --reply-channel slack --reply-to "#reports"
    ```

  </Step>
</Steps>

## Флаги

| Флаг | Описание |
| --- | --- |
| `--message \<text\>` | Сообщение для отправки (обязательно) |
| `--to \<dest\>` | Получить ключ сессии из целевого объекта (номер телефона, ID чата) |
| `--agent \<id\>` | Указать настроенного агента (использует его сессию `main`) |
| `--session-id \<id\>` | Использовать существующую сессию по ID |
| `--local` | Принудительно использовать встроенную среду выполнения на локальном компьютере (пропустить Gateway) |
| `--deliver` | Отправить ответ в чат-канал |
| `--channel \<name\>` | Канал доставки (WhatsApp, Telegram, Discord, Slack и т. д.) |
| `--reply-to \<target\>` | Переопределить целевой объект доставки |
| `--reply-channel \<name\>` | Переопределить канал доставки |
| `--reply-account \<id\>` | Переопределить ID учётной записи для доставки |
| `--thinking \<level\>` | Установить уровень "размышлений" (off, minimal, low, medium, high, xhigh) |
| `--verbose \<on\|full\|off\>` | Установить уровень детализации вывода |
| `--timeout \<seconds\>` | Переопределить тайм-аут агента |
| `--json` | Выводить данные в структурированном формате JSON |

## Поведение

- По умолчанию CLI работает **через Gateway**. Добавьте флаг `--local`, чтобы принудительно использовать встроенную среду выполнения на текущем компьютере.
- Если Gateway недоступен, CLI **переключается** на локальное выполнение.
- Выбор сессии: флаг `--to` получает ключ сессии (цели в виде групп/каналов сохраняют изоляцию; прямые чаты сводятся к `main`).
- Флаги уровня "размышлений" и детализации сохраняются в хранилище сессий.
- Вывод: по умолчанию — обычный текст, либо `--json` для структурированных данных + метаданных.

## Примеры

```bash
# Простой шаг с выводом в формате JSON
openclaw agent --to +15555550123 --message "Trace logs" --verbose on --json

# Шаг с указанием уровня "размышлений"
openclaw agent --session-id 1234 --message "Summarize inbox" --thinking medium

# Отправить в канал, отличный от канала сессии
openclaw agent --agent ops --message "Alert" --deliver --reply-channel telegram --reply-to "@admin"
```
