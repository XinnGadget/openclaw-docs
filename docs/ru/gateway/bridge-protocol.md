# Протокол моста (транспорт для устаревших узлов)

<Warning>
TCP-мост был **удален**. В текущих сборках OpenClaw отсутствует прослушиватель моста, а ключи конфигурации `bridge.*` больше не входят в схему. Эта страница сохранена только для исторических справок. Для всех клиентов узлов и операторов используйте [протокол шлюза](/gateway/protocol).
</Warning>

## Причины существования

- **Граница безопасности**: мост предоставляет небольшой белый список вместо полного набора API шлюза.
- **Сопряжение и идентификация узла**: допуск узла контролируется шлюзом и привязан к токену для каждого узла.
- **Удобство обнаружения**: узлы могут обнаруживать шлюзы через Bonjour в локальной сети или подключаться напрямую через tailnet.
- **Loopback WS**: полная плоскость управления WS остаётся локальной, если не туннелируется через SSH.

## Транспорт

- TCP, один JSON-объект в строке (JSONL).
- Опциональный TLS (когда `bridge.tls.enabled` имеет значение true).
- Исторический порт прослушивателя по умолчанию — `18790` (в текущих сборках TCP-мост не запускается).

Когда TLS включён, записи TXT для обнаружения содержат `bridgeTls=1` и `bridgeTlsSha256` в качестве несекретного подсказки. Обратите внимание, что записи TXT в Bonjour/mDNS не аутентифицированы; клиенты не должны рассматривать рекламируемый отпечаток как авторитетный пин без явного намерения пользователя или другой внеполосной проверки.

## Рукопожатие и сопряжение

1. Клиент отправляет `hello` с метаданными узла и токеном (если уже выполнено сопряжение).
2. Если сопряжение не выполнено, шлюз отвечает `error` (`NOT_PAIRED`/`UNAUTHORIZED`).
3. Клиент отправляет `pair-request`.
4. Шлюз ожидает подтверждения, затем отправляет `pair-ok` и `hello-ok`.

Исторически `hello-ok` возвращал `serverName` и мог включать `canvasHostUrl`.

## Фреймы

Клиент → Шлюз:

- `req` / `res`: RPC шлюза с ограниченной областью действия (чат, сессии, конфигурация, состояние, voicewake, skills.bins).
- `event`: сигналы узла (транскрипция голоса, запрос агента, подписка на чат, жизненный цикл exec).

Шлюз → Клиент:

- `invoke` / `invoke-res`: команды узла (`canvas.*`, `camera.*`, `screen.record`, `location.get`, `sms.send`).
- `event`: обновления чата для подписанных сессий.
- `ping` / `pong`: поддержание активности соединения.

Историческая реализация контроля белого списка находилась в `src/gateway/server-bridge.ts` (удалена).

## События жизненного цикла exec

Узлы могут отправлять события `exec.finished` или `exec.denied` для отображения активности system.run. Они сопоставляются с системными событиями в шлюзе. (Устаревшие узлы могут по-прежнему отправлять `exec.started`.)

Поля полезной нагрузки (все необязательные, если не указано иное):

- `sessionKey` (обязательно): сессия агента для получения системного события.
- `runId`: уникальный идентификатор exec для группировки.
- `command`: необработанная или форматированная строка команды.
- `exitCode`, `timedOut`, `success`, `output`: сведения о завершении (только для `exec.finished`).
- `reason`: причина отказа (только для `exec.denied`).

## Историческое использование tailnet

- Привязка моста к IP-адресу tailnet: `bridge.bind: "tailnet"` в `~/.openclaw/openclaw.json` (только исторически; `bridge.*` больше не действует).
- Клиенты подключаются через имя MagicDNS или IP-адрес tailnet.
- Bonjour **не** пересекает сети; при необходимости используйте ручной ввод хоста/порта или широкополосный DNS-SD.

## Версионирование

Мост имел **неявную версию v1** (без согласования минимальной/максимальной версии). Этот раздел предназначен только для исторических справок; текущие клиенты узлов и операторов используют WebSocket [протокол шлюза](/gateway/protocol).
