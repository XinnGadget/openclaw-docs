# Peekaboo Bridge (автоматизация UI в macOS)

OpenClaw может размещать **PeekabooBridge** в качестве локального брокера автоматизации UI с учётом разрешений. Это позволяет утилите командной строки `peekaboo` управлять автоматизацией UI, повторно используя разрешения TCC приложения macOS.

## Что это (и чем не является)

- **Хост**: OpenClaw.app может выступать в качестве хоста PeekabooBridge.
- **Клиент**: используйте CLI `peekaboo` (отдельная поверхность `openclaw ui ...` отсутствует).
- **UI**: визуальные наложения остаются в Peekaboo.app; OpenClaw — это тонкий хост-брокер.

## Включение моста

В приложении macOS:

- Настройки → **Включить Peekaboo Bridge**

При включении OpenClaw запускает локальный сервер UNIX-сокетов. Если функция отключена, хост останавливается, а `peekaboo` переключается на другие доступные хосты.

## Порядок обнаружения клиентов

Клиенты Peekaboo обычно пробуют хосты в следующем порядке:

1. Peekaboo.app (полный пользовательский опыт)
2. Claude.app (если установлено)
3. OpenClaw.app (тонкий брокер)

Используйте команду `peekaboo bridge status --verbose`, чтобы узнать, какой хост активен и какой путь к сокету используется. Вы можете переопределить его с помощью:

```bash
export PEEKABOO_BRIDGE_SOCKET=/path/to/bridge.sock
```

## Безопасность и разрешения

- Мост проверяет **цифровые подписи вызывающего кода**; применяется белый список TeamID (TeamID хоста Peekaboo + TeamID приложения OpenClaw).
- Запросы прерываются по таймауту примерно через 10 секунд.
- Если требуемые разрешения отсутствуют, мост возвращает чёткое сообщение об ошибке, а не запускает системные настройки.

## Поведение при создании снимков (автоматизация)

Снимки хранятся в памяти и автоматически удаляются через короткий промежуток времени. Если требуется более длительное хранение, повторно получите снимок с клиента.

## Устранение неполадок

- Если `peekaboo` сообщает "клиент моста не авторизован", убедитесь, что клиент правильно подписан, или запустите хост с `PEEKABOO_ALLOW_UNSIGNED_SOCKET_CLIENTS=1` **только в режиме отладки**.
- Если хосты не найдены, откройте одно из приложений-хостов (Peekaboo.app или OpenClaw.app) и убедитесь, что разрешения предоставлены.
