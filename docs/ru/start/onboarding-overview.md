# Обзор адаптации

В OpenClaw есть два пути адаптации. Оба настраивают аутентификацию, шлюз (Gateway) и необязательные чат-каналы — разница лишь в том, как вы взаимодействуете с процессом настройки.

## Какой путь выбрать?

|                | Адаптация через CLI                         | Адаптация через приложение для macOS      |
| -------------- | -------------------------------------- | ------------------------- |
| **Платформы**  | macOS, Linux, Windows (нативно или WSL2) | Только macOS                |
| **Интерфейс**  | Мастер в терминале                        | Пошаговый UI в приложении      |
| **Лучше подходит для**   | Серверов, headless-систем, полного контроля        | Настольных Mac, визуальной настройки |
| **Автоматизация** | `--non-interactive` для скриптов        | Только вручную               |
| **Команда**    | `openclaw onboard`                     | Запустить приложение            |

Большинству пользователей стоит начать с **адаптации через CLI** — она работает на всех платформах и даёт максимальный контроль.

## Что настраивается в процессе адаптации

Независимо от выбранного пути, адаптация настраивает:

1. **Поставщика модели и аутентификацию** — API-ключ, OAuth или токен настройки для выбранного поставщика.
2. **Рабочее пространство** — каталог для файлов агента, шаблонов начальной загрузки и памяти.
3. **Шлюз (Gateway)** — порт, адрес привязки, режим аутентификации.
4. **Каналы (необязательно)** — встроенные и предустановленные чат-каналы, такие как BlueBubbles, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams, Telegram, WhatsApp и другие.
5. **Демон (необязательно)** — фоновая служба, чтобы шлюз запускался автоматически.

## Адаптация через CLI

Выполните в любом терминале:

```bash
openclaw onboard
```

Добавьте `--install-daemon`, чтобы заодно установить фоновую службу за один шаг.

Полная справка: [Адаптация (CLI)](/start/wizard)
Документация по команде CLI: [`openclaw onboard`](/cli/onboard)

## Адаптация через приложение для macOS

Откройте приложение OpenClaw. При первом запуске мастер проведёт вас через те же шаги с помощью визуального интерфейса.

Полная справка: [Адаптация (приложение для macOS)](/start/onboarding)

## Пользовательские или неперечисленные поставщики

Если ваш поставщик не указан в процессе адаптации, выберите **Пользовательский поставщик (Custom Provider)** и введите:

- Режим совместимости с API (совместимый с OpenAI, совместимый с Anthropic или автоматическое определение).
- Базовый URL и API-ключ.
- ID модели и необязательный псевдоним.

Несколько пользовательских конечных точек могут сосуществовать — каждой присваивается собственный ID конечной точки.
