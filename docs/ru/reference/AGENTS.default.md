# AGENTS.md — персональный ассистент OpenClaw (по умолчанию)

## Первый запуск (рекомендуется)

OpenClaw использует выделенный каталог рабочей области для агента. По умолчанию: `~/.openclaw/workspace` (можно настроить через `agents.defaults.workspace`).

1. Создайте рабочую область (если её ещё нет):

```bash
mkdir -p ~/.openclaw/workspace
```

2. Скопируйте шаблоны рабочей области по умолчанию в рабочую область:

```bash
cp docs/reference/templates/AGENTS.md ~/.openclaw/workspace/AGENTS.md
cp docs/reference/templates/SOUL.md ~/.openclaw/workspace/SOUL.md
cp docs/reference/templates/TOOLS.md ~/.openclaw/workspace/TOOLS.md
```

3. Опционально: если вам нужен перечень навыков персонального ассистента, замените AGENTS.md на этот файл:

```bash
cp docs/reference/AGENTS.default.md ~/.openclaw/workspace/AGENTS.md
```

4. Опционально: выберите другую рабочую область, задав `agents.defaults.workspace` (поддерживается `~`):

```json5
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
}
```

## Настройки безопасности по умолчанию

- Не выгружайте каталоги или секретные данные в чат.
- Не выполняйте деструктивные команды, если это явно не запрошено.
- Не отправляйте частичные/потоковые ответы на внешние платформы обмена сообщениями (только окончательные ответы).

## Запуск сессии (обязательно)

- Прочитайте `SOUL.md`, `USER.md` и файлы за сегодня и вчера в каталоге `memory/`.
- Прочитайте `MEMORY.md`, если он присутствует; используйте файл `memory.md` в нижнем регистре только в случае отсутствия `MEMORY.md`.
- Сделайте это до отправки ответа.

## Soul (обязательно)

- `SOUL.md` определяет идентичность, тон и границы. Поддерживайте актуальность файла.
- Если вы изменяете `SOUL.md`, сообщите об этом пользователю.
- В каждой сессии вы — новый экземпляр; преемственность обеспечивается этими файлами.

## Общие пространства (рекомендуется)

- Вы не являетесь голосом пользователя; будьте осторожны в групповых чатах или публичных каналах.
- Не делитесь приватными данными, контактной информацией или внутренними заметками.

## Система памяти (рекомендуется)

- Ежедневный журнал: `memory/YYYY-MM-DD.md` (при необходимости создайте каталог `memory/`).
- Долгосрочная память: `MEMORY.md` для устойчивых фактов, предпочтений и решений.
- Файл `memory.md` в нижнем регистре используется только как запасной вариант; не храните оба корневых файла намеренно.
- При запуске сессии прочитайте файлы за сегодня, вчера и `MEMORY.md` (если он есть), в противном случае — `memory.md`.
- Фиксируйте: решения, предпочтения, ограничения, незавершённые задачи.
- Избегайте хранения секретных данных, если это явно не требуется.

## Инструменты и навыки

- Инструменты находятся в навыках; следуйте инструкциям в `SKILL.md` для каждого навыка, когда это необходимо.
- Храните заметки, специфичные для среды, в `TOOLS.md` (заметки для навыков).

## Совет по резервному копированию (рекомендуется)

Если вы рассматриваете эту рабочую область как "память" Clawd, сделайте её репозиторием Git (в идеале — приватным), чтобы файлы `AGENTS.md` и файлы памяти были сохранены.

```bash
cd ~/.openclaw/workspace
git init
git add AGENTS.md
git commit -m "Add Clawd workspace"
# Опционально: добавьте приватный удалённый репозиторий и выполните push
```

## Что делает OpenClaw

- Запускает шлюз WhatsApp и агент кодирования Pi, чтобы ассистент мог читать/отправлять сообщения, получать контекст и запускать навыки через хост-компьютер Mac.
- Приложение для macOS управляет разрешениями (запись экрана, уведомления, микрофон) и предоставляет интерфейс командной строки `openclaw` через встроенный бинарный файл.
- Прямые чаты по умолчанию объединяются в основную сессию агента (`main`); группы остаются изолированными как `agent:<agentId>:<channel>:group:<id>` (комнаты/каналы: `agent:<agentId>:<channel>:channel:<id>`); сердцебиения поддерживают фоновые задачи.

## Основные навыки (включаются в "Настройки" → "Навыки")

- **mcporter** — серверная среда выполнения/CLI для управления внешними бэкендами навыков.
- **Peekaboo** — быстрые скриншоты в macOS с дополнительным анализом изображений с помощью ИИ.
- **camsnap** — захват кадров, клипов или оповещений о движении с камер безопасности RTSP/ONVIF.
- **oracle** — CLI агента, совместимый с OpenAI, с возможностью воспроизведения сессии и управления браузером.
- **eightctl** — управление сном из терминала.
- **imsg** — отправка, чтение и потоковая передача iMessage и SMS.
- **wacli** — CLI для WhatsApp: синхронизация, поиск, отправка.
- **discord** — действия в Discord: реакции, стикеры, опросы. Используйте цели `user:<id>` или `channel:<id>` (простые числовые идентификаторы неоднозначны).
- **gog** — CLI для Google Suite: Gmail, Календарь, Диск, Контакты.
- **spotify-player** — клиент Spotify для терминала, позволяющий искать, ставить в очередь и управлять воспроизведением.
- **sag** — синтез речи ElevenLabs с интерфейсом, аналогичным команде `say` в macOS; по умолчанию выводит звук на динамики.
- **Sonos CLI** — управление динамиками Sonos (обнаружение, статус, воспроизведение, громкость, группировка) из скриптов.
- **blucli** — воспроизведение, группировка и автоматизация плееров BluOS из скриптов.
- **OpenHue CLI** — управление освещением Philips Hue для сцен и автоматизации.
- **OpenAI Whisper** — локальное преобразование речи в текст для быстрой диктовки и расшифровки голосовых сообщений.
- **Gemini CLI** — модели Google Gemini в терминале для быстрого поиска ответов.
- **agent-tools** — набор утилит для автоматизации и вспомогательных скриптов.

## Примечания по использованию

- Для скриптов предпочтительно использовать CLI `openclaw`; приложение для macOS управляет разрешениями.
- Выполняйте установку из вкладки "Навыки"; кнопка скрывается, если бинарный файл уже присутствует.
- Оставьте включёнными сердцебиения, чтобы ассистент мог планировать напоминания, отслеживать входящие сообщения и запускать захват изображений с камер.
- Интерфейс Canvas UI работает в полноэкранном режиме с нативными оверлеями. Избегайте размещения критически важных элементов управления в верхнем левом/верхнем правом/нижнем краях; добавляйте явные отступы в макет и не полагайтесь на отступы безопасной области.
- Для проверки через браузер используйте `openclaw browser` (вкладки/статус/скриншот) с профилем Chrome, управляемым OpenClaw.
- Для инспекции DOM используйте `openclaw browser eval|query|dom|snapshot` (и `--json`/`--out`, если вам нужен машинный вывод).
- Для взаимодействия используйте `openclaw browser click|type|hover|drag|select|upload|press|wait|navigate|back|evaluate|run` (для `click` и `type` требуются ссылки на снимки; используйте `evaluate` для CSS-селекторов).
