# Состояния иконки в строке меню

Автор: steipete · Обновлено: 2025-12-06 · Область применения: приложение для macOS (`apps/macos`)

- **Неактивное состояние (Idle):** стандартная анимация иконки (мигание, периодическое покачивание).
- **Приостановлено (Paused):** элемент статуса использует `appearsDisabled`; движение отсутствует.
- **Голосовой триггер (большие уши, Voice trigger):** детектор пробуждения по голосу вызывает `AppState.triggerVoiceEars(ttl: nil)`, когда распознаётся ключевое слово, при этом `earBoostActive=true` на время записи высказывания. Уши увеличиваются в размере (в 1,9 раза), появляются круглые отверстия в ушах для лучшей читаемости, затем возвращаются в исходное состояние через `stopVoiceEars()` после 1 секунды тишины. Активируется только через внутренний голосовой конвейер приложения.
- **Работа (агент запущен, Working):** значение `AppState.isWorking=true` запускает микродвижение "бег хвоста/лапок" (tail/leg scurry): более быстрое покачивание лапок и небольшой сдвиг во время выполнения работы. В настоящее время переключается при запуске агента WebChat; добавьте аналогичное переключение для других длительных задач при их реализации.

Точки подключения

- Пробуждение по голосу: во время выполнения runtime/tester вызывается `AppState.triggerVoiceEars(ttl: nil)` при срабатывании триггера и `stopVoiceEars()` после 1 секунды тишины, чтобы соответствовать окну захвата.
- Активность агента: установите `AppStateStore.shared.setWorking(true/false)` для периодов работы (уже реализовано для вызова агента WebChat). Старайтесь делать периоды короткими и сбрасывать их в блоках `defer`, чтобы избежать зависания анимаций.

Формы и размеры

- Базовая иконка рисуется в `CritterIconRenderer.makeIcon(blink:legWiggle:earWiggle:earScale:earHoles:)`.
- Масштаб ушей по умолчанию равен `1.0`; при голосовом усилении устанавливается `earScale=1.9` и переключается `earHoles=true` без изменения общего фрейма (шаблонное изображение 18×18 пт рендерится в буфер Retina размером 36×36 px).
- Для "бега" используется покачивание лапок до ~1,0 с небольшим горизонтальным колебанием; это добавляется к любому уже имеющемуся покачиванию в неактивном состоянии.

Примечания по поведению

- Нет внешнего переключателя CLI/брокера для ушей/работы; оставьте управление внутренними сигналами приложения, чтобы избежать случайного "махания".
- Старайтесь делать TTL короткими (<10 с), чтобы иконка быстро возвращалась в исходное состояние, если задача зависла.
