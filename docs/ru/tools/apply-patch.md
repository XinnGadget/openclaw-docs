# Инструмент apply_patch

Применяет изменения в файлах с использованием структурированного формата патча. Это оптимально для правок в нескольких файлах или нескольких фрагментах, когда один вызов `edit` будет ненадёжным.

Инструмент принимает единственную строку `input`, которая включает одну или несколько файловых операций:

```
*** Begin Patch
*** Add File: path/to/file.txt
+line 1
+line 2
*** Update File: src/app.ts
@@
-old line
+new line
*** Delete File: obsolete.txt
*** End Patch
```

## Параметры

- `input` (обязательно): полное содержимое патча, включая `*** Begin Patch` и `*** End Patch`.

## Примечания

- Пути в патчах поддерживают относительные пути (относительно директории рабочей области) и абсолютные пути.
- Параметр `tools.exec.applyPatch.workspaceOnly` по умолчанию имеет значение `true` (ограничено рабочей областью). Устанавливайте его в `false` только в том случае, если вы намеренно хотите, чтобы `apply_patch` выполнял запись или удаление за пределами директории рабочей области.
- Используйте `*** Move to:` внутри фрагмента `*** Update File:` для переименования файлов.
- `*** End of File` обозначает вставку только в конец файла (EOF), если это необходимо.
- Доступен по умолчанию для моделей OpenAI и OpenAI Codex. Установите `tools.exec.applyPatch.enabled: false`, чтобы отключить его.
- При необходимости можно ограничить использование по модели с помощью `tools.exec.applyPatch.allowModels`.
- Конфигурация находится только в разделе `tools.exec`.

## Пример

```json
{
  "tool": "apply_patch",
  "input": "*** Begin Patch\n*** Update File: src/index.ts\n@@\n-const foo = 1\n+const foo = 2\n*** End Patch"
}
```
