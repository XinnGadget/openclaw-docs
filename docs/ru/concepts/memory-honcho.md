# Память Honcho

[Honcho](https://honcho.dev) добавляет встроенную в ИИ память в OpenClaw. Сервис сохраняет диалоги в специализированной системе и со временем формирует модели пользователя и агента, обеспечивая контекст между сессиями — сверх того, что предоставляют файлы Markdown в рабочей области.

## Что предоставляется

- **Память между сессиями** — диалоги сохраняются после каждого хода, поэтому контекст сохраняется при перезапуске сессии, уплотнении данных и переключении каналов.
- **Моделирование пользователя** — Honcho ведёт профиль для каждого пользователя (предпочтения, факты, стиль общения) и для агента (характер, усвоенные модели поведения).
- **Семантический поиск** — поиск по наблюдениям из прошлых диалогов, а не только из текущей сессии.
- **Поддержка мультиагентности** — родительские агенты автоматически отслеживают порождённые суб-агенты, при этом родители добавляются в качестве наблюдателей в дочерние сессии.

## Доступные инструменты

Honcho регистрирует инструменты, которые агент может использовать во время диалога:

**Извлечение данных (быстро, без вызова LLM):**

| Инструмент | Что делает |
| --- | --- |
| `honcho_context` | Полное представление пользователя между сессиями |
| `honcho_search_conclusions` | Семантический поиск по сохранённым выводам |
| `honcho_search_messages` | Поиск сообщений между сессиями (с фильтрацией по отправителю, дате) |
| `honcho_session` | История и сводка текущей сессии |

**Вопросы и ответы (на базе LLM):**

| Инструмент | Что делает |
| --- | --- |
| `honcho_ask` | Задавать вопросы о пользователе. `depth='quick'` — для фактов, `'thorough'` — для синтеза |

## Начало работы

Установите плагин и выполните настройку:

```bash
openclaw plugins install @honcho-ai/openclaw-honcho
openclaw honcho setup
openclaw gateway --force
```

Команда настройки запросит ваши учётные данные API, запишет конфигурацию и при необходимости перенесёт существующие файлы памяти рабочей области.

<Info>
Honcho может работать полностью локально (самостоятельный хостинг) или через управляемый API на `api.honcho.dev`. Для варианта с самостоятельным хостингом внешние зависимости не требуются.
</Info>

## Конфигурация

Настройки находятся в `plugins.entries["openclaw-honcho"].config`:

```json5
{
  plugins: {
    entries: {
      "openclaw-honcho": {
        config: {
          apiKey: "your-api-key", // опустить для самостоятельного хостинга
          workspaceId: "openclaw", // изоляция памяти
          baseUrl: "https://api.honcho.dev",
        },
      },
    },
  },
}
```

Для экземпляров с самостоятельным хостингом укажите в `baseUrl` свой локальный сервер (например, `http://localhost:8000`) и опустите ключ API.

## Перенос существующей памяти

Если у вас есть существующие файлы памяти рабочей области (`USER.md`, `MEMORY.md`, `IDENTITY.md`, `memory/`, `canvas/`), команда `openclaw honcho setup` обнаружит их и предложит перенести.

<Info>
Перенос не приводит к удалению данных — файлы загружаются в Honcho. Исходные файлы никогда не удаляются и не перемещаются.
</Info>

## Как это работает

После каждого хода ИИ диалог сохраняется в Honcho. Сообщения как пользователя, так и агента фиксируются, что позволяет Honcho со временем формировать и уточнять свои модели.

Во время диалога инструменты Honcho запрашивают сервис на этапе `before_prompt_build`, внедряя соответствующий контекст до того, как модель увидит промпт. Это обеспечивает точные границы ходов и релевантное восстановление данных.

## Honcho против встроенной памяти

| | Встроенная / QMD | Honcho |
| --- | --- | --- |
| **Хранение** | Файлы Markdown в рабочей области | Специализированный сервис (локальный или размещённый) |
| **Между сессиями** | Через файлы памяти | Автоматически, встроенно |
| **Моделирование пользователя** | Вручную (запись в MEMORY.md) | Автоматические профили |
| **Поиск** | Векторный + ключевой (гибридный) | Семантический по наблюдениям |
| **Мультиагентность** | Не отслеживается | Отслеживание родительских/дочерних агентов |
| **Зависимости** | Нет (встроено) или бинарный файл QMD | Установка плагина |

Honcho и встроенная система памяти могут работать вместе. При настройке QMD становятся доступны дополнительные инструменты для поиска в локальных файлах Markdown наряду с памятью Honcho между сессиями.

## Команды CLI

```bash
openclaw honcho setup                        # Настроить ключ API и перенести файлы
openclaw honcho status                       # Проверить статус подключения
openclaw honcho ask <question>               # Запросить у Honcho информацию о пользователе
openclaw honcho search <query> [-k N] [-d D] # Семантический поиск по памяти
```
