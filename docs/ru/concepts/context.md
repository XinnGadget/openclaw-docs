# Контекст

"Контекст" — это **всё, что OpenClaw отправляет модели для выполнения запуска**. Он ограничен **окном контекста** модели (лимитом токенов).

Упрощённая модель для начинающих:

- **Системный промпт** (формируется OpenClaw): правила, инструменты, список навыков, время/время выполнения, а также внедряемые файлы рабочей области.
- **История диалога**: ваши сообщения + сообщения ассистента в рамках текущей сессии.
- **Вызовы инструментов/результаты + вложения**: вывод команд, чтение файлов, изображения/аудио и т. д.

Контекст — **не то же самое**, что "память": память может быть сохранена на диске и загружена позже; контекст — это то, что находится в текущем окне модели.

## Быстрый старт (проверка контекста)

- `/status` → быстрый просмотр "насколько заполнено моё окно?" + настройки сессии.
- `/context list` → что внедрено + приблизительные размеры (по файлам и суммарно).
- `/context detail` → более детальный разбор: по файлам, по размерам схем инструментов, по размерам записей навыков и размеру системного промпта.
- `/usage tokens` → добавление в обычные ответы нижнего колонтитула с использованием токенов для каждого ответа.
- `/compact` → суммирование более старой истории в компактную запись для освобождения места в окне.

См. также: [Слэш-команды](/tools/slash-commands), [Использование токенов и затраты](/reference/token-use), [Компактизация](/concepts/compaction).

## Что учитывается в окне контекста

Учитывается всё, что получает модель, в том числе:

- Системный промпт (все разделы).
- История диалога.
- Вызовы инструментов + результаты инструментов.
- Вложения/транскрипты (изображения/аудио/файлы).
- Суммарные данные компактизации и артефакты обрезки.
- "Обёртки" провайдера или скрытые заголовки (не видны, но учитываются).

## Как OpenClaw формирует системный промпт

Системный промпт **принадлежит OpenClaw** и перестраивается при каждом запуске. Он включает:

- Список инструментов + краткие описания.
- Список навыков (только метаданные; см. ниже).
- Расположение рабочей области.
- Время (UTC + преобразованное время пользователя, если настроено).
- Метаданные времени выполнения (хост/ОС/модель/мышление).
- Внедрённые загрузочные файлы рабочей области в разделе **Контекст проекта**.

Полный разбор: [Системный промпт](/concepts/system-prompt).

## Внедрённые файлы рабочей области (Контекст проекта)

По умолчанию OpenClaw внедряет фиксированный набор файлов рабочей области (если они присутствуют):

- `AGENTS.md`
- `SOUL.md`
- `TOOLS.md`
- `IDENTITY.md`
- `USER.md`
- `HEARTBEAT.md`
- `BOOTSTRAP.md` (только при первом запуске)

Большие файлы усекаются по отдельности с использованием параметра `agents.defaults.bootstrapMaxChars` (по умолчанию — 20 000 символов). OpenClaw также устанавливает общий лимит на внедрение загрузочных файлов с помощью параметра `agents.defaults.bootstrapTotalMaxChars` (по умолчанию — 150 000 символов). Команда `/context` показывает **исходный и внедрённый** размеры, а также факт усечения.

При усечении среда выполнения может внедрить в промпт блок предупреждения в разделе "Контекст проекта". Настройте это с помощью параметра `agents.defaults.bootstrapPromptTruncationWarning` (`off`, `once`, `always`; по умолчанию — `once`).

## Навыки: внедрение и загрузка по требованию

Системный промпт включает компактный **список навыков** (название + описание + расположение). Этот список имеет реальные накладные расходы.

Инструкции по навыкам **по умолчанию не включаются**. Ожидается, что модель будет использовать команду `read` для файла `SKILL.md` **только при необходимости**.

## Инструменты: две составляющие затрат

Инструменты влияют на контекст двумя способами:

1. **Текст списка инструментов** в системном промпте (то, что вы видите как "Инструменты").
2. **Схемы инструментов** (JSON). Они отправляются модели, чтобы она могла вызывать инструменты. Они учитываются в контексте, даже если вы не видите их в виде обычного текста.

Команда `/context detail` разбивает самые крупные схемы инструментов, чтобы вы могли увидеть, что занимает больше всего места.

## Команды, директивы и "встроенные сокращения"

Слэш-команды обрабатываются шлюзом. Существует несколько типов поведения:

- **Автономные команды**: сообщение, состоящее только из `/...`, выполняется как команда.
- **Директивы**: `/think`, `/verbose`, `/reasoning`, `/elevated`, `/model`, `/queue` удаляются до того, как модель увидит сообщение.
  - Сообщения, состоящие только из директив, сохраняют настройки сессии.
  - Встроенные директивы в обычном сообщении действуют как подсказки для конкретного сообщения.
- **Встроенные сокращения** (только для отправителей из белого списка): некоторые токены `/...` внутри обычного сообщения могут выполняться немедленно (например, "hey /status") и удаляются до того, как модель увидит оставшийся текст.

Подробности: [Слэш-команды](/tools/slash-commands).

## Сессии, компактизация и обрезка (что сохраняется)

То, что сохраняется между сообщениями, зависит от механизма:

- **Обычная история** сохраняется в транскрипте сессии до тех пор, пока не будет компактизирована/обрезана в соответствии с политикой.
- **Компактизация** сохраняет сводку в транскрипте и оставляет последние сообщения нетронутыми.
- **Обрезка** удаляет старые результаты инструментов из **промпта в памяти** для запуска, но не переписывает транскрипт.
