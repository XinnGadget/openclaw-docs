# Проверки работоспособности в macOS

Как узнать, находится ли связанный канал в работоспособном состоянии, с помощью приложения в строке меню.

## Строка меню

- Точка статуса теперь отражает состояние работоспособности Baileys:
  - Зелёный: связан + сокет недавно открыт.
  - Оранжевый: подключение/повторная попытка.
  - Красный: выход из системы или сбой проверки.
- Вторая строка отображает "связан · аутентификация 12 мин" или причину сбоя.
- Пункт меню "Выполнить проверку работоспособности" запускает проверку по запросу.

## Настройки

- На вкладке "Общие" появляется карточка "Работоспособность", в которой отображаются: время с момента аутентификации, путь/количество в хранилище сессий, время последней проверки, последний код ошибки/статуса, а также кнопки "Выполнить проверку работоспособности" / "Показать журналы".
- Используется кэшированный снимок, чтобы интерфейс загружался мгновенно и корректно работал в офлайн‑режиме.
- На вкладке **"Каналы"** отображается статус канала и элементы управления для WhatsApp/Telegram (QR‑код для входа, выход из системы, проверка, последнее отключение/ошибка).

## Как работает проверка

- Приложение запускает команду `openclaw health --json` через `ShellExecutor` примерно каждые 60 секунд и по запросу. Проверка загружает учётные данные и сообщает о статусе без отправки сообщений.
- Последний корректный снимок и последняя ошибка кэшируются отдельно, чтобы избежать мерцания; отображается отметка времени для каждого из них.

## Если есть сомнения

- Вы по‑прежнему можете использовать поток CLI в разделе [Работоспособность шлюза](/gateway/health) (`openclaw status`, `openclaw status --deep`, `openclaw health --json`) и просматривать журнал `/tmp/openclaw/openclaw-*.log` для `web-heartbeat` / `web-reconnect`.
