# Диффы

`diffs` — опциональный плагин с краткими встроенными системными инструкциями и сопутствующим навыком, который преобразует содержимое изменений в артефакт с диффами только для чтения для агентов.

Плагин принимает:

- текст `before` и `after`
- унифицированный `patch`

Может возвращать:

- URL-просмотрщика для отображения в canvas
- путь к отрендеренному файлу (PNG или PDF) для передачи в сообщении
- оба результата за один вызов

При активации плагин добавляет краткие инструкции по использованию в область системных подсказок, а также предоставляет подробный навык для случаев, когда агенту нужны более полные инструкции.

## Быстрый старт

1. Активируйте плагин.
2. Вызовите `diffs` с `mode: "view"` для потоков с приоритетом canvas.
3. Вызовите `diffs` с `mode: "file"` для потоков доставки файлов в чате.
4. Вызовите `diffs` с `mode: "both"`, если вам нужны оба артефакта.

## Активация плагина

```json5
{
  plugins: {
    entries: {
      diffs: {
        enabled: true,
      },
    },
  },
}
```

## Отключение встроенных системных инструкций

Если вы хотите оставить плагин `diffs` включённым, но отключить его встроенные подсказки в системных промтах, установите `plugins.entries.diffs.hooks.allowPromptInjection` в `false`:

```json5
{
  plugins: {
    entries: {
      diffs: {
        enabled: true,
        hooks: {
          allowPromptInjection: false,
        },
      },
    },
  },
}
```

Это заблокирует хук `before_prompt_build` плагина diffs, сохранив при этом сам плагин, инструмент и сопутствующий навык доступными.

Если вы хотите отключить и подсказки, и инструмент, просто деактивируйте плагин.

## Типичный рабочий процесс агента

1. Агент вызывает `diffs`.
2. Агент считывает поля `details`.
3. Агент либо:
   - открывает `details.viewerUrl` с помощью `canvas present`
   - отправляет `details.filePath` с `message`, используя `path` или `filePath`
   - выполняет оба действия

## Примеры входных данных

До и после:

```json
{
  "before": "# Hello\n\nOne",
  "after": "# Hello\n\nTwo",
  "path": "docs/example.md",
  "mode": "view"
}
```

Патч:

```json
{
  "patch": "diff --git a/src/example.ts b/src/example.ts\n--- a/src/example.ts\n+++ b/src/example.ts\n@@ -1 +1 @@\n-const x = 1;\n+const x = 2;\n",
  "mode": "both"
}
```

## Справочник по входным данным инструмента

Все поля необязательны, если не указано иное:

- `before` (`string`): исходный текст. Требуется вместе с `after`, если `patch` не указан.
- `after` (`string`): обновлённый текст. Требуется вместе с `before`, если `patch` не указан.
- `patch` (`string`): текст унифицированного диффа. Исключается вместе с `before` и `after`.
- `path` (`string`): отображаемое имя файла для режимов before и after.
- `lang` (`string`): подсказка для переопределения языка в режимах before и after. Неизвестные значения возвращаются к обычному тексту.
- `title` (`string`): переопределение заголовка просмотрщика.
- `mode` (`"view" | "file" | "both"`): режим вывода. По умолчанию — значение по умолчанию плагина `defaults.mode`.
  Устаревший псевдоним: `"image"` работает как `"file"` и по-прежнему принимается для обратной совместимости.
- `theme` (`"light" | "dark"`): тема просмотрщика. По умолчанию — значение по умолчанию плагина `defaults.theme`.
- `layout` (`"unified" | "split"`): макет дифффа. По умолчанию — значение по умолчанию плагина `defaults.layout`.
- `expandUnchanged` (`boolean`): развернуть неизменённые секции, если доступен полный контекст. Только для отдельного вызова (не является ключом по умолчанию для плагина).
- `fileFormat` (`"png" | "pdf"`): формат отрендеренного файла. По умолчанию — значение по умолчанию плагина `defaults.fileFormat`.
- `fileQuality` (`"standard" | "hq" | "print"`): предустановка качества для рендеринга PNG или PDF.
- `fileScale` (`number`): переопределение масштаба устройства (`1`–`4`).
- `fileMaxWidth` (`number`): максимальная ширина рендера в CSS-пикселях (`640`–`2400`).
- `ttlSeconds` (`number`): время жизни артефакта в секундах для просмотрщика и автономных файловых выходов. По умолчанию 1800, максимум 21600.
- `baseUrl` (`string`): переопределение источника URL просмотрщика. Переопределяет `viewerBaseUrl` плагина. Должен быть `http` или `https`, без запроса/хеша.

Устаревшие псевдонимы входных данных, по-прежнему принимаемые для обратной совместимости:

- `format` -> `fileFormat`
- `imageFormat` -> `fileFormat`
- `imageQuality` -> `fileQuality`
- `imageScale` -> `fileScale`
- `imageMaxWidth` -> `fileMaxWidth`

Проверка и ограничения:

- `before` и `after` — максимум 512 КиБ каждое.
- `patch` — максимум 2 МиБ.
- `path` — максимум 2048 байт.
- `lang` — максимум 128 байт.
- `title` — максимум 1024 байт.
- Ограничение сложности патча: максимум 128 файлов и 120 000 общих строк.
- Совместное использование `patch` и `before` или `after` отклоняется.
- Ограничения безопасности отрендеренных файлов (применяются к PNG и PDF):
  - `fileQuality: "standard"`: максимум 8 МП (8 000 000 отрендеренных пикселей).
  - `fileQuality: "hq"`: максимум 14 МП (14 000 000 отрендеренных пикселей).
  - `fileQuality: "print"`: максимум 24 МП (24 000 000 отрендеренных пикселей).
  - Для PDF также установлено ограничение в 50 страниц.

## Контракт деталей вывода

Инструмент возвращает структурированные метаданные в `details`.

Общие поля для режимов, создающих просмотрщик:

- `artifactId`
- `viewerUrl`
- `viewerPath`
- `title`
- `expiresAt`
- `inputKind`
- `fileCount`
- `mode`
- `context` (`agentId`, `sessionId`, `messageChannel`, `agentAccountId`, если доступны)

Поля файлов при рендеринге PNG или PDF:

- `artifactId`
- `expiresAt`
- `filePath`
- `path` (то же значение, что и `filePath`, для совместимости с инструментами сообщений)
- `fileBytes`
- `fileFormat`
- `fileQuality`
- `fileScale`
- `fileMaxWidth`

Также возвращаются псевдонимы совместимости для существующих вызывающих сторон:

- `format` (то же значение, что и `fileFormat`)
- `imagePath` (то же значение, что и `filePath`)
- `imageBytes` (то же значение, что и `fileBytes`)
- `imageQuality` (то же значение, что и `fileQuality`)
- `imageScale` (то же значение, что и `fileScale`)
- `imageMaxWidth` (то же значение, что и `fileMaxWidth`)

Краткое описание поведения режимов:

- `mode: "view"`: только поля просмотрщика.
- `mode: "file"`: только поля файла, без артефакта просмотрщика.
- `mode: "both"`: поля просмотрщика плюс поля файла. Если рендеринг файла завершается ошибкой, просмотрщик по-прежнему возвращает результат с `fileError` и псевдонимом совместимости `imageError`.

## Свёрнутые неизменённые секции

- Просмотрщик может отображать строки вида `N неизменённых строк`.
- Элементы управления развёртыванием для этих строк условны и не гарантируются для каждого типа входных данных.
- Элементы управления развёртыванием появляются, когда отрендеренный дифф содержит расширяемые контекстные данные — это типично для входных данных before и after.
- Для многих унифицированных входных данных патчей опущенные тела контекста недоступны в проанализированных фрагментах патча, поэтому строка может отображаться без элементов управления развёртыванием. Это ожидаемое поведение.
- `expandUnchanged` применяется только при наличии расширяемого контекста.

## Настройки плагина по умолчанию

Установите глобальные настройки плагина в `~/.openclaw/openclaw.json`:

```json5
{
  plugins: {
    entries: {
      diffs: {
        enabled: true,
        config: {
          defaults: {
            fontFamily: "Fira Code",
            fontSize: 15,
            lineSpacing: 1.6,
            layout: "unified",
            showLineNumbers: true,
            diffIndicators: "bars",
            wordWrap: true,
            background: true,
            theme: "dark",
            fileFormat: "png",
            fileQuality: "standard",
            fileScale: 2,
            fileMaxWidth: 960,
            mode: "both",
          },
        },
      },
    },
  },
}
```

Поддерживаемые настройки по умолчанию:

- `fontFamily`
- `fontSize`
- `lineSpacing`
- `layout`
- `showLineNumbers`
- `diffIndicators`
- `wordWrap`
- `background`
- `theme`
- `fileFormat`
- `fileQuality`
- `fileScale`
- `fileMaxWidth`
- `mode`

Явные параметры инструмента переопределяют эти настройки.

Постоянная конфигурация URL просмотрщика:

- `viewerBaseUrl` (`string`, необязательно)
  - Резервный вариант, принадлежащий плагину, для возвращаемых ссылок просмотрщика, когда вызов инструмента не передаёт `baseUrl`.
  - Должен быть `http` или `https`, без запроса/хеша.

Пример:

```json5
{
  plugins: {
    entries: {
      diffs: {
        enabled: true,
        config: {
          viewerBaseUrl: "https://gateway.example.com/openclaw",
        },
      },
    },
  },
}
```

## Конфигурация безопасности

- `security.allowRemoteViewer` (`boolean`, по умолчанию `false`)
  - `false`: запросы к маршрутам просмотрщика не из loopback отклоняются.
  - `true`: удалённые просмотрщики разрешены, если путь с токеном действителен.

Пример:

```json5
{
  plugins: {
    entries: {
      diffs: {
        enabled: true,
        config: {
          security: {
            allowRemoteViewer: false,
          },
        },
      },
    },
  },
}
```

## Жизненный цикл артефактов и хранение

- Артефакты хранятся во временной подпапке: `$TMPDIR/openclaw-diffs`.
- Метаданные артефакта просмотрщика содержат:
  - случайный ID артефакта (20 шестнадцатеричных символов)
  - случайный токен (48 шестнадцатеричных символов)
  - `createdAt` и `expiresAt`
  - сохранённый путь `viewer.html`
- Время жизни артефакта по умолчанию — 30 минут, если не указано иное.
- Максимально допустимое время жизни просмотрщика — 6 часов.
- Очистка выполняется по возможности после создания артефакта.
- Истекшие артефакты удаляются.
- Резервная очистка удаляет устаревшие папки старше 24 часов, если метаданные отсутствуют.

## URL просмотрщика и сетевое поведение

Маршрут просмотрщика:

- `/plugins/diffs/view/{artifactId}/{token}`

Активы просмотрщика:

- `/plugins/diffs/assets/viewer.js`
- `/plugins/diffs/assets/viewer-runtime.js`

Документ просмотрщика разрешает эти активы относительно URL просмотрщика, поэтому необязательный префикс пути `baseUrl` сохраняется и для запросов активов.

Поведение построения URL:

- Если в вызове инструмента указан `baseUrl`, он используется после строгой проверки.
