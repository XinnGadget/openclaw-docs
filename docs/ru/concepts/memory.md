# Обзор памяти

OpenClaw сохраняет данные, записывая **простые файлы Markdown** в рабочую область вашего агента. Модель "помнит" только то, что было сохранено на диск — скрытого состояния нет.

## Как это работает

У вашего агента есть три файла, связанных с памятью:

- **`MEMORY.md`** — долговременная память. Устойчивые факты, предпочтения и решения. Загружаются в начале каждой сессии DM.
- **`memory/YYYY-MM-DD.md`** — ежедневные заметки. Текущий контекст и наблюдения. Заметки за сегодня и вчера загружаются автоматически.
- **`DREAMS.md`** (экспериментальный, необязательный) — дневник сновидений и сводки сеансов "сновидений" для проверки человеком, включая обоснованные исторические записи для заполнения.

Эти файлы находятся в рабочей области агента (по умолчанию `~/.openclaw/workspace`).

<Tip>
Если вы хотите, чтобы агент что-то запомнил, просто попросите его: "Запомни, что я предпочитаю TypeScript". Он запишет это в соответствующий файл.
</Tip>

## Инструменты работы с памятью

У агента есть два инструмента для работы с памятью:

- **`memory_search`** — находит соответствующие заметки с помощью семантического поиска, даже если формулировка отличается от исходной.
- **`memory_get`** — читает конкретный файл памяти или диапазон строк.

Оба инструмента предоставляются активным плагином памяти (по умолчанию — `memory-core`).

## Плагин-компаньон Memory Wiki

Если вы хотите, чтобы долговременная память вела себя скорее как поддерживаемая база знаний, а не просто как необработанные заметки, используйте встроенный плагин `memory-wiki`.

`memory-wiki` компилирует устойчивые знания в хранилище вики со следующими возможностями:

- детерминированная структура страниц;
- структурированные утверждения и доказательства;
- отслеживание противоречий и актуальности;
- генерируемые панели мониторинга;
- скомпилированные дайджесты для агентов/компонентов среды выполнения;
- инструменты, встроенные в вики, такие как `wiki_search`, `wiki_get`, `wiki_apply` и `wiki_lint`.

Этот плагин не заменяет активный плагин памяти. Активный плагин памяти по-прежнему отвечает за извлечение, продвижение и "сновидения". `memory-wiki` добавляет к нему слой знаний с подробной информацией о происхождении данных.

См. [Memory Wiki](/plugins/memory-wiki).

## Поиск в памяти

Если настроен провайдер эмбеддингов, `memory_search` использует **гибридный поиск** — сочетает векторное сходство (семантическое значение) с сопоставлением ключевых слов (точные термины, такие как идентификаторы и символы кода). Это работает "из коробки", как только у вас появится API-ключ для любого поддерживаемого провайдера.

<Info>
OpenClaw автоматически определяет ваш провайдер эмбеддингов по доступным API-ключам. Если у вас настроен ключ OpenAI, Gemini, Voyage или Mistral, поиск в памяти включается автоматически.
</Info>

Подробности о том, как работает поиск, параметрах настройки и настройке провайдера, см. в разделе [Memory Search](/concepts/memory-search).

## Автоматическое очищение памяти

Перед тем как [компактизация](/concepts/compaction) суммирует вашу беседу, OpenClaw выполняет тихий ход, напоминающий агенту сохранить важный контекст в файлы памяти. Это включено по умолчанию — вам не нужно ничего настраивать.

<Tip>
Очищение памяти предотвращает потерю контекста во время компактизации. Если у вашего агента есть важные факты в беседе, которые ещё не записаны в файл, они будут автоматически сохранены до создания сводки.
</Tip>

## "Сновидения" (экспериментально)

"Сновидения" — это необязательный фоновый процесс консолидации памяти. Он собирает краткосрочные сигналы, оценивает кандидатов и продвигает в долговременную память (`MEMORY.md`) только соответствующие элементы.

Он предназначен для поддержания высокого уровня значимых данных в долговременной памяти:

- **По желанию**: отключено по умолчанию.
- **По расписанию**: при включении `memory-core` автоматически управляет одним повторяющимся заданием cron для полного сеанса "сновидений".
- **С пороговыми значениями**: продвижение возможно только при прохождении пороговых значений по оценке, частоте извлечения и разнообразию запросов.
- **С возможностью проверки**: сводки этапов и записи дневника записываются в `DREAMS.md` для проверки человеком.

Подробности о поведении этапов, сигналах оценки и дневнике "сновидений" см. в разделе [Dreaming (experimental)](/concepts/dreaming).

## Обоснованное заполнение и оперативное продвижение

Система "сновидений" теперь имеет два тесно связанных процесса проверки:

- **Оперативные "сновидения"** работают с краткосрочным хранилищем "сновидений" в `memory/.dreams/` и используются обычным глубоким этапом при принятии решения о том, что можно перенести в `MEMORY.md`.
- **Обоснованное заполнение** считывает исторические заметки `memory/YYYY-MM-DD.md` как отдельные дневные файлы и записывает структурированный результат проверки в `DREAMS.md`.

Обоснованное заполнение полезно, когда вы хотите воспроизвести старые заметки и проверить, что система считает устойчивым, без ручного редактирования `MEMORY.md`.

Когда вы используете:

```bash
openclaw memory rem-backfill --path ./memory --stage-short-term
```

обоснованные устойчивые кандидаты не продвигаются напрямую. Они помещаются в то же краткосрочное хранилище "сновидений", которое уже использует обычный глубокий этап. Это означает:

- `DREAMS.md` остаётся поверхностью для проверки человеком.
- краткосрочное хранилище остаётся поверхностью для ранжирования машиной.
- `MEMORY.md` по-прежнему заполняется только при глубоком продвижении.

Если вы решите, что воспроизведение было бесполезным, вы можете удалить подготовленные артефакты, не затрагивая обычные записи дневника или обычное состояние извлечения:

```bash
openclaw memory rem-backfill --rollback
openclaw memory rem-backfill --rollback-short-term
```

## CLI

```bash
openclaw memory status          # Проверить статус индекса и провайдера
openclaw memory search "query"  # Поиск из командной строки
openclaw memory index --force   # Перестроить индекс
```
