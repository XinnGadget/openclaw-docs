# Разбор местоположения канала

OpenClaw приводит общие местоположения из чат-каналов к единому формату:

- читаемый человеком текст, добавляемый к телу входящего сообщения;
- структурированные поля в полезной нагрузке контекста автоответа.

В настоящее время поддерживаются:

- **Telegram** (метки местоположения + места + актуальные местоположения);
- **WhatsApp** (locationMessage + liveLocationMessage);
- **Matrix** (`m.location` с `geo_uri`).

## Форматирование текста

Местоположение отображается в виде удобных строк без скобок:

- Метка:
  - `📍 48.858844, 2.294351 ±12 м`
- Названное место:
  - `📍 Эйфелева башня — Марсово поле, Париж (48.858844, 2.294351 ±12 м)`
- Актуальное местоположение:
  - `🛰 Актуальное местоположение: 48.858844, 2.294351 ±12 м`

Если в канале есть подпись/комментарий, она добавляется в следующей строке:

```
📍 48.858844, 2.294351 ±12 м
Встретимся здесь
```

## Поля контекста

При наличии местоположения в `ctx` добавляются следующие поля:

- `LocationLat` (число);
- `LocationLon` (число);
- `LocationAccuracy` (число, метры; необязательно);
- `LocationName` (строка; необязательно);
- `LocationAddress` (строка; необязательно);
- `LocationSource` (`pin | place | live`);
- `LocationIsLive` (булево значение).

## Примечания по каналам

- **Telegram**: места сопоставляются с `LocationName/LocationAddress`; для актуальных местоположений используется `live_period`.
- **WhatsApp**: `locationMessage.comment` и `liveLocationMessage.caption` добавляются в виде строки с подписью.
- **Matrix**: `geo_uri` разбирается как метка местоположения; высота игнорируется, а `LocationIsLive` всегда имеет значение `false`.
