# Плагины сообщества

Плагины сообщества — это сторонние пакеты, которые расширяют возможности OpenClaw за счёт новых каналов, инструментов, провайдеров или других функций. Они разрабатываются и поддерживаются сообществом, публикуются на [ClawHub](/tools/clawhub) или npm и устанавливаются с помощью одной команды.

ClawHub — это основной ресурс для поиска плагинов сообщества. Не отправляйте PR с документацией только для того, чтобы сделать ваш плагин доступным для поиска; вместо этого опубликуйте его на ClawHub.

```bash
openclaw plugins install <название-пакета>
```

OpenClaw сначала проверяет ClawHub, а затем автоматически обращается к npm.

## Список плагинов

### Codex App Server Bridge

Независимый мост OpenClaw для диалогов с Codex App Server. Привяжите чат к потоку Codex, общайтесь с ним с помощью обычного текста и управляйте им с помощью команд, встроенных в чат: возобновление, планирование, просмотр, выбор модели, сжатие и т. д.

- **npm:** `openclaw-codex-app-server`
- **репозиторий:** [github.com/pwrdrvr/openclaw-codex-app-server](https://github.com/pwrdrvr/openclaw-codex-app-server)

```bash
openclaw plugins install openclaw-codex-app-server
```

### DingTalk

Интеграция корпоративного робота с использованием режима Stream. Поддерживает текстовые сообщения, изображения и файлы через любой клиент DingTalk.

- **npm:** `@largezhou/ddingtalk`
- **репозиторий:** [github.com/largezhou/openclaw-dingtalk](https://github.com/largezhou/openclaw-dingtalk)

```bash
openclaw plugins install @largezhou/ddingtalk
```

### Lossless Claw (LCM)

Плагин для управления контекстом без потерь в OpenClaw. Суммирование диалогов на основе DAG с постепенным сжатием — сохраняет полную точность контекста при сокращении использования токенов.

- **npm:** `@martian-engineering/lossless-claw`
- **репозиторий:** [github.com/Martian-Engineering/lossless-claw](https://github.com/Martian-Engineering/lossless-claw)

```bash
openclaw plugins install @martian-engineering/lossless-claw
```

### Opik

Официальный плагин, который экспортирует трассировку агента в Opik. Отслеживайте поведение агента, затраты, токены, ошибки и т. д.

- **npm:** `@opik/opik-openclaw`
- **репозиторий:** [github.com/comet-ml/opik-openclaw](https://github.com/comet-ml/opik-openclaw)

```bash
openclaw plugins install @opik/opik-openclaw
```

### QQbot

Подключение OpenClaw к QQ через API QQ Bot. Поддерживает личные чаты, упоминания в группах, сообщения в каналах и мультимедийные файлы, включая голосовые сообщения, изображения, видео и файлы.

- **npm:** `@tencent-connect/openclaw-qqbot`
- **репозиторий:** [github.com/tencent-connect/openclaw-qqbot](https://github.com/tencent-connect/openclaw-qqbot)

```bash
openclaw plugins install @tencent-connect/openclaw-qqbot
```

### wecom

Плагин канала WeCom для OpenClaw от команды Tencent WeCom. Работает на основе постоянных соединений WebSocket бота WeCom, поддерживает прямые сообщения и групповые чаты, потоковые ответы, инициативные сообщения, обработку изображений и файлов, форматирование Markdown, встроенный контроль доступа, а также навыки работы с документами, встречами и сообщениями.

- **npm:** `@wecom/wecom-openclaw-plugin`
- **репозиторий:** [github.com/WecomTeam/wecom-openclaw-plugin](https://github.com/WecomTeam/wecom-openclaw-plugin)

```bash
openclaw plugins install @wecom/wecom-openclaw-plugin
```

## Отправьте свой плагин

Мы приветствуем плагины сообщества, которые полезны, задокументированы и безопасны в использовании.

<Steps>
  <Step title="Опубликовать на ClawHub или npm">
    Ваш плагин должен устанавливаться с помощью команды `openclaw plugins install \<название-пакета\>`.
    Опубликуйте его на [ClawHub](/tools/clawhub) (предпочтительно) или npm.
    Полное руководство см. в разделе [Создание плагинов](/plugins/building-plugins).
  </Step>

  <Step title="Разместить на GitHub">
    Исходный код должен находиться в публичном репозитории с документацией по настройке и системой отслеживания проблем.
  </Step>

  <Step title="Используйте PR с документацией только для изменений в исходных документах">
    Вам не нужен PR с документацией только для того, чтобы сделать ваш плагин доступным для поиска. Вместо этого опубликуйте его на ClawHub.

    Открывайте PR с документацией только в том случае, если в исходных документах OpenClaw необходимо внести фактические изменения в содержание, например, исправить руководство по установке или добавить межрепозиторную документацию, которая должна быть включена в основной набор документов.
  </Step>
</Steps>

## Требования к качеству

| Требование | Причина |
| --- | --- |
| Опубликован на ClawHub или npm | Пользователям необходимо, чтобы команда `openclaw plugins install` работала |
| Публичный репозиторий на GitHub | Проверка исходного кода, отслеживание проблем, прозрачность |
| Документация по настройке и использованию | Пользователям необходимо знать, как настроить плагин |
| Активное сопровождение | Недавние обновления или оперативное решение проблем |

Плагины с минимальными усилиями, неясной ответственностью или без сопровождения могут быть отклонены.
