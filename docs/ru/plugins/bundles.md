# Бандлы плагинов

OpenClaw может устанавливать плагины из трёх внешних экосистем: **Codex**, **Claude** и **Cursor**. Они называются **бандлами** — пакетами содержимого и метаданных, которые OpenClaw сопоставляет с нативными функциями, такими как навыки (skills), хуки (hooks) и инструменты MCP.

<Info>
  Бандлы **не** то же самое, что нативные плагины OpenClaw. Нативные плагины выполняются внутри процесса и могут регистрировать любые возможности. Бандлы — это пакеты содержимого с выборочным сопоставлением функций и более узкими границами доверия.
</Info>

## Зачем нужны бандлы

Многие полезные плагины публикуются в формате Codex, Claude или Cursor. Вместо того чтобы требовать от авторов переписывать их в виде нативных плагинов OpenClaw, OpenClaw обнаруживает эти форматы и сопоставляет поддерживаемое содержимое с набором нативных функций. Это означает, что вы можете установить пакет команд Claude или бандл навыков Codex и сразу начать его использовать.

## Установка бандла

<Steps>
  <Step title="Установка из каталога, архива или маркетплейса">
    ```bash
    # Локальный каталог
    openclaw plugins install ./my-bundle

    # Архив
    openclaw plugins install ./my-bundle.tgz

    # Маркетплейс Claude
    openclaw plugins marketplace list <название-маркетплейса>
    openclaw plugins install <название-плагина>@<название-маркетплейса>
    ```

  </Step>

  <Step title="Проверка обнаружения">
    ```bash
    openclaw plugins list
    openclaw plugins inspect <id>
    ```

    Бандлы отображаются как `Format: bundle` с подтипом `codex`, `claude` или `cursor`.

  </Step>

  <Step title="Перезапуск и использование">
    ```bash
    openclaw gateway restart
    ```

    Сопоставленные функции (навыки, хуки, инструменты MCP, настройки LSP по умолчанию) будут доступны в следующем сеансе.

  </Step>
</Steps>

## Что OpenClaw сопоставляет из бандлов

Не все функции бандла работают в OpenClaw на данный момент. Ниже указано, что поддерживается и что обнаружено, но ещё не подключено.

### Поддерживается сейчас

| Функция | Как происходит сопоставление | Применяется к |
| --- | --- | --- |
| Содержимое навыков | Корни навыков бандла загружаются как обычные навыки OpenClaw | Все форматы |
| Команды | `commands/` и `.cursor/commands/` обрабатываются как корни навыков | Claude, Cursor |
| Пакеты хуков | Макет в стиле OpenClaw: `HOOK.md` + `handler.ts` | Codex |
| Инструменты MCP | Конфигурация MCP бандла объединяется с встроенными настройками Pi; поддерживаемые серверы stdio и HTTP загружаются | Все форматы |
| Серверы LSP | Claude `.lsp.json` и объявленные в манифесте `lspServers` объединяются со встроенными настройками LSP Pi по умолчанию | Claude |
| Настройки | Claude `settings.json` импортируется как встроенные настройки Pi по умолчанию | Claude |

#### Содержимое навыков

- корни навыков бандла загружаются как обычные корни навыков OpenClaw;
- корни `commands` в Claude обрабатываются как дополнительные корни навыков;
- корни `.cursor/commands` в Cursor обрабатываются как дополнительные корни навыков.

Это означает, что файлы команд в формате Markdown для Claude работают через обычный загрузчик навыков OpenClaw. Команды в формате Markdown для Cursor работают по тому же пути.

#### Пакеты хуков

- корни хуков бандла работают **только** в том случае, если они используют обычный макет пакета хуков OpenClaw. На данный момент это в основном относится к случаю, совместимому с Codex:
  - `HOOK.md`;
  - `handler.ts` или `handler.js`.

#### MCP для Pi

- включённые бандлы могут предоставлять конфигурацию сервера MCP;
- OpenClaw объединяет конфигурацию MCP бандла с эффективными встроенными настройками Pi в качестве `mcpServers`;
- OpenClaw предоставляет поддерживаемые инструменты MCP бандла во время работы встроенного агента Pi, запуская серверы stdio или подключаясь к серверам HTTP;
- локальные для проекта настройки Pi по-прежнему применяются после настроек бандла по умолчанию, поэтому настройки рабочей области могут переопределить записи MCP бандла при необходимости;
- каталоги инструментов MCP бандла сортируются детерминированно перед регистрацией, поэтому изменения порядка в `listTools()` не нарушают блоки инструментов в кэше подсказок.

##### Транспорты

Серверы MCP могут использовать транспорт stdio или HTTP:

**Stdio** запускает дочерний процесс:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "command": "node",
        "args": ["server.js"],
        "env": { "PORT": "3000" }
      }
    }
  }
}
```

**HTTP** подключается к работающему серверу MCP через `sse` по умолчанию или через `streamable-http`, если это запрошено:

```json
{
  "mcp": {
    "servers": {
      "my-server": {
        "url": "http://localhost:3100/mcp",
        "transport": "streamable-http",
        "headers": {
          "Authorization": "Bearer ${MY_SECRET_TOKEN}"
        },
        "connectionTimeoutMs": 30000
      }
    }
  }
}
```

- `transport` может быть установлен как `"streamable-http"` или `"sse"`; если не указан, OpenClaw использует `sse`;
- разрешены только схемы URL `http:` и `https:`;
- значения `headers` поддерживают интерполяцию `${ENV_VAR}`;
- запись сервера с `command` и `url` одновременно отклоняется;
- учётные данные URL (userinfo и параметры запроса) удаляются из описаний инструментов и журналов;
- `connectionTimeoutMs` переопределяет тайм-аут подключения по умолчанию в 30 секунд для транспортов stdio и HTTP.

##### Именование инструментов

OpenClaw регистрирует инструменты MCP бандла с безопасными для провайдера именами в формате `serverName__toolName`. Например, сервер с ключом `"vigil-harbor"`, предоставляющий инструмент `memory_search`, регистрируется как `vigil-harbor__memory_search`.

- символы, не входящие в `A-Za-z0-9_-`, заменяются на `-`;
- префиксы серверов ограничены 30 символами;
- полные имена инструментов ограничены 64 символами;
- пустые имена серверов заменяются на `mcp`;
- конфликтующие очищенные имена различаются с помощью числовых суффиксов;
- окончательный порядок предоставляемых инструментов детерминирован по безопасному имени, чтобы обеспечить стабильность кэша при повторных поворотах Pi.

#### Встроенные настройки Pi

- `settings.json` в Claude импортируется как настройки встроенного Pi по умолчанию при включении бандла;
- OpenClaw очищает ключи переопределения оболочки перед их применением.

Очищенные ключи:

- `shellPath`;
- `shellCommandPrefix`.

#### Встроенный LSP для Pi

- включённые бандлы Claude могут предоставлять конфигурацию сервера LSP;
- OpenClaw загружает `.lsp.json` плюс любые пути `lspServers`, объявленные в манифесте;
- конфигурация LSP бандла объединяется с эффективными встроенными настройками LSP Pi по умолчанию;
- на данный момент можно запускать только поддерживаемые серверы LSP с транспортом stdio; неподдерживаемые транспорты по-прежнему отображаются в `openclaw plugins inspect <id>`.

### Обнаружено, но не выполняется

Эти элементы распознаются и отображаются в диагностике, но OpenClaw их не запускает:

- Claude `agents`, `hooks.json` (автоматизация), `outputStyles`;
- Cursor `.cursor/agents`, `.cursor/hooks.json`, `.cursor/rules`;
- метаданные Codex inline/app за пределами отчётов о возможностях.
