# Tavily

OpenClaw может использовать **Tavily** двумя способами:

- в качестве провайдера `web_search`;
- в качестве явных плагинных инструментов: `tavily_search` и `tavily_extract`.

Tavily — это поисковый API, разработанный для приложений с использованием ИИ, который возвращает структурированные результаты, оптимизированные для потребления LLM. Он поддерживает настраиваемую глубину поиска, фильтрацию по темам, фильтрацию по доменам, сводки ответов, сгенерированные ИИ, а также извлечение контента из URL-адресов (в том числе со страниц с рендерингом на JavaScript).

## Получение API-ключа

1. Создайте аккаунт Tavily на [tavily.com](https://tavily.com/).
2. Сгенерируйте API-ключ в панели управления.
3. Сохраните его в конфигурации либо установите переменную `TAVILY_API_KEY` в среде шлюза.

## Настройка поиска Tavily

```json5
{
  plugins: {
    entries: {
      tavily: {
        enabled: true,
        config: {
          webSearch: {
            apiKey: "tvly-...", // необязательно, если установлена переменная TAVILY_API_KEY
            baseUrl: "https://api.tavily.com",
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        provider: "tavily",
      },
    },
  },
}
```

Примечания:

- Выбор Tavily при настройке (onboarding) или выполнение команды `openclaw configure --section web` автоматически активирует встроенный плагин Tavily.
- Сохраняйте конфигурацию Tavily в `plugins.entries.tavily.config.webSearch.*`.
- `web_search` с Tavily поддерживает параметры `query` и `count` (до 20 результатов).
- Для использования специфических для Tavily параметров управления (таких как `search_depth`, `topic`, `include_answer` или фильтры по доменам) используйте `tavily_search`.

## Плагинные инструменты Tavily

### `tavily_search`

Используйте этот инструмент, если вам нужны специфические для Tavily параметры поиска вместо общего `web_search`.

| Параметр | Описание |
| ----------------- | --------------------------------------------------------------------- |
| `query` | Строка поискового запроса (не более 400 символов) |
| `search_depth` | `basic` (по умолчанию, сбалансированный режим) или `advanced` (максимальная релевантность, более медленный режим) |
| `topic` | `general` (по умолчанию), `news` (обновления в режиме реального времени) или `finance` |
| `max_results` | Количество результатов, от 1 до 20 (по умолчанию: 5) |
| `include_answer` | Включать сводку ответа, сгенерированную ИИ (по умолчанию: false) |
| `time_range` | Фильтрация по свежести: `day`, `week`, `month` или `year` |
| `include_domains` | Массив доменов, к которым нужно ограничить результаты |
| `exclude_domains` | Массив доменов, которые нужно исключить из результатов |

**Глубина поиска:**

| Глубина | Скорость | Релевантность | Для чего подходит |
| ---------- | ------ | --------- | ----------------------------------- |
| `basic` | Быстрее | Высокая | Общие запросы (по умолчанию) |
| `advanced` | Медленнее | Максимальная | Точная информация, конкретные факты, исследования |

### `tavily_extract`

Используйте этот инструмент для извлечения чистого контента из одного или нескольких URL-адресов. Он обрабатывает страницы с рендерингом на JavaScript и поддерживает разбиение на фрагменты с учётом запроса для целенаправленного извлечения данных.

| Параметр | Описание |
| ------------------- | ---------------------------------------------------------- |
| `urls` | Массив URL-адресов для извлечения (от 1 до 20 на запрос) |
| `query` | Переупорядочивание извлечённых фрагментов по релевантности к этому запросу |
| `extract_depth` | `basic` (по умолчанию, быстрый режим) или `advanced` (для страниц с большим количеством JavaScript) |
| `chunks_per_source` | Количество фрагментов на URL, от 1 до 5 (требуется указание `query`) |
| `include_images` | Включать URL-адреса изображений в результаты (по умолчанию: false) |

**Глубина извлечения:**

| Глубина | Когда использовать |
| ---------- | ----------------------------------------- |
| `basic` | Простые страницы — начните с этого варианта |
| `advanced` | SPA с рендерингом на JavaScript, динамический контент, таблицы |

Советы:

- Максимум 20 URL-адресов на запрос. Разбивайте большие списки на несколько вызовов.
- Используйте комбинацию `query` + `chunks_per_source`, чтобы получить только релевантный контент, а не полные страницы.
- Сначала попробуйте режим `basic`; переходите к `advanced`, если контент отсутствует или неполный.

## Выбор подходящего инструмента

| Задача | Инструмент |
| ------------------------------------ | ---------------- |
| Быстрый веб-поиск без специальных опций | `web_search` |
| Поиск с настройкой глубины, темы, ответами от ИИ | `tavily_search` |
| Извлечение контента из конкретных URL-адресов | `tavily_extract` |
