# WebChat (пользовательский интерфейс Gateway WebSocket)

Статус: пользовательский интерфейс чата на macOS/iOS (SwiftUI) взаимодействует напрямую с Gateway WebSocket.

## Что это такое

- Нативный пользовательский интерфейс чата для шлюза (без встроенного браузера и локального статического сервера).
- Использует те же сессии и правила маршрутизации, что и другие каналы.
- Детерминированная маршрутизация: ответы всегда возвращаются в WebChat.

## Быстрый старт

1. Запустите шлюз.
2. Откройте пользовательский интерфейс WebChat (приложение для macOS/iOS) или вкладку чата в Control UI.
3. Убедитесь, что настроен корректный путь аутентификации шлюза (по умолчанию — shared-secret, даже в режиме loopback).

## Как это работает (поведение)

- Пользовательский интерфейс подключается к Gateway WebSocket и использует `chat.history`, `chat.send` и `chat.inject`.
- `chat.history` ограничен для обеспечения стабильности: шлюз может усекать длинные текстовые поля, опускать объёмные метаданные и заменять слишком большие записи на `[chat.history omitted: message too large]`.
- `chat.history` также нормализуется для отображения: из видимого текста удаляются встроенные теги директив доставки, такие как `[[reply_to_*]]` и `[[audio_as_voice]]`, XML-полезные нагрузки вызовов инструментов в виде обычного текста (включая `<tool_call>...</tool_call>`, `<function_call>...</function_call>`, `<tool_calls>...</tool_calls>`, `<function_calls>...</function_calls>` и усечённые блоки вызовов инструментов), а также просочившиеся токены управления моделью в формате ASCII/полной ширины. Записи ассистента, весь видимый текст которых состоит только из точного тихого токена `NO_REPLY` / `no_reply`, опускаются.
- `chat.inject` добавляет заметку ассистента непосредственно в транскрипцию и транслирует её в пользовательский интерфейс (без запуска агента).
- Прерванные запуски могут оставлять частично видимый вывод ассистента в пользовательском интерфейсе.
- Шлюз сохраняет прерванный частичный текст ассистента в истории транскрипции, если существует буферизованный вывод, и помечает эти записи метаданными прерывания.
- История всегда загружается из шлюза (без отслеживания локальных файлов).
- Если шлюз недоступен, WebChat работает в режиме только для чтения.

## Панель инструментов агентов Control UI

- Панель инструментов `/agents` в Control UI имеет два отдельных представления:
  - **Available Right Now** использует `tools.effective(sessionKey=...)` и показывает, какие инструменты текущая сессия может фактически использовать во время выполнения, включая основные инструменты, плагины и инструменты, принадлежащие каналу.
  - **Tool Configuration** использует `tools.catalog` и сосредоточена на профилях, переопределениях и семантике каталога.
- Доступность во время выполнения ограничена сессией. Переключение сессий на одном и том же агенте может изменить список **Available Right Now**.
- Редактор конфигурации не подразумевает доступность во время выполнения; фактический доступ по-прежнему определяется приоритетом политик (`allow`/`deny`, переопределения для отдельных агентов и провайдеров/каналов).

## Удаленное использование

- В удалённом режиме шлюз WebSocket туннелируется через SSH/Tailscale.
- Вам не нужно запускать отдельный сервер WebChat.

## Справочная информация по конфигурации (WebChat)

Полная конфигурация: [Configuration](/gateway/configuration)

Параметры WebChat:

- `gateway.webchat.chatHistoryMaxChars`: максимальное количество символов для текстовых полей в ответах `chat.history`. Если запись транскрипции превышает этот лимит, шлюз усекает длинные текстовые поля и может заменить слишком большие сообщения заполнителем. Клиент также может отправить параметр `maxChars` для отдельного запроса, чтобы переопределить это значение по умолчанию для одного вызова `chat.history`.

Связанные глобальные параметры:

- `gateway.port`, `gateway.bind`: хост/порт WebSocket.
- `gateway.auth.mode`, `gateway.auth.token`, `gateway.auth.password`: аутентификация WebSocket с использованием shared-secret.
- `gateway.auth.allowTailscale`: при включении вкладка чата Control UI в браузере может использовать заголовки идентификации Tailscale Serve.
- `gateway.auth.mode: "trusted-proxy"`: аутентификация через обратный прокси для браузерных клиентов за идентификационно-ориентированным **не-loopback** прокси-сервером (см. [Trusted Proxy Auth](/gateway/trusted-proxy-auth)).
- `gateway.remote.url`, `gateway.remote.token`, `gateway.remote.password`: целевой удалённый шлюз.
- `session.*`: хранилище сессий и значения ключей по умолчанию.
