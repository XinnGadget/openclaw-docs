# CLI моделей

См. [/concepts/model-failover](/concepts/model-failover) для информации о ротации профилей аутентификации, периодах охлаждения и их взаимодействии с запасными вариантами.
Краткий обзор провайдеров + примеры: [/concepts/model-providers](/concepts/model-providers).

## Как работает выбор модели

OpenClaw выбирает модели в следующем порядке:

1. **Основная** модель (`agents.defaults.model.primary` или `agents.defaults.model`).
2. **Запасные варианты** в `agents.defaults.model.fallbacks` (по порядку).
3. **Отказоустойчивость аутентификации провайдера** срабатывает внутри провайдера до перехода к следующей модели.

Связанные параметры:

- `agents.defaults.models` — список разрешённых моделей / каталог моделей, которые может использовать OpenClaw (плюс псевдонимы).
- `agents.defaults.imageModel` используется **только в том случае**, если основная модель не может принимать изображения.
- `agents.defaults.pdfModel` используется инструментом `pdf`. Если параметр не задан, инструмент использует запасной вариант — `agents.defaults.imageModel`, затем разрешённую модель сеанса / модель по умолчанию.
- `agents.defaults.imageGenerationModel` используется общей функцией генерации изображений. Если параметр не задан, `image_generate` всё равно может определить провайдера по умолчанию с аутентификацией. Сначала пробуется текущий провайдер по умолчанию, затем остальные зарегистрированные провайдеры генерации изображений в порядке идентификаторов провайдера. Если вы задаёте конкретного провайдера / модель, также настройте аутентификацию / API-ключ этого провайдера.
- `agents.defaults.musicGenerationModel` используется общей функцией генерации музыки. Если параметр не задан, `music_generate` всё равно может определить провайдера по умолчанию с аутентификацией. Сначала пробуется текущий провайдер по умолчанию, затем остальные зарегистрированные провайдеры генерации музыки в порядке идентификаторов провайдера. Если вы задаёте конкретного провайдера / модель, также настройте аутентификацию / API-ключ этого провайдера.
- `agents.defaults.videoGenerationModel` используется общей функцией генерации видео. Если параметр не задан, `video_generate` всё равно может определить провайдера по умолчанию с аутентификацией. Сначала пробуется текущий провайдер по умолчанию, затем остальные зарегистрированные провайдеры генерации видео в порядке идентификаторов провайдера. Если вы задаёте конкретного провайдера / модель, также настройте аутентификацию / API-ключ этого провайдера.
- Настройки по умолчанию для отдельных агентов могут переопределить `agents.defaults.model` через `agents.list[].model` плюс привязки (см. [/concepts/multi-agent](/concepts/multi-agent)).

## Краткая политика работы с моделями

- Установите в качестве основной самую мощную модель последнего поколения, которая вам доступна.
- Используйте запасные варианты для задач, чувствительных к стоимости и задержкам, а также для чатов с невысокой степенью важности.
- Для агентов с поддержкой инструментов или при работе с ненадёжными входными данными избегайте использования более старых / менее мощных уровней моделей.

## Ввод в эксплуатацию (рекомендуется)

Если вы не хотите вручную редактировать конфигурацию, запустите ввод в эксплуатацию:

```bash
openclaw onboard
```

Это позволит настроить модель и аутентификацию для распространённых провайдеров, включая **подписку OpenAI Code (Codex)** (OAuth) и **Anthropic** (API-ключ или Claude CLI).

## Ключи конфигурации (обзор)

- `agents.defaults.model.primary` и `agents.defaults.model.fallbacks`
- `agents.defaults.imageModel.primary` и `agents.defaults.imageModel.fallbacks`
- `agents.defaults.pdfModel.primary` и `agents.defaults.pdfModel.fallbacks`
- `agents.defaults.imageGenerationModel.primary` и `agents.defaults.imageGenerationModel.fallbacks`
- `agents.defaults.videoGenerationModel.primary` и `agents.defaults.videoGenerationModel.fallbacks`
- `agents.defaults.models` (список разрешённых моделей + псевдонимы + параметры провайдера)
- `models.providers` (пользовательские провайдеры, записанные в `models.json`)

Ссылки на модели нормализуются к нижнему регистру. Псевдонимы провайдеров, такие как `z.ai/*`, нормализуются к `zai/*`.

Примеры конфигурации провайдеров (включая OpenCode) находятся в [/providers/opencode](/providers/opencode).

## "Модель не разрешена" (и почему ответы прекращаются)

Если задан `agents.defaults.models`, он становится **списком разрешённых моделей** для `/model` и для переопределений сеанса. Когда пользователь выбирает модель, которой нет в этом списке, OpenClaw возвращает:

```
Модель "provider/model" не разрешена. Используйте /model для вывода списка доступных моделей.
```

Это происходит **до** генерации обычного ответа, поэтому может показаться, что сообщение "не было отправлено". Чтобы исправить это, можно:

- добавить модель в `agents.defaults.models`;
- очистить список разрешённых моделей (удалить `agents.defaults.models`);
- выбрать модель из `/model list`.

Пример конфигурации списка разрешённых моделей:

```json5
{
  agent: {
    model: { primary: "anthropic/claude-sonnet-4-6" },
    models: {
      "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
      "anthropic/claude-opus-4-6": { alias: "Opus" },
    },
  },
}
```

## Переключение моделей в чате (`/model`)

Вы можете переключать модели для текущего сеанса без перезапуска:

```
/model
/model list
/model 3
/model openai/gpt-5.4
/model status
```

Примечания:

- `/model` (и `/model list`) — это компактный нумерованный селектор (семейство моделей + доступные провайдеры).
- В Discord `/model` и `/models` открывают интерактивный селектор с выпадающими списками провайдеров и моделей, а также шагом "Отправить".
- `/model <#>` выбирает модель из этого селектора.
- `/model` немедленно сохраняет новый выбор для сеанса.
- Если агент бездействует, следующий запуск сразу использует новую модель.
- Если запуск уже активен, OpenClaw помечает переключение в реальном времени как ожидающее и переключается на новую модель только в точке чистого повтора.
- Если активность инструмента или вывод ответа уже начались, ожидающее переключение может оставаться в очереди до следующей возможности повтора или следующего хода пользователя.
- `/model status` — это подробный вид (кандидаты аутентификации и, при настройке, конечная точка провайдера `baseUrl` + режим `api`).
- Ссылки на модели анализируются путём разделения по **первому** символу `/`. При вводе `/model <ref>` используйте формат `provider/model`.
- Если идентификатор модели сам содержит `/` (в стиле OpenRouter), необходимо указать префикс провайдера (например: `/model openrouter/moonshotai/kimi-k2`).
- Если вы опустите провайдера, OpenClaw разрешит ввод в следующем порядке:
  1. совпадение по псевдониму;
  2. уникальное совпадение с настроенным провайдером для этого точного идентификатора модели без префикса;
  3. устаревший запасной вариант — провайдер по умолчанию.
     Если этот провайдер больше не предоставляет настроенную модель по умолчанию, OpenClaw вместо этого переключается на первого настроенного провайдера / модель, чтобы избежать использования устаревшего удалённого провайдера по умолчанию.

Полное описание поведения команд / конфигурации: [Команды с косой чертой](/tools/slash-commands).

## Команды CLI

```bash
openclaw models list
openclaw models status
openclaw models set <provider/model>
openclaw models set-image <provider/model>

openclaw models aliases list
openclaw models aliases add <alias> <provider/model>
openclaw models aliases remove <alias>

openclaw models fallbacks list
openclaw models fallbacks add <provider/model>
openclaw models fallbacks remove <provider/model>
openclaw models fallbacks clear

openclaw models image-fallbacks list
openclaw models image-fallbacks add <provider/model>
openclaw models image-fallbacks remove <provider/model>
openclaw models image-fallbacks clear
```

`openclaw models` (без подкоманды) — это сокращение для `models status`.

### `models list`

По умолчанию показывает настроенные модели. Полезные флаги:

- `--all`: полный каталог;
- `--local`: только локальные провайдеры;
- `--provider <name>`: фильтрация по провайдеру;
- `--plain`: одна модель в строке;
- `--json`: машиночитаемый вывод.

### `models status`

Показывает разрешённую основную модель, запасные варианты, модель для изображений и обзор аутентификации настроенных провайдеров. Также отображает статус истечения срока действия OAuth для профилей, найденных в хранилище аутентификации (
