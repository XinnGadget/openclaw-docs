# Канал QA

`qa-channel` — это встроенный синтетический транспорт сообщений для автоматизированного QA в OpenClaw.

Это не производственный канал. Он предназначен для тестирования того же интерфейса плагина канала, который используют реальные транспорты, при этом состояние остаётся детерминированным и полностью поддающимся анализу.

## Что он делает сейчас

- Грамматика целевого класса Slack:
  - `dm:<пользователь>`
  - `channel:<комната>`
  - `thread:<комната>/<поток>`
- Синтетическая шина на основе HTTP для:
  - внедрения входящих сообщений
  - захвата исходящих транскриптов
  - создания потоков
  - реакций
  - правок
  - удалений
  - действий по поиску и чтению
- Встроенный инструмент самопроверки на стороне хоста, который формирует отчёт в формате Markdown

## Конфигурация

```json
{
  "channels": {
    "qa-channel": {
      "baseUrl": "http://127.0.0.1:43123",
      "botUserId": "openclaw",
      "botDisplayName": "OpenClaw QA",
      "allowFrom": ["*"],
      "pollTimeoutMs": 1000
    }
  }
}
```

Поддерживаемые ключи учётной записи:

- `baseUrl`
- `botUserId`
- `botDisplayName`
- `pollTimeoutMs`
- `allowFrom`
- `defaultTo`
- `actions.messages`
- `actions.reactions`
- `actions.search`
- `actions.threads`

## Запуск

Текущий вертикальный срез:

```bash
pnpm qa:e2e
```

Сейчас это выполняется через встроенный расширение `qa-lab`. Команда запускает шину QA в репозитории, загружает встроенный срез среды выполнения `qa-channel`, выполняет детерминированную самопроверку и формирует отчёт в формате Markdown в каталоге `.artifacts/qa-e2e/`.

Приватный пользовательский интерфейс отладчика:

```bash
pnpm qa:lab:up
```

Эта команда собирает сайт QA, запускает шлюз на базе Docker и стек QA Lab, а также выводит URL QA Lab. На этом сайте можно выбирать сценарии, определять полосу модели, запускать отдельные выполнения и отслеживать результаты в режиме реального времени.

Полный набор тестов QA на основе репозитория:

```bash
pnpm openclaw qa suite
```

Эта команда запускает приватный отладчик QA по локальному URL, отдельно от поставляемого пакета пользовательского интерфейса Control UI.

## Область применения

Текущая область применения намеренно ограничена:

- шина + транспорт плагина
- грамматика маршрутизации с потоками
- действия с сообщениями, принадлежащие каналу
- формирование отчётов в формате Markdown
- сайт QA на базе Docker с элементами управления выполнением

В дальнейшей работе планируется добавить:

- выполнение матрицы провайдер/модель
- более широкий поиск сценариев
- оркестрацию, встроенную в OpenClaw
