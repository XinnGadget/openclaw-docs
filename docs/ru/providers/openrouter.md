# OpenRouter

OpenRouter предоставляет **унифицированный API**, который направляет запросы к множеству моделей через единую конечную точку и API-ключ. Он совместим с OpenAI, поэтому большинство SDK OpenAI работают после смены базового URL.

## Настройка CLI

```bash
openclaw onboard --auth-choice openrouter-api-key
```

## Фрагмент конфигурации

```json5
{
  env: { OPENROUTER_API_KEY: "sk-or-..." },
  agents: {
    defaults: {
      model: { primary: "openrouter/auto" },
    },
  },
}
```

## Примечания

- Ссылки на модели имеют формат `openrouter/<провайдер>/<модель>`.
- При подключении по умолчанию используется `openrouter/auto`. Позже можно переключиться на конкретную модель с помощью команды `openclaw models set openrouter/<провайдер>/<модель>`.
- Дополнительные варианты моделей и провайдеров см. в разделе [/concepts/model-providers](/concepts/model-providers).
- OpenRouter использует токен Bearer с вашим API-ключом на внутреннем уровне.
- При реальных запросах к OpenRouter (`https://openrouter.ai/api/v1`) OpenClaw также добавляет задокументированные заголовки атрибуции приложения OpenRouter: `HTTP-Referer: https://openclaw.ai`, `X-OpenRouter-Title: OpenClaw` и `X-OpenRouter-Categories: cli-agent`.
- На проверенных маршрутах OpenRouter ссылки на модели Anthropic также сохраняют специфические для OpenRouter маркеры `cache_control` от Anthropic, которые OpenClaw использует для более эффективного повторного использования кэша подсказок в блоках подсказок для системы/разработчика.
- Если вы перенаправите провайдера OpenRouter на какой-либо другой прокси/базовый URL, OpenClaw не будет вставлять эти специфические для OpenRouter заголовки или маркеры кэша Anthropic.
- OpenRouter по-прежнему работает через прокси-путь, совместимый с OpenAI, поэтому собственные параметры формирования запросов только для OpenAI, такие как `serviceTier`, `Responses store`, полезные нагрузки, совместимые с рассуждениями OpenAI, и подсказки для кэша подсказок, не пересылаются.
- Ссылки OpenRouter, основанные на Gemini, остаются на прокси-пути Gemini: OpenClaw сохраняет там очистку сигнатур мыслей Gemini, но не включает собственную проверку воспроизведения Gemini или перезапись начальной загрузки.
- На поддерживаемых маршрутах, отличных от `auto`, OpenClaw сопоставляет выбранный уровень рассуждений с полезными нагрузками рассуждений прокси OpenRouter. Неподдерживаемые подсказки модели и `openrouter/auto` пропускают эту инъекцию рассуждений.
- Если вы передаёте маршрутизацию провайдера OpenRouter в параметрах модели, OpenClaw пересылает её как метаданные маршрутизации OpenRouter до запуска общих обёрток потока.
