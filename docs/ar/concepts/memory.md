# نظرة عامة على الذاكرة

يتذكر OpenClaw الأشياء من خلال كتابة **ملفات Markdown عادية** في مساحة عمل
الوكيل الخاصة بك. لا "يتذكر" النموذج إلا ما يتم حفظه على القرص -- ولا توجد
أي حالة مخفية.

## كيف يعمل

يحتوي وكيلك على ثلاثة ملفات مرتبطة بالذاكرة:

- **`MEMORY.md`** -- الذاكرة طويلة الأمد. حقائق وتفضيلات وقرارات دائمة.
  يتم تحميلها عند بداية كل جلسة رسالة مباشرة.
- **`memory/YYYY-MM-DD.md`** -- ملاحظات يومية. سياق وملاحظات جارية.
  يتم تحميل ملاحظات اليوم والأمس تلقائيًا.
- **`DREAMS.md`** (تجريبي، اختياري) -- يوميات الأحلام وملخصات جولات الأحلام
  لمراجعة البشر، بما في ذلك إدخالات الاستكمال التاريخي المؤسس.

توجد هذه الملفات في مساحة عمل الوكيل (الافتراضي `~/.openclaw/workspace`).

<Tip>
إذا كنت تريد أن يتذكر وكيلك شيئًا، فقط اطلب منه ذلك: "تذكر أنني
أفضل TypeScript." سيكتب ذلك في الملف المناسب.
</Tip>

## أدوات الذاكرة

يحتوي الوكيل على أداتين للعمل مع الذاكرة:

- **`memory_search`** -- يعثر على الملاحظات ذات الصلة باستخدام البحث الدلالي،
  حتى عندما تختلف الصياغة عن الأصل.
- **`memory_get`** -- يقرأ ملف ذاكرة محددًا أو نطاقًا من الأسطر.

يتم توفير كلتا الأداتين بواسطة إضافة الذاكرة النشطة (الافتراضي: `memory-core`).

## إضافة Memory Wiki المرافقة

إذا كنت تريد أن تتصرف الذاكرة الدائمة بشكل أقرب إلى قاعدة معرفة مُدارة بدلًا من
مجرد ملاحظات خام، فاستخدم الإضافة المدمجة `memory-wiki`.

يقوم `memory-wiki` بتجميع المعرفة الدائمة في مستودع wiki مع:

- بنية صفحات حتمية
- ادعاءات وأدلة منظمة
- تتبع التناقض والحداثة
- لوحات معلومات مُولدة
- ملخصات مُجمعة لمستهلكي الوكيل/وقت التشغيل
- أدوات أصلية لـ wiki مثل `wiki_search` و`wiki_get` و`wiki_apply` و`wiki_lint`

وهو لا يستبدل إضافة الذاكرة النشطة. تظل إضافة الذاكرة النشطة مسؤولة
عن الاسترجاع والترقية والأحلام. يضيف `memory-wiki` طبقة معرفة غنية
بالمصدر إلى جانبها.

راجع [Memory Wiki](/ar/plugins/memory-wiki).

## البحث في الذاكرة

عند تكوين موفر تضمين، يستخدم `memory_search` **بحثًا هجينًا**
-- يجمع بين التشابه المتجهي (المعنى الدلالي) ومطابقة الكلمات المفتاحية
(المصطلحات الدقيقة مثل المعرّفات ورموز الشيفرة). يعمل هذا مباشرة بمجرد أن
يتوفر لديك مفتاح API لأي موفر مدعوم.

<Info>
يكتشف OpenClaw موفر التضمين تلقائيًا من مفاتيح API المتاحة. إذا
كان لديك مفتاح OpenAI أو Gemini أو Voyage أو Mistral مضبوطًا، يتم
تفعيل البحث في الذاكرة تلقائيًا.
</Info>

للتفاصيل حول كيفية عمل البحث وخيارات الضبط وإعداد الموفر، راجع
[Memory Search](/ar/concepts/memory-search).

## تفريغ الذاكرة التلقائي

قبل أن يقوم [الضغط](/ar/concepts/compaction) بتلخيص محادثتك، يشغّل OpenClaw
دورة صامتة تذكّر الوكيل بحفظ السياق المهم في ملفات الذاكرة.
هذا مفعّل افتراضيًا -- ولا تحتاج إلى ضبط أي شيء.

<Tip>
يمنع تفريغ الذاكرة فقدان السياق أثناء الضغط. إذا كانت لدى وكيلك
حقائق مهمة في المحادثة ولم تُكتب بعد في ملف، فسيتم حفظها
تلقائيًا قبل حدوث التلخيص.
</Tip>

## الأحلام (تجريبي)

الأحلام هي عملية دمج اختيارية في الخلفية للذاكرة. فهي تجمع
إشارات قصيرة الأمد، وتقيّم المرشحين، ولا ترفع إلا العناصر المؤهلة إلى
الذاكرة طويلة الأمد (`MEMORY.md`).

وقد صُممت للحفاظ على ارتفاع جودة الذاكرة طويلة الأمد:

- **اختيارية**: معطلة افتراضيًا.
- **مجدولة**: عند تفعيلها، يدير `memory-core` تلقائيًا مهمة cron متكررة
  واحدة لجولة أحلام كاملة.
- **معتمدة على العتبات**: يجب أن تجتاز الترقيات بوابات النتيجة وتكرار
  الاسترجاع وتنوع الاستعلام.
- **قابلة للمراجعة**: تتم كتابة ملخصات المراحل وإدخالات اليوميات في `DREAMS.md`
  لمراجعة البشر.

للاطلاع على سلوك المراحل وإشارات التقييم وتفاصيل يوميات الأحلام، راجع
[Dreaming (experimental)](/ar/concepts/dreaming).

## الاستكمال المؤسس والترقية المباشرة

يحتوي نظام الأحلام الآن على مسارين متقاربين للمراجعة:

- تعمل **الأحلام المباشرة** انطلاقًا من مخزن الأحلام قصير الأمد تحت
  `memory/.dreams/`، وهو ما تستخدمه المرحلة العميقة العادية عند تقرير ما
  يمكن أن ينتقل إلى `MEMORY.md`.
- يقرأ **الاستكمال المؤسس** ملاحظات `memory/YYYY-MM-DD.md` التاريخية
  كملفات أيام مستقلة ويكتب مخرجات مراجعة منظمة في `DREAMS.md`.

يكون الاستكمال المؤسس مفيدًا عندما تريد إعادة تشغيل الملاحظات القديمة
وفحص ما يعتقد النظام أنه دائم دون تحرير `MEMORY.md` يدويًا.

عند استخدام:

```bash
openclaw memory rem-backfill --path ./memory --stage-short-term
```

لا تتم ترقية المرشحين الدائمين المؤسسين مباشرة. بل يتم تجهيزهم في
مخزن الأحلام قصير الأمد نفسه الذي تستخدمه المرحلة العميقة العادية بالفعل. وهذا
يعني:

- يبقى `DREAMS.md` سطح المراجعة البشرية.
- يبقى المخزن قصير الأمد سطح الترتيب المواجه للآلة.
- لا يزال `MEMORY.md` يُكتب فقط بواسطة الترقية العميقة.

إذا قررت أن إعادة التشغيل لم تكن مفيدة، يمكنك إزالة العناصر المجهزة
من دون المساس بإدخالات اليوميات العادية أو حالة الاسترجاع العادية:

```bash
openclaw memory rem-backfill --rollback
openclaw memory rem-backfill --rollback-short-term
```

## CLI

```bash
openclaw memory status          # تحقّق من حالة الفهرس والموفر
openclaw memory search "query"  # ابحث من سطر الأوامر
openclaw memory index --force   # أعد بناء الفهرس
```
