# Конфигурация навыков

Большая часть конфигурации загрузки/установки навыков располагается в разделе `skills` файла `~/.openclaw/openclaw.json`. Видимость навыков для конкретных агентов задаётся в разделах `agents.defaults.skills` и `agents.list[].skills`.

```json5
{
  skills: {
    allowBundled: ["gemini", "peekaboo"],
    load: {
      extraDirs: ["~/Projects/agent-scripts/skills", "~/Projects/oss/some-skill-pack/skills"],
      watch: true,
      watchDebounceMs: 250,
    },
    install: {
      preferBrew: true,
      nodeManager: "npm", // npm | pnpm | yarn | bun (среда выполнения Gateway по-прежнему Node; использование bun не рекомендуется)
    },
    entries: {
      "image-lab": {
        enabled: true,
        apiKey: { source: "env", provider: "default", id: "GEMINI_API_KEY" }, // или строка в открытом виде
        env: {
          GEMINI_API_KEY: "GEMINI_KEY_HERE",
        },
      },
      peekaboo: { enabled: true },
      sag: { enabled: false },
    },
  },
}
```

Для встроенных функций генерации/редактирования изображений предпочтительно использовать `agents.defaults.imageGenerationModel` вместе с основным инструментом `image_generate`. Раздел `skills.entries.*` предназначен только для пользовательских или сторонних рабочих процессов с навыками.

Если вы выбираете конкретного поставщика/модель для работы с изображениями, также настройте аутентификацию/API-ключ для этого поставщика. Типичные примеры: `GEMINI_API_KEY` или `GOOGLE_API_KEY` для `google/*`, `OPENAI_API_KEY` для `openai/*` и `FAL_KEY` для `fal/*`.

Примеры:

- Настройка в стиле Native Nano Banana: `agents.defaults.imageGenerationModel.primary: "google/gemini-3.1-flash-image-preview"`
- Настройка Native fal: `agents.defaults.imageGenerationModel.primary: "fal/fal-ai/flux/dev"`

## Списки разрешённых навыков для агентов

Используйте конфигурацию агента, если требуется сохранить одинаковые корни навыков для машины/рабочей области, но задать разный набор видимых навыков для каждого агента.

```json5
{
  agents: {
    defaults: {
      skills: ["github", "weather"],
    },
    list: [
      { id: "writer" }, // наследует значения по умолчанию → github, weather
      { id: "docs", skills: ["docs-search"] }, // заменяет значения по умолчанию
      { id: "locked-down", skills: [] }, // без навыков
    ],
  },
}
```

Правила:

- `agents.defaults.skills`: общий базовый список разрешённых навыков для агентов, которые не задают `agents.list[].skills`.
- Опустите `agents.defaults.skills`, чтобы по умолчанию не ограничивать навыки.
- `agents.list[].skills`: явный итоговый набор навыков для данного агента; он не объединяется со значениями по умолчанию.
- `agents.list[].skills: []`: не предоставлять никаких навыков для данного агента.

## Поля

- Встроенные корни навыков всегда включают `~/.openclaw/skills`, `~/.agents/skills`, `<workspace>/.agents/skills` и `<workspace>/skills`.
- `allowBundled`: необязательный список разрешённых **встроенных** навыков. Если задан, только перечисленные встроенные навыки считаются допустимыми (навыки управления, агентов и рабочей области не затрагиваются).
- `load.extraDirs`: дополнительные каталоги навыков для сканирования (наименьший приоритет).
- `load.watch`: отслеживать папки с навыками и обновлять снимок навыков (по умолчанию: true).
- `load.watchDebounceMs`: задержка для событий наблюдателя за навыками в миллисекундах (по умолчанию: 250).
- `install.preferBrew`: предпочитать установщики brew, если они доступны (по умолчанию: true).
- `install.nodeManager`: предпочтение установщика Node (`npm` | `pnpm` | `yarn` | `bun`, по умолчанию: npm).
  Это влияет только на **установку навыков**; среда выполнения Gateway должна оставаться Node (использование Bun не рекомендуется для WhatsApp/Telegram).
  - `openclaw setup --node-manager` имеет более узкий охват и в настоящее время поддерживает `npm`, `pnpm` или `bun`. Вручную задайте `skills.install.nodeManager: "yarn"`, если хотите использовать Yarn для установки навыков.
- `entries.<skillKey>`: переопределения для отдельных навыков.
- `agents.defaults.skills`: необязательный список разрешённых навыков по умолчанию, наследуемый агентами, которые не задают `agents.list[].skills`.
- `agents.list[].skills`: необязательный итоговый список разрешённых навыков для конкретного агента; явные списки заменяют унаследованные значения по умолчанию, а не объединяются с ними.

Поля для отдельных навыков:

- `enabled`: установите значение `false`, чтобы отключить навык, даже если он встроен/установлен.
- `env`: переменные окружения, внедряемые для запуска агента (только если они ещё не заданы).
- `apiKey`: необязательное удобство для навыков, которые объявляют основную переменную окружения.
  Поддерживает строку в открытом виде или объект SecretRef (`{ source, provider, id }`).

## Примечания

- Ключи в разделе `entries` по умолчанию соответствуют имени навыка. Если навык определяет `metadata.openclaw.skillKey`, используйте этот ключ вместо него.
- Приоритет загрузки: `<workspace>/skills` → `<workspace>/.agents/skills` → `~/.agents/skills` → `~/.openclaw/skills` → встроенные навыки → `skills.load.extraDirs`.
- Изменения навыков применяются на следующем шаге работы агента, если включён наблюдатель.

### Навыки в песочнице + переменные окружения

Когда сессия находится в **песочнице**, процессы навыков выполняются внутри Docker. Песочница **не** наследует `process.env` хоста.

Используйте один из вариантов:

- `agents.defaults.sandbox.docker.env` (или для конкретного агента `agents.list[].sandbox.docker.env`)
- внедрите переменные окружения в собственный образ песочницы

Глобальные `env` и `skills.entries.<skill>.env/apiKey` применяются только к запускам на **хосте**.
