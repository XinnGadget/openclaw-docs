# Onboarding (CLI)

Настройка через CLI — **рекомендуемый** способ установки OpenClaw на macOS, Linux или Windows (через WSL2; настоятельно рекомендуется).
Она позволяет в рамках одного пошагового процесса настроить локальный шлюз (Gateway) или подключение к удалённому шлюзу, а также каналы, навыки и параметры рабочей области по умолчанию.

```bash
openclaw onboard
```

<Info>
Самый быстрый способ начать чат: откройте Control UI (настройка каналов не требуется). Запустите `openclaw dashboard` и общайтесь в браузере. Документация: [Dashboard](/web/dashboard).
</Info>

Чтобы позже перенастроить систему:

```bash
openclaw configure
openclaw agents add <name>
```

<Note>
Параметр `--json` не подразумевает неинтерактивный режим. Для скриптов используйте `--non-interactive`.
</Note>

<Tip>
Процесс onboarding через CLI включает этап поиска в интернете, где вы можете выбрать провайдера, например: Brave, DuckDuckGo, Exa, Firecrawl, Gemini, Grok, Kimi, MiniMax Search, Ollama Web Search, Perplexity, SearXNG или Tavily. Некоторым провайдерам требуется API-ключ, другие работают без ключа. Вы также можете настроить это позже с помощью команды `openclaw configure --section web`. Документация: [Web tools](/tools/web).
</Tip>

## QuickStart vs Advanced

Процесс onboarding начинается с выбора между **QuickStart** (настройки по умолчанию) и **Advanced** (полный контроль).

<Tabs>
  <Tab title="QuickStart (настройки по умолчанию)">
    - Локальный шлюз (loopback)
    - Рабочая область по умолчанию (или существующая рабочая область)
    - Порт шлюза **18789**
    - Аутентификация шлюза **Token** (автоматически генерируется, даже для loopback)
    - Политика инструментов по умолчанию для новых локальных установок: `tools.profile: "coding"` (существующая явная конфигурация сохраняется)
    - Изоляция DM по умолчанию: при локальной настройке записывается `session.dmScope: "per-channel-peer"`, если значение не задано. Подробности: [CLI Setup Reference](/start/wizard-cli-reference#outputs-and-internals)
    - Exposure через Tailscale **выключен**
    - Для Telegram и WhatsApp DM по умолчанию используется **allowlist** (вам будет предложено ввести номер телефона)
  </Tab>
  <Tab title="Advanced (полный контроль)">
    - Предоставляет доступ ко всем этапам (режим, рабочая область, шлюз, каналы, демон, навыки).
  </Tab>
</Tabs>

## Что настраивает onboarding

**Локальный режим (по умолчанию)** проводит вас через следующие этапы:

1. **Модель/Аутентификация** — выберите любого поддерживаемого провайдера/поток аутентификации (API-ключ, OAuth или ручная аутентификация, специфичная для провайдера), включая Custom Provider (совместимый с OpenAI, совместимый с Anthropic или автоматическое определение неизвестного). Выберите модель по умолчанию.
   Примечание по безопасности: если этот агент будет запускать инструменты или обрабатывать содержимое вебхуков/хуков, предпочтительно использовать самую мощную модель последнего поколения и строго ограничить политику инструментов. Более слабые/старые модели легче подвергнуть prompt-инъекциям.
   Для неинтерактивных запусков параметр `--secret-input-mode ref` сохраняет ссылки, поддерживаемые окружением, в профилях аутентификации вместо текстовых значений API-ключа.
   В неинтерактивном режиме `ref` переменная окружения провайдера должна быть задана; передача встроенных флагов ключа без этой переменной окружения приводит к быстрой ошибке.
   В интерактивном режиме выбор режима секретной ссылки позволяет указать либо переменную окружения, либо настроенную ссылку провайдера (`file` или `exec`) с быстрой предварительной проверкой перед сохранением.
   Для Anthropic интерактивный onboarding/настройка предлагает **Anthropic Claude CLI** в качестве предпочтительного локального пути и **Anthropic API key** в качестве рекомендуемого пути для продакшена. Настройка токена Anthropic также остаётся доступным поддерживаемым способом аутентификации через токен.
2. **Рабочая область** — местоположение файлов агента (по умолчанию `~/.openclaw/workspace`). Засевает загрузочные файлы.
3. **Шлюз (Gateway)** — порт, адрес привязки, режим аутентификации, exposure через Tailscale.
   В интерактивном режиме с токенами выберите хранение токена в виде обычного текста по умолчанию или перейдите на SecretRef.
   Неинтерактивный путь SecretRef для токена: `--gateway-token-ref-env <ENV_VAR>`.
4. **Каналы** — встроенные и упакованные чат-каналы, такие как BlueBubbles, Discord, Feishu, Google Chat, Mattermost, Microsoft Teams, QQ Bot, Signal, Slack, Telegram, WhatsApp и другие.
5. **Демон** — устанавливает LaunchAgent (macOS), юнит systemd для пользователя (Linux/WSL2) или нативную задачу планировщика Windows с резервным вариантом папки автозагрузки для пользователя.
   Если для аутентификации через токен требуется токен и `gateway.auth.token` управляется через SecretRef, установка демона проверяет его, но не сохраняет разрешённый токен в метаданных окружения службы супервизора.
   Если для аутентификации через токен требуется токен, а настроенная ссылка SecretRef не разрешена, установка демона блокируется с практическими рекомендациями.
   Если настроены и `gateway.auth.token`, и `gateway.auth.password`, а `gateway.auth.mode` не задан, установка демона блокируется до явного задания режима.
6. **Проверка работоспособности** — запускает шлюз (Gateway) и проверяет, что он работает.
7. **Навыки** — устанавливает рекомендуемые навыки и необязательные зависимости.

<Note>
Повторный запуск onboarding **не удаляет** ничего, если вы явно не выберете **Reset** (или не передадите параметр `--reset`).
Параметр CLI `--reset` по умолчанию затрагивает конфигурацию, учётные данные и сессии; используйте `--reset-scope full`, чтобы включить рабочую область.
Если конфигурация недействительна или содержит устаревшие ключи, onboarding предложит вам сначала запустить `openclaw doctor`.
</Note>

**Удалённый режим** только настраивает локальный клиент для подключения к шлюзу (Gateway) на другом хосте.
Он **не устанавливает** и **не изменяет** ничего на удалённом хосте.

## Добавление ещё одного агента

Используйте `openclaw agents add <name>`, чтобы создать отдельного агента со своей рабочей областью, сессиями и профилями аутентификации. Запуск без параметра `--workspace` запускает процесс onboarding.

Что задаётся:

- `agents.list[].name`
- `agents.list[].workspace`
- `agents.list[].agentDir`

Примечания:

- Рабочие области по умолчанию следуют формату `~/.openclaw/workspace-<agentId>`.
- Добавьте `bindings`, чтобы маршрутизировать входящие сообщения (это может сделать onboarding).
- Неинтерактивные флаги: `--model`, `--agent-dir`, `--bind`, `--non-interactive`.
