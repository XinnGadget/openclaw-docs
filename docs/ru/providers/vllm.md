# vLLM

vLLM может предоставлять модели с открытым исходным кодом (и некоторые пользовательские модели) через **HTTP API, совместимый с OpenAI**. OpenClaw может подключаться к vLLM с использованием API `openai-completions`.

OpenClaw также может **автоматически обнаруживать** доступные модели в vLLM, если вы активируете эту функцию с помощью переменной `VLLM_API_KEY` (любое значение подойдёт, если на вашем сервере не настроена аутентификация) и не задаёте явную запись `models.providers.vllm`.

## Быстрый старт

1. Запустите vLLM с сервером, совместимым с OpenAI.

Ваш базовый URL должен предоставлять endpoints `/v1` (например, `/v1/models`, `/v1/chat/completions`). Обычно vLLM работает на:

- `http://127.0.0.1:8000/v1`

2. Активируйте функцию (любое значение подойдёт, если аутентификация не настроена):

```bash
export VLLM_API_KEY="vllm-local"
```

3. Выберите модель (замените на один из ID моделей в вашем vLLM):

```json5
{
  agents: {
    defaults: {
      model: { primary: "vllm/your-model-id" },
    },
  },
}
```

## Обнаружение моделей (неявный провайдер)

Когда задана переменная `VLLM_API_KEY` (или существует профиль аутентификации) и вы **не** задаёте `models.providers.vllm`, OpenClaw выполнит запрос:

- `GET http://127.0.0.1:8000/v1/models`

…и преобразует полученные ID в записи моделей.

Если вы явно зададите `models.providers.vllm`, автоматическое обнаружение будет пропущено, и вам придётся задать модели вручную.

## Явная конфигурация (ручное задание моделей)

Используйте явную конфигурацию, когда:

- vLLM работает на другом хосте/порту.
- Вы хотите зафиксировать значения `contextWindow`/`maxTokens`.
- Ваш сервер требует реальный API-ключ (или вы хотите управлять заголовками).

```json5
{
  models: {
    providers: {
      vllm: {
        baseUrl: "http://127.0.0.1:8000/v1",
        apiKey: "${VLLM_API_KEY}",
        api: "openai-completions",
        models: [
          {
            id: "your-model-id",
            name: "Local vLLM Model",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 128000,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

## Устранение неполадок

- Проверьте доступность сервера:

```bash
curl http://127.0.0.1:8000/v1/models
```

- Если запросы завершаются с ошибками аутентификации, задайте реальный `VLLM_API_KEY`, соответствующий конфигурации вашего сервера, или явно настройте провайдер в разделе `models.providers.vllm`.

## Поведение в стиле прокси

vLLM рассматривается как бэкенд в стиле прокси, совместимый с OpenAI `/v1`, а не как нативный endpoint OpenAI.

- формирование запросов, специфичное только для OpenAI, здесь не применяется;
- отсутствуют `service_tier`, Responses `store`, подсказки для кэширования промтов и формирование полезной нагрузки, совместимой с рассуждениями OpenAI;
- скрытые заголовки атрибуции OpenClaw (`originator`, `version`, `User-Agent`) не добавляются к пользовательским базовым URL vLLM.
