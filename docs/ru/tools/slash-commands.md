# Команды с косой чертой

Команды обрабатываются шлюзом (Gateway). Большинство команд нужно отправлять как **отдельное** сообщение, начинающееся с `/`.
Команда чата bash только для хоста использует `! <cmd>` (с псевдонимом `/bash <cmd>`).

Существуют две связанные системы:

- **Команды**: отдельные сообщения вида `/...`.
- **Директивы**: `/think`, `/fast`, `/verbose`, `/reasoning`, `/elevated`, `/exec`, `/model`, `/queue`.
  - Директивы удаляются из сообщения до того, как его увидит модель.
  - В обычных чат-сообщениях (не состоящих только из директив) они трактуются как "встроенные подсказки" и **не** сохраняют настройки сессии.
  - В сообщениях, состоящих только из директив (сообщение содержит исключительно директивы), они сохраняются в сессии и возвращаются с подтверждением.
  - Директивы применяются только для **авторизованных отправителей**. Если задано `commands.allowFrom`, это единственный используемый белый список; в противном случае авторизация осуществляется через белые списки каналов/сопряжение плюс `commands.useAccessGroups`.
  - Для неавторизованных отправителей директивы трактуются как обычный текст.

Также есть несколько **встроенных сокращений** (только для отправителей из белого списка/авторизованных): `/help`, `/commands`, `/status`, `/whoami` (`/id`).
Они выполняются немедленно, удаляются до того, как модель увидит сообщение, а оставшийся текст продолжает обрабатываться в обычном режиме.

## Конфигурация

```json5
{
  commands: {
    native: "auto",
    nativeSkills: "auto",
    text: true,
    bash: false,
    bashForegroundMs: 2000,
    config: false,
    mcp: false,
    plugins: false,
    debug: false,
    restart: true,
    ownerAllowFrom: ["discord:123456789012345678"],
    ownerDisplay: "raw",
    ownerDisplaySecret: "${OWNER_ID_HASH_SECRET}",
    allowFrom: {
      "*": ["user1"],
      discord: ["user:123"],
    },
    useAccessGroups: true,
  },
}
```

- `commands.text` (по умолчанию `true`) включает разбор `/...` в чат-сообщениях.
  - На платформах без нативных команд (WhatsApp/WebChat/Signal/iMessage/Google Chat/Microsoft Teams) текстовые команды работают, даже если вы установите это значение в `false`.
- `commands.native` (по умолчанию `"auto"`) регистрирует нативные команды.
  - Авто: включено для Discord/Telegram; выключено для Slack (пока вы не добавите команды с косой чертой); игнорируется для провайдеров без нативной поддержки.
  - Задайте `channels.discord.commands.native`, `channels.telegram.commands.native` или `channels.slack.commands.native`, чтобы переопределить настройки для конкретного провайдера (bool или `"auto"`).
  - Значение `false` удаляет ранее зарегистрированные команды в Discord/Telegram при запуске. Команды Slack управляются в приложении Slack и не удаляются автоматически.
- `commands.nativeSkills` (по умолчанию `"auto"`) регистрирует команды **навыков** нативно, если это поддерживается.
  - Авто: включено для Discord/Telegram; выключено для Slack (Slack требует создания команды с косой чертой для каждого навыка).
  - Задайте `channels.discord.commands.nativeSkills`, `channels.telegram.commands.nativeSkills` или `channels.slack.commands.nativeSkills`, чтобы переопределить настройки для конкретного провайдера (bool или `"auto"`).
- `commands.bash` (по умолчанию `false`) включает `! <cmd>` для запуска команд оболочки хоста (`/bash <cmd>` — псевдоним; требует белых списков `tools.elevated`).
- `commands.bashForegroundMs` (по умолчанию `2000`) управляет тем, как долго bash ожидает перед переключением в фоновый режим (`0` — переключение в фоновый режим немедленно).
- `commands.config` (по умолчанию `false`) включает `/config` (чтение/запись `openclaw.json`).
- `commands.mcp` (по умолчанию `false`) включает `/mcp` (чтение/запись конфигурации MCP, управляемой OpenClaw, в разделе `mcp.servers`).
- `commands.plugins` (по умолчанию `false`) включает `/plugins` (обнаружение плагинов/статус, а также управление установкой, включением/отключением).
- `commands.debug` (по умолчанию `false`) включает `/debug` (переопределения только во время выполнения).
- `commands.restart` (по умолчанию `true`) включает `/restart` плюс действия инструмента перезапуска шлюза.
- `commands.ownerAllowFrom` (необязательно) задаёт явный белый список владельцев для поверхностей команд/инструментов только для владельцев. Это отдельно от `commands.allowFrom`.
- `commands.ownerDisplay` управляет тем, как идентификаторы владельцев отображаются в системном промпте: `raw` или `hash`.
- `commands.ownerDisplaySecret` (необязательно) задаёт секрет HMAC, используемый, когда `commands.ownerDisplay="hash"`.
- `commands.allowFrom` (необязательно) задаёт белый список для авторизации команд для каждого провайдера. При настройке это единственный источник авторизации для команд и директив (белые списки каналов/сопряжение и `commands.useAccessGroups` игнорируются). Используйте `"*"` для глобального значения по умолчанию; ключи для конкретного провайдера переопределяют его.
- `commands.useAccessGroups` (по умолчанию `true`) применяет белые списки/политики для команд, когда `commands.allowFrom` не задан.

## Список команд

Актуальный источник истины:

- встроенные базовые команды берутся из `src/auto-reply/commands-registry.shared.ts`;
- сгенерированные команды dock берутся из `src/auto-reply/commands-registry.data.ts`;
- команды плагинов берутся из вызовов `registerCommand()` плагинов;
- фактическая доступность на вашем шлюзе по-прежнему зависит от флагов конфигурации, поверхности канала и установленных/включённых плагинов.

### Встроенные базовые команды

Доступные на данный момент встроенные команды:

- `/new [model]` начинает новую сессию; `/reset` — псевдоним для сброса.
- `/compact [instructions]` сокращает контекст сессии. См. [/concepts/compaction](/concepts/compaction).
- `/stop` прерывает текущий запуск.
- `/session idle <duration|off>` и `/session max-age <duration|off>` управляют истечением привязки потока.
- `/think <off|minimal|low|medium|high|xhigh>` устанавливает уровень размышлений. Псевдонимы: `/thinking`, `/t`.
- `/verbose on|off|full` включает/выключает подробный вывод. Псевдоним: `/v`.
- `/fast [status|on|off]` показывает или устанавливает быстрый режим.
- `/reasoning [on|off|stream]` включает/выключает видимость рассуждений. Псевдоним: `/reason`.
- `/elevated [on|off|ask|full]` включает/выключает повышенный режим. Псевдоним: `/elev`.
- `/exec host=<auto|sandbox|gateway|node> security=<deny|allowlist|full> ask=<off|on-miss|always> node=<id>` показывает или устанавливает значения по умолчанию для exec.
- `/model [name|#|status]` показывает или устанавливает модель.
- `/models [provider] [page] [limit=<n>|size=<n>|all]` выводит список провайдеров или моделей для провайдера.
- `/queue <mode>` управляет поведением очереди (`steer`, `interrupt`, `followup`, `collect`, `steer-backlog`), а также такими опциями, как `debounce:2s cap:25 drop:summarize`.
- `/help` показывает краткое справочное сообщение.
- `/commands` показывает сгенерированный каталог команд.
- `/tools [compact|verbose]` показывает, что текущий агент может использовать прямо сейчас.
- `/status` показывает статус во время выполнения, включая использование/квоту провайдера, если доступно.
- `/tasks` выводит список активных/недавних фоновых задач для текущей сессии.
- `/context [list|detail|json]` объясняет, как собирается контекст.
- `/export-session [path]` экспортирует текущую сессию в HTML. Псевдоним: `/export`.
- `/whoami` показывает ваш идентификатор отправителя. Псевдоним: `/id`.
- `/skill <name> [input]` запускает навык по имени.
- `/allowlist [list|add|remove] ...` управляет записями белого списка. Только текст.
- `/approve <id> <decision>` разрешает запросы на утверждение exec.
- `/btw <question>` задаёт побочный вопрос без изменения будущего контекста сессии. См. [/tools/btw](/tools/btw).
- `/subagents list|kill|log|info|send|steer|spawn` управляет запусками субагентов для текущей сессии.
- `/acp spawn|cancel|steer|close|sessions|status|set-mode|set|cwd|permissions|timeout|model|reset-options|doctor|install|help` управляет сессиями ACP и параметрами во время выполнения.
- `/focus <target>` привязывает текущий поток Discord или тему/беседу Telegram к цели сессии.
- `/unfocus` удаляет текущую привязку.
- `/agents` выводит список привязанных к потоку агентов для текущей сессии.
- `/kill <id|#|all>` прерывает один или все запущенные субагенты.
- `/steer <id|#> <message>` отправляет управляющие команды запущенному субагенту. Псевдоним: `/tell`.
- `/config show|get|set|unset` читает или записывает `openclaw.json`. Только для владельца. Требуется `commands.config: true`.
- `/mcp show|get|set|unset` читает или записывает конфигурацию сервера MCP, управляемую OpenClaw, в разделе `mcp.servers`. Только для владельца. Требуется `commands.mcp: true`.
- `/plugins list|inspect|show|get|install|enable|disable` проверяет или изменяет состояние плагина. `/plugin` — псевдоним. Только для владельца при записи. Требуется `commands.plugins: true`.
- `/debug show|set|unset|reset` управляет переопределениями конфигурации только во время выполнения. Только для владельца. Требуется `commands.debug: true`.
- `/usage off|tokens|full|cost` управляет футером использования для каждого ответа или выводит локальную сводку затрат.
- `/tts on|off|status|provider|limit|summary|audio|help` управляет TTS. См. [/tools/tts](/tools/tts).
- `/restart` перезапускает OpenClaw, если включено. По умолчанию включено; установите `commands.restart: false`, чтобы отключить.
- `/activation mention|always` устанавливает режим активации группы.
- `/send on|off|inherit` устанавливает политику отправки. Только для владельца.
- `/bash <command>` запускает команду оболочки хоста. Только текст. Псевдоним: `! <command>`. Требуется `commands.bash: true` плюс белые списки `tools.elevated`.
- `!poll [sessionId]` проверяет фоновое задание bash.
- `!stop [sessionId]` останавливает фоновое задание bash.

### Сгенерированные команды dock

Команды dock генерируются из плагинов каналов с поддержкой нативных команд. Текущий комплект:

- `/dock-discord` (псевдоним: `/dock_discord`);
- `/dock-mattermost` (псевдоним: `/dock_mattermost`);
- `/dock-slack` (псевдоним: `/dock_slack`);
- `/dock-telegram` (псевдоним: `/dock_telegram`).

### Команды встроенных плагинов

Встроенные плагины могут добавлять дополнительные команды с косой чертой. Текущие встроенные команды в этом репозитории:

- `/dreaming [on|off|status|help]` включает/выключает "сны" памяти. См. [Dreaming](/concepts/dreaming).
- `/pair [qr|status|pending|approve|cleanup|notify]` управляет потоком сопряжения/настройки устройства. См. [Pairing](/channels/pairing).
- `/phone status|arm <camera|screen|writes|all> [duration]|disarm` временно активирует команды узла телефона с высоким риском.
- `/voice status|list [limit]|set <voiceId|name>` управляет конфигурацией голоса Talk. В Discord имя нативной команды — `/talkvoice`.
- `/card ...` отправляет предустановки богатых карточек LINE. См. [LINE](/channels/line).
- Команды только для QQBot:
  - `/bot-ping`;
  - `/bot-version`;
  - `/bot-help`;
  - `/bot-upgrade`;
  - `/bot-logs`.

### Динамические команды навыков

Навыки, вызываемые пользователем, также представлены в виде команд с косой чертой:

- `/skill <name> [input]` всегда работает как универсальный вход.
- навыки также могут отображаться как прямые команды, например `/prose`, когда навык/плагин их регистрирует.
- регистрация нативных команд навыков управляется с помощью `commands.nativeSkills` и `channels.<provider>.commands.nativeSkills`.

Примечания:

- Команды принимают необязательный `:` между командой и аргументами (например, `/think: high`, `/send: on`, `/help:`).
- `/new <model>` принимает псевдоним модели, `provider/model` или имя провайдера (нечёткое сопоставление); если совпадения нет, текст трактуется как тело сообщения.
- Для полной разбивки использования провайдера используйте `openclaw status --usage`.
- `/allowlist add|remove` требует `commands.config=true` и учитывает `configWrites` канала.
- В каналах с несколькими учётными записями нацеленные на конфигурацию `/allowlist --account <id>` и `/config set channels.<provider>.accounts.<id>...` также учитывают `configWrites` целевой учётной записи.
- `/usage` управляет футером использования для каждого ответа; `/usage cost` выводит локальную сводку затрат из журналов сессий OpenClaw.
- `/restart` включён по умолчанию; установите `commands.restart: false`, чтобы отключить его.
- `/plugins install <spec>` принимает те же спецификации плагинов, что и `openclaw plugins install`: локальный путь/архив, пакет npm или `clawhub:<pkg>`.
- `/plugins enable|disable` обновляет конфигурацию плагина и может запросить перезапуск.
- Нативная команда только для Discord: `/vc join|leave|status` управляет голосовыми каналами (требуется `channels.discord.voice` и нативные команды; недоступна в текстовом виде).
- Команды привязки потоков Discord (`/focus`, `/unfocus`, `/agents`, `/session idle`, `/session max-age`) требуют, чтобы эффективная привязка потоков была включена (`session.threadBindings.enabled` и/или `channels.discord.threadBindings.enabled`).
- Справочник команд ACP и поведение во время выполнения: [ACP Agents](/tools/acp-agents).
- `/verbose` предназначен для отладки и дополнительной видимости; в обычном использовании его следует **выключать**.
- `/fast on|off` сохраняет переопределение сессии. Используйте опцию `inherit` в пользовательском интерфейсе сессий, чтобы сбросить его и вернуться к значениям по умолчанию в конфигурации.
- `/fast` зависит от провайдера: OpenAI/OpenAI Codex сопоставляют его с `service_tier=priority` в нативных конечных точках Responses, а прямые публичные запросы к Anthropic, включая трафик с OAuth-аутентификацией, отправляемый на `api.anthropic.com`, сопоставляют его с `service_tier=auto` или `standard_only`. См. [OpenAI](/providers/openai) и [Anthropic](/providers/anthropic).
- Сводные данные о сбоях инструментов по-прежнему отображаются, когда это актуально, но подробный текст сбоя включается только тогда, когда `/verbose` имеет значение `on` или `full`.
- `/reasoning` (и `/verbose`) опасны в групповых настройках: они могут раскрыть внутренние рассуждения или вывод инструмента, который вы не планировали раскрывать. Лучше оставлять их выключенными, особенно в групповых чатах.
- `/model` немедленно сохраняет новую модель сессии.
- Если агент бездействует, следующий запуск использует её сразу.
- Если запуск уже активен, OpenClaw помечает переключение в реальном времени как ожидающее и перезапускает с новой моделью только в точке чистой повторной попытки.
- Если активность инструмента или вывод ответа уже начались, ожидающее переключение может оставаться в очереди до более поздней возможности повторной попытки или следующего хода пользователя.
- **Быстрый путь**: сообщения, содержащие только команды, от отправителей из белого списка обрабатываются немедленно (в обход очереди и модели).
- **Управление упоминаниями в группах**: сообщения, содержащие только команды, от отправителей из белого списка обходят требования к упоминаниям.
- **Встроенные сокращения (только для отправителей из белого списка)**: некоторые команды также работают, если они встроены в обычное сообщение, и удаляются до того, как модель увидит оставшийся текст.
  - Пример: `hey /status` запускает ответ о статусе, а оставшийся текст продолжает обрабатываться в обычном режиме.
- В настоящее время: `/help`, `/commands`, `/status`, `/whoami` (`/id`).
- Неавторизованные сообщения, содержащие только команды, молча игнорируются, а встроенные токены `/...` трактуются как обычный текст.
- **Команды навыков**: навыки, вызываемые пользователем (`user-invocable`), представлены в виде команд с косой чертой. Имена очищаются до `a-z0-9_` (максимум 32 символа); при конфликтах добавляются числовые суффиксы (например, `_2`).
  - `/skill <name> [input]` запускает навык по имени (полезно, когда ограничения нативных команд не позволяют создавать команды для каждого навыка).
  - По умолчанию команды навыков пересылаются модели как обычный запрос.
  - Навыки могут опционально объявлять `command-dispatch: tool`, чтобы направлять команду непосредственно в инструмент (детерминированно, без модели).
  - Пример: `/prose` (плагин OpenProse) — см. [OpenProse](/prose).
- **Аргументы нативных команд**: Discord использует автозаполнение для динамических опций (и меню кнопок, когда вы опускаете обязательные аргументы). Telegram и Slack показывают меню кнопок, когда команда поддерживает выбор и вы опускаете аргумент.

## `/tools`

`/tools` отвечает на вопрос во время выполнения, а не на вопрос конфигурации: **что этот агент может использовать прямо сейчас в текущем разговоре**.

- По умолчанию `/tools` компактен и оптимизирован для быстрого просмотра.
- `/tools verbose` добавляет краткие описания.
- Поверхности нативных команд, поддерживающие аргументы, предоставляют тот же переключатель режимов, что и `compact|verbose`.
- Результаты ограничены сессией, поэтому изменение агента, канала, потока, авторизации отправителя или модели может изменить вывод.
- `/tools` включает инструменты, которые фактически доступны во время выполнения, в том числе базовые инструменты, подключённые инструменты плагинов и инструменты, принадлежащие каналу.

Для редактирования профиля и переопределений используйте панель инструментов пользовательского интерфейса управления или поверхности конфигурации/каталога вместо того, чтобы рассматривать `/tools` как статический каталог.

## Поверхности использования (что где отображается)

- **Использование/квота провайдера** (пример: "Claude осталось 80 %") отображается в `/status` для текущего провайдера модели, когда отслеживание использования включено. OpenClaw нормализует окна провайдера до `% осталось`; для MiniMax поля с оставшимся процентом инвертируются перед отображением, а ответы `model_remains` отдают предпочтение записи модели чата плюс метке плана с тегом модели.
- **Строки токенов/кэша** в `/status` могут опираться на последнюю запись использования транскрипции, когда моментальный снимок живой сессии скуден. Существующие ненулевые значения во время выполнения по-прежнему имеют приоритет, а резервное копирование из транскрипции также может восстановить активную метку модели во время выполнения плюс более крупный итог, ориентированный на промпт, когда сохранённые итоги отсутствуют или меньше.
- **Токены/стоимость для каждого ответа** управляется с помощью `/usage off|tokens|full` (добавляется к обычным ответам).
- `/model status` относится к **моделям/авторизации/конечным точкам**, а не к использованию.

## Выбор модели (`/model`)

`/model` реализован как директива.

Примеры:

```
/model
/model list
/model 3
/model openai/gpt-5.4
/model opus@anthropic:default
/model status
```

Примечания:

- `/model` и `/model list` показывают компактный пронумерованный селектор (семейство моделей + доступные провайдеры).
- В Discord `/model` и `/models` открывают интерактивный селектор с выпадающими списками провайдеров и моделей плюс шагом "Отправить".
- `/model <#>` выбирает из этого селектора (и по возможности отдаёт предпочтение текущему провайдеру).
- `/model status` показывает подробный вид, включая настроенную конечную точку провайдера (`baseUrl`) и режим API (`api`), если доступно.

## Переопределения для отладки

`/debug` позволяет устанавливать переопределения конфигурации **только во время выполнения** (в памяти, а не на диске). Только для владельца. По умолчанию отключено; включите с помощью `commands.debug: true`.

Примеры:

```
/debug show
/debug set messages.responsePrefix="[openclaw]"
/debug set channels.whatsapp.allowFrom=["+1555","+4477"]
/debug unset messages.responsePrefix
/debug reset
```

Примечания:

- Переопределения применяются немедленно к новым чтениям конфигурации, но **не** записываются в `openclaw.json`.
- Используйте `/debug reset`, чтобы очистить все переопределения и вернуться к конфигурации на диске.

## Обновления конфигурации

`/config` записывает в вашу конфигурацию на диске (`openclaw.json`). Только для владельца. По умолчанию отключено; включите с помощью `commands.config: true`.

Примеры:

```
/config show
/config show messages.responsePrefix
/config get messages.responsePrefix
/config set messages.responsePrefix="[openclaw]"
/config unset messages.responsePrefix
```

Примечания:

- Конфигурация проверяется перед записью; недопустимые изменения отклоняются.
- Обновления `/config` сохраняются после перезапусков.

## Обновления MCP

`/mcp` записывает определения серверов MCP, управляемые OpenClaw, в разделе `mcp.servers`. Только для владельца. По умолчанию отключено; включите с помощью `commands.mcp: true`.

Примеры:

```text
/mcp show
/mcp show context7
/mcp set context7={"command":"uvx","args":["context7-mcp"]}
/mcp unset context7
```

Примечания:

- `/mcp` сохраняет конфигурацию в конфигурации OpenClaw, а не в настройках проекта, принадлежащих Pi.
- Адаптеры во время выполнения решают, какие транспорты фактически выполнимы.

## Обновления плагинов

`/plugins` позволяет операторам проверять обнаруженные плагины и переключать включение в конфигурации. Потоки только для чтения могут использовать `/plugin` в качестве псевдонима. По умолчанию отключено; включите с помощью `commands.plugins: true`.

Примеры:

```text
/plugins
/plugins list
/plugin show context7
/plugins enable context7
/plugins disable context7
```

Примечания:

- `/plugins list` и `/plugins show` используют реальное обнаружение плагинов для текущего рабочего пространства плюс конфигурацию на диске.
- `/plugins enable|disable` обновляет только конфигурацию плагина; он не устанавливает и не удаляет плагины.
- После изменений включения/отключения перезапустите шлюз, чтобы применить их.

## Примечания о поверхностях

- **Текстовые команды** выполняются в обычном чате (в личных сообщениях используется `main`, в группах — своя сессия).
- **Нативные команды** используют изолированные сессии:
  - Discord: `agent:<agentId>:discord:slash:<userId>`;
  - Slack: `agent:<agentId>:slack:slash:<userId>` (префикс настраивается через `channels.slack.slashCommand.sessionPrefix`);
  - Telegram: `telegram:slash:<userId>` (нацелено на сессию чата через `CommandTargetSessionKey`).
- **`/stop`** нацелен на активную чат-сессию, поэтому может прервать текущий запуск.
- **Slack**: `channels.slack.slashCommand` по-прежнему поддерживается для одной команды в стиле `/openclaw`. Если вы включите `commands.native`, вам нужно будет создать одну команду Slack с косой чертой для каждой встроенной команды (с теми же именами, что и в `/help`). Меню аргументов команд для Slack доставляются в виде эфемерных кнопок Block Kit.
  - Исключение для нативных команд Slack: зарегистрируйте `/agentstatus` (не `/status`), потому что Slack резервирует `/status`. Текстовая команда `/status` по-прежнему работает в сообщениях Slack.

## Побочные вопросы BTW

`/btw` — это быстрый **побочный вопрос** о текущей сессии.

В отличие от обычного чата:

- он использует текущую сессию в качестве фонового контекста;
- выполняется как отдельный **без инструментальный** разовый вызов;
- не изменяет будущий контекст сессии;
- не записывается в историю транскрипции;
- доставляется как живой побочный результат вместо обычного сообщения ассистента.

Это делает `/btw` полезным, когда вам нужно временное уточнение, пока основная задача продолжается.

Пример:

```text
/btw что мы сейчас делаем?
```

См. [BTW Side Questions](/tools/btw) для полного описания поведения и деталей пользовательского опыта.
