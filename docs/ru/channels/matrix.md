# Matrix

Matrix — это встроенный плагин канала для OpenClaw.
Он использует официальный `matrix-js-sdk` и поддерживает личные сообщения (DM), комнаты, потоки (threads), медиа, реакции, опросы, передачу местоположения и E2EE.

## Встроенный плагин

Matrix поставляется как встроенный плагин в текущих релизах OpenClaw, поэтому для обычных сборок отдельная установка не требуется.

Если вы используете более старую сборку или пользовательскую установку, в которой Matrix отсутствует, установите его вручную:

Установка из npm:

```bash
openclaw plugins install @openclaw/matrix
```

Установка из локальной копии:

```bash
openclaw plugins install ./path/to/local/matrix-plugin
```

См. [Плагины](/tools/plugin) для ознакомления с поведением плагинов и правилами установки.

## Настройка

1. Убедитесь, что плагин Matrix доступен:
   - В текущих сборках OpenClaw он уже включён.
   - В более старых или пользовательских установках его можно добавить вручную с помощью команд выше.
2. Создайте учётную запись Matrix на своём сервере.
3. Настройте `channels.matrix`, указав одно из следующих сочетаний:
   - `homeserver` + `accessToken`;
   - `homeserver` + `userId` + `password`.
4. Перезапустите шлюз.
5. Начните личное сообщение с ботом или пригласите его в комнату.
   - Новые приглашения в Matrix работают только в том случае, если `channels.matrix.autoJoin` их разрешает.

Интерактивные пути настройки:

```bash
openclaw channels add
openclaw configure --section channels
```

Мастер настройки Matrix запрашивает:

- URL сервера homeserver;
- метод аутентификации: токен доступа или пароль;
- идентификатор пользователя (только для аутентификации по паролю);
- необязательное имя устройства;
- необходимость включения E2EE;
- необходимость настройки доступа к комнатам и автоматического присоединения по приглашениям.

Основные особенности мастера:

- Если переменные окружения для аутентификации Matrix уже существуют, а учётная запись ещё не сохранена в конфигурации, мастер предлагает использовать переменные окружения для сохранения аутентификации.
- Имена учётных записей нормализуются до идентификатора учётной записи. Например, `Ops Bot` становится `ops-bot`.
- В список разрешённых DM можно напрямую добавить `@user:server`; отображаемые имена работают только в том случае, если при поиске в каталоге найдено ровно одно совпадение.
- В список разрешённых комнат можно напрямую добавить идентификаторы комнат и псевдонимы. Рекомендуется использовать `!room:server` или `#alias:server`; неразрешённые имена игнорируются во время выполнения при разрешении списка разрешённых.
- В режиме автоматического присоединения по приглашениям используйте только стабильные цели приглашения: `!roomId:server`, `#alias:server` или `*`. Простые имена комнат отклоняются.
- Чтобы разрешить имена комнат перед сохранением, используйте `openclaw channels resolve --channel matrix "Project Room"`.

<Предупреждение>
По умолчанию `channels.matrix.autoJoin` имеет значение `off`.

Если вы не зададите это значение, бот не будет присоединяться к приглашённым комнатам или новым приглашениям в стиле DM, поэтому он не появится в новых группах или приглашённых DM, пока вы не присоединитесь вручную.

Задайте `autoJoin: "allowlist"` вместе с `autoJoinAllowlist`, чтобы ограничить приглашения, которые он принимает, или задайте `autoJoin: "always"`, если хотите, чтобы бот присоединялся ко всем приглашениям.

В режиме `allowlist` параметр `autoJoinAllowlist` принимает только `!roomId:server`, `#alias:server` или `*`.
</Предупреждение>

Пример списка разрешённых:

```json5
{
  channels: {
    matrix: {
      autoJoin: "allowlist",
      autoJoinAllowlist: ["!ops:example.org", "#support:example.org"],
      groups: {
        "!ops:example.org": {
          requireMention: true,
        },
      },
    },
  },
}
```

Присоединение ко всем приглашениям:

```json5
{
  channels: {
    matrix: {
      autoJoin: "always",
    },
  },
}
```

Минимальная настройка на основе токена:

```json5
{
  channels: {
    matrix: {
      enabled: true,
      homeserver: "https://matrix.example.org",
      accessToken: "syt_xxx",
      dm: { policy: "pairing" },
    },
  },
}
```

Настройка на основе пароля (токен кэшируется после входа):

```json5
{
  channels: {
    matrix: {
      enabled: true,
      homeserver: "https://matrix.example.org",
      userId: "@bot:example.org",
      password: "replace-me", // pragma: allowlist secret
      deviceName: "OpenClaw Gateway",
    },
  },
}
```

Matrix хранит кэшированные учётные данные в `~/.openclaw/credentials/matrix/`.
Для учётной записи по умолчанию используется `credentials.json`, для именованных учётных записей — `credentials-<account>.json`.
Если кэшированные учётные данные существуют, OpenClaw считает, что Matrix настроен для настройки, диагностики и обнаружения статуса канала, даже если текущая аутентификация не задана напрямую в конфигурации.

Эквиваленты переменных окружения (используются, если ключ конфигурации не задан):

- `MATRIX_HOMESERVER`;
- `MATRIX_ACCESS_TOKEN`;
- `MATRIX_USER_ID`;
- `MATRIX_PASSWORD`;
- `MATRIX_DEVICE_ID`;
- `MATRIX_DEVICE_NAME`.

Для учётных записей, отличных от учётной записи по умолчанию, используйте переменные окружения с областью действия учётной записи:

- `MATRIX_<ACCOUNT_ID>_HOMESERVER`;
- `MATRIX_<ACCOUNT_ID>_ACCESS_TOKEN`;
- `MATRIX_<ACCOUNT_ID>_USER_ID`;
- `MATRIX_<ACCOUNT_ID>_PASSWORD`;
- `MATRIX_<ACCOUNT_ID>_DEVICE_ID`;
- `MATRIX_<ACCOUNT_ID>_DEVICE_NAME`.

Пример для учётной записи `ops`:

- `MATRIX_OPS_HOMESERVER`;
- `MATRIX_OPS_ACCESS_TOKEN`.

Для нормализованного идентификатора учётной записи `ops-bot` используйте:

- `MATRIX_OPS_X2D_BOT_HOMESERVER`;
- `MATRIX_OPS_X2D_BOT_ACCESS_TOKEN`.

Matrix экранирует знаки препинания в идентификаторах учётных записей, чтобы избежать конфликтов переменных окружения с областью действия.
Например, `-` становится `_X2D_`, поэтому `ops-prod` сопоставляется с `MATRIX_OPS_X2D_PROD_*`.

Интерактивный мастер предлагает использовать переменные окружения только в том случае, если эти переменные уже присутствуют, а выбранная учётная запись ещё не имеет сохранённой аутентификации Matrix в конфигурации.

## Пример конфигурации

Ниже приведена практическая базовая конфигурация с сопряжением DM, списком разрешённых комнат и включённым E2EE:

```json5
{
  channels: {
    matrix: {
      enabled: true,
      homeserver: "https://matrix.example.org",
      accessToken: "syt_xxx",
      encryption: true,

      dm: {
        policy: "pairing",
        sessionScope: "per-room",
        threadReplies: "off",
      },

      groupPolicy: "allowlist",
      groupAllowFrom: ["@admin:example.org"],
      groups: {
        "!roomid:example.org": {
          requireMention: true,
        },
      },

      autoJoin: "allowlist",
      autoJoinAllowlist: ["!roomid:example.org"],
      threadReplies: "inbound",
      replyToMode: "off",
      streaming: "partial",
    },
  },
}
```

`autoJoin` применяется ко всем приглашениям в Matrix, включая приглашения в стиле DM. OpenClaw не может надёжно классифицировать приглашённую комнату как DM или группу во время приглашения, поэтому все приглашения сначала проходят через `autoJoin`. `dm.policy` применяется после того, как бот присоединился и комната классифицирована как DM.
