# Запланированные задачи (Cron)

Cron — это встроенный планировщик в Gateway. Он сохраняет задания, пробуждает агент в нужное время и может передавать результаты обратно в канал чата или на эндпоинт вебхука.

## Быстрый старт

```bash
# Добавить одноразовое напоминание
openclaw cron add \
  --name "Напоминание" \
  --at "2026-02-01T16:00:00Z" \
  --session main \
  --system-event "Напоминание: проверить черновик документации по cron" \
  --wake now \
  --delete-after-run

# Проверить свои задания
openclaw cron list

# Посмотреть историю запусков
openclaw cron runs --id <job-id>
```

## Как работает cron

- Cron работает **внутри процесса Gateway** (не внутри модели).
- Задания сохраняются в `~/.openclaw/cron/jobs.json`, поэтому при перезапуске расписания не теряются.
- Все выполнения cron создают записи [фоновой задачи](/automation/tasks).
- Одноразовые задания (`--at`) по умолчанию автоматически удаляются после успешного выполнения.
- Изолированные запуски cron по возможности закрывают отслеживаемые вкладки браузера/процессы для сессии `cron:<jobId>` после завершения запуска, чтобы автоматизация браузера не оставляла "осиротевших" процессов.
- Изолированные запуски cron также защищают от устаревших ответов-подтверждений. Если первый результат — это лишь промежуточное обновление статуса (`on it`, `pulling everything together` и аналогичные подсказки), и никакой дочерний субагент по-прежнему не отвечает за окончательный ответ, OpenClaw повторно запрашивает фактический результат перед отправкой.

Согласование задач для cron осуществляется во время выполнения: активная задача cron остаётся активной, пока среда выполнения cron отслеживает выполнение задания, даже если старая строка дочерней сессии всё ещё существует. Как только среда выполнения перестаёт управлять заданием и истекает 5-минутное льготное окно, обслуживание может пометить задачу как `lost`.

## Типы расписаний

| Тип | Флаг CLI | Описание |
| --- | --- | --- |
| `at` | `--at` | Одноразовая отметка времени (ISO 8601 или относительная, например, `20m`) |
| `every` | `--every` | Фиксированный интервал |
| `cron` | `--cron` | Выражение cron из 5 или 6 полей с необязательным параметром `--tz` |

Отметки времени без указания часового пояса считаются UTC. Добавьте `--tz America/New_York`, чтобы использовать локальное время.

Повторяющиеся выражения, запускаемые в начале часа, автоматически сдвигаются на срок до 5 минут, чтобы снизить пики нагрузки. Используйте `--exact`, чтобы принудительно установить точное время, или `--stagger 30s` для явного указания окна.

## Стили выполнения

| Стиль | Значение `--session` | Где выполняется | Для чего подходит |
| --- | --- | --- | --- |
| Основная сессия | `main` | В следующем цикле heartbeat | Напоминания, системные события |
| Изолированная | `isolated` | В выделенной сессии `cron:<jobId>` | Отчёты, фоновые задачи |
| Текущая сессия | `current` | Привязана ко времени создания | Периодическая работа с учётом контекста |
| Пользовательская сессия | `session:custom-id` | Постоянная именованная сессия | Рабочие процессы, использующие историю |

Задания **основной сессии** ставят в очередь системное событие и при необходимости пробуждают heartbeat (`--wake now` или `--wake next-heartbeat`). **Изолированные** задания запускают выделенный цикл агента с новой сессией. **Пользовательские сессии** (`session:xxx`) сохраняют контекст между запусками, позволяя реализовывать рабочие процессы вроде ежедневных стендапов, опирающихся на предыдущие сводки.

Для изолированных заданий завершение работы среды выполнения теперь включает попытку очистки браузера для этой сессии cron. Ошибки очистки игнорируются, поэтому фактический результат cron остаётся приоритетным.

Когда изолированные запуски cron управляют субагентами, приоритет отдаётся окончательному результату дочернего элемента, а не устаревшему промежуточному тексту родительского элемента. Если дочерние элементы всё ещё выполняются, OpenClaw подавляет частичное обновление родительского элемента вместо того, чтобы объявлять его.

### Параметры полезной нагрузки для изолированных заданий

- `--message`: текст подсказки (обязательно для изолированных заданий);
- `--model` / `--thinking`: переопределение модели и уровня мышления;
- `--light-context`: пропустить внедрение файла начальной загрузки рабочей области;
- `--tools exec,read`: ограничить инструменты, которые может использовать задание.

Параметр `--model` использует выбранную разрешённую модель для задания. Если запрошенная модель не разрешена, cron записывает предупреждение и возвращается к выбору модели по умолчанию для агента/задания. Настроенные цепочки отката по-прежнему применяются, но простое переопределение модели без явного списка отката для каждого задания больше не добавляет основную модель агента в качестве скрытого дополнительного целевого объекта для повторной попытки.

Приоритет выбора модели для изолированных заданий:

1. Переопределение модели через хук Gmail (если запуск произошёл через Gmail и такое переопределение разрешено).
2. Параметр `model` в полезной нагрузке для задания.
3. Переопределение модели в сохранённой сессии cron.
4. Выбор модели по умолчанию для агента.

Быстрый режим также следует выбранному в реальном времени варианту. Если в конфигурации выбранной модели есть `params.fastMode`, изолированный cron использует его по умолчанию. Переопределение `fastMode` в сохранённой сессии по-прежнему имеет приоритет над конфигурацией в любом направлении.

Если изолированный запуск сталкивается с переключением модели в реальном времени, cron повторяет попытку с новым провайдером/моделью и сохраняет выбранный вариант перед повторной попыткой. Если переключение также включает новый профиль аутентификации, cron сохраняет переопределение профиля аутентификации. Попытки ограничены: после первоначальной попытки и двух повторных попыток переключения cron прерывает выполнение, чтобы избежать бесконечного цикла.

## Доставка и вывод

| Режим | Что происходит |
| --- | --- |
| `announce` | Отправка сводки в целевой канал (по умолчанию для изолированных заданий) |
| `webhook` | Отправка завершённого пакета данных на URL через POST |
| `none` | Только внутреннее использование, без доставки |

Используйте `--announce --channel telegram --to "-1001234567890"` для доставки в канал. Для тем форума Telegram используйте `-1001234567890:topic:123`. Для целей Slack/Discord/Mattermost следует использовать явные префиксы (`channel:<id>`, `user:<id>`).

Для изолированных заданий, управляемых cron, путь окончательной доставки определяется исполнителем. Агенту предлагается вернуть сводку в виде обычного текста, которая затем отправляется через `announce`, `webhook` или сохраняется внутри системы для `none`. Параметр `--no-deliver` не возвращает доставку агенту; он сохраняет выполнение внутри системы.

Если исходная задача явно указывает на необходимость отправки сообщения внешнему получателю, агент должен указать, кому/куда должно быть отправлено сообщение, в своём выводе, вместо того чтобы пытаться отправить его напрямую.

Уведомления об ошибках следуют отдельному пути назначения:

- `cron.failureDestination` задаёт глобальное значение по умолчанию для уведомлений об ошибках.
- `job.delivery.failureDestination` переопределяет его для конкретного задания.
- Если ни один из параметров не задан и задание уже доставляется через `announce`, уведомления об ошибках теперь возвращаются к основной цели объявления.
- `delivery.failureDestination` поддерживается только для заданий с `sessionTarget="isolated"`, если основной режим доставки не является `webhook`.

## Примеры CLI

Одноразовое напоминание (основная сессия):

```bash
openclaw cron add \
  --name "Проверка календаря" \
  --at "20m" \
  --session main \
  --system-event "Следующий heartbeat: проверить календарь." \
  --wake now
```

Повторяющееся изолированное задание с доставкой:

```bash
openclaw cron add \
  --name "Утренний брифинг" \
  --cron "0 7 * * *" \
  --tz "America/Los_Angeles" \
  --session isolated \
  --message "Свести воедино обновления за ночь." \
  --announce \
  --channel slack \
  --to "channel:C1234567890"
```

Изолированное задание с переопределением модели и уровня мышления:

```bash
openclaw cron add \
  --name "Глубокий анализ" \
  --cron "0 6 * * 1" \
  --tz "America/Los_Angeles" \
  --session isolated \
  --message "Еженедельный глубокий анализ хода проекта." \
  --model "opus" \
  --thinking high \
  --announce
```
