# Обнаружение tool-loop

OpenClaw может предотвращать застревание агентов в повторяющихся шаблонах вызовов инструментов.
Защита **по умолчанию отключена**.

Включайте её только там, где это необходимо, — при строгих настройках она может блокировать легитимные повторяющиеся вызовы.

## Зачем это нужно

- Обнаруживать повторяющиеся последовательности, не приводящие к прогрессу.
- Обнаруживать высокочастотные циклы без результата (один и тот же инструмент, одни и те же входные данные, повторяющиеся ошибки).
- Обнаруживать определённые шаблоны повторяющихся вызовов для известных инструментов с опросом (polling).

## Блок конфигурации

Глобальные значения по умолчанию:

```json5
{
  tools: {
    loopDetection: {
      enabled: false,
      historySize: 30,
      warningThreshold: 10,
      criticalThreshold: 20,
      globalCircuitBreakerThreshold: 30,
      detectors: {
        genericRepeat: true,
        knownPollNoProgress: true,
        pingPong: true,
      },
    },
  },
}
```

Переопределение для отдельных агентов (необязательно):

```json5
{
  agents: {
    list: [
      {
        id: "safe-runner",
        tools: {
          loopDetection: {
            enabled: true,
            warningThreshold: 8,
            criticalThreshold: 16,
          },
        },
      },
    ],
  },
}
```

### Поведение полей

- `enabled`: главный переключатель. Значение `false` означает, что обнаружение циклов не выполняется.
- `historySize`: количество последних вызовов инструментов, сохраняемых для анализа.
- `warningThreshold`: порог, после которого шаблон классифицируется как предупредительный (только предупреждение).
- `criticalThreshold`: порог для блокировки повторяющихся шаблонов циклов.
- `globalCircuitBreakerThreshold`: глобальный порог срабатывания механизма прерывания при отсутствии прогресса.
- `detectors.genericRepeat`: обнаруживает повторяющиеся шаблоны с одним и тем же инструментом и параметрами.
- `detectors.knownPollNoProgress`: обнаруживает известные шаблоны опроса (polling) без изменения состояния.
- `detectors.pingPong`: обнаруживает чередующиеся шаблоны типа "пинг-понг".

## Рекомендуемая настройка

- Начните с `enabled: true`, оставив значения по умолчанию без изменений.
- Соблюдайте порядок порогов: `warningThreshold < criticalThreshold < globalCircuitBreakerThreshold`.
- Если возникают ложные срабатывания:
  - увеличьте `warningThreshold` и/или `criticalThreshold`;
  - (опционально) увеличьте `globalCircuitBreakerThreshold`;
  - отключите только тот детектор, который вызывает проблемы;
  - уменьшите `historySize`, чтобы снизить строгость учёта исторического контекста.

## Журналы и ожидаемое поведение

При обнаружении цикла OpenClaw регистрирует событие цикла и блокирует или ограничивает следующий цикл вызовов инструментов — в зависимости от серьёзности ситуации.
Это защищает пользователей от неконтролируемого расхода токенов и зависаний, сохраняя при этом обычный доступ к инструментам.

- Сначала отдавайте предпочтение предупреждению и временному ограничению.
- Усиливайте меры только при накоплении повторяющихся свидетельств.

## Примечания

- `tools.loopDetection` объединяется с переопределениями на уровне агента.
- Конфигурация для отдельного агента полностью переопределяет или расширяет глобальные значения.
- Если конфигурация отсутствует, защитные механизмы остаются отключёнными.
