# الحلم (تجريبي)

الحلم هو نظام دمج الذاكرة في الخلفية ضمن `memory-core`.
يساعد OpenClaw على نقل الإشارات القوية قصيرة الأمد إلى ذاكرة دائمة مع
الحفاظ على أن تكون العملية قابلة للتفسير والمراجعة.

الحلم **اختياري** وهو معطّل افتراضيًا.

## ما الذي يكتبه الحلم

يحتفظ الحلم بنوعين من المخرجات:

- **حالة الآلة** في `memory/.dreams/` (مخزن الاستدعاء، إشارات المراحل، نقاط تحقق الاستيعاب، الأقفال).
- **مخرجات قابلة للقراءة البشرية** في `DREAMS.md` (أو `dreams.md` الموجود) وملفات تقارير المراحل الاختيارية ضمن `memory/dreaming/<phase>/YYYY-MM-DD.md`.

لا تزال الترقية إلى الذاكرة طويلة الأمد تكتب فقط إلى `MEMORY.md`.

## نموذج المراحل

يستخدم الحلم ثلاث مراحل تعاونية:

| المرحلة | الغرض | الكتابة الدائمة |
| ----- | ----------------------------------------- | ----------------- |
| الخفيفة | فرز المواد قصيرة الأمد الحديثة وتجهيزها | لا |
| العميقة  | تقييم المرشحين الدائمين وترقيتهم      | نعم (`MEMORY.md`) |
| REM   | التأمل في الأنماط والأفكار المتكررة     | لا |

هذه المراحل تفاصيل تنفيذ داخلية وليست "أوضاعًا" منفصلة يضبطها المستخدم.

### المرحلة الخفيفة

تستوعب المرحلة الخفيفة إشارات الذاكرة اليومية الحديثة وآثار الاستدعاء، وتزيل التكرارات منها،
وتجهز أسطرًا مرشحة.

- تقرأ من حالة الاستدعاء قصيرة الأمد، وملفات الذاكرة اليومية الحديثة، ونصوص الجلسات المنقحة عند توفرها.
- تكتب كتلة `## Light Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات تعزيز لاستخدامها لاحقًا في الترتيب العميق.
- لا تكتب أبدًا إلى `MEMORY.md`.

### المرحلة العميقة

تقرر المرحلة العميقة ما الذي يصبح ذاكرة طويلة الأمد.

- ترتب المرشحين باستخدام تقييم مرجّح وبوابات عتبة.
- تتطلب اجتياز `minScore` و`minRecallCount` و`minUniqueQueries`.
- تعيد إماهة المقتطفات من الملفات اليومية الحية قبل الكتابة، لذلك يتم تخطي المقتطفات القديمة/المحذوفة.
- تُلحق الإدخالات المُرقّاة إلى `MEMORY.md`.
- تكتب ملخص `## Deep Sleep` داخل `DREAMS.md` وتكتب اختياريًا إلى `memory/dreaming/deep/YYYY-MM-DD.md`.

### مرحلة REM

تستخرج مرحلة REM الأنماط والإشارات التأملية.

- تبني ملخصات للموضوعات والتأملات من الآثار قصيرة الأمد الحديثة.
- تكتب كتلة `## REM Sleep` مُدارة عندما يتضمن التخزين مخرجات مضمّنة.
- تسجل إشارات تعزيز REM المستخدمة في الترتيب العميق.
- لا تكتب أبدًا إلى `MEMORY.md`.

## استيعاب نصوص الجلسات

يمكن للحلم استيعاب نصوص الجلسات المنقحة ضمن مادة الحلم. عند
توفر النصوص، يتم تمريرها إلى المرحلة الخفيفة إلى جانب إشارات الذاكرة
اليومية وآثار الاستدعاء. تتم تنقيح المحتوى الشخصي والحساس
قبل الاستيعاب.

## يوميات الأحلام

يحتفظ الحلم أيضًا بـ **يوميات أحلام** سردية في `DREAMS.md`.
بعد أن تتوفر مادة كافية لكل مرحلة، يقوم `memory-core` بتشغيل
دورة وكيل فرعي في الخلفية بأفضل جهد (باستخدام نموذج وقت التشغيل الافتراضي)
ويُلحق إدخال يوميات قصيرًا.

هذه اليوميات مخصصة للقراءة البشرية في واجهة Dreams، وليست مصدرًا للترقية.

## إشارات الترتيب العميق

يستخدم الترتيب العميق ست إشارات أساسية مرجحة بالإضافة إلى تعزيز المراحل:

| الإشارة | الوزن | الوصف |
| ------------------- | ------ | ------------------------------------------------- |
| التكرار           | 0.24   | عدد الإشارات قصيرة الأمد التي راكمها الإدخال |
| الصلة           | 0.30   | متوسط جودة الاسترجاع للإدخال           |
| تنوع الاستعلامات     | 0.15   | سياقات الاستعلام/اليوم المميزة التي أظهرته      |
| الحداثة             | 0.15   | درجة حداثة متناقصة بمرور الوقت                      |
| الدمج       | 0.10   | قوة التكرار عبر أيام متعددة                     |
| الثراء المفاهيمي | 0.06   | كثافة وسوم المفاهيم من المقتطف/المسار             |

تضيف إصابات المرحلتين الخفيفة وREM دفعة صغيرة متناقصة بمرور الوقت من
`memory/.dreams/phase-signals.json`.

## الجدولة

عند التفعيل، يدير `memory-core` تلقائيًا مهمة cron واحدة لاكتساح حلم كامل.
ينفذ كل اكتساح المراحل بالترتيب: الخفيفة -> REM -> العميقة.

سلوك الوتيرة الافتراضية:

| الإعداد              | الافتراضي     |
| -------------------- | ----------- |
| `dreaming.frequency` | `0 3 * * *` |

## البدء السريع

فعّل الحلم:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

فعّل الحلم مع وتيرة اكتساح مخصصة:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

## أمر الشرطة المائلة

```
/dreaming status
/dreaming on
/dreaming off
/dreaming help
```

## سير عمل CLI

استخدم ترقية CLI للمعاينة أو التطبيق اليدوي:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

يستخدم `memory promote` اليدوي عتبات المرحلة العميقة افتراضيًا ما لم يتم تجاوزها
بعلامات CLI.

اشرح لماذا قد تتم أو لا تتم ترقية مرشح معيّن:

```bash
openclaw memory promote-explain "router vlan"
openclaw memory promote-explain "router vlan" --json
```

عاين تأملات REM، والحقائق المرشحة، ومخرجات الترقية العميقة بدون
كتابة أي شيء:

```bash
openclaw memory rem-harness
openclaw memory rem-harness --json
```

## القيم الافتراضية الأساسية

توجد جميع الإعدادات ضمن `plugins.entries.memory-core.config.dreaming`.

| المفتاح         | الافتراضي     |
| ----------- | ----------- |
| `enabled`   | `false`     |
| `frequency` | `0 3 * * *` |

سياسة المراحل، والعتبات، وسلوك التخزين هي تفاصيل تنفيذ داخلية
(وليست إعدادات موجهة للمستخدم).

راجع [مرجع إعدادات الذاكرة](/ar/reference/memory-config#dreaming-experimental)
للاطلاع على القائمة الكاملة للمفاتيح.

## واجهة Dreams

عند التفعيل، تعرض علامة تبويب **Dreams** في Gateway ما يلي:

- حالة تفعيل الحلم الحالية
- الحالة على مستوى المرحلة ووجود الاكتساح المُدار
- أعداد الذاكرة قصيرة الأمد وطويلة الأمد والمُرقّاة اليوم
- توقيت التشغيل المجدول التالي
- قارئ يوميات أحلام قابل للتوسيع ومدعوم بواسطة `doctor.memory.dreamDiary`
