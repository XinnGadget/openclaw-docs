# Веб (шлюз)

Шлюз предоставляет небольшой **управляющий UI для браузера** (Vite + Lit) через тот же порт, что и WebSocket шлюза:

- по умолчанию: `http://<host>:18789/`
- необязательный префикс: задайте `gateway.controlUi.basePath` (например, `/openclaw`)

Возможности описаны в разделе [Управляющий UI](/web/control-ui).
В этой странице речь идёт о режимах привязки, безопасности и веб-интерфейсах.

## Вебхуки

Если `hooks.enabled=true`, шлюз также предоставляет небольшую конечную точку вебхука на том же HTTP-сервере.
Смотрите [Конфигурация шлюза](/gateway/configuration) → `hooks` для информации об аутентификации и полезных нагрузках.

## Конфигурация (включено по умолчанию)

Управляющий UI **включён по умолчанию**, если присутствуют ресурсы (`dist/control-ui`).
Вы можете управлять им через конфигурацию:

```json5
{
  gateway: {
    controlUi: { enabled: true, basePath: "/openclaw" }, // basePath необязателен
  },
}
```

## Доступ через Tailscale

### Интегрированный Serve (рекомендуется)

Оставьте шлюз на loopback и позвольте Tailscale Serve проксировать его:

```json5
{
  gateway: {
    bind: "loopback",
    tailscale: { mode: "serve" },
  },
}
```

Затем запустите шлюз:

```bash
openclaw gateway
```

Откройте:

- `https://<magicdns>/` (или указанный вами `gateway.controlUi.basePath`)

### Привязка к Tailnet + токен

```json5
{
  gateway: {
    bind: "tailnet",
    controlUi: { enabled: true },
    auth: { mode: "token", token: "your-token" },
  },
}
```

Затем запустите шлюз (в этом примере без loopback используется аутентификация по токену с общим секретом):

```bash
openclaw gateway
```

Откройте:

- `http://<tailscale-ip>:18789/` (или указанный вами `gateway.controlUi.basePath`)

### Открытый интернет (Funnel)

```json5
{
  gateway: {
    bind: "loopback",
    tailscale: { mode: "funnel" },
    auth: { mode: "password" }, // или OPENCLAW_GATEWAY_PASSWORD
  },
}
```

## Примечания по безопасности

- Аутентификация шлюза требуется по умолчанию (токен, пароль, доверенный прокси или заголовки идентификации Tailscale Serve, если они включены).
- Для привязок не через loopback **по-прежнему требуется** аутентификация шлюза. На практике это означает аутентификацию по токену/паролю или использование обратного прокси с учётом идентификации с `gateway.auth.mode: "trusted-proxy"`.
- Мастер по умолчанию создаёт аутентификацию с общим секретом и обычно генерирует токен шлюза (даже для loopback).
- В режиме с общим секретом UI отправляет `connect.params.auth.token` или `connect.params.auth.password`.
- В режимах с идентификацией, таких как Tailscale Serve или `trusted-proxy`, проверка аутентификации WebSocket выполняется на основе заголовков запроса.
- Для развёртываний управляющего UI не через loopback явно задайте `gateway.controlUi.allowedOrigins` (полные источники). Без этого запуск шлюза по умолчанию будет отклонён.
- `gateway.controlUi.dangerouslyAllowHostHeaderOriginFallback=true` включает режим отката к источнику по заголовку Host, но это опасное снижение уровня безопасности.
- При использовании Serve заголовки идентификации Tailscale могут удовлетворять требованиям аутентификации управляющего UI/WebSocket, если `gateway.auth.allowTailscale` имеет значение `true` (токен/пароль не требуются).
  Конечные точки HTTP API не используют эти заголовки идентификации Tailscale; вместо этого они следуют обычному режиму HTTP-аутентификации шлюза. Задайте `gateway.auth.allowTailscale: false`, чтобы требовать явные учётные данные. Смотрите [Tailscale](/gateway/tailscale) и [Безопасность](/gateway/security). Этот поток без токенов предполагает, что хост шлюза доверен.
- `gateway.tailscale.mode: "funnel"` требует `gateway.auth.mode: "password"` (общий пароль).

## Сборка UI

Шлюз обслуживает статические файлы из `dist/control-ui`. Соберите их с помощью:

```bash
pnpm ui:build # автоматически устанавливает зависимости UI при первом запуске
```
