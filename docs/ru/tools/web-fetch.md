# Web Fetch

Инструмент `web_fetch` выполняет простой HTTP-запрос GET и извлекает читаемый контент (преобразует HTML в Markdown или текст). Он **не** выполняет JavaScript.

Для сайтов с активным использованием JavaScript или страниц, защищённых авторизацией, используйте вместо этого [Web Browser](/tools/browser).

## Быстрый старт

`web_fetch` **включён по умолчанию** — дополнительная настройка не требуется. Агент может вызвать его сразу:

```javascript
await web_fetch({ url: "https://example.com/article" });
```

## Параметры инструмента

| Параметр | Тип | Описание |
| ------------- | -------- | ---------------------------------------- |
| `url` | `string` | URL для загрузки (обязательный параметр, только http/https) |
| `extractMode` | `string` | `"markdown"` (по умолчанию) или `"text"` |
| `maxChars` | `number` | Ограничить вывод указанным количеством символов |

## Принцип работы

<Steps>
  <Step title="Загрузка">
    Отправляет HTTP-запрос GET с User-Agent, похожим на Chrome, и заголовком `Accept-Language`. Блокирует частные/внутренние имена хостов и повторно проверяет перенаправления.
  </Step>
  <Step title="Извлечение">
    Применяет Readability (извлечение основного контента) к полученному HTML-ответу.
  </Step>
  <Step title="Запасной вариант (опционально)">
    Если извлечение с помощью Readability не удалось и настроен Firecrawl, выполняется повторная попытка через API Firecrawl в режиме обхода ботов.
  </Step>
  <Step title="Кэширование">
    Результаты кэшируются на 15 минут (настраивается), чтобы сократить количество повторных запросов к одному и тому же URL.
  </Step>
</Steps>

## Настройка

```json5
{
  tools: {
    web: {
      fetch: {
        enabled: true, // по умолчанию: true
        provider: "firecrawl", // опционально; пропустите для автоматического определения
        maxChars: 50000, // максимальное количество символов в выводе
        maxCharsCap: 50000, // жёсткое ограничение для параметра maxChars
        maxResponseBytes: 2000000, // максимальный размер загружаемых данных до усечения
        timeoutSeconds: 30,
        cacheTtlMinutes: 15,
        maxRedirects: 3,
        readability: true, // использовать извлечение с помощью Readability
        userAgent: "Mozilla/5.0 ...", // переопределить User-Agent
      },
    },
  },
}
```

## Запасной вариант с Firecrawl

Если извлечение с помощью Readability не удалось, `web_fetch` может переключиться на [Firecrawl](/tools/firecrawl) для обхода ботов и более качественного извлечения данных:

```json5
{
  tools: {
    web: {
      fetch: {
        provider: "firecrawl", // опционально; пропустите для автоматического определения по доступным учётным данным
      },
    },
  },
  plugins: {
    entries: {
      firecrawl: {
        enabled: true,
        config: {
          webFetch: {
            apiKey: "fc-...", // опционально, если задана переменная окружения FIRECRAWL_API_KEY
            baseUrl: "https://api.firecrawl.dev",
            onlyMainContent: true,
            maxAgeMs: 86400000, // длительность кэширования (1 день)
            timeoutSeconds: 60,
          },
        },
      },
    },
  },
}
```

`plugins.entries.firecrawl.config.webFetch.apiKey` поддерживает объекты SecretRef. Устаревшая конфигурация `tools.web.fetch.firecrawl.*` автоматически переносится с помощью `openclaw doctor --fix`.

<Note>
  Если Firecrawl включён, а его SecretRef не разрешён и нет запасного варианта через переменную окружения `FIRECRAWL_API_KEY`, запуск шлюза немедленно завершается с ошибкой.
</Note>

<Note>
  Переопределения `baseUrl` для Firecrawl ограничены: они должны использовать `https://` и официальный хост Firecrawl (`api.firecrawl.dev`).
</Note>

Текущее поведение во время выполнения:

- `tools.web.fetch.provider` явно выбирает поставщика для запасного варианта загрузки.
- Если параметр `provider` опущен, OpenClaw автоматически определяет первого готового поставщика загрузки веб-данных по доступным учётным данным. На текущий момент встроенный поставщик — Firecrawl.
- Если Readability отключён, `web_fetch` сразу переходит к выбранному запасному поставщику. Если поставщик недоступен, операция завершается с ошибкой.

## Ограничения и безопасность

- Значение `maxChars` ограничивается параметром `tools.web.fetch.maxCharsCap`.
- Тело ответа ограничивается по размеру параметром `maxResponseBytes` до начала парсинга; слишком большие ответы усекаются с выводом предупреждения.
- Частные/внутренние имена хостов блокируются.
- Перенаправления проверяются и ограничиваются параметром `maxRedirects`.
- `web_fetch` работает по принципу "максимальных усилий" — для некоторых сайтов требуется [Web Browser](/tools/browser).

## Профили инструментов

Если вы используете профили инструментов или белые списки, добавьте `web_fetch` или `group:web`:

```json5
{
  tools: {
    allow: ["web_fetch"],
    // или: allow: ["group:web"] (включает web_fetch, web_search и x_search)
  },
}
```
