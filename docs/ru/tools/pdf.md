# Инструмент PDF

`pdf` анализирует один или несколько PDF-документов и возвращает текст.

Краткое описание поведения:

- Режим нативного провайдера для поставщиков моделей Anthropic и Google.
- Резервный режим извлечения для остальных провайдеров (сначала извлекается текст, затем — изображения страниц, если это необходимо).
- Поддерживает одиночный (`pdf`) или множественный (`pdfs`) ввод, максимум 10 PDF-файлов за вызов.

## Доступность

Инструмент регистрируется только в том случае, если OpenClaw может определить конфигурацию модели с поддержкой PDF для агента:

1. `agents.defaults.pdfModel`.
2. Резерв — `agents.defaults.imageModel`.
3. Резерв — разрешённая для сессии/модель по умолчанию для агента.
4. Если провайдеры с нативной поддержкой PDF требуют аутентификации, они имеют приоритет перед общими резервными вариантами с поддержкой изображений.

Примечания о доступности:

- Цепочка резервирования учитывает аутентификацию. Настроенный `provider/model` учитывается только в том случае, если OpenClaw может аутентифицировать этого провайдера для агента.
- Провайдеры с нативной поддержкой PDF на текущий момент — **Anthropic** и **Google**.
- Если у разрешённого провайдера для сессии/по умолчанию уже настроена модель с поддержкой зрения/PDF, инструмент PDF повторно использует её до перехода к другим провайдерам с поддержкой аутентификации.

## Справочная информация по вводу данных

- `pdf` (`string`): путь к одному PDF-файлу или URL.
- `pdfs` (`string[]`): несколько путей к PDF-файлам или URL, всего до 10.
- `prompt` (`string`): промпт для анализа, по умолчанию — `Analyze this PDF document.` ("Проанализируй этот PDF-документ").
- `pages` (`string`): фильтр страниц, например `1-5` или `1,3,7-9`.
- `model` (`string`): необязательная переопределение модели (`provider/model`).
- `maxBytesMb` (`number`): ограничение размера PDF-файла в МБ (на один файл).

Примечания по вводу:

- `pdf` и `pdfs` объединяются и дедуплицируются перед загрузкой.
- Если ввод PDF-данных не предоставлен, инструмент выдаёт ошибку.
- `pages` обрабатывается как номера страниц с отсчётом от 1, дедуплицируется, сортируется и ограничивается максимально допустимым числом страниц.
- По умолчанию `maxBytesMb` берётся из `agents.defaults.pdfMaxBytesMb` или равен `10`.

## Поддерживаемые ссылки на PDF

- Локальный путь к файлу (включая расширение `~`).
- URL с протоколом `file://`.
- URL с протоколами `http://` и `https://`.

Примечания по ссылкам:

- Другие схемы URI (например, `ftp://`) отклоняются с ошибкой `unsupported_pdf_reference`.
- В режиме песочницы удалённые URL с протоколом `http(s)` отклоняются.
- Если включена политика работы только с файлами в рабочей области, локальные пути к файлам за пределами разрешённых корней отклоняются.

## Режимы выполнения

### Режим нативного провайдера

Нативный режим используется для провайдеров `anthropic` и `google`. Инструмент отправляет необработанные байты PDF напрямую в API провайдера.

Ограничения нативного режима:

- Параметр `pages` не поддерживается. Если он задан, инструмент возвращает ошибку.
- Поддерживается ввод нескольких PDF-файлов: каждый PDF отправляется как нативный блок документа / встроенная часть PDF перед промтом.

### Резервный режим извлечения

Резервный режим используется для провайдеров без нативной поддержки.

Порядок действий:

1. Извлечение текста с выбранных страниц (до `agents.defaults.pdfMaxPages`, по умолчанию — `20`).
2. Если длина извлечённого текста меньше 200 символов, выбранные страницы преобразуются в изображения PNG и включаются в запрос.
3. Извлечённое содержимое вместе с промтом отправляется выбранной модели.

Детали резервного режима:

- При извлечении изображений страниц используется бюджет пикселей в размере 4 000 000.
- Если целевая модель не поддерживает ввод изображений и текст извлечь не удалось, инструмент выдаёт ошибку.
- Если извлечение текста прошло успешно, но для извлечения изображений требуется поддержка зрения в модели, работающей только с текстом, OpenClaw отбрасывает отрендеренные изображения и продолжает работу с извлечённым текстом.
- Для резервного режима извлечения требуется `pdfjs-dist` (и `@napi-rs/canvas` для рендеринга изображений).

## Конфигурация

```json5
{
  agents: {
    defaults: {
      pdfModel: {
        primary: "anthropic/claude-opus-4-6",
        fallbacks: ["openai/gpt-5.4-mini"],
      },
      pdfMaxBytesMb: 10,
      pdfMaxPages: 20,
    },
  },
}
```

Полные сведения о полях см. в [Справочнике по конфигурации](/gateway/configuration-reference).

## Сведения о выводе

Инструмент возвращает текст в `content[0].text` и структурированные метаданные в `details`.

Распространённые поля `details`:

- `model`: разрешённая ссылка на модель (`provider/model`).
- `native`: `true` для режима нативного провайдера, `false` для резервного режима.
- `attempts`: число неудачных попыток резервного режима до успешного выполнения.

Поля путей:

- Ввод одного PDF-файла: `details.pdf`.
- Ввод нескольких PDF-файлов: `details.pdfs[]` с записями `pdf`.
- Метаданные перезаписи пути в песочнице (если применимо): `rewrittenFrom`.

## Поведение при ошибках

- Отсутствие ввода PDF: выдаётся ошибка `pdf required: provide a path or URL to a PDF document` ("Требуется PDF: укажите путь или URL к PDF-документу").
- Слишком много PDF-файлов: возвращается структурированная ошибка в `details.error = "too_many_pdfs"`.
- Неподдерживаемая схема ссылки: возвращается `details.error = "unsupported_pdf_reference"`.
- Нативный режим с параметром `pages`: выдаётся чёткая ошибка `pages is not supported with native PDF providers` ("Параметр pages не поддерживается с нативными провайдерами PDF").

## Примеры

Один PDF-файл:

```json
{
  "pdf": "/tmp/report.pdf",
  "prompt": "Summarize this report in 5 bullets"
}
```

Несколько PDF-файлов:

```json
{
  "pdfs": ["/tmp/q1.pdf", "/tmp/q2.pdf"],
  "prompt": "Compare risks and timeline changes across both documents"
}
```

Резервная модель с фильтрацией страниц:

```json
{
  "pdf": "https://example.com/report.pdf",
  "pages": "1-3,7",
  "model": "openai/gpt-5.4-mini",
  "prompt": "Extract only customer-impacting incidents"
}
```
