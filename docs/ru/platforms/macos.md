# Сопутствующее приложение OpenClaw для macOS (панель меню + брокер шлюза)

Приложение для macOS — это **сопутствующее приложение в панели меню** для OpenClaw. Оно управляет разрешениями, подключается к шлюзу локально (через launchd или вручную) и предоставляет агенту возможности macOS в качестве узла.

## Что оно делает

- Отображает собственные уведомления и статус в панели меню.
- Управляет запросами TCC (уведомления, доступность, запись экрана, микрофон, распознавание речи, автоматизация/AppleScript).
- Запускает шлюз или подключается к нему (локальный или удалённый).
- Предоставляет инструменты, доступные только в macOS (Canvas, Camera, запись экрана, `system.run`).
- Запускает локальную службу хоста узла в **удалённом** режиме (launchd) и останавливает её в **локальном** режиме.
- При необходимости размещает **PeekabooBridge** для автоматизации пользовательского интерфейса.
- Устанавливает глобальный CLI (`openclaw`) по запросу через npm, pnpm или bun (приложение отдаёт предпочтение npm, затем pnpm, затем bun; Node остаётся рекомендуемой средой выполнения шлюза).

## Локальный и удалённый режимы

- **Локальный** (по умолчанию): приложение подключается к работающему локальному шлюзу, если он есть; в противном случае оно активирует службу launchd через `openclaw gateway install`.
- **Удалённый**: приложение подключается к шлюзу через SSH/Tailscale и никогда не запускает локальный процесс.
  Приложение запускает локальную **службу хоста узла**, чтобы удалённый шлюз мог подключиться к этому Mac.
  Приложение не запускает шлюз как дочерний процесс.
  При обнаружении шлюза теперь отдаётся предпочтение именам Tailscale MagicDNS, а не сырым IP-адресам tailnet, поэтому приложение для Mac восстанавливается более надёжно при изменении IP-адресов tailnet.

## Управление launchd

Приложение управляет LaunchAgent для каждого пользователя с меткой `ai.openclaw.gateway` (или `ai.openclaw.<profile>` при использовании `--profile`/`OPENCLAW_PROFILE`; устаревшие `com.openclaw.*` по-прежнему выгружаются).

```bash
launchctl kickstart -k gui/$UID/ai.openclaw.gateway
launchctl bootout gui/$UID/ai.openclaw.gateway
```

Замените метку на `ai.openclaw.<profile>`, если используете именованный профиль.

Если LaunchAgent не установлен, активируйте его из приложения или выполните команду `openclaw gateway install`.

## Возможности узла (macOS)

Приложение для macOS представляет себя как узел. Распространённые команды:

- Canvas: `canvas.present`, `canvas.navigate`, `canvas.eval`, `canvas.snapshot`, `canvas.a2ui.*`
- Camera: `camera.snap`, `camera.clip`
- Screen: `screen.record`
- System: `system.run`, `system.notify`

Узел передаёт карту `permissions`, чтобы агенты могли определить, что разрешено.

Служба узла + IPC приложения:

- Когда служба хоста узла без интерфейса работает (удаленный режим), она подключается к шлюзу через WebSocket в качестве узла.
- `system.run` выполняется в приложении macOS (контекст UI/TCC) через локальный Unix-сокет; запросы и вывод остаются в приложении.

Схема (SCI):

```
Шлюз -> Служба узла (WS)
                 |  IPC (UDS + токен + HMAC + TTL)
                 v
             Приложение для Mac (UI + TCC + system.run)
```

## Утверждения выполнения (`system.run`)

`system.run` контролируется **утверждениями выполнения** в приложении для macOS (Настройки → Утверждения выполнения).
Безопасность, запросы и белый список хранятся локально на Mac в:

```
~/.openclaw/exec-approvals.json
```

Пример:

```json
{
  "version": 1,
  "defaults": {
    "security": "deny",
    "ask": "on-miss"
  },
  "agents": {
    "main": {
      "security": "allowlist",
      "ask": "on-miss",
      "allowlist": [{ "pattern": "/opt/homebrew/bin/rg" }]
    }
  }
}
```

Примечания:

- Записи в `allowlist` — это шаблоны glob для разрешённых путей к бинарным файлам.
- Сырой текст команды оболочки, содержащий управляющие символы или синтаксис расширения (`&&`, `||`, `;`, `|`, `` ` ``, `$`, `<`, `>`, `(`, `)`), считается пропуском белого списка и требует явного утверждения (или добавления в белый список бинарного файла оболочки).
- Выбор "Всегда разрешать" в запросе добавляет эту команду в белый список.
- Переопределения окружения для `system.run` фильтруются (удаляются `PATH`, `DYLD_*`, `LD_*`, `NODE_OPTIONS`, `PYTHON*`, `PERL*`, `RUBYOPT`, `SHELLOPTS`, `PS4`), а затем объединяются с окружением приложения.
- Для оболочек (`bash|sh|zsh ... -c/-lc`) переопределения окружения в рамках запроса сокращаются до небольшого явного белого списка (`TERM`, `LANG`, `LC_*`, `COLORTERM`, `NO_COLOR`, `FORCE_COLOR`).
- Для решений "всегда разрешать" в режиме белого списка известные обёртки диспетчера (`env`, `nice`, `nohup`, `stdbuf`, `timeout`) сохраняют пути к внутренним исполняемым файлам вместо путей к обёрткам. Если распаковка небезопасна, запись в белый список автоматически не сохраняется.

## Глубокие ссылки

Приложение регистрирует схему URL `openclaw://` для локальных действий.

### `openclaw://agent`

Запускает запрос `agent` шлюза.

```bash
open 'openclaw://agent?message=Hello%20from%20deep%20link'
```

Параметры запроса:

- `message` (обязательно)
- `sessionKey` (необязательно)
- `thinking` (необязательно)
- `deliver` / `to` / `channel` (необязательно)
- `timeoutSeconds` (необязательно)
- `key` (необязательно, ключ для режима без участия пользователя)

Безопасность:

- Без `key` приложение запрашивает подтверждение.
- Без `key` приложение устанавливает ограничение на длину сообщения для запроса подтверждения и игнорирует `deliver` / `to` / `channel`.
- При наличии действительного `key` выполнение происходит без участия пользователя (предназначено для личной автоматизации).

## Процесс ознакомления (типичный)

1. Установите и запустите **OpenClaw.app**.
2. Выполните проверку разрешений (запросы TCC).
3. Убедитесь, что активен **локальный** режим и шлюз запущен.
4. Установите CLI, если вам нужен доступ через терминал.

## Размещение каталога состояния (macOS)

Не помещайте каталог состояния OpenClaw в iCloud или другие папки, синхронизируемые с облаком.
Пути с синхронизацией могут увеличивать задержку и иногда вызывать конфликты блокировки файлов/синхронизации для сеансов и учётных данных.

Предпочтительнее использовать локальный путь без синхронизации, например:

```bash
OPENCLAW_STATE_DIR=~/.openclaw
```

Если `openclaw doctor` обнаружит состояние в:

- `~/Library/Mobile Documents/com~apple~CloudDocs/...`
- `~/Library/CloudStorage/...`

он выдаст предупреждение и порекомендует вернуться к локальному пути.

## Отладка подключения к шлюзу (CLI для macOS)

Используйте отладочный CLI, чтобы протестировать ту же логику рукопожатия и обнаружения шлюза через WebSocket, которую использует приложение для macOS, без запуска приложения.

```bash
cd apps/macos
swift run openclaw-mac connect --json
swift run openclaw-mac discover --timeout 3000 --json
```

Опции подключения:

- `--url <ws://host:port>`: переопределение конфигурации
- `--mode <local|remote>`: разрешение из конфигурации (по умолчанию: конфигурация или локальный)
- `--probe`: принудительная проверка работоспособности
- `--timeout <ms>`: тайм-аут запроса (по умолчанию: `15000`)
- `--json`: структурированный вывод для сравнения

Опции обнаружения:

- `--include-local`: включить шлюзы, которые будут отфильтрованы как "локальные"
- `--timeout <ms>`: общее окно обнаружения (по умолчанию: `2000`)
- `--json`: структурированный вывод для сравнения
