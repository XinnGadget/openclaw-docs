# Генерация изображений

Инструмент `image_generate` позволяет агенту создавать и редактировать изображения с использованием настроенных вами провайдеров. Сгенерированные изображения автоматически доставляются в виде медиа-вложений в ответе агента.

<Note>
Инструмент отображается только при наличии хотя бы одного провайдера генерации изображений. Если вы не видите `image_generate` среди инструментов вашего агента, настройте `agents.defaults.imageGenerationModel` или укажите API-ключ провайдера.
</Note>

## Быстрый старт

1. Укажите API-ключ хотя бы для одного провайдера (например, `OPENAI_API_KEY` или `GEMINI_API_KEY`).
2. При необходимости задайте предпочтительную модель:

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: {
        primary: "openai/gpt-image-1",
      },
    },
  },
}
```

3. Задайте агенту запрос: "Сгенерируй изображение дружелюбного лобстера-маскота".

Агент автоматически вызывает `image_generate`. Разрешения для использования инструмента не требуются — он включён по умолчанию при наличии провайдера.

## Поддерживаемые провайдеры

| Провайдер | Модель по умолчанию | Поддержка редактирования | API-ключ |
| -------- | -------------------------------- | ---------------------------------- | ----------------------------------------------------- |
| OpenAI | `gpt-image-1` | Да (до 5 изображений) | `OPENAI_API_KEY` |
| Google | `gemini-3.1-flash-image-preview` | Да | `GEMINI_API_KEY` или `GOOGLE_API_KEY` |
| fal | `fal-ai/flux/dev` | Да | `FAL_KEY` |
| MiniMax | `image-01` | Да (ссылка на объект) | `MINIMAX_API_KEY` или OAuth для MiniMax (`minimax-portal`) |
| ComfyUI | `workflow` | Да (1 изображение, настроено через workflow) | `COMFY_API_KEY` или `COMFY_CLOUD_API_KEY` для облачной версии |
| Vydra | `grok-imagine` | Нет | `VYDRA_API_KEY` |

Используйте `action: "list"`, чтобы проверить доступные провайдеры и модели во время выполнения:

```
/tool image_generate action=list
```

## Параметры инструмента

| Параметр | Тип | Описание |
| ------------- | -------- | ------------------------------------------------------------------------------------- |
| `prompt` | string | Промпт для генерации изображения (обязателен для `action: "generate"`) |
| `action` | string | `"generate"` (по умолчанию) или `"list"` для проверки провайдеров |
| `model` | string | Переопределение провайдера/модели, например `openai/gpt-image-1` |
| `image` | string | Путь к одному эталонному изображению или URL для режима редактирования |
| `images` | string[] | Несколько эталонных изображений для режима редактирования (до 5) |
| `size` | string | Указание размера: `1024x1024`, `1536x1024`, `1024x1536`, `1024x1792`, `1792x1024` |
| `aspectRatio` | string | Соотношение сторон: `1:1`, `2:3`, `3:2`, `3:4`, `4:3`, `4:5`, `5:4`, `9:16`, `16:9`, `21:9` |
| `resolution` | string | Указание разрешения: `1K`, `2K` или `4K` |
| `count` | number | Количество изображений для генерации (1–4) |
| `filename` | string | Подсказка для имени выходного файла |

Не все провайдеры поддерживают все параметры. Если провайдер-запасной вариант поддерживает близкий вариант геометрии вместо точно запрошенного, OpenClaw преобразует параметры в ближайший поддерживаемый размер, соотношение сторон или разрешение перед отправкой. Действительно неподдерживаемые переопределения по-прежнему указываются в результате работы инструмента.

Результаты работы инструмента содержат отчёт о применённых настройках. Если OpenClaw преобразует геометрию при переключении на запасной провайдер, возвращаемые значения `size`, `aspectRatio` и `resolution` отражают то, что было фактически отправлено, а `details.normalization` фиксирует преобразование от запрошенных параметров к применённым.

## Конфигурация

### Выбор модели

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: {
        primary: "openai/gpt-image-1",
        fallbacks: ["google/gemini-3.1-flash-image-preview", "fal/fal-ai/flux/dev"],
      },
    },
  },
}
```

### Порядок выбора провайдера

При генерации изображения OpenClaw пробует провайдеры в следующем порядке:

1. Параметр **`model`** из вызова инструмента (если агент его указал).
2. **`imageGenerationModel.primary`** из конфигурации.
3. **`imageGenerationModel.fallbacks`** — по порядку.
4. **Автообнаружение** — использует только провайдеры с настроенной аутентификацией:
   - сначала текущий провайдер по умолчанию;
   - затем остальные зарегистрированные провайдеры генерации изображений — в порядке их идентификаторов.

Если провайдер выдаёт ошибку (ошибка аутентификации, ограничение частоты запросов и т. д.), автоматически пробуется следующий кандидат. Если все попытки завершились ошибкой, в отчёте будут детали по каждой из них.

Примечания:

- Автообнаружение учитывает аутентификацию. Провайдер по умолчанию попадает в список кандидатов только тогда, когда OpenClaw может успешно пройти аутентификацию для этого провайдера.
- Автообнаружение включено по умолчанию. Установите `agents.defaults.mediaGenerationAutoProviderFallback: false`, если хотите, чтобы генерация изображений использовала только явно указанные записи `model`, `primary` и `fallbacks`.
- Используйте `action: "list"`, чтобы проверить зарегистрированные в данный момент провайдеры, их модели по умолчанию и подсказки по переменным окружения для аутентификации.

### Редактирование изображений

OpenAI, Google, fal, MiniMax и ComfyUI поддерживают редактирование эталонных изображений. Передайте путь к эталонному изображению или URL:

```
"Сгенерируй акварельную версию этого фото" + image: "/path/to/photo.jpg"
```

OpenAI и Google поддерживают до 5 эталонных изображений через параметр `images`. fal, MiniMax и ComfyUI — только 1.

Генерация изображений в MiniMax доступна через оба встроенных способа аутентификации MiniMax:

- `minimax/image-01` — для настройки через API-ключ;
- `minimax-portal/image-01` — для настройки через OAuth.

## Возможности провайдеров

| Возможность | OpenAI | Google | fal | MiniMax | ComfyUI | Vydra |
| --------------------- | -------------------- | -------------------- | ------------------- | -------------------------- | ---------------------------------- | ------- |
| Генерация | Да (до 4) | Да (до 4) | Да (до 4) | Да (до 9) | Да (выходы, определённые workflow) | Да (1) |
| Редактирование/ссылка | Да (до 5 изображений) | Да (до 5 изображений) | Да (1 изображение) | Да (1 изображение, ссылка на объект) | Да (1 изображение, настроено через workflow) | Нет |
| Управление размером | Да | Да | Да | Нет | Нет | Нет |
| Соотношение сторон | Нет | Да | Да (только генерация) | Да | Нет | Нет |
| Разрешение (1K/2K/4K) | Нет | Да | Да | Нет | Нет | Нет |
