# Аудио / голосовые заметки (2026-01-17)

## Что работает

- **Понимание медиа (аудио)**: если понимание аудио включено (или определено автоматически), OpenClaw:
  1. Находит первое вложение с аудио (локальный путь или URL) и при необходимости загружает его.
  2. Применяет ограничение `maxBytes` перед отправкой в каждую запись модели.
  3. Запускает первую подходящую запись модели по порядку (провайдер или CLI).
  4. Если происходит сбой или пропуск (из-за размера/тайм-аута), пробует следующую запись.
  5. В случае успеха заменяет `Body` на блок `[Audio]` и устанавливает `{{Transcript}}`.
- **Разбор команд**: когда транскрипция успешна, `CommandBody`/`RawBody` устанавливаются в транскрипцию, чтобы команды со слэшем продолжали работать.
- **Подробное логирование**: в режиме `--verbose` записывается, когда запускается транскрипция и когда происходит замена тела сообщения.

## Автоматическое определение (по умолчанию)

Если вы **не настраиваете модели** и `tools.media.audio.enabled` **не** установлено в `false`,
OpenClaw автоматически определяет параметры в следующем порядке и останавливается на первом рабочем варианте:

1. **Активная модель ответа**, если её провайдер поддерживает понимание аудио.
2. **Локальные CLI** (если установлены):
   - `sherpa-onnx-offline` (требуется `SHERPA_ONNX_MODEL_DIR` с encoder/decoder/joiner/tokens);
   - `whisper-cli` (из `whisper-cpp`; использует `WHISPER_CPP_MODEL` или встроенную компактную модель);
   - `whisper` (Python CLI; загружает модели автоматически).
3. **Gemini CLI** (`gemini`) с использованием `read_many_files`.
4. **Аутентификация провайдера**:
   - Сначала пробуются настроенные записи `models.providers.*`, поддерживающие аудио;
   - Встроенный порядок отката: OpenAI → Groq → Deepgram → Google → Mistral.

Чтобы отключить автоматическое определение, установите `tools.media.audio.enabled: false`.
Чтобы настроить параметры, установите `tools.media.audio.models`.

Примечание: определение бинарных файлов выполняется с максимально возможной точностью на macOS/Linux/Windows; убедитесь, что CLI находится в `PATH` (мы расширяем `~`), либо укажите явный CLI-модель с полным путём к команде.

## Примеры конфигурации

### Провайдер + откат на CLI (OpenAI + Whisper CLI)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        maxBytes: 20971520,
        models: [
          { provider: "openai", model: "gpt-4o-mini-transcribe" },
          {
            type: "cli",
            command: "whisper",
            args: ["--model", "base", "{{MediaPath}}"],
            timeoutSeconds: 45,
          },
        ],
      },
    },
  },
}
```

### Только провайдер с ограничением по области видимости

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        scope: {
          default: "allow",
          rules: [{ action: "deny", match: { chatType: "group" } }],
        },
        models: [{ provider: "openai", model: "gpt-4o-mini-transcribe" }],
      },
    },
  },
}
```

### Только провайдер (Deepgram)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        models: [{ provider: "deepgram", model: "nova-3" }],
      },
    },
  },
}
```

### Только провайдер (Mistral Voxtral)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        models: [{ provider: "mistral", model: "voxtral-mini-latest" }],
      },
    },
  },
}
```

### Отправка транскрипции в чат (по желанию)

```json5
{
  tools: {
    media: {
      audio: {
        enabled: true,
        echoTranscript: true, // по умолчанию false
        echoFormat: '📝 "{transcript}"', // необязательно, поддерживает {transcript}
        models: [{ provider: "openai", model: "gpt-4o-mini-transcribe" }],
      },
    },
  },
}
```

## Примечания и ограничения

- Аутентификация провайдера следует стандартному порядку аутентификации моделей (профили аутентификации, переменные окружения, `models.providers.*.apiKey`).
- Детали настройки Groq: [Groq](/providers/groq).
- Deepgram использует `DEEPGRAM_API_KEY`, когда используется `provider: "deepgram"`.
- Детали настройки Deepgram: [Deepgram (транскрипция аудио)](/providers/deepgram).
- Детали настройки Mistral: [Mistral](/providers/mistral).
- Провайдеры аудио могут переопределить `baseUrl`, `headers` и `providerOptions` через `tools.media.audio`.
- Ограничение размера по умолчанию — 20 МБ (`tools.media.audio.maxBytes`). Аудио, превышающее этот размер, пропускается для данной модели, и пробуется следующая запись.
- Короткие/пустые аудиофайлы размером менее 1024 байт пропускаются до транскрипции провайдером/CLI.
- Значение `maxChars` для аудио по умолчанию **не задано** (полная транскрипция). Установите `tools.media.audio.maxChars` или `maxChars` для каждой записи, чтобы обрезать вывод.
- По умолчанию для OpenAI используется `gpt-4o-mini-transcribe`; установите `model: "gpt-4o-transcribe"` для более высокой точности.
- Используйте `tools.media.audio.attachments` для обработки нескольких голосовых заметок (`mode: "all"` + `maxAttachments`).
- Транскрипция доступна для шаблонов как `{{Transcript}}`.
- `tools.media.audio.echoTranscript` по умолчанию отключён; включите его, чтобы отправить подтверждение транскрипции обратно в исходный чат до обработки агентом.
- `tools.media.audio.echoFormat` позволяет настроить текст подтверждения (заполнитель: `{transcript}`).
- Вывод CLI ограничен (5 МБ); старайтесь, чтобы вывод CLI был лаконичным.

### Поддержка прокси-окружения

Транскрипция аудио на основе провайдера учитывает стандартные переменные окружения для исходящего прокси:

- `HTTPS_PROXY`
- `HTTP_PROXY`
- `https_proxy`
- `http_proxy`

Если переменные окружения прокси не заданы, используется прямое исходящее соединение. Если конфигурация прокси некорректна, OpenClaw выводит предупреждение и переходит к прямому получению данных.

## Обнаружение упоминаний в группах

Когда для группового чата установлено `requireMention: true`, OpenClaw теперь транскрибирует аудио **до** проверки на упоминания. Это позволяет обрабатывать голосовые заметки, даже если они содержат упоминания.

**Как это работает:**

1. Если голосовое сообщение не имеет текстового тела и в группе требуются упоминания, OpenClaw выполняет "предварительную" транскрипцию.
2. Транскрипция проверяется на наличие шаблонов упоминаний (например, `@BotName`, триггеры-эмодзи).
3. Если упоминание найдено, сообщение проходит через полный конвейер ответов.
4. Транскрипция используется для обнаружения упоминаний, поэтому голосовые заметки могут пройти проверку на упоминания.

**Поведение при откате:**

- Если транскрипция не удаётся во время предварительной проверки (тайм-аут, ошибка API и т. д.), сообщение обрабатывается на основе обнаружения упоминаний только в текстовой части.
- Это гарантирует, что смешанные сообщения (текст + аудио) никогда не будут ошибочно отброшены.

**Отказ от предварительной проверки для группы/темы Telegram:**

- Установите `channels.telegram.groups.<chatId>.disableAudioPreflight: true`, чтобы пропустить проверку транскрипции на упоминания для этой группы.
- Установите `channels.telegram.groups.<chatId>.topics.<threadId>.disableAudioPreflight`, чтобы переопределить настройки для темы (`true` — пропустить, `false` — принудительно включить).
- По умолчанию установлено `false` (предварительная проверка включена, когда выполняются условия с упоминаниями).

**Пример:** пользователь отправляет голосовую заметку со словами "Hey @Claude, what's the weather?" в группу Telegram с `requireMention: true`. Голосовая заметка транскрибируется, упоминание обнаруживается, и агент отвечает.
