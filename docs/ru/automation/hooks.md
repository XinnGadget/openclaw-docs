# Хуки

Хуки — это небольшие скрипты, которые запускаются при возникновении определённых событий внутри Gateway. Они автоматически обнаруживаются в каталогах и могут быть проверены с помощью команды `openclaw hooks`.

В OpenClaw есть два типа хуков:

- **Внутренние хуки** (эта страница): запускаются внутри Gateway при возникновении событий агента, таких как `/new`, `/reset`, `/stop` или событий жизненного цикла.
- **Вебхуки**: внешние HTTP-эндпоинты, позволяющие другим системам запускать работу в OpenClaw. См. [Вебхуки](/automation/cron-jobs#webhooks).

Хуки также могут быть включены в плагины. Команда `openclaw hooks list` отображает как автономные хуки, так и хуки, управляемые плагинами.

## Быстрый старт

```bash
# Вывести список доступных хуков
openclaw hooks list

# Включить хук
openclaw hooks enable session-memory

# Проверить статус хука
openclaw hooks check

# Получить подробную информацию
openclaw hooks info session-memory
```

## Типы событий

| Событие | Когда срабатывает |
| ------------------------ | ------------------------------------------------ |
| `command:new` | Выполнена команда `/new` |
| `command:reset` | Выполнена команда `/reset` |
| `command:stop` | Выполнена команда `/stop` |
| `command` | Любое событие команды (общий слушатель) |
| `session:compact:before` | Перед тем как сжатие суммирует историю |
| `session:compact:after` | После завершения сжатия |
| `session:patch` | При изменении свойств сессии |
| `agent:bootstrap` | Перед внедрением файлов начальной загрузки рабочей области |
| `gateway:startup` | После запуска каналов и загрузки хуков |
| `message:received` | Входящее сообщение из любого канала |
| `message:transcribed` | После завершения транскрипции аудио |
| `message:preprocessed` | После завершения обработки медиа и ссылок |
| `message:sent` | Отправленное исходящее сообщение |

## Написание хуков

### Структура хука

Каждый хук — это каталог, содержащий два файла:

```
my-hook/
├── HOOK.md          # Метаданные + документация
└── handler.ts       # Реализация обработчика
```

### Формат HOOK.md

```markdown
---
name: my-hook
description: "Краткое описание того, что делает этот хук"
metadata:
  { "openclaw": { "emoji": "🔗", "events": ["command:new"], "requires": { "bins": ["node"] } } }
---

# Мой хук

Здесь размещается подробная документация.
```

**Поля метаданных** (`metadata.openclaw`):

| Поле | Описание |
| ---------- | ---------------------------------------------------- |
| `emoji` | Эмодзи для отображения в CLI |
| `events` | Массив событий, на которые нужно подписаться |
| `export` | Именованный экспорт для использования (по умолчанию `"default"`) |
| `os` | Необходимые платформы (например, `["darwin", "linux"]`) |
| `requires` | Необходимые `bins`, `anyBins`, `env` или пути к `config` |
| `always` | Обход проверок соответствия (булево значение) |
| `install` | Методы установки |

### Реализация обработчика

```typescript
const handler = async (event) => {
  if (event.type !== "command" || event.action !== "new") {
    return;
  }

  console.log(`[my-hook] Новая команда запущена`);
  // Ваша логика здесь

  // При необходимости отправить сообщение пользователю
  event.messages.push("Хук выполнен!");
};

export default handler;
```

Каждое событие включает: `type`, `action`, `sessionKey`, `timestamp`, `messages` (добавляйте сюда сообщения для отправки пользователю) и `context` (данные, специфичные для события).

### Ключевые моменты контекста событий

**События команд** (`command:new`, `command:reset`): `context.sessionEntry`, `context.previousSessionEntry`, `context.commandSource`, `context.workspaceDir`, `context.cfg`.

**События сообщений** (`message:received`): `context.from`, `context.content`, `context.channelId`, `context.metadata` (данные, специфичные для провайдера, включая `senderId`, `senderName`, `guildId`).

**События сообщений** (`message:sent`): `context.to`, `context.content`, `context.success`, `context.channelId`.

**События сообщений** (`message:transcribed`): `context.transcript`, `context.from`, `context.channelId`, `context.mediaPath`.

**События сообщений** (`message:preprocessed`): `context.bodyForAgent` (окончательное обогащённое тело), `context.from`, `context.channelId`.

**События начальной загрузки** (`agent:bootstrap`): `context.bootstrapFiles` (изменяемый массив), `context.agentId`.

**События патча сессии** (`session:patch`): `context.sessionEntry`, `context.patch` (только изменённые поля), `context.cfg`. Только привилегированные клиенты могут запускать события патча.

**События сжатия**: `session:compact:before` включает `messageCount`, `tokenCount`. `session:compact:after` добавляет `compactedCount`, `summaryLength`, `tokensBefore`, `tokensAfter`.

## Обнаружение хуков

Хуки обнаруживаются в следующих каталогах в порядке возрастания приоритета переопределения:

1. **Встроенные хуки**: поставляются вместе с OpenClaw.
2. **Хуки плагинов**: хуки, включённые в установленные плагины.
3. **Управляемые хуки**: `~/.openclaw/hooks/` (установленные пользователем, общие для всех рабочих областей). Дополнительные каталоги из `hooks.internal.load.extraDirs` имеют такой же приоритет.
4. **Хуки рабочей области**: `<workspace>/hooks/` (для каждого агента, по умолчанию отключены до явного включения).

Хуки рабочей области могут добавлять новые имена хуков, но не могут переопределять встроенные, управляемые или предоставленные плагинами хуки с тем же именем.

### Пакеты хуков

Пакеты хуков — это npm-пакеты, которые экспортируют хуки через `openclaw.hooks` в `package.json`. Установка выполняется командой:

```bash
openclaw plugins install <path-or-spec>
```

Спецификации npm относятся только к реестру (имя пакета + необязательная точная версия или тег дистрибутива). Спецификации Git/URL/файла и диапазоны semver не принимаются.

## Встроенные хуки

| Хук | События | Что делает |
| --------------------- | ------------------------------ | ----------------------------------------------------- |
| session-memory | `command:new`, `command:reset` | Сохраняет контекст сессии в `<workspace>/memory/` |
| bootstrap-extra-files | `agent:bootstrap` | Внедряет дополнительные файлы начальной загрузки из шаблонов glob |
| command-logger | `command` | Записывает все команды в `~/.openclaw/logs/commands.log` |
| boot-md | `gateway:startup` | Запускает `BOOT.md` при запуске шлюза |

Включите любой встроенный хук:

```bash
openclaw hooks enable <hook-name>
```

### Детали session-memory

Извлекает последние 15 сообщений пользователя и ассистента, генерирует описательный слаг имени файла с помощью LLM и сохраняет в `<workspace>/memory/YYYY-MM-DD-slug.md`. Требуется настройка `workspace.dir`.

### Конфигурация bootstrap-extra-files

```json
{
  "hooks": {
    "internal": {
      "entries": {
        "bootstrap-extra-files": {
          "enabled": true,
          "paths": ["packages/*/AGENTS.md", "packages/*/TOOLS.md"]
        }
      }
    }
  }
}
```

Пути разрешаются относительно рабочей области. Загружаются только распознанные базовые имена файлов начальной загрузки (`AGENTS.md`, `SOUL.md`, `TOOLS.md`, `IDENTITY.md`, `USER.md`, `HEARTBEAT.md`, `BOOTSTRAP.md`, `MEMORY.md`).

## Хуки плагинов

Плагины могут регистрировать хуки через Plugin SDK для более глубокой интеграции: перехвата вызовов инструментов, изменения подсказок, управления потоком сообщений и т. д. Plugin SDK предоставляет 28 хуков, охватывающих разрешение моделей, жизненный цикл агента, поток сообщений, выполнение инструментов, координацию субагентов и жизненный цикл шлюза.
