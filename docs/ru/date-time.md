# Дата и время

По умолчанию в OpenClaw для меток времени транспортировки используется **локальное время хоста**, а **часовой пояс пользователя** — только в системном промте.
Метки времени провайдера сохраняются, чтобы инструменты сохраняли свою родную семантику (текущее время доступно через `session_status`).

## Конверты сообщений (по умолчанию — локальные)

Входящие сообщения оборачиваются меткой времени (с точностью до минуты):

```
[Provider ... 2026-01-05 16:26 PST] текст сообщения
```

Метка времени в конверте по умолчанию **является локальной для хоста**, независимо от часового пояса провайдера.

Вы можете переопределить это поведение:

```json5
{
  agents: {
    defaults: {
      envelopeTimezone: "local", // "utc" | "local" | "user" | часовой пояс IANA
      envelopeTimestamp: "on", // "on" | "off"
      envelopeElapsed: "on", // "on" | "off"
    },
  },
}
```

- `envelopeTimezone: "utc"` использует UTC.
- `envelopeTimezone: "local"` использует часовой пояс хоста.
- `envelopeTimezone: "user"` использует `agents.defaults.userTimezone` (в качестве запасного варианта используется часовой пояс хоста).
- Используйте явный часовой пояс IANA (например, `"America/Chicago"`) для фиксированной зоны.
- `envelopeTimestamp: "off"` удаляет абсолютные метки времени из заголовков конверта.
- `envelopeElapsed: "off"` удаляет суффиксы прошедшего времени (в стиле `+2m`).

### Примеры

**Локальный (по умолчанию):**

```
[WhatsApp +1555 2026-01-18 00:19 PST] привет
```

**Часовой пояс пользователя:**

```
[WhatsApp +1555 2026-01-18 00:19 CST] привет
```

**Включено отображение прошедшего времени:**

```
[WhatsApp +1555 +30s 2026-01-18T05:19Z] продолжение
```

## Системный промт: текущая дата и время

Если часовой пояс пользователя известен, системный промт включает специальный раздел **Текущая дата и время** только с **часовым поясом** (без формата времени), чтобы сохранить стабильность кэширования промта:

```
Часовой пояс: America/Chicago
```

Когда агенту нужно текущее время, используйте инструмент `session_status`; карточка статуса включает строку с меткой времени.

## Строки системных событий (по умолчанию — локальные)

Очередные системные события, вставленные в контекст агента, имеют префикс с меткой времени, использующей тот же выбор часового пояса, что и конверты сообщений (по умолчанию — локальный для хоста):

```
System: [2026-01-12 12:19:17 PST] Модель переключена.
```

### Настройка часового пояса пользователя и формата

```json5
{
  agents: {
    defaults: {
      userTimezone: "America/Chicago",
      timeFormat: "auto", // auto | 12 | 24
    },
  },
}
```

- `userTimezone` задаёт **локальный часовой пояс пользователя** для контекста промта.
- `timeFormat` управляет **отображением в 12-часовом или 24-часовом формате** в промте. Значение `auto` следует настройкам ОС.

## Определение формата времени (auto)

Когда `timeFormat: "auto"`, OpenClaw проверяет настройки ОС (macOS/Windows) и в качестве запасного варианта использует форматирование по локали. Обнаруженное значение **кэшируется для каждого процесса**, чтобы избежать повторных системных вызовов.

## Полезные нагрузки инструментов и коннекторы (исходное время провайдера + нормализованные поля)

Инструменты каналов возвращают **метки времени в родном формате провайдера** и добавляют нормализованные поля для обеспечения согласованности:

- `timestampMs`: миллисекунды эпохи (UTC);
- `timestampUtc`: строка в формате ISO 8601 (UTC).

Исходные поля провайдера сохраняются, чтобы ничего не было потеряно.

- Slack: строки, похожие на эпоху, из API;
- Discord: метки времени в формате UTC ISO;
- Telegram/WhatsApp: числовые метки времени или метки времени в формате ISO, специфичные для провайдера.

Если вам нужно локальное время, преобразуйте его на последующих этапах, используя известный часовой пояс.
