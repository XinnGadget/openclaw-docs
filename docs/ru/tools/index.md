# Инструменты и плагины

Всё, что агент делает помимо генерации текста, выполняется с помощью **инструментов**.
Инструменты — это то, с помощью чего агент читает файлы, выполняет команды, просматривает веб-страницы, отправляет сообщения и взаимодействует с устройствами.

## Инструменты, навыки и плагины

В OpenClaw есть три взаимодействующих между собой уровня:

<Steps>
  <Step title="Инструменты — то, что вызывает агент">
    Инструмент — это типизированная функция, которую может вызвать агент (например, `exec`, `browser`, `web_search`, `message`). OpenClaw поставляется с набором **встроенных инструментов**, а плагины могут регистрировать дополнительные.

    Агент воспринимает инструменты как структурированные определения функций, передаваемые в API модели.
  </Step>

  <Step title="Навыки учат агента, когда и как действовать">
    Навык — это файл Markdown (`SKILL.md`), внедряемый в системный промпт. Навыки дают агенту контекст, ограничения и пошаговые инструкции для эффективного использования инструментов. Навыки хранятся в вашем рабочем пространстве, в общих папках или входят в состав плагинов.

    [Справочник по навыкам](/tools/skills) | [Создание навыков](/tools/creating-skills)
  </Step>

  <Step title="Плагины объединяют всё вместе">
    Плагин — это пакет, который может регистрировать любую комбинацию возможностей: каналы, провайдеры моделей, инструменты, навыки, речь, транскрибацию в реальном времени, голосовую связь в реальном времени, понимание мультимедиа, генерацию изображений, генерацию видео, получение данных из сети, поиск в сети и многое другое. Некоторые плагины являются **основными** (поставляются с OpenClaw), другие — **внешними** (публикуются сообществом на npm).

    [Установка и настройка плагинов](/tools/plugin) | [Создание собственных плагинов](/plugins/building-plugins)
  </Step>
</Steps>

## Встроенные инструменты

Эти инструменты поставляются с OpenClaw и доступны без установки каких-либо плагинов:

| Инструмент | Что делает | Страница |
| --- | --- | --- |
| `exec` / `process` | Выполняет команды оболочки, управляет фоновыми процессами | [Exec](/tools/exec) |
| `code_execution` | Выполняет удалённый анализ на Python в песочнице | [Code Execution](/tools/code-execution) |
| `browser` | Управляет браузером Chromium (навигация, клик, скриншот) | [Browser](/tools/browser) |
| `web_search` / `x_search` / `web_fetch` | Ищет в сети, ищет посты в X, получает содержимое страниц | [Web](/tools/web) |
| `read` / `write` / `edit` | Операции ввода-вывода с файлами в рабочем пространстве | |
| `apply_patch` | Патчи для файлов с несколькими фрагментами | [Apply Patch](/tools/apply-patch) |
| `message` | Отправляет сообщения по всем каналам | [Agent Send](/tools/agent-send) |
| `canvas` | Управляет node Canvas (представление, вычисление, снимок) | |
| `nodes` | Обнаруживает и нацеливается на сопряжённые устройства | |
| `cron` / `gateway` | Управляет запланированными заданиями; проверяет, исправляет, перезапускает или обновляет шлюз | |
| `image` / `image_generate` | Анализирует или генерирует изображения | [Image Generation](/tools/image-generation) |
| `music_generate` | Генерирует музыкальные треки | [Music Generation](/tools/music-generation) |
| `video_generate` | Генерирует видео | [Video Generation](/tools/video-generation) |
| `tts` | Однократное преобразование текста в речь | [TTS](/tools/tts) |
| `sessions_*` / `subagents` / `agents_list` | Управление сессиями, статусом и оркестрацией субагентов | [Sub-agents](/tools/subagents) |
| `session_status` | Лёгковесное считывание в стиле `/status` и переопределение модели для сессии | [Session Tools](/concepts/session-tool) |

Для работы с изображениями используйте `image` для анализа и `image_generate` для генерации или редактирования. Если вы используете провайдера изображений, отличного от стандартного (`openai/*`, `google/*`, `fal/*` и т. д.), сначала настройте аутентификацию/API-ключ этого провайдера.

Для работы с музыкой используйте `music_generate`. Если вы используете провайдера музыки, отличного от стандартного (`google/*`, `minimax/*` и т. д.), сначала настройте аутентификацию/API-ключ этого провайдера.

Для работы с видео используйте `video_generate`. Если вы используете провайдера видео, отличного от стандартного (`qwen/*` и т. д.), сначала настройте аутентификацию/API-ключ этого провайдера.

Для генерации аудио в рамках рабочего процесса используйте `music_generate`, если плагин (например, ComfyUI) регистрирует эту функцию. Это отдельно от `tts`, который отвечает за преобразование текста в речь.

`session_status` — это облегчённый инструмент для получения статуса/считывания в группе сессий. Он отвечает на вопросы в стиле `/status` о текущей сессии и может при необходимости установить переопределение модели для сессии; `model=default` сбрасывает это переопределение. Как и `/status`, он может дополнять разреженные счётчики токенов/кэша и метку активной модели среды выполнения из последней записи об использовании транскрипции.

`gateway` — это инструмент времени выполнения только для владельца, предназначенный для операций со шлюзом:

* `config.schema.lookup` — получение поддерева конфигурации для одного пути перед редактированием;
* `config.get` — получение текущего снимка конфигурации + хеш;
* `config.patch` — частичное обновление конфигурации с перезапуском;
* `config.apply` — только для полной замены конфигурации;
* `update.run` — явное самообновление + перезапуск.

Для частичных изменений предпочтительно использовать `config.schema.lookup`, а затем `config.patch`. Используйте `config.apply` только в том случае, если вы намеренно заменяете всю конфигурацию. Инструмент также не позволяет изменять `tools.exec.ask` или `tools.exec.security`; устаревшие псевдонимы `tools.bash.*` нормализуются до тех же защищённых путей выполнения.

### Инструменты, предоставляемые плагинами

Плагины могут регистрировать дополнительные инструменты. Несколько примеров:

* [Lobster](/tools/lobster) — типизированное время выполнения рабочего процесса с возобновляемым утверждением;
* [LLM Task](/tools/llm-task) — шаг LLM только для JSON с целью получения структурированного вывода;
* [Music Generation](/tools/music-generation) — общий инструмент `music_generate` с провайдерами, поддерживающими рабочие процессы;
* [Diffs](/tools/diffs) — просмотрщик и рендерер различий;
* [OpenProse](/prose) — оркестрация рабочих процессов с приоритетом Markdown.

## Настройка инструментов

### Списки разрешений и запретов

Управляйте тем, какие инструменты может вызывать агент, с помощью `tools.allow` / `tools.deny` в конфигурации. Запрет всегда имеет приоритет над разрешением.

```json5
{
  tools: {
    allow: ["group:fs", "browser", "web_search"],
    deny: ["exec"],
  },
}
```

### Профили инструментов

`tools.profile` задаёт базовый список разрешений до применения `allow`/`deny`. Переопределение для агента: `agents.list[].tools.profile`.

| Профиль | Что включает |
| --- | --- |
| `full` | Без ограничений (то же самое, что не задано) |
| `coding` | `group:fs`, `group:runtime`, `group:web`, `group:sessions`, `group:memory`, `cron`, `image`, `image_generate`, `music_generate`, `video_generate` |
| `messaging` | `group:messaging`, `sessions_list`, `sessions_history`, `sessions_send`, `session_status` |
| `minimal` | Только `session_status` |

### Группы инструментов

Используйте сокращения `group:*` в списках разрешений/запретов:

| Группа | Инструменты |
| --- | --- |
| `group:runtime` | exec, process, code_execution (`bash` принимается как псевдоним для `exec`) |
| `group:fs` | read, write, edit, apply_patch |
| `group:sessions` | sessions_list, sessions_history, sessions_send, sessions_spawn, sessions_yield, subagents, session_status |
| `group:memory` | memory_search, memory_get |
| `group:web` | web_search, x_search, web_fetch |
| `group:ui` | browser, canvas |
| `group:automation` | cron, gateway |
| `group:messaging` | message |
| `group:nodes` | nodes |
| `group:agents` | agents_list |
| `group:media` | image, image_generate, music_generate, video_generate, tts |
| `group:openclaw` | Все встроенные инструменты OpenClaw (не включает инструменты плагинов) |

`sessions_history` возвращает ограниченный, отфильтрованный с точки зрения безопасности вид данных. Он удаляет теги размышлений, каркас `<relevant-memories>`, полезные нагрузки XML для вызовов инструментов в виде обычного текста (включая `<tool_call>...</tool_call>`, `<function_call>...</function_call>`, `<tool_calls>...</tool_calls>`, `<function_calls>...</function_calls>` и усечённые блоки вызовов инструментов), понижает уровень каркаса вызовов инструментов, удаляет просочившиеся ASCII-символы/полноширинные токены управления моделью и некорректный XML вызовов инструментов MiniMax из текста ассистента, затем применяет редактирование/усечение и, возможно, заполнители для слишком больших строк вместо того, чтобы выдавать необработанный дамп транскрипции.

### Ограничения для конкретных провайдеров

Используйте `tools.byProvider`, чтобы ограничить инструменты для конкретных провайдеров без изменения глобальных настроек по умолчанию:

```json5
{
  tools: {
    profile: "coding",
    byProvider: {
      "google-antigravity": { profile: "minimal" },
    },
  },
}
```
