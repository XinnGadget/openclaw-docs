# Zalo (Bot API)

Статус: экспериментальный. Поддержка прямых сообщений (DM) реализована. Раздел "[Возможности](#vozmozhnosti)" ниже отражает текущее поведение ботов на маркетплейсе.

## Встроенный плагин

В текущих версиях OpenClaw Zalo поставляется как встроенный плагин, поэтому для стандартных сборок отдельная установка не требуется.

Если вы используете более старую сборку или кастомную установку, в которой Zalo отсутствует, установите его вручную:

- Через CLI: `openclaw plugins install @openclaw/zalo`
- Или из локального репозитория: `openclaw plugins install ./path/to/local/zalo-plugin`
- Подробности: [Плагины](/tools/plugin)

## Быстрая настройка (для начинающих)

1. Убедитесь, что плагин Zalo доступен:
   - В текущих стандартных сборках OpenClaw он уже включён.
   - В более старых или кастомных сборках его можно добавить вручную с помощью команд выше.
2. Задайте токен:
   - Через переменную окружения: `ZALO_BOT_TOKEN=...`
   - Или в конфигурации: `channels.zalo.accounts.default.botToken: "..."`.
3. Перезапустите шлюз (или завершите настройку).
4. Доступ к прямым сообщениям по умолчанию осуществляется через сопряжение; подтвердите код сопряжения при первом контакте.

Минимальная конфигурация:

```json5
{
  channels: {
    zalo: {
      enabled: true,
      accounts: {
        default: {
          botToken: "12345689:abc-xyz",
          dmPolicy: "pairing",
        },
      },
    },
  },
}
```

## Что это такое

Zalo — приложение для обмена сообщениями, ориентированное на Вьетнам; его Bot API позволяет шлюзу запускать бота для общения один на один.
Это хороший вариант для поддержки или уведомлений, где требуется детерминированная маршрутизация обратно в Zalo.

На этой странице описано текущее поведение OpenClaw для **ботов Zalo Bot Creator / маркетплейса**.
**Боты Zalo Official Account (OA)** — это отдельный продукт Zalo, и их поведение может отличаться.

- Канал Zalo Bot API, управляемый шлюзом.
- Детерминированная маршрутизация: ответы возвращаются в Zalo; модель никогда не выбирает каналы.
- Прямые сообщения используют основную сессию агента.
- Раздел "[Возможности](#vozmozhnosti)" ниже показывает текущую поддержку ботов на маркетплейсе.

## Настройка (быстрый путь)

### 1) Создание токена бота (Zalo Bot Platform)

1. Перейдите на [https://bot.zaloplatforms.com](https://bot.zaloplatforms.com) и войдите в систему.
2. Создайте нового бота и настройте его параметры.
3. Скопируйте полный токен бота (обычно в формате `numeric_id:secret`). Для ботов на маркетплейсе используемый во время выполнения токен может появиться в приветственном сообщении бота после создания.

### 2) Настройка токена (через окружение или конфигурацию)

Пример:

```json5
{
  channels: {
    zalo: {
      enabled: true,
      accounts: {
        default: {
          botToken: "12345689:abc-xyz",
          dmPolicy: "pairing",
        },
      },
    },
  },
}
```

Если позже вы перейдёте на платформу ботов Zalo, где доступны группы, вы можете явно добавить конфигурацию для групп, например `groupPolicy` и `groupAllowFrom`. Для текущего поведения ботов на маркетплейсе см. раздел "[Возможности](#vozmozhnosti)".

Вариант через окружение: `ZALO_BOT_TOKEN=...` (работает только для учётной записи по умолчанию).

Поддержка нескольких учётных записей: используйте `channels.zalo.accounts` с токенами для каждой учётной записи и необязательным параметром `name`.

3. Перезапустите шлюз. Zalo запускается, когда токен разрешён (через окружение или конфигурацию).
4. Доступ к прямым сообщениям по умолчанию осуществляется через сопряжение. Подтвердите код при первом контакте с ботом.

## Как это работает (поведение)

- Входящие сообщения нормализуются в общий конверт канала с плейсхолдерами для медиа.
- Ответы всегда направляются обратно в тот же чат Zalo.
- По умолчанию используется длинный опрос (long-polling); режим вебхука доступен с параметром `channels.zalo.webhookUrl`.

## Ограничения

- Исходящий текст разбивается на фрагменты по 2000 символов (ограничение API Zalo).
- Загрузка и выгрузка медиа ограничена параметром `channels.zalo.mediaMaxMb` (по умолчанию — 5 МБ).
- Потоковая передача заблокирована по умолчанию из-за ограничения в 2000 символов, что делает её менее полезной.

## Контроль доступа (прямые сообщения)

### Доступ к прямым сообщениям

- По умолчанию: `channels.zalo.dmPolicy = "pairing"`. Неизвестные отправители получают код сопряжения; сообщения игнорируются до подтверждения (коды истекают через 1 час).
- Подтверждение через:
  - `openclaw pairing list zalo`
  - `openclaw pairing approve zalo <CODE>`
- Сопряжение — это обмен токенами по умолчанию. Подробности: [Сопряжение](/channels/pairing)
- `channels.zalo.allowFrom` принимает числовые идентификаторы пользователей (поиск по имени пользователя недоступен).

## Контроль доступа (группы)

Для **ботов Zalo Bot Creator / маркетплейса** поддержка групп на практике отсутствовала, поскольку бота нельзя было добавить в группу.

Это означает, что ключи конфигурации, связанные с группами, ниже присутствуют в схеме, но не могут быть использованы для ботов на маркетплейсе:

- `channels.zalo.groupPolicy` управляет обработкой входящих сообщений в группах: `open | allowlist | disabled`.
- `channels.zalo.groupAllowFrom` ограничивает идентификаторы отправителей, которые могут активировать бота в группах.
- Если `groupAllowFrom` не задан, Zalo использует `allowFrom` для проверки отправителей.
- Примечание о времени выполнения: если `channels.zalo` полностью отсутствует, во время выполнения по умолчанию используется `groupPolicy="allowlist"` в целях безопасности.

Значения политики групп (когда доступ к группам доступен на вашей платформе бота):

- `groupPolicy: "disabled"` — блокирует все сообщения в группах.
- `groupPolicy: "open"` — разрешает любым участникам группы (с ограничением по упоминаниям).
- `groupPolicy: "allowlist"` — по умолчанию блокировка; принимаются только разрешённые отправители.

Если вы используете другую платформу ботов Zalo и подтвердили работоспособность групп, документируйте это отдельно, а не предполагайте, что это соответствует потоку ботов на маркетплейсе.

## Длинный опрос (long-polling) против вебхука

- По умолчанию: длинный опрос (не требуется публичный URL).
- Режим вебхука: задайте `channels.zalo.webhookUrl` и `channels.zalo.webhookSecret`.
  - Секрет вебхука должен содержать от 8 до 256 символов.
  - URL вебхука должен использовать HTTPS.
  - Zalo отправляет события с заголовком `X-Bot-Api-Secret-Token` для проверки.
  - HTTP-сервер шлюза обрабатывает запросы вебхука по `channels.zalo.webhookPath` (по умолчанию — путь URL вебхука).
  - Запросы должны использовать `Content-Type: application/json` (или типы медиа `+json`).
  - Дублирующиеся события (`event_name + message_id`) игнорируются в течение короткого окна повтора.
  - Интенсивный трафик ограничивается по пути/источнику и может возвращать HTTP 429.

**Примечание:** getUpdates (опрос) и вебхук взаимоисключающие согласно документации API Zalo.

## Поддерживаемые типы сообщений

Для краткой справки по поддержке см. раздел "[Возможности](#vozmozhnosti)". Ниже приведены примечания, где поведение требует дополнительного контекста.

- **Текстовые сообщения:** полная поддержка с разбиением на фрагменты по 2000 символов.
- **Обычные URL в тексте:** ведут себя как обычный текстовый ввод.
- **Предварительный просмотр ссылок / карточки богатых ссылок:** см. статус ботов на маркетплейсе в разделе "[Возможности](#vozmozhnosti)"; они не всегда вызывали ответ.
- **Сообщения с изображениями:** см. статус ботов на маркетплейсе в разделе "[Возможности](#vozmozhnosti)"; обработка входящих изображений была ненадёжной (индикатор набора текста без окончательного ответа).
- **Стикеры:** см. статус ботов на маркетплейсе в разделе "[Возможности](#vozmozhnosti)".
- **Голосовые заметки / аудиофайлы / видео / общие вложения файлов:** см. статус ботов на маркетплейсе в разделе "[Возможности](#vozmozhnosti)".
- **Неподдерживаемые типы:** регистрируются (например, сообщения от защищённых пользователей).
