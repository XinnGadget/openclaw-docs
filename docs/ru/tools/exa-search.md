# Поиск Exa

OpenClaw поддерживает [Exa AI](https://exa.ai/) в качестве провайдера `web_search`. Exa предлагает режимы нейронного, ключевого и гибридного поиска со встроенным извлечением контента (выделения, текст, сводки).

## Получение API-ключа

<Steps>
  <Step title="Создайте учётную запись">
    Зарегистрируйтесь на [exa.ai](https://exa.ai/) и сгенерируйте API-ключ в своей панели управления.
  </Step>
  <Step title="Сохраните ключ">
    Установите `EXA_API_KEY` в среде Gateway либо настройте с помощью команды:

    ```bash
    openclaw configure --section web
    ```

  </Step>
</Steps>

## Конфигурация

```json5
{
  plugins: {
    entries: {
      exa: {
        config: {
          webSearch: {
            apiKey: "exa-...", // необязательно, если задан EXA_API_KEY
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        provider: "exa",
      },
    },
  },
}
```

**Альтернатива с использованием переменных окружения:** задайте `EXA_API_KEY` в среде Gateway. При установке шлюза поместите его в `~/.openclaw/.env`.

## Параметры инструмента

| Параметр | Описание |
| ------------- | ----------------------------------------------------------------------------- |
| `query` | Поисковый запрос (обязательный) |
| `count` | Количество возвращаемых результатов (1–100) |
| `type` | Режим поиска: `auto`, `neural`, `fast`, `deep`, `deep-reasoning` или `instant` |
| `freshness` | Фильтр по времени: `day`, `week`, `month` или `year` |
| `date_after` | Результаты после указанной даты (ГГГГ-ММ-ДД) |
| `date_before` | Результаты до указанной даты (ГГГГ-ММ-ДД) |
| `contents` | Параметры извлечения контента (см. ниже) |

### Извлечение контента

Exa может возвращать извлечённый контент вместе с результатами поиска. Чтобы активировать эту функцию, передайте объект `contents`:

```javascript
await web_search({
  query: "transformer architecture explained",
  type: "neural",
  contents: {
    text: true, // текст всей страницы
    highlights: { numSentences: 3 }, // ключевые предложения
    summary: true, // сводка, сгенерированная ИИ
  },
});
```

| Параметр извлечения контента | Тип | Описание |
| --------------- | --------------------------------------------------------------------- | ---------------------- |
| `text` | `boolean \| { maxCharacters }` | Извлечь текст всей страницы |
| `highlights` | `boolean \| { maxCharacters, query, numSentences, highlightsPerUrl }` | Извлечь ключевые предложения |
| `summary` | `boolean \| { query }` | Сводка, сгенерированная ИИ |

### Режимы поиска

| Режим | Описание |
| ---------------- | --------------------------------- |
| `auto` | Exa выбирает оптимальный режим (по умолчанию) |
| `neural` | Семантический поиск (по смыслу) |
| `fast` | Быстрый поиск по ключевым словам |
| `deep` | Тщательный углублённый поиск |
| `deep-reasoning` | Углублённый поиск с логическим анализом |
| `instant` | Самые быстрые результаты |

## Примечания

- Если параметр `contents` не указан, Exa по умолчанию использует `{ highlights: true }`, поэтому в результатах будут представлены выдержки с ключевыми предложениями.
- В результатах сохраняются поля `highlightScores` и `summary` из ответа API Exa, если они доступны.
- Описания результатов формируются сначала на основе выдержек, затем — сводки, затем — полного текста (в зависимости от того, что доступно).
- Параметры `freshness` и `date_after`/`date_before` нельзя использовать одновременно — выберите один режим фильтрации по времени.
- За один запрос можно получить до 100 результатов (с учётом ограничений, установленных для типа поиска в Exa).
- По умолчанию результаты кэшируются на 15 минут (можно настроить через `cacheTtlMinutes`).
- Exa — официальная интеграция с API, предоставляющая ответы в структурированном формате JSON.
