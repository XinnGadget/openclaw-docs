# Локальные модели

Работа с локальными моделями возможна, но OpenClaw требует большого контекста и надёжной защиты от инъекций промтов. Маломощные GPU обрезают контекст и снижают уровень безопасности. Рекомендуется использовать **≥2 полностью укомплектованных Mac Studio или эквивалентную GPU-систему (стоимость ~30 000 $ и выше)**. Один GPU с **24 ГБ** памяти подойдёт только для несложных промтов, при этом задержка будет выше. Используйте **наиболее крупную/полную версию модели, которую сможете запустить**; агрессивно квантованные или "малые" чекпоинты повышают риск инъекций промтов (см. [Безопасность](/gateway/security)).

Если вы хотите максимально простую локальную настройку, начните с [Ollama](/providers/ollama) и команды `openclaw onboard`. Эта страница содержит руководство по настройке более продвинутых локальных стеков и пользовательских локальных серверов, совместимых с OpenAI.

## Рекомендовано: LM Studio + крупная локальная модель (Responses API)

Лучший на текущий момент локальный стек. Загрузите крупную модель в LM Studio (например, полную версию Qwen, DeepSeek или Llama), включите локальный сервер (по умолчанию `http://127.0.0.1:1234`) и используйте Responses API, чтобы отделить рассуждения от итогового текста.

```json5
{
  agents: {
    defaults: {
      model: { primary: “lmstudio/my-local-model” },
      models: {
        “anthropic/claude-opus-4-6”: { alias: “Opus” },
        “lmstudio/my-local-model”: { alias: “Local” },
      },
    },
  },
  models: {
    mode: “merge”,
    providers: {
      lmstudio: {
        baseUrl: “http://127.0.0.1:1234/v1”,
        apiKey: “lmstudio”,
        api: “openai-responses”,
        models: [
          {
            id: “my-local-model”,
            name: “Local Model”,
            reasoning: false,
            input: [“text”],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 196608,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

**Чек-лист настройки**

- Установите LM Studio: [https://lmstudio.ai](https://lmstudio.ai).
- В LM Studio загрузите **самую крупную доступную версию модели** (избегайте "малых"/сильно квантованных вариантов), запустите сервер и убедитесь, что модель отображается по адресу `http://127.0.0.1:1234/v1/models`.
- Замените `my-local-model` на фактический ID модели, отображаемый в LM Studio.
- Держите модель загруженной — холодная загрузка увеличивает задержку при запуске.
- При необходимости скорректируйте `contextWindow`/`maxTokens` в соответствии с вашей версией LM Studio.
- Для WhatsApp используйте Responses API, чтобы отправлять только итоговый текст.

Настройте размещённые модели даже при работе с локальными — используйте `models.mode: "merge"`, чтобы резервные варианты оставались доступными.

### Гибридная конфигурация: основная — размещённая, резервная — локальная

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-sonnet-4-6",
        fallbacks: ["lmstudio/my-local-model", "anthropic/claude-opus-4-6"],
      },
      models: {
        "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
        "lmstudio/my-local-model": { alias: "Local" },
        "anthropic/claude-opus-4-6": { alias: "Opus" },
      },
    },
  },
  models: {
    mode: "merge",
    providers: {
      lmstudio: {
        baseUrl: "http://127.0.0.1:1234/v1",
        apiKey: "lmstudio",
        api: "openai-responses",
        models: [
          {
            id: "my-local-model",
            name: "Local Model",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 196608,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

### Приоритет локальных моделей с резервом на размещённых

Поменяйте местами основную и резервную модели; оставьте тот же блок providers и `models.mode: "merge"`, чтобы иметь возможность переключиться на Sonnet или Opus, если локальный сервер недоступен.

### Региональное размещение / маршрутизация данных

- Размещённые варианты MiniMax/Kimi/GLM также доступны на OpenRouter с эндпоинтами, привязанными к региону (например, размещённые в США). Выберите региональный вариант, чтобы трафик оставался в выбранной юрисдикции, при этом продолжая использовать `models.mode: "merge"` для резервных вариантов Anthropic/OpenAI.
- Исключительно локальная работа обеспечивает максимальную конфиденциальность; региональная маршрутизация размещённых сервисов — это компромисс, когда вам нужны функции провайдера, но вы хотите контролировать поток данных.

## Другие локальные прокси, совместимые с OpenAI

vLLM, LiteLLM, OAI-proxy или пользовательские шлюзы работают, если они предоставляют эндпоинт в стиле OpenAI `/v1`. Замените блок provider выше на ваш эндпоинт и ID модели:

```json5
{
  models: {
    mode: "merge",
    providers: {
      local: {
        baseUrl: "http://127.0.0.1:8000/v1",
        apiKey: "sk-local",
        api: "openai-responses",
        models: [
          {
            id: "my-local-model",
            name: "Local Model",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 120000,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

Оставьте `models.mode: "merge"`, чтобы размещённые модели оставались доступными в качестве резервных.

**Примечания по поведению для локальных/проксированных бэкендов `/v1`**

- OpenClaw рассматривает их как прокси-маршруты, совместимые с OpenAI, а не как нативные эндпоинты OpenAI.
- Нативные механизмы формирования запросов OpenAI здесь не применяются: нет `service_tier`, нет Responses `store`, нет формирования полезной нагрузки, совместимой с рассуждениями OpenAI, и нет подсказок для кэша промтов.
- Скрытые заголовки атрибуции OpenClaw (`originator`, `version`, `User-Agent`) не добавляются к этим пользовательским прокси-URL.

**Примечания по совместимости для более строгих бэкендов, совместимых с OpenAI**

- Некоторые серверы принимают только строку `messages[].content` в Chat Completions, а не структурированные массивы частей контента. Установите `models.providers.<provider>.models[].compat.requiresStringContent: true` для таких эндпоинтов.
- Некоторые менее мощные или более строгие локальные бэкенды нестабильно работают с полной формой промта агента OpenClaw, особенно при включении схем инструментов. Если бэкенд работает для небольших прямых вызовов `/v1/chat/completions`, но не работает с обычными запросами агента OpenClaw, сначала попробуйте установить `models.providers.<provider>.models[].compat.supportsTools: false`.
- Если бэкенд по-прежнему не работает только при более крупных запусках OpenClaw, проблема, как правило, связана с пропускной способностью модели/сервера или ошибкой бэкенда, а не с транспортным уровнем OpenClaw.

## Устранение неполадок

- Может ли шлюз подключиться к прокси? Выполните команду `curl http://127.0.0.1:1234/v1/models`.
- Модель в LM Studio выгружена? Перезагрузите её — холодная загрузка часто является причиной "зависаний".
