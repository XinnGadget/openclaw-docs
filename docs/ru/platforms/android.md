## Руководство по подключению

Приложение узла Android ⇄ (mDNS/NSD + WebSocket) ⇄ **Шлюз**

Android подключается напрямую к WebSocket шлюза и использует сопряжение устройств (`role: node`).

Для Tailscale или публичных хостов Android требует защищённой конечной точки:

- Предпочтительно: Tailscale Serve / Funnel с `https://<magicdns>` / `wss://<magicdns>`
- Также поддерживается: любой другой URL шлюза `wss://` с реальным TLS-конечной точкой
- Открытый текст `ws://` по-прежнему поддерживается для частных LAN-адресов / хостов `.local`, а также `localhost`, `127.0.0.1` и моста эмулятора Android (`10.0.2.2`)

### Предварительные условия

- Вы можете запустить шлюз на "главной" машине.
- Устройство/эмулятор Android может получить доступ к WebSocket шлюза:
  - Та же LAN с mDNS/NSD, **или**
  - Та же Tailscale tailnet с использованием Wide-Area Bonjour / unicast DNS-SD (см. ниже), **или**
  - Ручной ввод хоста/порта шлюза (запасной вариант)
- Сопряжение через tailnet/публичный мобильный доступ **не** использует необработанные IP-адреса tailnet `ws://`. Вместо этого используйте Tailscale Serve или другой URL `wss://`.
- Вы можете запустить CLI (`openclaw`) на машине шлюза (или через SSH).

### 1) Запустите шлюз

```bash
openclaw gateway --port 18789 --verbose
```

Убедитесь, что в логах есть что-то вроде:

- `listening on ws://0.0.0.0:18789`

Для удалённого доступа Android через Tailscale предпочтительнее использовать Serve/Funnel вместо необработанного привязки к tailnet:

```bash
openclaw gateway --tailscale serve
```

Это предоставит Android защищённую конечную точку `wss://` / `https://`. Простая настройка `gateway.bind: "tailnet"` недостаточна для первого сопряжения Android удалённо, если вы отдельно не завершите TLS.

### 2) Проверьте обнаружение (необязательно)

С машины шлюза:

```bash
dns-sd -B _openclaw-gw._tcp local.
```

Дополнительные заметки по отладке: [Bonjour](/gateway/bonjour).

Если вы также настроили домен обнаружения для широкой области, сравните с:

```bash
openclaw gateway discover --json
```

Это покажет `local.` плюс настроенный домен для широкой области за один проход и использует разрешённую конечную точку службы вместо подсказок только в TXT.

#### Обнаружение через tailnet (Вена ⇄ Лондон) с использованием unicast DNS-SD

Обнаружение Android NSD/mDNS не пересекает сети. Если ваш узел Android и шлюз находятся в разных сетях, но подключены через Tailscale, используйте Wide-Area Bonjour / unicast DNS-SD.

Само по себе обнаружение недостаточно для сопряжения Android через tailnet/публичный доступ. Обнаруженный маршрут всё равно требует защищённой конечной точки (`wss://` или Tailscale Serve):

1. Настройте зону DNS-SD (например, `openclaw.internal.`) на хосте шлюза и опубликуйте записи `_openclaw-gw._tcp`.
2. Настройте разделённый DNS Tailscale для выбранного домена, указывающего на этот DNS-сервер.

Подробности и пример конфигурации CoreDNS: [Bonjour](/gateway/bonjour).

### 3) Подключение с Android

В приложении Android:

- Приложение поддерживает соединение со шлюзом через **фоновый сервис** (постоянное уведомление).
- Откройте вкладку **Connect**.
- Используйте режим **Setup Code** или **Manual**.
- Если обнаружение заблокировано, используйте ручной ввод хоста/порта в **Advanced controls**. Для частных LAN-хостов по-прежнему работает `ws://`. Для хостов Tailscale/публичных хостов включите TLS и используйте конечную точку `wss://` / Tailscale Serve.

После первого успешного сопряжения Android автоматически переподключается при запуске:

- Ручная конечная точка (если включена), в противном случае
- Последний обнаруженный шлюз (по возможности).

### 4) Одобрение сопряжения (CLI)

На машине шлюза:

```bash
openclaw devices list
openclaw devices approve <requestId>
openclaw devices reject <requestId>
```

Подробности о сопряжении: [Сопряжение](/channels/pairing).

### 5) Проверка подключения узла

- Через статус узлов:

  ```bash
  openclaw nodes status
  ```

- Через шлюз:

  ```bash
  openclaw gateway call node.list --params "{}"
  ```

### 6) Чат и история

Вкладка Chat в Android поддерживает выбор сессии (по умолчанию `main`, а также другие существующие сессии):

- История: `chat.history` (нормализованная для отображения; теги директив встроены и удалены из видимого текста, текстовые полезные нагрузки вызовов инструментов (включая `<tool_call>...</tool_call>`, `<function_call>...</function_call>`, `<tool_calls>...</tool_calls>`, `<function_calls>...</function_calls>` и усечённые блоки вызовов инструментов) удалены, просочившиеся ASCII/полноширинные токены управления моделью удалены, строки помощника с чистыми молчаливыми токенами, такие как точное `NO_REPLY` / `no_reply`, опущены, а слишком большие строки могут быть заменены заполнителями).
- Отправка: `chat.send`
- Обновления (по возможности): `chat.subscribe` → `event:"chat"`

### 7) Canvas и камера

#### Хост Canvas шлюза (рекомендуется для веб-контента)

Если вы хотите, чтобы узел отображал реальный HTML/CSS/JS, который агент может редактировать на диске, укажите узлу хост Canvas шлюза.

Примечание: узлы загружают Canvas с HTTP-сервера шлюза (тот же порт, что и `gateway.port`, по умолчанию `18789`).

1. Создайте `~/.openclaw/workspace/canvas/index.html` на хосте шлюза.

2. Перейдите к нему на узле (LAN):

```bash
openclaw nodes invoke --node "<Android Node>" --command canvas.navigate --params '{"url":"http://<gateway-hostname>.local:18789/__openclaw__/canvas/"}'
```

Tailnet (необязательно): если оба устройства подключены к Tailscale, используйте имя MagicDNS или IP tailnet вместо `.local`, например `http://<gateway-magicdns>:18789/__openclaw__/canvas/`.

Этот сервер внедряет клиент live-reload в HTML и перезагружает при изменении файлов. Хост A2UI находится по адресу `http://<gateway-host>:18789/__openclaw__/a2ui/`.

Команды Canvas (только для переднего плана):

- `canvas.eval`, `canvas.snapshot`, `canvas.navigate` (используйте `{"url":""}` или `{"url":"/"}`, чтобы вернуться к стандартному шаблону). `canvas.snapshot` возвращает `{ format, base64 }` (по умолчанию `format="jpeg"`).
- A2UI: `canvas.a2ui.push`, `canvas.a2ui.reset` (`canvas.a2ui.pushJSONL` — устаревший псевдоним).
