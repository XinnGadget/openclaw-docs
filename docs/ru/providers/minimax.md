# MiniMax

Поставщик MiniMax в OpenClaw по умолчанию использует **MiniMax M2.7**.

MiniMax также предоставляет:

- встроенную синтез речи через T2A v2;
- встроенное понимание изображений через `MiniMax-VL-01`;
- встроенную генерацию музыки через `music-2.5+`;
- встроенный `web_search` через API поиска MiniMax Coding Plan.

Разделение поставщиков:

- `minimax`: поставщик текста через API-ключ, а также встроенная генерация изображений, понимание изображений, синтез речи и веб-поиск;
- `minimax-portal`: поставщик текста через OAuth, а также встроенная генерация и понимание изображений.

## Линейка моделей

- `MiniMax-M2.7`: модель рассуждений, размещённая по умолчанию.
- `MiniMax-M2.7-highspeed`: более быстрый уровень рассуждений M2.7.
- `image-01`: модель генерации изображений (создание изображений и редактирование "изображение в изображение").

## Генерация изображений

Плагин MiniMax регистрирует модель `image-01` для инструмента `image_generate`. Она поддерживает:

- **Генерацию "текст в изображение"** с управлением соотношением сторон.
- **Редактирование "изображение в изображение"** (с опорой на объект) с управлением соотношением сторон.
- До **9 выходных изображений** за запрос.
- До **1 опорного изображения** за запрос на редактирование.
- Поддерживаемые соотношения сторон: `1:1`, `16:9`, `4:3`, `3:2`, `2:3`, `3:4`, `9:16`, `21:9`.

Чтобы использовать MiniMax для генерации изображений, задайте его в качестве поставщика генерации изображений:

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: { primary: "minimax/image-01" },
    },
  },
}
```

Плагин использует тот же `MINIMAX_API_KEY` или OAuth-аутентификацию, что и текстовые модели. Дополнительная настройка не требуется, если MiniMax уже настроен.

И `minimax`, и `minimax-portal` регистрируют `image_generate` с одной и той же моделью `image-01`. В настройках с API-ключом используется `MINIMAX_API_KEY`; в настройках с OAuth можно использовать встроенный путь аутентификации `minimax-portal`.

При подключении или настройке API-ключа явные записи `models.providers.minimax` приводят к тому, что OpenClaw материализует `MiniMax-M2.7` и `MiniMax-M2.7-highspeed` с `input: ["text", "image"]`.

Встроенный каталог текстовых моделей MiniMax остаётся только текстовыми метаданными до тех пор, пока не будет создана явная конфигурация поставщика. Понимание изображений предоставляется отдельно через медиапоставщика `MiniMax-VL-01`, принадлежащего плагину.

См. раздел [Генерация изображений](/tools/image-generation) для ознакомления с общими параметрами инструмента, выбором поставщика и поведением при отказоустойчивости.

## Генерация музыки

Встроенный плагин `minimax` также регистрирует генерацию музыки через общий инструмент `music_generate`.

- Модель музыки по умолчанию: `minimax/music-2.5+`.
- Также поддерживаются `minimax/music-2.5` и `minimax/music-2.0`.
- Параметры управления подсказкой: `lyrics`, `instrumental`, `durationSeconds`.
- Формат вывода: `mp3`.
- Запуски с поддержкой сеансов отделяются через общий поток задач/статусов, включая `action: "status"`.

Чтобы использовать MiniMax в качестве поставщика музыки по умолчанию:

```json5
{
  agents: {
    defaults: {
      musicGenerationModel: {
        primary: "minimax/music-2.5+",
      },
    },
  },
}
```

См. раздел [Генерация музыки](/tools/music-generation) для ознакомления с общими параметрами инструмента, выбором поставщика и поведением при отказоустойчивости.

## Генерация видео

Встроенный плагин `minimax` также регистрирует генерацию видео через общий инструмент `video_generate`.

- Модель видео по умолчанию: `minimax/MiniMax-Hailuo-2.3`.
- Режимы: "текст в видео" и потоки с опорой на одно изображение.
- Поддерживаются `aspectRatio` и `resolution`.

Чтобы использовать MiniMax в качестве поставщика видео по умолчанию:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "minimax/MiniMax-Hailuo-2.3",
      },
    },
  },
}
```

См. раздел [Генерация видео](/tools/video-generation) для ознакомления с общими параметрами инструмента, выбором поставщика и поведением при отказоустойчивости.

## Понимание изображений

Плагин MiniMax регистрирует понимание изображений отдельно от текстового каталога:

- `minimax`: модель изображений по умолчанию `MiniMax-VL-01`;
- `minimax-portal`: модель изображений по умолчанию `MiniMax-VL-01`.

Именно поэтому автоматическая маршрутизация медиа может использовать понимание изображений MiniMax, даже если встроенный каталог текстовых поставщиков по-прежнему показывает только текстовые ссылки на M2.7 для чата.

## Веб-поиск

Плагин MiniMax также регистрирует `web_search` через API поиска MiniMax Coding Plan.

- Идентификатор поставщика: `minimax`.
- Структурированные результаты: заголовки, URL, фрагменты, связанные запросы.
- Рекомендуемая переменная окружения: `MINIMAX_CODE_PLAN_KEY`.
- Допустимый псевдоним переменной окружения: `MINIMAX_CODING_API_KEY`.
- Резервный вариант совместимости: `MINIMAX_API_KEY`, если она уже указывает на токен плана кодирования.
- Повторное использование региона: `plugins.entries.minimax.config.webSearch.region`, затем `MINIMAX_API_HOST`, затем базовые URL поставщика MiniMax.
- Поиск остаётся на идентификаторе поставщика `minimax`; настройка OAuth для CN/глобального доступа может косвенно управлять регионом через `models.providers.minimax-portal.baseUrl`.

Конфигурация находится в разделе `plugins.entries.minimax.config.webSearch.*`.
См. раздел [Поиск MiniMax](/tools/minimax-search).

## Выбор настройки

### MiniMax OAuth (Coding Plan) — рекомендуется

**Лучше всего подходит для:** быстрой настройки с MiniMax Coding Plan через OAuth, API-ключ не требуется.

Пройдите аутентификацию с явным выбором регионального OAuth:

```bash
openclaw onboard --auth-choice minimax-global-oauth
# или
openclaw onboard --auth-choice minimax-cn-oauth
```

Соответствие выбора:

- `minimax-global-oauth`: пользователи из других стран (`api.minimax.io`);
- `minimax-cn-oauth`: пользователи в Китае (`api.minimaxi.com`).

См. README пакета плагина MiniMax в репозитории OpenClaw для получения подробной информации.

### MiniMax M2.7 (API-ключ)

**Лучше всего подходит для:** размещённого MiniMax с API, совместимым с Anthropic.

Настройка через CLI:

- Интерактивное подключение:

```bash
openclaw onboard --auth-choice minimax-global-api
# или
openclaw onboard --auth-choice minimax-cn-api
```

- `minimax-global-api`: пользователи из других стран (`api.minimax.io`);
- `minimax-cn-api`: пользователи в Китае (`api.minimaxi.com`).
