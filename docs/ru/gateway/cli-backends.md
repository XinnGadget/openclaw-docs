# Бэкенды CLI (резервный рантайм)

OpenClaw может запускать **локальные AI CLI** в качестве **резерва только для текста**, когда провайдеры API недоступны, ограничены по лимитам или временно работают некорректно. Это сделано намеренно консервативно:

- **Инструменты OpenClaw не внедряются напрямую**, но бэкенды с `bundleMcp: true` могут получать инструменты шлюза через мост loopback MCP.
- **Потоковая передача JSONL** для CLI, которые её поддерживают.
- **Поддерживаются сессии** (чтобы последующие обращения оставались согласованными).
- **Изображения могут передаваться**, если CLI принимает пути к изображениям.

Это реализовано как **страховочная сеть**, а не как основной путь. Используйте это, когда вам нужны "всегда работающие" текстовые ответы без зависимости от внешних API.

Если вам нужен полноценный рантайм с управлением сессиями ACP, фоновыми задачами, привязкой потоков/диалогов и постоянными внешними сессиями кодирования, используйте вместо этого [ACP Agents](/tools/acp-agents). Бэкенды CLI не являются ACP.

## Быстрый старт для начинающих

Вы можете использовать Codex CLI **без какой-либо конфигурации** (встроенный плагин OpenAI регистрирует бэкенд по умолчанию):

```bash
openclaw agent --message "hi" --model codex-cli/gpt-5.4
```

Если ваш шлюз работает под launchd/systemd и PATH минимален, добавьте только путь к команде:

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "codex-cli": {
          command: "/opt/homebrew/bin/codex",
        },
      },
    },
  },
}
```

Вот и всё. Никаких ключей, никакой дополнительной конфигурации аутентификации, кроме самой CLI.

Если вы используете встроенный бэкенд CLI в качестве **основного поставщика сообщений** на хосте шлюза, OpenClaw теперь автоматически загружает соответствующий встроенный плагин, когда ваша конфигурация явно ссылается на этот бэкенд в ссылке на модель или в `agents.defaults.cliBackends`.

## Использование в качестве резерва

Добавьте бэкенд CLI в список резервов, чтобы он запускался только в случае сбоя основных моделей:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "anthropic/claude-opus-4-6",
        fallbacks: ["codex-cli/gpt-5.4"],
      },
      models: {
        "anthropic/claude-opus-4-6": { alias: "Opus" },
        "codex-cli/gpt-5.4": {},
      },
    },
  },
}
```

Примечания:

- Если вы используете `agents.defaults.models` (белый список), вы должны включить в него модели вашего бэкенда CLI.
- Если основной провайдер выдаёт ошибку (аутентификация, лимиты, тайм-ауты), OpenClaw попробует использовать бэкенд CLI.

## Обзор конфигурации

Все бэкенды CLI находятся в разделе:

```
agents.defaults.cliBackends
```

Каждая запись идентифицируется по **ID провайдера** (например, `codex-cli`, `my-cli`). ID провайдера становится левой частью ссылки на модель:

```
<provider>/<model>
```

### Пример конфигурации

```json5
{
  agents: {
    defaults: {
      cliBackends: {
        "codex-cli": {
          command: "/opt/homebrew/bin/codex",
        },
        "my-cli": {
          command: "my-cli",
          args: ["--json"],
          output: "json",
          input: "arg",
          modelArg: "--model",
          modelAliases: {
            "claude-opus-4-6": "opus",
            "claude-sonnet-4-6": "sonnet",
          },
          sessionArg: "--session",
          sessionMode: "existing",
          sessionIdFields: ["session_id", "conversation_id"],
          systemPromptArg: "--system",
          // CLI в стиле Codex могут указывать на файл с подсказкой вместо этого:
          // systemPromptFileConfigArg: "-c",
          // systemPromptFileConfigKey: "model_instructions_file",
          systemPromptWhen: "first",
          imageArg: "--image",
          imageMode: "repeat",
          serialize: true,
        },
      },
    },
  },
}
```

## Как это работает

1. **Выбирает бэкенд** на основе префикса провайдера (`codex-cli/...`).
2. **Формирует системную подсказку**, используя ту же подсказку OpenClaw + контекст рабочей области.
3. **Запускает CLI** с идентификатором сессии (если поддерживается), чтобы история оставалась согласованной.
4. **Парсит вывод** (JSON или обычный текст) и возвращает итоговый текст.
5. **Сохраняет идентификаторы сессий** для каждого бэкенда, чтобы последующие обращения использовали ту же сессию CLI.

<Note>
Встроенный бэкенд Anthropic `claude-cli` снова поддерживается. Сотрудники Anthropic сообщили нам, что использование Claude CLI в стиле OpenClaw снова разрешено, поэтому OpenClaw рассматривает использование `claude -p` как санкционированное для этой интеграции, если Anthropic не опубликует новую политику.
</Note>

Встроенный бэкенд OpenAI `codex-cli` передаёт системную подсказку OpenClaw через переопределение конфигурации Codex `model_instructions_file` (`-c model_instructions_file="..."`). Codex не предоставляет флаг в стиле Claude `--append-system-prompt`, поэтому OpenClaw записывает собранную подсказку во временный файл для каждой новой сессии Codex CLI.

## Сессии

- Если CLI поддерживает сессии, задайте `sessionArg` (например, `--session-id`) или `sessionArgs` (заполнитель `{sessionId}`), если идентификатор нужно вставить в несколько флагов.
- Если CLI использует **подкоманду возобновления** с другими флагами, задайте `resumeArgs` (заменяет `args` при возобновлении) и, при необходимости, `resumeOutput` (для не-JSON возобновлений).
- `sessionMode`:
  - `always`: всегда отправлять идентификатор сессии (новый UUID, если ничего не сохранено).
  - `existing`: отправлять идентификатор сессии только если он был сохранён ранее.
  - `none`: никогда не отправлять идентификатор сессии.

Примечания по сериализации:

- `serialize: true` сохраняет порядок выполнения в рамках одного канала.
- Большинство CLI сериализуют в рамках одного канала провайдера.
- OpenClaw удаляет сохранённые сессии CLI при изменении состояния аутентификации бэкенда, включая повторный вход, ротацию токенов или изменение учётных данных профиля аутентификации.

## Изображения (проброс)

Если ваш CLI принимает пути к изображениям, задайте `imageArg`:

```json5
imageArg: "--image",
imageMode: "repeat"
```

OpenClaw запишет изображения в формате base64 во временные файлы. Если `imageArg` задан, эти пути передаются в качестве аргументов CLI. Если `imageArg` отсутствует, OpenClaw добавляет пути к файлам в подсказку (инъекция пути), что достаточно для CLI, которые автоматически загружают локальные файлы по простым путям.

## Входы / выходы

- `output: "json"` (по умолчанию) пытается распарсить JSON и извлечь текст + идентификатор сессии.
- Для вывода JSON в Gemini CLI OpenClaw считывает текст ответа из `response`, а использование — из `stats`, когда `usage` отсутствует или пуст.
- `output: "jsonl"` парсит потоки JSONL (например, Codex CLI `--json`) и извлекает итоговое сообщение агента плюс идентификаторы сессии, если они присутствуют.
- `output: "text"` рассматривает stdout как итоговый ответ.

Режимы ввода:

- `input: "arg"` (по умолчанию) передаёт подсказку как последний аргумент CLI.
- `input: "stdin"` отправляет подсказку через stdin.
- Если подсказка очень длинная и задано `maxPromptArgChars`, используется stdin.
