# OpenAI

OpenAI предоставляет API для разработчиков для работы с моделями GPT. Codex поддерживает **вход через ChatGPT** для доступа по подписке или **вход по API-ключу** для доступа на основе использования. Для облачной версии Codex требуется вход через ChatGPT. OpenAI явно поддерживает использование подписки через OAuth во внешних инструментах и рабочих процессах, таких как OpenClaw.

## Стиль взаимодействия по умолчанию

OpenClaw может добавлять небольшой специфичный для OpenAI оверлей промпта для запусков `openai/*` и `openai-codex/*`. По умолчанию оверлей поддерживает дружелюбный, коллаборативный, лаконичный и прямой стиль общения, а также немного более эмоциональную выразительность, не заменяя при этом базовый промпт системы OpenClaw. Дружелюбный оверлей также допускает использование эмодзи, если это естественно вписывается в контекст, при этом сохраняя общую лаконичность вывода.

Ключ конфигурации:

`plugins.entries.openai.config.personality`

Допустимые значения:

- `"friendly"`: значение по умолчанию; включает специфичный для OpenAI оверлей.
- `"on"`: псевдоним для `"friendly"`.
- `"off"`: отключает оверлей и использует только базовый промпт OpenClaw.

Область применения:

- Применяется к моделям `openai/*`.
- Применяется к моделям `openai-codex/*`.
- Не влияет на других провайдеров.

Такое поведение включено по умолчанию. Если вы хотите сохранить значение `"friendly"` при будущих изменениях локальной конфигурации, укажите его явно:

```json5
{
  plugins: {
    entries: {
      openai: {
        config: {
          personality: "friendly",
        },
      },
    },
  },
}
```

### Отключение оверлея промпта OpenAI

Если вы хотите использовать неизменный базовый промпт OpenClaw, установите для оверлея значение `"off"`:

```json5
{
  plugins: {
    entries: {
      openai: {
        config: {
          personality: "off",
        },
      },
    },
  },
}
```

Вы также можете установить это значение напрямую с помощью CLI конфигурации:

```bash
openclaw config set plugins.entries.openai.config.personality off
```

OpenClaw нормализует это значение без учёта регистра во время выполнения, поэтому значения вроде `"Off"` также отключают дружелюбный оверлей.

## Вариант А: API-ключ OpenAI (платформа OpenAI)

**Оптимально для:** прямого доступа к API и тарификации на основе использования.
Получите свой API-ключ в панели управления OpenAI.

Краткое описание маршрута:

- `openai/gpt-5.4` = прямой маршрут API платформы OpenAI
- Требуется `OPENAI_API_KEY` (или эквивалентная конфигурация провайдера OpenAI)
- В OpenClaw вход через ChatGPT/Codex осуществляется через `openai-codex/*`, а не через `openai/*`

### Настройка через CLI

```bash
openclaw onboard --auth-choice openai-api-key
# или в неинтерактивном режиме
openclaw onboard --openai-api-key "$OPENAI_API_KEY"
```

### Фрагмент конфигурации

```json5
{
  env: { OPENAI_API_KEY: "sk-..." },
  agents: { defaults: { model: { primary: "openai/gpt-5.4" } } },
}
```

В текущей документации API OpenAI указаны модели `gpt-5.4` и `gpt-5.4-pro` для прямого использования API OpenAI. OpenClaw передаёт обе через путь ответов `openai/*`. OpenClaw намеренно исключает устаревшую строку `openai/gpt-5.3-codex-spark`, поскольку прямые вызовы API OpenAI отклоняют её в реальном трафике.

OpenClaw **не** предоставляет доступ к `openai/gpt-5.3-codex-spark` через прямой путь API OpenAI. В `pi-ai` по-прежнему есть встроенная строка для этой модели, но текущие запросы API OpenAI отклоняют её. Spark рассматривается в OpenClaw как доступный только в Codex.

## Генерация изображений

Встроенный плагин `openai` также регистрирует генерацию изображений через общий инструмент `image_generate`.

- Модель изображений по умолчанию: `openai/gpt-image-1`
- Генерация: до 4 изображений за запрос
- Режим редактирования: включён, до 5 эталонных изображений
- Поддерживается параметр `size`
- Текущее ограничение, специфичное для OpenAI: OpenClaw в настоящее время не передаёт переопределения `aspectRatio` или `resolution` в API изображений OpenAI

Чтобы использовать OpenAI в качестве провайдера изображений по умолчанию:

```json5
{
  agents: {
    defaults: {
      imageGenerationModel: {
        primary: "openai/gpt-image-1",
      },
    },
  },
}
```

См. раздел [Генерация изображений](/tools/image-generation) для ознакомления с параметрами общего инструмента, выбором провайдера и поведением при отказоустойчивости.

## Генерация видео

Встроенный плагин `openai` также регистрирует генерацию видео через общий инструмент `video_generate`.

- Модель видео по умолчанию: `openai/sora-2`
- Режимы: текст-в-видео, изображение-в-видео и потоки редактирования с одним видео/эталоном
- Текущие ограничения: 1 изображение или 1 эталонное видео на вход
- Текущее ограничение, специфичное для OpenAI: OpenClaw в настоящее время передаёт только переопределения `size` для нативной генерации видео OpenAI. Неподдерживаемые дополнительные переопределения, такие как `aspectRatio`, `resolution`, `audio` и `watermark`, игнорируются и сообщаются как предупреждение инструмента.

Чтобы использовать OpenAI в качестве провайдера видео по умолчанию:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "openai/sora-2",
      },
    },
  },
}
```

См. раздел [Генерация видео](/tools/video-generation) для ознакомления с параметрами общего инструмента, выбором провайдера и поведением при отказоустойчивости.

## Вариант Б: Подписка OpenAI Code (Codex)

**Оптимально для:** использования доступа по подписке ChatGPT/Codex вместо API-ключа.
Для облачной версии Codex требуется вход через ChatGPT, в то время как CLI Codex поддерживает вход через ChatGPT или по API-ключу.

Краткое описание маршрута:

- `openai-codex/gpt-5.4` = маршрут OAuth ChatGPT/Codex
- Используется вход через ChatGPT/Codex, а не прямой API-ключ платформы OpenAI
- Ограничения на стороне провайдера для `openai-codex/*` могут отличаться от веб-версии/приложения ChatGPT

### Настройка через CLI (OAuth Codex)

```bash
# Запустить OAuth Codex в мастере
openclaw onboard --auth-choice openai-codex

# Или запустить OAuth напрямую
openclaw models auth login --provider openai-codex
```

### Фрагмент конфигурации (подписка Codex)

```json5
{
  agents: { defaults: { model: { primary: "openai-codex/gpt-5.4" } } },
}
```

В текущей документации Codex от OpenAI модель `gpt-5.4` указана как текущая модель Codex. OpenClaw сопоставляет её с `openai-codex/gpt-5.4` для использования OAuth ChatGPT/Codex.

Этот маршрут намеренно отделён от `openai/gpt-5.4`. Если вам нужен прямой путь API платформы OpenAI, используйте `openai/*` с API-ключом. Если вам нужен вход через ChatGPT/Codex, используйте `openai-codex/*`.

Если при регистрации используется существующий вход в CLI Codex, эти учётные данные остаются под управлением CLI Codex. По истечении срока действия OpenClaw сначала повторно считывает внешний источник Codex и, если провайдер может обновить их, записывает обновлённые учётные данные обратно в хранилище Codex, вместо того чтобы брать их под свой контроль в отдельной копии, предназначенной только для OpenClaw.

Если ваша учётная запись Codex имеет доступ к Codex Spark, OpenClaw также поддерживает:

- `openai-codex/gpt-5.3-codex-spark`

OpenClaw рассматривает Codex Spark как доступный только в Codex. Он не предоставляет прямой путь API-ключа `openai/gpt-5.3-codex-spark`.

OpenClaw также сохраняет `openai-codex/gpt-5.3-codex-spark`, когда `pi-ai` обнаруживает его. Рассматривайте это как зависящее
