# Ollama

Ollama — это локальная среда выполнения LLM, которая упрощает запуск моделей с открытым исходным кодом на вашем компьютере. OpenClaw интегрируется с нативным API Ollama (`/api/chat`), поддерживает потоковую передачу данных и вызов инструментов, а также может автоматически обнаруживать локальные модели Ollama, если вы указали `OLLAMA_API_KEY` (или профиль аутентификации) и не задали явную запись `models.providers.ollama`.

<Warning>
**Пользователи удалённого Ollama**: не используйте URL, совместимый с OpenAI `/v1` (`http://host:11434/v1`) с OpenClaw. Это нарушает работу вызова инструментов, и модели могут выводить сырой JSON инструментов в виде обычного текста. Вместо этого используйте нативный URL API Ollama: `baseUrl: "http://host:11434"` (без `/v1`).
</Warning>

## Быстрый старт

### Онбординг (рекомендуется)

Самый быстрый способ настроить Ollama — через онбординг:

```bash
openclaw onboard
```

Выберите **Ollama** из списка провайдеров. Онбординг выполнит следующие действия:

1. Запросит базовый URL Ollama, по которому доступен ваш экземпляр (по умолчанию `http://127.0.0.1:11434`).
2. Позволит выбрать **Cloud + Local** (облачные и локальные модели) или **Local** (только локальные модели).
3. Запустит поток входа в систему через браузер, если вы выбрали **Cloud + Local** и не вошли в систему на ollama.com.
4. Обнаружит доступные модели и предложит значения по умолчанию.
5. Автоматически загрузит выбранную модель, если её нет локально.

Также поддерживается неинтерактивный режим:

```bash
openclaw onboard --non-interactive \
  --auth-choice ollama \
  --accept-risk
```

При желании можно указать пользовательский базовый URL или модель:

```bash
openclaw onboard --non-interactive \
  --auth-choice ollama \
  --custom-base-url "http://ollama-host:11434" \
  --custom-model-id "qwen3.5:27b" \
  --accept-risk
```

### Ручная настройка

1. Установите Ollama: [https://ollama.com/download](https://ollama.com/download)

2. Загрузите локальную модель, если хотите выполнять локальный инференс:

```bash
ollama pull gemma4
# или
ollama pull gpt-oss:20b
# или
ollama pull llama3.3
```

3. Если вы также хотите использовать облачные модели, войдите в систему:

```bash
ollama signin
```

4. Запустите онбординг и выберите `Ollama`:

```bash
openclaw onboard
```

- **Local**: только локальные модели
- **Cloud + Local**: локальные модели плюс облачные модели

Облачные модели, такие как `kimi-k2.5:cloud`, `minimax-m2.7:cloud` и `glm-5.1:cloud`, **не** требуют локальной команды `ollama pull`.

В настоящее время OpenClaw предлагает:

- локальная модель по умолчанию: `gemma4`
- облачные модели по умолчанию: `kimi-k2.5:cloud`, `minimax-m2.7:cloud`, `glm-5.1:cloud`

5. Если вы предпочитаете ручную настройку, включите Ollama для OpenClaw напрямую (подходит любое значение; Ollama не требует реального ключа):

```bash
# Установите переменную окружения
export OLLAMA_API_KEY="ollama-local"

# Или настройте в файле конфигурации
openclaw config set models.providers.ollama.apiKey "ollama-local"
```

6. Проверьте или переключитесь на другие модели:

```bash
openclaw models list
openclaw models set ollama/gemma4
```

7. Или задайте значение по умолчанию в конфигурации:

```json5
{
  agents: {
    defaults: {
      model: { primary: "ollama/gemma4" },
    },
  },
}
```

## Обнаружение моделей (неявный провайдер)

Когда вы задаёте `OLLAMA_API_KEY` (или профиль аутентификации) и **не** определяете `models.providers.ollama`, OpenClaw обнаруживает модели из локального экземпляра Ollama по адресу `http://127.0.0.1:11434`:

- Отправляет запросы к `/api/tags`
- Использует запросы `/api/show` (по возможности), чтобы прочитать `contextWindow` и определить возможности (включая работу с изображениями), если они доступны
- Модели с возможностью `vision`, указанной в `/api/show`, помечаются как поддерживающие изображения (`input: ["text", "image"]`), поэтому OpenClaw автоматически добавляет изображения в промпт для этих моделей
- Помечает `reasoning` с помощью эвристики по имени модели (`r1`, `reasoning`, `think`)
- Устанавливает `maxTokens` равным максимальному количеству токенов, используемому Ollama в OpenClaw
- Устанавливает все затраты в `0`

Это позволяет избежать ручного ввода моделей, сохраняя каталог синхронизированным с локальным экземпляром Ollama.

Чтобы увидеть доступные модели:

```bash
ollama list
openclaw models list
```

Чтобы добавить новую модель, просто загрузите её с помощью Ollama:

```bash
ollama pull mistral
```

Новая модель будет автоматически обнаружена и станет доступной для использования.

Если вы явно зададите `models.providers.ollama`, автоматическое обнаружение будет пропущено, и вам придётся определять модели вручную (см. ниже).

## Конфигурация

### Базовая настройка (неявное обнаружение)

Самый простой способ включить Ollama — через переменную окружения:

```bash
export OLLAMA_API_KEY="ollama-local"
```

### Явная настройка (ручные модели)

Используйте явную конфигурацию, когда:

- Ollama работает на другом хосте/порту.
- Вы хотите принудительно задать определённые окна контекста или списки моделей.
- Вы хотите полностью вручную определить модели.

```json5
{
  models: {
    providers: {
      ollama: {
        baseUrl: "http://ollama-host:11434",
        apiKey: "ollama-local",
        api: "ollama",
        models: [
          {
            id: "gpt-oss:20b",
            name: "GPT-OSS 20B",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 8192,
            maxTokens: 8192 * 10
          }
        ]
      }
    }
  }
}
```

Если задана переменная `OLLAMA_API_KEY`, вы можете не указывать `apiKey` в записи провайдера, и OpenClaw заполнит её для проверки доступности.

### Пользовательский базовый URL (явная конфигурация)

Если Ollama работает на другом хосте или порту (явная конфигурация отключает автоматическое обнаружение, поэтому модели нужно определять вручную):

```json5
{
  models: {
    providers: {
      ollama: {
        apiKey: "ollama-local",
        baseUrl: "http://ollama-host:11434", // Без /v1 — используйте нативный URL API Ollama
        api: "ollama", // Задайте явно, чтобы гарантировать нативное поведение вызова инструментов
      },
    },
  },
}
```

<Warning>
Не добавляйте `/v1` к URL. Путь `/v1` использует режим, совместимый с OpenAI, где вызов инструментов ненадёжен. Используйте базовый URL Ollama без суффикса пути.
</Warning>

### Выбор модели

После настройки все ваши модели Ollama будут доступны:

```json5
{
  agents: {
    defaults: {
      model: {
        primary: "ollama/gpt-oss:20b",
        fallbacks: ["ollama/llama3.3", "ollama/qwen2.5-coder:32b"],
      },
    },
  },
}
```
