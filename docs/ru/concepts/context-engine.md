# Механизм контекста

**Механизм контекста** управляет тем, как OpenClaw формирует контекст модели для каждого запуска.
Он определяет, какие сообщения включать, как суммировать более старую историю и как управлять контекстом на границах субагентов.

В OpenClaw встроен механизм `legacy`. Плагины могут регистрировать альтернативные механизмы, которые заменяют активный жизненный цикл механизма контекста.

## Быстрый старт

Проверьте, какой механизм активен:

```bash
openclaw doctor
# или проверьте конфигурацию напрямую:
cat ~/.openclaw/openclaw.json | jq '.plugins.slots.contextEngine'
```

### Установка плагина механизма контекста

Плагины механизма контекста устанавливаются так же, как и любые другие плагины OpenClaw. Сначала установите, затем выберите механизм в слоте:

```bash
# Установить из npm
openclaw plugins install @martian-engineering/lossless-claw

# Или установить из локального пути (для разработки)
openclaw plugins install -l ./my-context-engine
```

Затем включите плагин и выберите его в качестве активного механизма в конфигурации:

```json5
// openclaw.json
{
  plugins: {
    slots: {
      contextEngine: "lossless-claw", // должно совпадать с ID механизма, зарегистрированного плагином
    },
    entries: {
      "lossless-claw": {
        enabled: true,
        // Конфигурация, специфичная для плагина (см. документацию плагина)
      },
    },
  },
}
```

Перезапустите шлюз после установки и настройки.

Чтобы вернуться к встроенному механизму, установите для `contextEngine` значение `"legacy"` (или полностью удалите ключ — `"legacy"` используется по умолчанию).

## Как это работает

Каждый раз, когда OpenClaw запускает подсказку модели, механизм контекста участвует в четырёх этапах жизненного цикла:

1. **Ingest (приём)** — вызывается, когда в сессию добавляется новое сообщение. Механизм может сохранить или проиндексировать сообщение в своём хранилище данных.
2. **Assemble (сборка)** — вызывается перед каждым запуском модели. Механизм возвращает упорядоченный набор сообщений (и необязательный `systemPromptAddition`), которые укладываются в лимит токенов.
3. **Compact (сжатие)** — вызывается, когда окно контекста заполнено или когда пользователь запускает `/compact`. Механизм суммирует более старую историю, чтобы освободить место.
4. **After turn (после хода)** — вызывается после завершения запуска. Механизм может сохранить состояние, запустить фоновое сжатие или обновить индексы.

### Жизненный цикл субагента (необязательно)

В настоящее время OpenClaw вызывает один хук жизненного цикла субагента:

- **onSubagentEnded** — очистка после завершения сессии субагента или её удаления.

Хук `prepareSubagentSpawn` является частью интерфейса для будущего использования, но среда выполнения пока его не вызывает.

### Добавление системной подсказки

Метод `assemble` может возвращать строку `systemPromptAddition`. OpenClaw добавляет её в начало системной подсказки для запуска. Это позволяет механизмам внедрять динамические указания по извлечению информации, инструкции по поиску или подсказки с учётом контекста без необходимости использования статических файлов рабочей области.

## Устаревший механизм

Встроенный механизм `legacy` сохраняет исходное поведение OpenClaw:

- **Ingest**: нет действий (менеджер сессии напрямую обрабатывает сохранение сообщений).
- **Assemble**: проход без изменений (существующий конвейер sanitize → validate → limit в среде выполнения обрабатывает сборку контекста).
- **Compact**: делегируется встроенному сжатию с суммированием, которое создаёт единое резюме более старых сообщений и оставляет недавние сообщения без изменений.
- **After turn**: нет действий.

Устаревший механизм не регистрирует инструменты и не предоставляет `systemPromptAddition`.

Если параметр `plugins.slots.contextEngine` не задан (или задан как `"legacy"`), этот механизм используется автоматически.

## Плагины механизмов

Плагин может зарегистрировать механизм контекста с помощью API плагина:

```ts
import { buildMemorySystemPromptAddition } from "openclaw/plugin-sdk/core";

export default function register(api) {
  api.registerContextEngine("my-engine", () => ({
    info: {
      id: "my-engine",
      name: "My Context Engine",
      ownsCompaction: true,
    },

    async ingest({ sessionId, message, isHeartbeat }) {
      // Сохранить сообщение в своём хранилище данных
      return { ingested: true };
    },

    async assemble({ sessionId, messages, tokenBudget, availableTools, citationsMode }) {
      // Вернуть сообщения, которые укладываются в бюджет
      return {
        messages: buildContext(messages, tokenBudget),
        estimatedTokens: countTokens(messages),
        systemPromptAddition: buildMemorySystemPromptAddition({
          availableTools: availableTools ?? new Set(),
          citationsMode,
        }),
      };
    },

    async compact({ sessionId, force }) {
      // Суммировать более старый контекст
      return { ok: true, compacted: true };
    },
  }));
}
```

Затем включите его в конфигурации:

```json5
{
  plugins: {
    slots: {
      contextEngine: "my-engine",
    },
    entries: {
      "my-engine": {
        enabled: true,
      },
    },
  },
}
```

### Интерфейс ContextEngine

Обязательные элементы:

| Элемент | Тип | Назначение |
| --- | --- | --- |
| `info` | Свойство | ID механизма, название, версия и указание на то, управляет ли он сжатием |
| `ingest(params)` | Метод | Сохранить одно сообщение |
| `assemble(params)` | Метод | Собрать контекст для запуска модели (возвращает `AssembleResult`) |
| `compact(params)` | Метод | Суммировать/сократить контекст |

`assemble` возвращает `AssembleResult` со следующими полями:

- `messages` — упорядоченные сообщения, которые будут отправлены модели.
- `estimatedTokens` (обязательно, `number`) — оценка механизма общего количества токенов в собранном контексте. OpenClaw использует это значение для принятия решений о пороге сжатия и для диагностической отчётности.
- `systemPromptAddition` (необязательно, `string`) — добавляется в начало системной подсказки.

Необязательные элементы:

| Элемент | Тип | Назначение |
| --- | --- | --- |
| `bootstrap(params)` | Метод | Инициализировать состояние механизма для сессии. Вызывается один раз, когда механизм впервые видит сессию (например, для импорта истории). |
| `ingestBatch(params)` | Метод | Принять завершённый ход как пакет. Вызывается после завершения запуска со всеми сообщениями этого хода сразу. |
| `afterTurn(params)` | Метод | Работа на этапе жизненного цикла после запуска (сохранение состояния, запуск фонового сжатия). |
| `prepareSubagentSpawn(params)` | Метод | Настройка общего состояния для дочерней сессии. |
| `onSubagentEnded(params)` | Метод | Очистка после завершения работы субагента. |
| `dispose()` | Метод | Освобождение ресурсов. Вызывается при завершении работы шлюза или перезагрузке плагина — не для каждой сессии. |

### ownsCompaction

`ownsCompaction` определяет, остаётся ли включённым встроенное автоматическое сжатие Pi в рамках попытки:

- `true` — механизм управляет поведением сжатия. OpenClaw отключает встроенное автоматическое сжатие Pi для этого запуска, и реализация `compact()` механизма отвечает за `/compact`, восстановление при переполнении и любое упреждающее сжатие, которое он хочет выполнить в `afterTurn()`.
- `false` или не задано — встроенное автоматическое сжатие Pi может по-прежнему выполняться во время выполнения подсказки, но метод `compact()` активного механизма всё равно вызывается для `/compact` и восстановления при переполнении.

`ownsCompaction: false` **не** означает, что OpenClaw автоматически возвращается к пути сжатия устаревшего механизма.

Это означает, что существуют два допустимых шаблона для плагинов:

- **Режим владения** — реализовать собственный алгоритм сжатия и установить `ownsCompaction: true`.
- **Режим делегирования** — установить `ownsCompaction: false` и в `compact()` вызвать `delegateCompactionToRuntime(...)` из `openclaw/plugin-sdk/core`, чтобы использовать встроенное поведение сжатия OpenClaw.

Пустой метод `compact()` небезопасен для активного механизма, не являющегося владельцем, поскольку он отключает обычный путь `/compact` и восстановления при переполнении для этого слота механизма.

## Справочная информация по конфигурации

```json5
{
  plugins: {
    slots: {
      // Выбрать активный механизм контекста. По умолчанию: "legacy".
      // Установить ID плагина, чтобы использовать механизм плагина.
      contextEngine: "legacy",
    },
  },
}
```
