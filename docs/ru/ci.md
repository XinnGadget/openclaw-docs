# Конвейер CI

CI запускается при каждом пуше в ветку `main` и при каждом pull request. Используется интеллектуальное определение областей действия, чтобы пропускать ресурсоёмкие задания, если изменились только несвязанные области.

## Обзор заданий

| Задание | Назначение | Когда запускается |
| --- | --- | --- |
| `preflight` | Выявление изменений только в документации, изменённых областей действия, изменённых расширений и сборка манифеста CI | Всегда при пушах и PR, не являющихся черновиками |
| `security-fast` | Поиск закрытых ключей, аудит рабочих процессов с помощью `zizmor`, аудит зависимостей для продакшена | Всегда при пушах и PR, не являющихся черновиками |
| `build-artifacts` | Однократная сборка `dist/` и Control UI, загрузка повторно используемых артефактов для последующих заданий | При изменениях, релевантных для Node |
| `checks-fast-core` | Быстрые проверки корректности в Linux, такие как проверки bundled/plugin-contract/protocol | При изменениях, релевантных для Node |
| `checks-fast-extensions` | Агрегация полос шардов расширений после завершения `checks-fast-extensions-shard` | При изменениях, релевантных для Node |
| `extension-fast` | Целевые тесты только для изменённых встроенных плагинов | При обнаружении изменений в расширениях |
| `check` | Основные локальные ворота в CI: `pnpm check` плюс `pnpm build:strict-smoke` | При изменениях, релевантных для Node |
| `check-additional` | Защита архитектуры, границ, циклов импорта, а также набор тестов для регрессии шлюза | При изменениях, релевантных для Node |
| `build-smoke` | Дымовые тесты Built-CLI и проверка памяти при запуске | При изменениях, релевантных для Node |
| `checks` | Более ресурсоёмкие полосы для Linux Node: полные тесты, тесты каналов и проверка совместимости с Node 22 (только для пушей) | При изменениях, релевантных для Node |
| `check-docs` | Проверка форматирования документации, линтинг и поиск битых ссылок | При изменениях в документации |
| `skills-python` | Ruff + pytest для навыков на Python | При изменениях, релевантных для навыков на Python |
| `checks-windows` | Тестовые полосы, специфичные для Windows | При изменениях, релевантных для Windows |
| `macos-node` | Полоса тестов TypeScript для macOS с использованием общих собранных артефактов | При изменениях, релевантных для macOS |
| `macos-swift` | Линтинг, сборка и тесты для приложения macOS на Swift | При изменениях, релевантных для macOS |
| `android` | Матрица сборки и тестов для Android | При изменениях, релевантных для Android |

## Порядок быстрого отказа

Задания упорядочены так, чтобы дешёвые проверки завершались с ошибкой до запуска ресурсоёмких:

1. `preflight` определяет, какие полосы вообще существуют. Логика `docs-scope` и `changed-scope` — это шаги внутри этого задания, а не отдельные задания.
2. `security-fast`, `check`, `check-additional`, `check-docs` и `skills-python` быстро завершаются с ошибкой, не дожидаясь более ресурсоёмких заданий по сборке артефактов и матрице платформ.
3. `build-artifacts` выполняется параллельно с быстрыми полосами для Linux, чтобы последующие потребители могли начать работу, как только будет готова общая сборка.
4. После этого запускаются более ресурсоёмкие полосы для платформ и сред выполнения: `checks-fast-core`, `checks-fast-extensions`, `extension-fast`, `checks`, `checks-windows`, `macos-node`, `macos-swift` и `android`.

Логика областей действия находится в `scripts/ci-changed-scope.mjs` и покрыта юнит-тестами в `src/scripts/ci-changed-scope.test.ts`.
Отдельный рабочий процесс `install-smoke` повторно использует тот же скрипт областей действия через своё собственное задание `preflight`. Он вычисляет `run_install_smoke` на основе более узкого сигнала changed-smoke, поэтому дымовые тесты Docker/install выполняются только при изменениях, связанных с установкой, упаковкой и контейнерами.

При пушах матрица `checks` добавляет полосу `compat-node22`, доступную только для пушей. При pull request эта полоса пропускается, и матрица остаётся сосредоточенной на обычных тестовых полосах и полосах каналов.

## Раннеры

| Раннер | Задания |
| --- | --- |
| `blacksmith-16vcpu-ubuntu-2404` | `preflight`, `security-fast`, `build-artifacts`, проверки для Linux, проверки документации, навыки на Python, `android` |
| `blacksmith-32vcpu-windows-2025` | `checks-windows` |
| `macos-latest` | `macos-node`, `macos-swift` |

## Эквиваленты локальных команд

```bash
pnpm check          # типы + линтинг + форматирование
pnpm build:strict-smoke
pnpm check:import-cycles
pnpm test:gateway:watch-regression
pnpm test           # тесты vitest
pnpm test:channels
pnpm check:docs     # форматирование документации + линтинг + поиск битых ссылок
pnpm build          # сборка dist, когда важны полосы артефактов CI/build-smoke
```
