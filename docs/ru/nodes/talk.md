# Режим разговора

Режим разговора — это непрерывный цикл голосового диалога:

1. Прослушивание речи.
2. Отправка расшифровки в модель (основной сеанс, chat.send).
3. Ожидание ответа.
4. Озвучивание ответа через настроенного провайдера Talk (`talk.speak`).

## Поведение (macOS)

- **Постоянно отображаемый оверлей** во время активации режима разговора.
- Переходы между фазами: **прослушивание → обдумывание → озвучивание**.
- При **короткой паузе** (окно тишины) текущая расшифровка отправляется.
- Ответы **записываются в WebChat** (так же, как при вводе текста).
- **Прерывание при речи** (включено по умолчанию): если пользователь начинает говорить, когда ассистент озвучивает ответ, воспроизведение останавливается, а метка времени прерывания фиксируется для следующего запроса.

## Голосовые директивы в ответах

Ассистент может начать свой ответ с **одной строки в формате JSON** для управления голосом:

```json
{ "voice": "<voice-id>", "once": true }
```

Правила:

- Учитывается только первая непустая строка.
- Неизвестные ключи игнорируются.
- `once: true` применяется только к текущему ответу.
- Если `once` не указан, голос становится новым значением по умолчанию для режима разговора.
- Строка JSON удаляется перед воспроизведением TTS.

Поддерживаемые ключи:

- `voice` / `voice_id` / `voiceId`
- `model` / `model_id` / `modelId`
- `speed`, `rate` (WPM), `stability`, `similarity`, `style`, `speakerBoost`
- `seed`, `normalize`, `lang`, `output_format`, `latency_tier`
- `once`

## Конфигурация (`~/.openclaw/openclaw.json`)

```json5
{
  talk: {
    voiceId: "elevenlabs_voice_id",
    modelId: "eleven_v3",
    outputFormat: "mp3_44100_128",
    apiKey: "elevenlabs_api_key",
    silenceTimeoutMs: 1500,
    interruptOnSpeech: true,
  },
}
```

Значения по умолчанию:

- `interruptOnSpeech`: true
- `silenceTimeoutMs`: если не задано, Talk использует значение паузы по умолчанию для платформы перед отправкой расшифровки (`700 мс на macOS и Android, 900 мс на iOS`)
- `voiceId`: используется значение `ELEVENLABS_VOICE_ID` / `SAG_VOICE_ID` (или первый голос ElevenLabs, если доступен API-ключ)
- `modelId`: по умолчанию `eleven_v3`, если не задано
- `apiKey`: используется значение `ELEVENLABS_API_KEY` (или профиль оболочки шлюза, если доступен)
- `outputFormat`: по умолчанию `pcm_44100` на macOS/iOS и `pcm_24000` на Android (укажите `mp3_*`, чтобы принудительно использовать потоковую передачу MP3)

## Интерфейс пользователя на macOS

- Переключатель в строке меню: **Talk**
- Вкладка конфигурации: группа **Talk Mode** (идентификатор голоса + переключатель прерывания)
- Оверлей:
  - **Прослушивание**: облако пульсирует в соответствии с уровнем микрофона
  - **Обдумывание**: анимация погружения
  - **Озвучивание**: расходящиеся кольца
  - Щелчок по облаку: остановка озвучивания
  - Щелчок по X: выход из режима разговора

## Примечания

- Требуется разрешение на использование речи и микрофона.
- Используется `chat.send` с ключом сеанса `main`.
- Шлюз осуществляет воспроизведение в режиме Talk через `talk.speak` с использованием активного провайдера Talk. На Android в качестве запасного варианта используется локальная система TTS, только если этот RPC недоступен.
- Для `eleven_v3` значение `stability` должно быть `0.0`, `0.5` или `1.0`; другие модели принимают значения от `0` до `1`.
- Значение `latency_tier` должно быть в диапазоне от `0` до `4`, если оно задано.
- На Android поддерживаются форматы вывода `pcm_16000`, `pcm_22050`, `pcm_24000` и `pcm_44100` для потоковой передачи AudioTrack с низкой задержкой.
