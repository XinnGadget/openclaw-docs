# Анализ медиаконтента — входящие данные (2026-01-17)

OpenClaw может **суммировать входящий медиаконтент** (изображения/аудио/видео) до запуска конвейера ответов. Система автоматически определяет наличие локальных инструментов или ключей провайдера, а также позволяет отключить или настроить эту функцию. Если анализ отключён, модели по-прежнему получают исходные файлы/URL-адреса в обычном режиме.

Поведение, специфичное для конкретного провайдера, регистрируется с помощью плагинов провайдера, в то время как ядро OpenClaw управляет общей конфигурацией `tools.media`, порядком резервных механизмов и интеграцией с конвейером ответов.

## Цели

- Опционально: предварительно обрабатывать входящий медиаконтент, преобразуя его в краткий текст для ускорения маршрутизации и улучшения парсинга команд.
- Всегда сохранять передачу исходного медиаконтента модели.
- Поддерживать **API провайдеров** и **резервные механизмы через CLI**.
- Позволить использование нескольких моделей с упорядоченным резервным механизмом (ошибки/размер/тайм-аут).

## Общее поведение

1. Собрать входящие вложения (`MediaPaths`, `MediaUrls`, `MediaTypes`).
2. Для каждой включённой возможности (изображение/аудио/видео) выбрать вложения согласно политике (по умолчанию — **первое**).
3. Выбрать первую подходящую запись модели (с учётом размера, возможностей и аутентификации).
4. Если модель не сработала или размер медиаконтента превышает допустимый, **перейти к следующей записи**.
5. В случае успеха:
   - `Body` преобразуется в блок `[Image]`, `[Audio]` или `[Video]`.
   - Для аудио устанавливается `{{Transcript}}`; при парсинге команд используется текст подписи, если она есть, в противном случае — транскрипция.
   - Подписи сохраняются как `User text:` внутри блока.

Если анализ не удался или отключён, **процесс формирования ответа продолжается** с использованием исходного тела сообщения и вложений.

## Обзор конфигурации

`tools.media` поддерживает **общие модели** и переопределения для каждой возможности:

- `tools.media.models`: общий список моделей (используйте `capabilities` для ограничения).
- `tools.media.image` / `tools.media.audio` / `tools.media.video`:
  - значения по умолчанию (`prompt`, `maxChars`, `maxBytes`, `timeoutSeconds`, `language`);
  - переопределения для провайдера (`baseUrl`, `headers`, `providerOptions`);
  - опции аудио Deepgram через `tools.media.audio.providerOptions.deepgram`;
  - управление выводом транскрипции аудио (`echoTranscript`, по умолчанию `false`; `echoFormat`);
  - необязательный **список моделей для каждой возможности** (имеет приоритет перед общими моделями);
  - политика обработки вложений (`attachments`: `mode`, `maxAttachments`, `prefer`);
  - `scope` (опциональное ограничение по каналу/типу чата/ключу сессии).
- `tools.media.concurrency`: максимальное количество одновременно выполняемых операций (по умолчанию **2**).

```json5
{
  tools: {
    media: {
      models: [
        /* общий список */
      ],
      image: {
        /* необязательные переопределения */
      },
      audio: {
        /* необязательные переопределения */
        echoTranscript: true,
        echoFormat: '📝 "{transcript}"',
      },
      video: {
        /* необязательные переопределения */
      },
    },
  },
}
```

### Записи моделей

Каждая запись в `models[]` может быть **провайдером** или **CLI**:

```json5
{
  type: "provider", // по умолчанию, если не указано
  provider: "openai",
  model: "gpt-5.4-mini",
  prompt: "Опишите изображение не более чем в 500 символах.",
  maxChars: 500,
  maxBytes: 10485760,
  timeoutSeconds: 60,
  capabilities: ["image"], // необязательно, используется для мультимодальных записей
  profile: "vision-profile",
  preferredProfile: "vision-fallback",
}
```

```json5
{
  type: "cli",
  command: "gemini",
  args: [
    "-m",
    "gemini-3-flash",
    "--allowed-tools",
    "read_file",
    "Прочитайте медиафайл по пути {{MediaPath}} и опишите его не более чем в {{MaxChars}} символах.",
  ],
  maxChars: 500,
  maxBytes: 52428800,
  timeoutSeconds: 120,
  capabilities: ["video", "image"],
}
```

Шаблоны CLI также могут использовать:

- `{{MediaDir}}` (каталог, содержащий медиафайл);
- `{{OutputDir}}` (временный каталог, созданный для этого запуска);
- `{{OutputBase}}` (базовый путь к временному файлу без расширения).

## Значения по умолчанию и ограничения

Рекомендуемые значения по умолчанию:

- `maxChars`: **500** для изображений/видео (кратко, удобно для команд);
- `maxChars`: **не задано** для аудио (полная транскрипция, если не установлен лимит);
- `maxBytes`:
  - изображение: **10 МБ**;
  - аудио: **20 МБ**;
  - видео: **50 МБ**.

Правила:

- Если размер медиаконтента превышает `maxBytes`, эта модель пропускается, и **пробуется следующая модель**.
- Аудиофайлы размером менее **1024 байт** считаются пустыми/повреждёнными и пропускаются до транскрипции через провайдера/CLI.
- Если модель возвращает текст длиннее `maxChars`, вывод обрезается.
- `prompt` по умолчанию — простое "Опишите {медиаконтент}." с указанием `maxChars` (только для изображений/видео).
- Если активная основная модель для изображений уже поддерживает зрение нативно, OpenClaw **пропускает блок сводки `[Image]`** и передаёт исходное изображение модели.
- Если `<capability>.enabled: true`, но модели не настроены, OpenClaw пробует **активную модель ответа**, если её провайдер поддерживает эту возможность.

### Автоматическое определение анализа медиаконтента (по умолчанию)

Если `tools.media.<capability>.enabled` **не** установлено в `false` и модели не настроены, OpenClaw автоматически определяет подходящий вариант в следующем порядке и **останавливается на первом рабочем варианте**:

1. **Активная модель ответа**, если её провайдер поддерживает эту возможность.
2. **`agents.defaults.imageModel`** (основная/резервная ссылка, только для изображений).
3. **Локальные CLI** (только для аудио, если установлены):
   - `sherpa-onnx-offline` (требуется `SHERPA_ONNX_MODEL_DIR` с encoder/decoder/joiner/tokens);
   - `whisper-cli` (`whisper-cpp`; использует `WHISPER_CPP_MODEL` или встроенный компактный модель);
   - `whisper` (CLI на Python; автоматически загружает модели).
4. **Gemini CLI** (`gemini`) с использованием `read_many_files`.
5. **Аутентификация провайдера**:
   - Настроенные записи `models.providers.*`, поддерживающие эту возможность, пробуются до встроенного порядка резервных механизмов.
   - Провайдеры конфигурации только для изображений с моделью, поддерживающей изображения, автоматически регистрируются для анализа медиаконтента, даже если они не являются встроенным плагином провайдера.
   - Встроенный порядок резервных механизмов:
     - Аудио: OpenAI → Groq → Deepgram → Google → Mistral;
     - Изображения: OpenAI → Anthropic → Google → MiniMax → MiniMax Portal → Z.AI;
     - Видео: Google → Qwen → Moonshot.

Чтобы отключить автоматическое определение, установите:

```json5
{
  tools: {
    media: {
      audio: {
        enabled: false,
      },
    },
  },
}
```

Примечание: определение бинарных файлов выполняется с наилучшими усилиями на macOS/Linux/Windows; убедитесь, что CLI находится в `PATH` (мы расширяем `~`), или укажите явную модель CLI с полным путём к команде.

### Поддержка прокси-окружения (модели провайдера)

Когда включён анализ аудио и видео медиаконтента на основе провайдера, OpenClaw учитывает стандартные переменные окружения прокси для HTTP-запросов к провайдерам:

- `HTTPS_PROXY`;
- `HTTP_PROXY`;
- `https_proxy`;
- `http_proxy`.

Если переменные окружения прокси не заданы, анализ медиаконтента использует прямое подключение.
Если значение прокси некорректно, OpenClaw выводит предупреждение и переходит к прямому подключению
