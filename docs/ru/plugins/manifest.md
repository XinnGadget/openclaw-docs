# Манифест плагина (openclaw.plugin.json)

Эта страница посвящена **нативному манифесту плагина OpenClaw**.

О совместимых форматах пакетов см. [Пакеты плагинов](/plugins/bundles).

Совместимые форматы пакетов используют разные файлы манифеста:

- Пакет Codex: `.codex-plugin/plugin.json`
- Пакет Claude: `.claude-plugin/plugin.json` или макет компонента Claude по умолчанию без манифеста
- Пакет Cursor: `.cursor-plugin/plugin.json`

OpenClaw также автоматически определяет эти форматы пакетов, но они не проверяются на соответствие схеме `openclaw.plugin.json`, описанной здесь.

Для совместимых пакетов OpenClaw в настоящее время считывает метаданные пакета, а также объявленные корневые элементы навыков, корневые элементы команд Claude, значения по умолчанию для `settings.json` в пакете Claude, значения по умолчанию LSP в пакете Claude и поддерживаемые пакеты хуков, если макет соответствует ожиданиям среды выполнения OpenClaw.

Каждый нативный плагин OpenClaw **должен** содержать файл `openclaw.plugin.json` в **корне плагина**. OpenClaw использует этот манифест для проверки конфигурации **без выполнения кода плагина**. Отсутствие манифеста или его некорректность расцениваются как ошибка плагина и препятствуют проверке конфигурации.

См. полное руководство по системе плагинов: [Плагины](/tools/plugin).
О модели возможностей и текущих рекомендациях по внешней совместимости: [Модель возможностей](/plugins/architecture#public-capability-model).

## Для чего предназначен этот файл

`openclaw.plugin.json` — это метаданные, которые OpenClaw считывает до загрузки кода плагина.

Используйте его для:

- идентификации плагина;
- проверки конфигурации;
- метаданных аутентификации и начальной настройки, которые должны быть доступны без запуска среды выполнения плагина;
- метаданных псевдонимов и автоматического включения, которые должны разрешаться до загрузки среды выполнения плагина;
- кратких метаданных о принадлежности к семейству моделей, которые должны автоматически активировать плагин до загрузки среды выполнения;
- статических снимков принадлежности к возможностям, используемых для настройки совместимости пакетов и покрытия контрактов;
- метаданных конфигурации для конкретных каналов, которые должны объединяться с каталогом и поверхностями проверки без загрузки среды выполнения;
- подсказок для интерфейса конфигурации.

Не используйте его для:

- регистрации поведения во время выполнения;
- объявления точек входа кода;
- метаданных установки npm.

Они должны находиться в коде плагина и в `package.json`.

## Минимальный пример

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

## Расширенный пример

```json
{
  "id": "openrouter",
  "name": "OpenRouter",
  "description": "Плагин провайдера OpenRouter",
  "version": "1.0.0",
  "providers": ["openrouter"],
  "modelSupport": {
    "modelPrefixes": ["router-"]
  },
  "cliBackends": ["openrouter-cli"],
  "providerAuthEnvVars": {
    "openrouter": ["OPENROUTER_API_KEY"]
  },
  "providerAuthAliases": {
    "openrouter-coding": "openrouter"
  },
  "channelEnvVars": {
    "openrouter-chatops": ["OPENROUTER_CHATOPS_TOKEN"]
  },
  "providerAuthChoices": [
    {
      "provider": "openrouter",
      "method": "api-key",
      "choiceId": "openrouter-api-key",
      "choiceLabel": "Ключ API OpenRouter",
      "groupId": "openrouter",
      "groupLabel": "OpenRouter",
      "optionKey": "openrouterApiKey",
      "cliFlag": "--openrouter-api-key",
      "cliOption": "--openrouter-api-key <key>",
      "cliDescription": "Ключ API OpenRouter",
      "onboardingScopes": ["text-inference"]
    }
  ],
  "uiHints": {
    "apiKey": {
      "label": "Ключ API",
      "placeholder": "sk-or-v1-...",
      "sensitive": true
    }
  },
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "apiKey": {
        "type": "string"
      }
    }
  }
}
```

## Справочник полей верхнего уровня

| Поле | Обязательно | Тип | Значение |
| --- | --- | --- | --- |
| `id` | Да | `string` | Канонический идентификатор плагина. Используется в `plugins.entries.<id>`. |
| `configSchema` | Да | `object` | Встроенная JSON-схема для конфигурации этого плагина. |
| `enabledByDefault` | Нет | `true` | Помечает встроенный плагин как включённый по умолчанию. Чтобы оставить плагин отключённым по умолчанию, опустите это поле или задайте любое значение, отличное от `true`. |
| `legacyPluginIds` | Нет | `string[]` | Устаревшие идентификаторы, которые нормализуются до этого канонического идентификатора плагина. |
| `autoEnableWhenConfiguredProviders` | Нет | `string[]` | Идентификаторы провайдеров, при упоминании которых в аутентификации, конфигурации или ссылках на модели плагин должен автоматически включаться. |
| `kind` | Нет | `"memory"` \| `"context-engine"` | Объявляет эксклюзивный тип плагина, используемый в `plugins.slots.*`. |
| `channels` | Нет | `string[]` | Идентификаторы каналов, принадлежащих этому плагину. Используются для обнаружения и проверки конфигурации. |
| `providers` | Нет | `string[]` | Идентификаторы провайдеров, принадлежащих этому плагину. |
| `modelSupport` | Нет | `object` | Краткие метаданные семейства моделей, принадлежащие манифесту, используемые для автоматической загрузки плагина до запуска среды выполнения. |
| `cliBackends` | Нет | `string[]` | Идентификаторы бэкендов вывода CLI, принадлежащих этому плагину. Используются для автоматической активации при запуске из явных ссылок на конфигурацию. |
| `providerAuthEnvVars` | Нет | `Record<string, string[]>` | Простые метаданные среды аутентификации провайдера, которые OpenClaw может проверить без загрузки кода плагина. |
| `providerAuthAliases` | Нет | `Record<string, string>` | Идентификаторы провайдеров, которые должны использовать другой идентификатор провайдера для поиска аутентификации, например, провайдер кодирования, который использует базовый ключ API провайдера и профили аутентификации. |
| `channelEnvVars` | Нет | `Record<string, string[]>` | Простые метаданные среды канала, которые OpenClaw может проверить без загрузки кода плагина. Используйте их для настройки канала на основе среды или поверхностей аутентификации, которые должны видеть общие помощники запуска/конфигурации. |
| `providerAuthChoices` | Нет | `object[]` | Простые метаданные выбора аутентификации для средств выбора начальной настройки, разрешения предпочтительного провайдера и простой настройки флагов CLI. |
| `contracts` | Нет | `object` | Статический снимок возможностей пакета для речи, транскрипции в реальном времени, голоса в реальном времени, понимания медиа, генерации изображений, генерации музыки, генерации видео, извлечения данных из веб-страниц, веб-поиска и владения инструментами. |
| `channelConfigs` | Нет | `Record<string, object>` | Метаданные конфигурации канала, принадлежащие манифесту, объединяются с поверхностями обнаружения и проверки до загрузки среды выполнения. |
| `skills` | Нет | `string[]` | Каталоги навыков для загрузки относительно корня плагина. |
| `name` | Нет | `string` | Человекочитаемое название плагина. |
| `description` | Нет | `string` | Краткое описание, отображаемое на поверхностях плагина. |
| `version` | Нет | `string` | Информационная версия плагина. |
| `uiHints` | Нет | `Record<string, object>` | Метки интерфейса, плейсхолдеры и подсказки о конфиденциальности для полей конфигурации. |

## Справочник providerAuthChoices

Каждая запись `providerAuthChoices` описывает один вариант начальной настройки или аутентификации. OpenClaw считывает это до загрузки среды выполнения провайдера.

| Поле | Обязательно | Тип | Значение |
| --- | --- | --- | --- |
| `provider` | Да | `string` | Идентификатор провайдера, к которому относится этот выбор. |
| `method` | Да | `string` | Идентификатор метода аутентификации для диспетчеризации. |
| `choiceId` | Да | `string` | Стабильный идентификатор выбора аутентификации, используемый в процессах начальной настройки и CLI. |
