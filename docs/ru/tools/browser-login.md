# Вход в браузер + публикация в X/Twitter

## Ручной вход (рекомендуется)

Если сайт требует авторизации, **войдите вручную** в профиле браузера **хоста** (браузере OpenClaw).

**Не** передавайте модели свои учётные данные. Автоматизированный вход часто активирует защиту от ботов и может привести к блокировке аккаунта.

Вернуться к основной документации по браузеру: [Browser](/tools/browser).

## Какой профиль Chrome используется?

OpenClaw управляет **выделенным профилем Chrome** (с именем `openclaw`, интерфейс с оранжевым оттенком). Он отделён от вашего повседневного профиля браузера.

Для вызовов инструмента браузера агентом:

- Вариант по умолчанию: агент должен использовать изолированный браузер `openclaw`.
- Используйте `profile="user"` только в тех случаях, когда важны существующие авторизованные сессии и пользователь находится за компьютером, чтобы подтвердить любой запрос на подключение.
- Если у вас несколько пользовательских профилей браузера, явно укажите профиль вместо того, чтобы полагаться на автоматическое определение.

Есть два простых способа получить к нему доступ:

1. **Попросите агента открыть браузер**, а затем войдите в систему самостоятельно.
2. **Откройте его через CLI**:

```bash
openclaw browser start
openclaw browser open https://x.com
```

Если у вас несколько профилей, передайте параметр `--browser-profile <name>` (по умолчанию используется `openclaw`).

## X/Twitter: рекомендуемый порядок действий

- **Чтение/поиск/ветки обсуждений:** используйте браузер **хоста** (ручной вход).
- **Публикация обновлений:** используйте браузер **хоста** (ручной вход).

## Песочница + доступ к браузеру хоста

Сессии браузера в песочнице **с большей вероятностью** активируют обнаружение ботов. Для X/Twitter (и других сайтов с жёсткими ограничениями) предпочтительнее использовать браузер **хоста**.

Если агент работает в песочнице, инструмент браузера по умолчанию использует песочницу. Чтобы разрешить управление хостом:

```json5
{
  agents: {
    defaults: {
      sandbox: {
        mode: "non-main",
        browser: {
          allowHostControl: true,
        },
      },
    },
  },
}
```

Затем нацельтесь на браузер хоста:

```bash
openclaw browser open https://x.com --browser-profile openclaw --target host
```

Либо отключите песочницу для агента, который публикует обновления.
