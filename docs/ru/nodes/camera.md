# Захват изображений с камеры (для агента)

OpenClaw поддерживает **захват изображений с камеры** для рабочих процессов агента:

- **Узел iOS** (подключён через Gateway): захват **фотографии** (`jpg`) или **короткого видеоклипа** (`mp4`, с необязательной аудиодорожкой) через `node.invoke`.
- **Узел Android** (подключён через Gateway): захват **фотографии** (`jpg`) или **короткого видеоклипа** (`mp4`, с необязательной аудиодорожкой) через `node.invoke`.
- **Приложение для macOS** (узел через Gateway): захват **фотографии** (`jpg`) или **короткого видеоклипа** (`mp4`, с необязательной аудиодорожкой) через `node.invoke`.

Весь доступ к камере контролируется через **настройки, управляемые пользователем**.

## Узел iOS

### Настройка пользователя (по умолчанию включена)

- Вкладка "Настройки iOS" → **Камера** → **Разрешить использование камеры** (`camera.enabled`):
  - По умолчанию: **включено** (отсутствие ключа трактуется как включённое состояние).
  - Если отключено: команды `camera.*` возвращают `CAMERA_DISABLED`.

### Команды (через Gateway `node.invoke`)

- `camera.list`
  - Полезная нагрузка ответа:
    - `devices`: массив объектов `{ id, name, position, deviceType }`.

- `camera.snap`
  - Параметры:
    - `facing`: `front|back` (по умолчанию: `front`).
    - `maxWidth`: число (необязательно; по умолчанию `1600` на узле iOS).
    - `quality`: `0..1` (необязательно; по умолчанию `0.9`).
    - `format`: в настоящее время `jpg`.
    - `delayMs`: число (необязательно; по умолчанию `0`).
    - `deviceId`: строка (необязательно; из `camera.list`).
  - Полезная нагрузка ответа:
    - `format: "jpg"`.
    - `base64: "<...>"`.
    - `width`, `height`.
  - Защита полезной нагрузки: фотографии пережимаются, чтобы размер base64-полезной нагрузки не превышал 5 МБ.

- `camera.clip`
  - Параметры:
    - `facing`: `front|back` (по умолчанию: `front`).
    - `durationMs`: число (по умолчанию `3000`, ограничено максимумом в `60000`).
    - `includeAudio`: булево значение (по умолчанию `true`).
    - `format`: в настоящее время `mp4`.
    - `deviceId`: строка (необязательно; из `camera.list`).
  - Полезная нагрузка ответа:
    - `format: "mp4"`.
    - `base64: "<...>"`.
    - `durationMs`.
    - `hasAudio`.

### Требование нахождения в переднем плане

Как и в случае с `canvas.*`, узел iOS разрешает команды `camera.*` только **в переднем плане**. Вызовы в фоновом режиме возвращают `NODE_BACKGROUND_UNAVAILABLE`.

### Помощник CLI (временные файлы + MEDIA)

Самый простой способ получить вложения — использовать помощник CLI, который записывает декодированные медиафайлы во временный файл и выводит `MEDIA:<path>`.

Примеры:

```bash
openclaw nodes camera snap --node <id>               # по умолчанию: обе камеры (передняя и задняя) (2 строки MEDIA)
openclaw nodes camera snap --node <id> --facing front
openclaw nodes camera clip --node <id> --duration 3000
openclaw nodes camera clip --node <id> --no-audio
```

Примечания:

- `nodes camera snap` по умолчанию использует **обе** камеры, чтобы предоставить агенту оба вида.
- Выходные файлы являются временными (в каталоге временных файлов ОС), если вы не создадите собственную оболочку.

## Узел Android

### Настройка пользователя Android (по умолчанию включена)

- Лист настроек Android → **Камера** → **Разрешить использование камеры** (`camera.enabled`):
  - По умолчанию: **включено** (отсутствие ключа трактуется как включённое состояние).
  - Если отключено: команды `camera.*` возвращают `CAMERA_DISABLED`.

### Разрешения

- Для Android требуются разрешения во время выполнения:
  - `CAMERA` для `camera.snap` и `camera.clip`.
  - `RECORD_AUDIO` для `camera.clip`, когда `includeAudio=true`.

Если разрешения отсутствуют, приложение запросит их при возможности; если они отклонены, запросы `camera.*` завершатся ошибкой `*_PERMISSION_REQUIRED`.

### Требование нахождения в переднем плане на Android

Как и в случае с `canvas.*`, узел Android разрешает команды `camera.*` только **в переднем плане**. Вызовы в фоновом режиме возвращают `NODE_BACKGROUND_UNAVAILABLE`.

### Команды Android (через Gateway `node.invoke`)

- `camera.list`
  - Полезная нагрузка ответа:
    - `devices`: массив объектов `{ id, name, position, deviceType }`.

### Защита полезной нагрузки

Фотографии пережимаются, чтобы размер base64-полезной нагрузки не превышал 5 МБ.

## Приложение для macOS

### Настройка пользователя (по умолчанию отключена)

В сопутствующем приложении для macOS есть флажок:

- **Настройки → Общие → Разрешить использование камеры** (`openclaw.cameraEnabled`):
  - По умолчанию: **отключено**.
  - Если отключено: запросы к камере возвращают сообщение "Камера отключена пользователем".

### Помощник CLI (вызов узла)

Используйте основной CLI `openclaw` для вызова команд камеры на узле macOS.

Примеры:

```bash
openclaw nodes camera list --node <id>            # вывести список идентификаторов камер
openclaw nodes camera snap --node <id>            # выводит MEDIA:<path>
openclaw nodes camera snap --node <id> --max-width 1280
openclaw nodes camera snap --node <id> --delay-ms 2000
openclaw nodes camera snap --node <id> --device-id <id>
openclaw nodes camera clip --node <id> --duration 10s          # выводит MEDIA:<path>
openclaw nodes camera clip --node <id> --duration-ms 3000      # выводит MEDIA:<path> (устаревший флаг)
openclaw nodes camera clip --node <id> --device-id <id>
openclaw nodes camera clip --node <id> --no-audio
```

Примечания:

- `openclaw nodes camera snap` по умолчанию использует `maxWidth=1600`, если не указано иное.
- На macOS `camera.snap` ожидает `delayMs` (по умолчанию 2000 мс) после прогрева/настройки экспозиции перед съёмкой.
- Полезные нагрузки фотографий пережимаются, чтобы размер base64 не превышал 5 МБ.

## Безопасность и практические ограничения

- Доступ к камере и микрофону вызывает стандартные системные запросы разрешений (и требует указания строк использования в Info.plist).
- Длительность видеоклипов ограничена (в настоящее время `<= 60 с`), чтобы избежать слишком больших полезных нагрузок узла (накладные расходы base64 + ограничения на размер сообщений).

## Видео с экрана macOS (на уровне ОС)

Для записи **видео с экрана** (не с камеры) используйте сопутствующее приложение для macOS:

```bash
openclaw nodes screen record --node <id> --duration 10s --fps 15   # выводит MEDIA:<path>
```

Примечания:

- Требуется разрешение macOS на **запись экрана** (TCC).
