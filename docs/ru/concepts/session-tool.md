# Инструменты работы с сессиями

OpenClaw предоставляет агентам инструменты для работы между сессиями, проверки состояния и оркестрации суб-агентов.

## Доступные инструменты

| Инструмент | Что делает |
| --- | --- |
| `sessions_list` | Выводит список сессий с возможностью применения фильтров (по типу, давности) |
| `sessions_history` | Читает расшифровку конкретной сессии |
| `sessions_send` | Отправляет сообщение в другую сессию и при необходимости ожидает ответа |
| `sessions_spawn` | Создаёт изолированную сессию суб-агента для фоновой работы |
| `sessions_yield` | Завершает текущий ход и ожидает результатов от суб-агента |
| `subagents` | Выводит список, управляет или завершает созданные суб-агенты для текущей сессии |
| `session_status` | Показывает карточку в стиле `/status` и при необходимости устанавливает переопределение модели для сессии |

## Вывод списка и чтение сессий

`sessions_list` возвращает сессии с их ключом, типом, каналом, моделью, количеством токенов и временными метками. Можно фильтровать по типу (`main`, `group`, `cron`, `hook`, `node`) или давности (`activeMinutes`).

`sessions_history` извлекает расшифровку диалога для конкретной сессии. По умолчанию результаты работы инструментов не включаются — чтобы их увидеть, нужно передать параметр `includeTools: true`. Возвращаемый результат намеренно ограничен и проходит фильтрацию с точки зрения безопасности:

- текст ассистента нормализуется перед извлечением:
  - удаляются теги мышления;
  - удаляются блоки каркаса `<relevant-memories>` / `<relevant_memories>`;
  - удаляются блоки XML-полезной нагрузки вызовов инструментов в виде обычного текста, такие как `<tool_call>...</tool_call>`, `<function_call>...</function_call>`, `<tool_calls>...</tool_calls>` и `<function_calls>...</function_calls>`, включая усечённые полезные нагрузки, которые никогда не закрываются корректно;
  - удаляются упрощённые каркасы вызовов/результатов инструментов, такие как `[Tool Call: ...]`, `[Tool Result ...]` и `[Historical context ...]`;
  - удаляются просочившиеся токены управления моделью, такие как `<|assistant|>`, другие ASCII-токены `<|...|>` и полноширинные варианты `<｜...｜>`;
  - удаляются некорректные XML-вызовы инструментов MiniMax, такие как `<invoke ...>` / `</minimax:tool_call>`;
- текст, похожий на учётные данные/токены, редактируется перед возвратом;
- длинные текстовые блоки усекаются;
- при очень больших объёмах истории могут быть удалены более старые строки или заменена чрезмерно большая строка на `[sessions_history omitted: message too large]`;
- инструмент сообщает флаги сводки, такие как `truncated`, `droppedMessages`, `contentTruncated`, `contentRedacted` и `bytes`.

Оба инструмента принимают либо **ключ сессии** (например, `"main"`), либо **ID сессии** из предыдущего вызова списка.

Если вам нужна точная побайтовая расшифровка, проверьте файл расшифровки на диске, а не рассматривайте `sessions_history` как необработанный дамп.

## Отправка сообщений между сессиями

`sessions_send` доставляет сообщение в другую сессию и при необходимости ожидает ответа:

- **Отправка без ожидания ответа:** установите `timeoutSeconds: 0`, чтобы поставить в очередь и немедленно вернуться.
- **Ожидание ответа:** установите тайм-аут и получите ответ встроено.

После того как целевая сессия ответит, OpenClaw может запустить **цикл обратной связи**, в котором агенты обмениваются сообщениями (до 5 ходов). Целевой агент может отправить `REPLY_SKIP`, чтобы завершить цикл досрочно.

## Вспомогательные инструменты для состояния и оркестрации

`session_status` — это облегчённый инструмент, эквивалентный `/status`, для текущей или другой видимой сессии. Он сообщает об использовании, времени, состоянии модели/среды выполнения и связанном контексте фоновой задачи, если он есть. Как и `/status`, он может заполнять разреженные счётчики токенов/кэша из последней записи об использовании расшифровки, а `model=default` сбрасывает переопределение для сессии.

`sessions_yield` намеренно завершает текущий ход, чтобы следующее сообщение могло быть ожидаемым последующим событием. Используйте его после создания суб-агентов, если хотите, чтобы результаты завершения приходили в виде следующего сообщения, а не создавали циклы опроса.

`subagents` — это вспомогательный инструмент для управления уже созданными суб-агентами OpenClaw. Он поддерживает:

- `action: "list"` — для просмотра активных/недавних запусков;
- `action: "steer"` — для отправки дополнительных указаний работающему дочернему агенту;
- `action: "kill"` — для остановки одного дочернего агента или всех (`all`).

## Создание суб-агентов

`sessions_spawn` создаёт изолированную сессию для фоновой задачи. Она всегда неблокирующая — немедленно возвращается с `runId` и `childSessionKey`.

Ключевые параметры:

- `runtime: "subagent"` (по умолчанию) или `"acp"` для внешних агентов с обвязкой.
- `model` и `thinking` — переопределения для дочерней сессии.
- `thread: true` — привязка создания к потоку чата (Discord, Slack и т. д.).
- `sandbox: "require"` — принудительное включение песочницы для дочерней сессии.

Суб-агенты конечного уровня по умолчанию не получают инструменты работы с сессиями. Когда `maxSpawnDepth >= 2`, суб-агенты-оркестраторы уровня 1 дополнительно получают `sessions_spawn`, `subagents`, `sessions_list` и `sessions_history`, чтобы они могли управлять своими дочерними агентами. Суб-агенты конечного уровня по-прежнему не получают инструменты рекурсивной оркестрации.

После завершения шаг объявления отправляет результат в канал запрашивающего. Доставка результата сохраняет привязку к потоку/теме, если это возможно. Если источник завершения идентифицирует только канал, OpenClaw всё равно может использовать сохранённый маршрут сессии запрашивающего (`lastChannel` / `lastTo`) для прямой доставки.

Для поведения, специфичного для ACP, см. [ACP Agents](/tools/acp-agents).

## Видимость

Инструменты работы с сессиями ограничены по области видимости, чтобы ограничить то, что может видеть агент:

| Уровень | Область видимости |
| --- | --- |
| `self` | Только текущая сессия |
| `tree` | Текущая сессия + созданные суб-агенты |
| `agent` | Все сессии для этого агента |
| `all` | Все сессии (между агентами, если настроено) |

По умолчанию используется `tree`. Сессии в песочнице ограничены `tree` независимо от конфигурации.
