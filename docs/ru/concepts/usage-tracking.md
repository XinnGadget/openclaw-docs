# Отслеживание использования

## Что это такое

- Получает данные об использовании/квоте провайдера напрямую из их конечных точек отслеживания использования.
- Нет расчётной стоимости; используются только окна, предоставленные провайдером.
- Вывод статуса в удобочитаемом виде нормализуется до формата "X % осталось", даже если вышестоящий API сообщает об израсходованной квоте, оставшейся квоте или только о необработанных счётчиках.
- Команды `/status` и `session_status` на уровне сессии могут использовать последнюю запись об использовании в транскрипте, если моментальный снимок текущей сессии неполный. Этот запасной вариант заполняет отсутствующие счётчики токенов/кэша, может восстановить метку активной модели среды выполнения и отдаёт предпочтение общему количеству, ориентированному на промпт, если метаданные сессии отсутствуют или меньше. Существующие ненулевые текущие значения по-прежнему имеют приоритет.

## Где отображается

- `/status` в чатах: карточка статуса с эмодзи, содержащая токены сессии + расчётная стоимость (только для API-ключа). Использование провайдера отображается для **текущего провайдера модели**, если доступно, в виде нормализованного окна "X % осталось".
- `/usage off|tokens|full` в чатах: подвал с данными об использовании для каждого ответа (при использовании OAuth отображаются только токены).
- `/usage cost` в чатах: сводка локальной стоимости, агрегированная из журналов сессий OpenClaw.
- CLI: `openclaw status --usage` выводит полную разбивку по провайдерам.
- CLI: `openclaw channels list` выводит тот же моментальный снимок использования вместе с конфигурацией провайдера (используйте `--no-usage`, чтобы пропустить).
- Строка меню macOS: раздел "Использование" в "Контексте" (если доступен).

## Провайдеры и учётные данные

- **Anthropic (Claude)**: токены OAuth в профилях аутентификации.
- **GitHub Copilot**: токены OAuth в профилях аутентификации.
- **Gemini CLI**: токены OAuth в профилях аутентификации.
  - Данные об использовании в формате JSON возвращаются к `stats`; `stats.cached` нормализуется до `cacheRead`.
- **OpenAI Codex**: токены OAuth в профилях аутентификации (при наличии используется accountId).
- **MiniMax**: API-ключ или профиль аутентификации MiniMax OAuth. OpenClaw рассматривает `minimax`, `minimax-cn` и `minimax-portal` как один и тот же интерфейс квоты MiniMax, отдаёт предпочтение сохранённому MiniMax OAuth при наличии, а в противном случае возвращается к `MINIMAX_CODE_PLAN_KEY`, `MINIMAX_CODING_API_KEY` или `MINIMAX_API_KEY`. Поля `usage_percent` / `usagePercent` в MiniMax означают **оставшуюся** квоту, поэтому OpenClaw инвертирует их перед отображением; поля, основанные на подсчёте, имеют приоритет при наличии.
  - Метки окна плана кодирования берутся из полей часов/минут провайдера при наличии, затем возвращаются к диапазону `start_time` / `end_time`.
  - Если конечная точка плана кодирования возвращает `model_remains`, OpenClaw отдаёт предпочтение записи о чат-модели, выводит метку окна из временных меток, если явные поля `window_hours` / `window_minutes` отсутствуют, и включает название модели в метку плана.
- **Xiaomi MiMo**: API-ключ через env/config/auth store (`XIAOMI_API_KEY`).
- **z.ai**: API-ключ через env/config/auth store.

Использование скрыто, если не удаётся разрешить пригодные для использования учётные данные провайдера. Провайдеры могут предоставлять логику аутентификации использования, специфичную для плагина; в противном случае OpenClaw возвращается к сопоставлению учётных данных OAuth/API-ключа из профилей аутентификации, переменных окружения или конфигурации.
