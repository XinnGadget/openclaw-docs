# بيان الإضافة (openclaw.plugin.json)

هذه الصفحة مخصّصة فقط لـ **بيان الإضافة الأصلي في OpenClaw**.

للاطلاع على تخطيطات الحزم المتوافقة، راجع [حزم الإضافات](/ar/plugins/bundles).

تستخدم تنسيقات الحزم المتوافقة ملفات بيان مختلفة:

- حزمة Codex: `.codex-plugin/plugin.json`
- حزمة Claude: `.claude-plugin/plugin.json` أو تخطيط مكوّن Claude الافتراضي
  بدون بيان
- حزمة Cursor: `.cursor-plugin/plugin.json`

يكتشف OpenClaw هذه التخطيطات المتوافقة للحزم تلقائيًا أيضًا، لكنه لا يتحقق منها
مقابل schema ‏`openclaw.plugin.json` الموصوف هنا.

بالنسبة إلى الحزم المتوافقة، يقرأ OpenClaw حاليًا بيانات تعريف الحزمة بالإضافة إلى
جذور Skills المعلنة، وجذور أوامر Claude، وقيم Claude bundle ‏`settings.json` الافتراضية،
وقيم Claude bundle ‏LSP الافتراضية، وحزم الخطافات المدعومة عندما يطابق التخطيط
توقعات وقت تشغيل OpenClaw.

يجب أن تشحن كل إضافة أصلية في OpenClaw **ملف `openclaw.plugin.json`** داخل
**جذر الإضافة**. يستخدم OpenClaw هذا البيان للتحقق من الإعدادات
**من دون تنفيذ كود الإضافة**. وتُعامل البيانات المفقودة أو غير الصالحة على أنها
أخطاء في الإضافة وتمنع التحقق من الإعدادات.

راجع الدليل الكامل لنظام الإضافات: [الإضافات](/ar/tools/plugin).
وللاطلاع على نموذج القدرات الأصلي وإرشادات التوافق الخارجي الحالية:
[نموذج القدرات](/ar/plugins/architecture#public-capability-model).

## ما الذي يفعله هذا الملف

`openclaw.plugin.json` هو بيانات التعريف التي يقرأها OpenClaw قبل أن يحمّل
كود الإضافة.

استخدمه من أجل:

- هوية الإضافة
- التحقق من الإعدادات
- بيانات تعريف المصادقة والتهيئة التي يجب أن تكون متاحة من دون تشغيل
  وقت تشغيل الإضافة
- بيانات تعريف الأسماء المستعارة والتمكين التلقائي التي يجب أن تُحل قبل تحميل وقت تشغيل الإضافة
- بيانات تعريف مختصرة مملوكة لعائلة النموذج يجب أن تفعّل
  الإضافة تلقائيًا قبل تحميل وقت التشغيل
- لقطات ثابتة لملكية القدرات تُستخدم في توصيل التوافق المضمّن
  وتغطية العقود
- بيانات تعريف إعدادات خاصة بالقناة يجب أن تُدمج في أسطح الفهرسة والتحقق
  من دون تحميل وقت التشغيل
- تلميحات واجهة المستخدم للإعدادات

لا تستخدمه من أجل:

- تسجيل سلوك وقت التشغيل
- التصريح بنقاط دخول الكود
- بيانات تعريف تثبيت npm

فهذه تنتمي إلى كود الإضافة الخاص بك وإلى `package.json`.

## مثال بسيط

```json
{
  "id": "voice-call",
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {}
  }
}
```

## مثال غني

```json
{
  "id": "openrouter",
  "name": "OpenRouter",
  "description": "إضافة مزوّد OpenRouter",
  "version": "1.0.0",
  "providers": ["openrouter"],
  "modelSupport": {
    "modelPrefixes": ["router-"]
  },
  "cliBackends": ["openrouter-cli"],
  "providerAuthEnvVars": {
    "openrouter": ["OPENROUTER_API_KEY"]
  },
  "channelEnvVars": {
    "openrouter-chatops": ["OPENROUTER_CHATOPS_TOKEN"]
  },
  "providerAuthChoices": [
    {
      "provider": "openrouter",
      "method": "api-key",
      "choiceId": "openrouter-api-key",
      "choiceLabel": "مفتاح API لـ OpenRouter",
      "groupId": "openrouter",
      "groupLabel": "OpenRouter",
      "optionKey": "openrouterApiKey",
      "cliFlag": "--openrouter-api-key",
      "cliOption": "--openrouter-api-key <key>",
      "cliDescription": "مفتاح API لـ OpenRouter",
      "onboardingScopes": ["text-inference"]
    }
  ],
  "uiHints": {
    "apiKey": {
      "label": "مفتاح API",
      "placeholder": "sk-or-v1-...",
      "sensitive": true
    }
  },
  "configSchema": {
    "type": "object",
    "additionalProperties": false,
    "properties": {
      "apiKey": {
        "type": "string"
      }
    }
  }
}
```

## مرجع الحقول العليا

| الحقل                               | مطلوب | النوع                             | معناه                                                                                                                                                                                                |
| ----------------------------------- | ------ | --------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `id`                                | نعم    | `string`                          | معرّف الإضافة المعياري. هذا هو المعرّف المستخدم في `plugins.entries.<id>`.                                                                                                                          |
| `configSchema`                      | نعم    | `object`                          | JSON Schema مضمّنة لإعدادات هذه الإضافة.                                                                                                                                                             |
| `enabledByDefault`                  | لا     | `true`                            | يحدّد أن الإضافة المضمّنة مفعّلة افتراضيًا. احذف الحقل، أو عيّن أي قيمة غير `true`، لترك الإضافة معطلة افتراضيًا.                                                                                 |
| `legacyPluginIds`                   | لا     | `string[]`                        | معرّفات قديمة تُطبَّع إلى معرّف الإضافة المعياري هذا.                                                                                                                                               |
| `autoEnableWhenConfiguredProviders` | لا     | `string[]`                        | معرّفات المزوّدين التي يجب أن تفعّل هذه الإضافة تلقائيًا عندما تشير المصادقة أو الإعدادات أو مراجع النماذج إليها.                                                                                  |
| `kind`                              | لا     | `"memory"` \| `"context-engine"`  | يصرّح بنوع إضافة حصري يُستخدم بواسطة `plugins.slots.*`.                                                                                                                                             |
| `channels`                          | لا     | `string[]`                        | معرّفات القنوات التي تمتلكها هذه الإضافة. تُستخدم للاكتشاف والتحقق من الإعدادات.                                                                                                                    |
| `providers`                         | لا     | `string[]`                        | معرّفات المزوّدين التي تمتلكها هذه الإضافة.                                                                                                                                                          |
| `modelSupport`                      | لا     | `object`                          | بيانات تعريف مختصرة مملوكة للبيان لعائلة النموذج تُستخدم لتحميل الإضافة تلقائيًا قبل وقت التشغيل.                                                                                                  |
| `cliBackends`                       | لا     | `string[]`                        | معرّفات الواجهة الخلفية للاستدلال في CLI التي تمتلكها هذه الإضافة. تُستخدم للتفعيل التلقائي عند بدء التشغيل من مراجع الإعدادات الصريحة.                                                            |
| `providerAuthEnvVars`               | لا     | `Record<string, string[]>`        | بيانات تعريف خفيفة لمصادقة المزوّد يمكن لـ OpenClaw فحصها من دون تحميل كود الإضافة.                                                                                                                |
| `channelEnvVars`                    | لا     | `Record<string, string[]>`        | بيانات تعريف خفيفة لمتغيرات بيئة القنوات يمكن لـ OpenClaw فحصها من دون تحميل كود الإضافة. استخدم هذا لأسطح إعداد القنوات أو المصادقة المعتمدة على البيئة التي يجب أن تراها أدوات البدء/الإعداد العامة. |
| `providerAuthChoices`               | لا     | `object[]`                        | بيانات تعريف خفيفة لخيارات المصادقة لمنتقيات التهيئة، وحل المزوّد المفضّل، وتوصيل أعلام CLI البسيطة.                                                                                              |
| `contracts`                         | لا     | `object`                          | لقطة ثابتة لقدرات الحزمة المضمّنة للكلام، والنسخ الفوري، والصوت الفوري، وفهم الوسائط، وتوليد الصور، وتوليد الموسيقى، وتوليد الفيديو، وجلب الويب، والبحث في الويب، وملكية الأدوات.               |
| `channelConfigs`                    | لا     | `Record<string, object>`          | بيانات تعريف مملوكة للبيان لإعدادات القنوات تُدمج في أسطح الاكتشاف والتحقق قبل تحميل وقت التشغيل.                                                                                                  |
| `skills`                            | لا     | `string[]`                        | أدلة Skills المطلوب تحميلها، نسبةً إلى جذر الإضافة.                                                                                                                                                 |
| `name`                              | لا     | `string`                          | اسم الإضافة المقروء بشريًا.                                                                                                                                                                          |
| `description`                       | لا     | `string`                          | ملخص قصير يظهر في أسطح الإضافة.                                                                                                                                                                      |
| `version`                           | لا     | `string`                          | إصدار معلوماتي للإضافة.                                                                                                                                                                              |
| `uiHints`                           | لا     | `Record<string, object>`          | تسميات واجهة المستخدم، والعناصر النائبة، وتلميحات الحساسية لحقول الإعدادات.                                                                                                                        |

## مرجع `providerAuthChoices`

يصف كل إدخال في `providerAuthChoices` خيار تهيئة أو مصادقة واحدًا.
يقرأ OpenClaw هذا قبل تحميل وقت تشغيل المزوّد.

| الحقل                 | مطلوب | النوع                                            | معناه                                                                                            |
| --------------------- | ------ | ------------------------------------------------ | ------------------------------------------------------------------------------------------------ |
| `provider`            | نعم    | `string`                                         | معرّف المزوّد الذي ينتمي إليه هذا الخيار.                                                        |
| `method`              | نعم    | `string`                                         | معرّف طريقة المصادقة المطلوب التوجيه إليها.                                                      |
| `choiceId`            | نعم    | `string`                                         | معرّف ثابت لخيار المصادقة تستخدمه التهيئة وتدفقات CLI.                                           |
| `choiceLabel`         | لا     | `string`                                         | تسمية موجهة للمستخدم. إذا حُذفت، يعود OpenClaw إلى `choiceId`.                                   |
| `choiceHint`          | لا     | `string`                                         | نص مساعد قصير للمنتقي.                                                                           |
| `assistantPriority`   | لا     | `number`                                         | تُرتَّب القيم الأدنى أولًا في المنتقيات التفاعلية التي يقودها المساعد.                           |
| `assistantVisibility` | لا     | `"visible"` \| `"manual-only"`                   | إخفاء الخيار من منتقيات المساعد مع السماح بالاختيار اليدوي عبر CLI.                              |
| `deprecatedChoiceIds` | لا     | `string[]`                                       | معرّفات خيارات قديمة يجب أن تعيد توجيه المستخدمين إلى خيار الاستبدال هذا.                        |
| `groupId`             | لا     | `string`                                         | معرّف مجموعة اختياري لتجميع الخيارات ذات الصلة.                                                  |
| `groupLabel`          | لا     | `string`                                         | تسمية موجهة للمستخدم لتلك المجموعة.                                                              |
| `groupHint`           | لا     | `string`                                         | نص مساعد قصير للمجموعة.                                                                          |
| `optionKey`           | لا     | `string`                                         | مفتاح خيار داخلي لتدفقات المصادقة البسيطة ذات العلم الواحد.                                      |
| `cliFlag`             | لا     | `string`                                         | اسم علم CLI، مثل `--openrouter-api-key`.                                                         |
| `cliOption`           | لا     | `string`                                         | الصيغة الكاملة لخيار CLI، مثل `--openrouter-api-key <key>`.                                      |
| `cliDescription`      | لا     | `string`                                         | الوصف المستخدم في مساعدة CLI.                                                                    |
| `onboardingScopes`    | لا     | `Array<"text-inference" \| "image-generation">`  | أسطح التهيئة التي يجب أن يظهر فيها هذا الخيار. إذا حُذفت، فالقيمة الافتراضية هي `["text-inference"]`. |

## مرجع `uiHints`

`uiHints` هي خريطة من أسماء حقول الإعدادات إلى تلميحات عرض صغيرة.

```json
{
  "uiHints": {
    "apiKey": {
      "label": "مفتاح API",
      "help": "يُستخدم لطلبات OpenRouter",
      "placeholder": "sk-or-v1-...",
      "sensitive": true
    }
  }
}
```

يمكن أن يتضمن كل تلميح حقل ما يلي:

| الحقل         | النوع      | معناه                                 |
| ------------- | ---------- | ------------------------------------- |
| `label`       | `string`   | تسمية الحقل الموجهة للمستخدم.         |
| `help`        | `string`   | نص مساعد قصير.                        |
| `tags`        | `string[]` | وسوم واجهة مستخدم اختيارية.           |
| `advanced`    | `boolean`  | يحدّد أن الحقل متقدّم.                |
| `sensitive`   | `boolean`  | يحدّد أن الحقل سرّي أو حسّاس.         |
| `placeholder` | `string`   | نص العنصر النائب لحقول النماذج.       |

## مرجع `contracts`

استخدم `contracts` فقط لبيانات تعريف ملكية القدرات الثابتة التي يمكن لـ OpenClaw
قراءتها من دون استيراد وقت تشغيل الإضافة.

```json
{
  "contracts": {
    "speechProviders": ["openai"],
    "realtimeTranscriptionProviders": ["openai"],
    "realtimeVoiceProviders": ["openai"],
    "mediaUnderstandingProviders": ["openai", "openai-codex"],
    "imageGenerationProviders": ["openai"],
    "videoGenerationProviders": ["qwen"],
    "webFetchProviders": ["firecrawl"],
    "webSearchProviders": ["gemini"],
    "tools": ["firecrawl_search", "firecrawl_scrape"]
  }
}
```

كل قائمة اختيارية:

| الحقل                            | النوع      | معناه                                                    |
| -------------------------------- | ---------- | -------------------------------------------------------- |
| `speechProviders`                | `string[]` | معرّفات مزوّدي الكلام التي تمتلكها هذه الإضافة.          |
| `realtimeTranscriptionProviders` | `string[]` | معرّفات مزوّدي النسخ الفوري التي تمتلكها هذه الإضافة.    |
| `realtimeVoiceProviders`         | `string[]` | معرّفات مزوّدي الصوت الفوري التي تمتلكها هذه الإضافة.    |
| `mediaUnderstandingProviders`    | `string[]` | معرّفات مزوّدي فهم الوسائط التي تمتلكها هذه الإضافة.     |
| `imageGenerationProviders`       | `string[]` | معرّفات مزوّدي توليد الصور التي تمتلكها هذه الإضافة.     |
| `videoGenerationProviders`       | `string[]` | معرّفات مزوّدي توليد الفيديو التي تمتلكها هذه الإضافة.   |
| `webFetchProviders`              | `string[]` | معرّفات مزوّدي جلب الويب التي تمتلكها هذه الإضافة.       |
| `webSearchProviders`             | `string[]` | معرّفات مزوّدي البحث في الويب التي تمتلكها هذه الإضافة.  |
| `tools`                          | `string[]` | أسماء أدوات الوكيل التي تمتلكها هذه الإضافة لفحوصات العقود المضمّنة. |

## مرجع `channelConfigs`

استخدم `channelConfigs` عندما تحتاج إضافة قناة إلى بيانات تعريف إعدادات خفيفة قبل
تحميل وقت التشغيل.

```json
{
  "channelConfigs": {
    "matrix": {
      "schema": {
        "type": "object",
        "additionalProperties": false,
        "properties": {
          "homeserverUrl": { "type": "string" }
        }
      },
      "uiHints": {
        "homeserverUrl": {
          "label": "عنوان URL للخادم المنزلي",
          "placeholder": "https://matrix.example.com"
        }
      },
      "label": "Matrix",
      "description": "اتصال خادم Matrix المنزلي",
      "preferOver": ["matrix-legacy"]
    }
  }
}
```

يمكن أن يتضمن كل إدخال قناة ما يلي:

| الحقل         | النوع                     | معناه                                                                                  |
| ------------- | ------------------------ | --------------------------------------------------------------------------------------- |
| `schema`      | `object`                 | JSON Schema لـ `channels.<id>`. مطلوب لكل إدخال إعدادات قناة مُصرّح به.                |
| `uiHints`     | `Record<string, object>` | تسميات/عناصر نائبة/تلميحات حساسية اختيارية لواجهة المستخدم لقسم إعدادات تلك القناة.   |
| `label`       | `string`                 | تسمية القناة المدمجة في أسطح المنتقي والفحص عندما لا تكون بيانات تعريف وقت التشغيل جاهزة. |
| `description` | `string`                 | وصف قصير للقناة لأسطح الفحص والفهرسة.                                                  |
| `preferOver`  | `string[]`               | معرّفات إضافات قديمة أو أقل أولوية يجب أن تتفوق عليها هذه القناة في أسطح الاختيار.    |

## مرجع `modelSupport`

استخدم `modelSupport` عندما يجب على OpenClaw استنتاج إضافة المزوّد الخاصة بك من
معرّفات نماذج مختصرة مثل `gpt-5.4` أو `claude-sonnet-4.6` قبل تحميل وقت تشغيل الإضافة.

```json
{
  "modelSupport": {
    "modelPrefixes": ["gpt-", "o1", "o3", "o4"],
    "modelPatterns": ["^computer-use-preview"]
  }
}
```

يطبّق OpenClaw هذا الترتيب:

- تستخدم مراجع `provider/model` الصريحة بيانات تعريف `providers` المملوكة للبيان
- تتقدم `modelPatterns` على `modelPrefixes`
- إذا طابقت إضافة غير مضمّنة وإضافة مضمّنة معًا، فإن الإضافة غير المضمّنة
  تفوز
- يُتجاهل الغموض المتبقي إلى أن يحدد المستخدم أو الإعدادات مزوّدًا

الحقول:

| الحقل           | النوع      | معناه                                                                        |
| --------------- | ---------- | ----------------------------------------------------------------------------- |
| `modelPrefixes` | `string[]` | بادئات تُطابَق باستخدام `startsWith` مع معرّفات النماذج المختصرة.            |
| `modelPatterns` | `string[]` | مصادر Regex تُطابَق مع معرّفات النماذج المختصرة بعد إزالة لاحقة الملف التعريفي. |

مفاتيح القدرات القديمة على المستوى الأعلى مهجورة. استخدم `openclaw doctor --fix`
لنقل `speechProviders`, `realtimeTranscriptionProviders`,
`realtimeVoiceProviders`, `mediaUnderstandingProviders`,
`imageGenerationProviders`, `videoGenerationProviders`,
`webFetchProviders`, و `webSearchProviders` تحت `contracts`; لم يعد
تحميل البيان العادي يتعامل مع تلك الحقول العليا على أنها ملكية
للقدرات.

## البيان مقابل package.json

يؤدي الملفان وظائف مختلفة:

| الملف                  | استخدمه من أجل                                                                                                                      |
| ---------------------- | ----------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw.plugin.json` | الاكتشاف، والتحقق من الإعدادات، وبيانات تعريف خيارات المصادقة، وتلميحات واجهة المستخدم التي يجب أن توجد قبل تشغيل كود الإضافة     |
| `package.json`         | بيانات تعريف npm، وتثبيت التبعيات، وكتلة `openclaw` المستخدمة لنقاط الدخول، وضبط التثبيت، والإعداد، أو بيانات تعريف الفهرس      |

إذا لم تكن متأكدًا من مكان انتماء جزء من بيانات التعريف، فاستخدم هذه القاعدة:

- إذا كان يجب على OpenClaw معرفته قبل تحميل كود الإضافة، فضعه في `openclaw.plugin.json`
- وإذا كان متعلقًا بالتغليف، أو ملفات الدخول، أو سلوك تثبيت npm، فضعه في `package.json`

### حقول package.json التي تؤثر في الاكتشاف

توجد بعض بيانات تعريف الإضافة قبل وقت التشغيل عمدًا في `package.json` ضمن
كتلة `openclaw` بدلًا من `openclaw.plugin.json`.

أمثلة مهمة:

| الحقل                                                             | معناه                                                                                                                                 |
| ----------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------------------------- |
| `openclaw.extensions`                                             | يصرّح بنقاط دخول الإضافة الأصلية.                                                                                                     |
| `openclaw.setupEntry`                                             | نقطة دخول خفيفة للإعداد فقط تُستخدم أثناء التهيئة وبدء تشغيل القنوات المؤجل.                                                          |
| `openclaw.channel`                                                | بيانات تعريف خفيفة لفهرس القنوات مثل التسميات، ومسارات المستندات، والأسماء المستعارة، ونسخة الاختيار.                                 |
| `openclaw.channel.configuredState`                                | بيانات تعريف خفيفة لفاحص الحالة المُهيأة يمكنه الإجابة عن "هل يوجد إعداد يعتمد على البيئة فقط بالفعل؟" من دون تحميل وقت تشغيل القناة الكامل. |
| `openclaw.channel.persistedAuthState`                             | بيانات تعريف خفيفة لفاحص المصادقة المحفوظة يمكنه الإجابة عن "هل يوجد أي تسجيل دخول بالفعل؟" من دون تحميل وقت تشغيل القناة الكامل.     |
| `openclaw.install.npmSpec` / `openclaw.install.localPath`         | تلميحات التثبيت/التحديث للإضافات المضمّنة والمنشورة خارجيًا.                                                                           |
| `openclaw.install.defaultChoice`                                  | مسار التثبيت المفضّل عندما تتوفر مصادر تثبيت متعددة.                                                                                   |
| `openclaw.install.minHostVersion`                                 | أقل إصدار مدعوم من مضيف OpenClaw، باستخدام حد semver أدنى مثل `>=2026.3.22`.                                                        |
| `openclaw.install.allowInvalidConfigRecovery`                     | يسمح بمسار استرداد ضيق لإعادة تثبيت الإضافات المضمّنة عندما تكون الإعدادات غير صالحة.                                                |
| `openclaw.startup.deferConfiguredChannelFullLoadUntilAfterListen` | يسمح بتحميل أسطح القناة الخاصة بالإعداد فقط قبل الإضافة الكاملة للقناة أثناء بدء التشغيل.                                             |

يُفرض `openclaw.install.minHostVersion` أثناء التثبيت وتحميل سجل
البيانات. تُرفض القيم غير الصالحة؛ أما القيم الأحدث لكنها صالحة فتتجاوز
الإضافة على المضيفين الأقدم.

إن `openclaw.install.allowInvalidConfigRecovery` ضيق عمدًا. فهو
لا يجعل الإعدادات المعطوبة عشوائيًا قابلة للتثبيت. في الوقت الحالي، يتيح فقط
لتدفقات التثبيت الاسترداد من إخفاقات ترقية محددة لإضافات مضمّنة قديمة، مثل
غياب مسار إضافة مضمّنة أو وجود إدخال قديم `channels.<id>` لتلك الإضافة
المضمّنة نفسها. وما تزال أخطاء الإعدادات غير المرتبطة تمنع التثبيت وتوجّه
المشغّلين إلى `openclaw doctor --fix`.

إن `openclaw.channel.persistedAuthState` هو بيانات تعريف حزمة لوحدة
تحقق صغيرة:

```json
{
  "openclaw": {
    "channel": {
      "id": "whatsapp",
      "persistedAuthState": {
        "specifier": "./auth-presence",
        "exportName": "hasAnyWhatsAppAuth"
      }
    }
  }
}
```

استخدمه عندما تحتاج تدفقات الإعداد أو doctor أو الحالة المُهيأة إلى
فحص مصادقة خفيف بنعم/لا قبل تحميل إضافة القناة الكاملة. يجب أن يكون
التصدير الهدف دالة صغيرة تقرأ الحالة المحفوظة فقط؛ ولا تمرره عبر شريط
وقت تشغيل القناة الكامل.

يتبع `openclaw.channel.configuredState` الصيغة نفسها لفحوصات
الحالة المُهيأة الخفيفة المعتمدة على البيئة فقط:

```json
{
  "openclaw": {
    "channel": {
      "id": "telegram",
      "configuredState": {
        "specifier": "./configured-state",
        "exportName": "hasTelegramConfiguredState"
      }
    }
  }
}
```

استخدمه عندما تستطيع القناة الإجابة عن الحالة المُهيأة من البيئة أو من مدخلات صغيرة
أخرى غير وقت التشغيل. وإذا كان الفحص يحتاج إلى حل كامل للإعدادات أو
إلى وقت تشغيل القناة الحقيقي، فأبقِ ذلك المنطق داخل خطاف الإضافة
`config.hasConfiguredState` بدلًا من ذلك.

## متطلبات JSON Schema

- **يجب على كل إضافة شحن JSON Schema**، حتى لو لم تكن تقبل أي إعدادات.
- schema فارغة مقبولة (مثلًا `{ "type": "object", "additionalProperties": false }`).
- يتم التحقق من الـ schemas وقت قراءة/كتابة الإعدادات، وليس في وقت التشغيل.

## سلوك التحقق

- مفاتيح `channels.*` غير المعروفة هي **أخطاء**، ما لم يكن معرّف القناة مصرّحًا به
  بواسطة بيان إضافة.
- يجب أن تشير `plugins.entries.<id>` و `plugins.allow` و `plugins.deny` و `plugins.slots.*`
  إلى معرّفات إضافات **قابلة للاكتشاف**. المعرفات غير المعروفة هي **أخطاء**.
- إذا كانت الإضافة مثبّتة لكنها تحتوي على بيان أو schema معطوبة أو مفقودة،
  يفشل التحقق ويبلغ Doctor عن خطأ الإضافة.
- إذا وُجدت إعدادات للإضافة لكن الإضافة **معطلة**، فسيتم الاحتفاظ بالإعدادات
  ويظهر **تحذير** في Doctor + السجلات.

راجع [مرجع الإعدادات](/ar/gateway/configuration) للاطلاع على schema الكاملة لـ `plugins.*`.

## ملاحظات

- البيان **مطلوب للإضافات الأصلية في OpenClaw**، بما في ذلك التحميلات المحلية من نظام الملفات.
- ما يزال وقت التشغيل يحمّل وحدة الإضافة بشكل منفصل؛ والبيان مخصص فقط
  للاكتشاف + التحقق.
- تُحلّل البيانات الأصلية باستخدام JSON5، لذا تُقبل التعليقات والفواصل اللاحقة
  والمفاتيح غير الموضوعة بين علامات اقتباس طالما أن القيمة النهائية ما تزال كائنًا.
- لا يقرأ محمّل البيان إلا الحقول الموثقة في البيان. تجنب إضافة
  مفاتيح عليا مخصصة هنا.
- `providerAuthEnvVars` هو مسار بيانات التعريف الخفيف لفحوصات المصادقة، والتحقق
  من علامات البيئة، والأسطح المماثلة لمصادقة المزوّد التي لا ينبغي أن تشغّل وقت تشغيل الإضافة
  فقط لفحص أسماء البيئة.
- `channelEnvVars` هو مسار بيانات التعريف الخفيف للتراجع إلى بيئة الصدفة،
  ومطالبات الإعداد، والأسطح المماثلة للقنوات التي لا ينبغي أن تشغّل وقت تشغيل الإضافة
  فقط لفحص أسماء البيئة.
- `providerAuthChoices` هو مسار بيانات التعريف الخفيف لمنتقيات خيارات المصادقة،
  وحل `--auth-choice`، وربط المزوّد المفضّل، وتسجيل أعلام CLI البسيطة أثناء التهيئة
  قبل تحميل وقت تشغيل المزوّد. أما بالنسبة إلى بيانات تعريف المعالج وقت التشغيل
  التي تتطلب كود المزوّد، فراجع
  [خطافات وقت تشغيل المزوّد](/ar/plugins/architecture#provider-runtime-hooks).
- تُختار أنواع الإضافات الحصرية عبر `plugins.slots.*`.
  - يُختار `kind: "memory"` بواسطة `plugins.slots.memory`.
  - ويُختار `kind: "context-engine"` بواسطة `plugins.slots.contextEngine`
    (الافتراضي: `legacy` المضمّن).
- يمكن حذف `channels` و `providers` و `cliBackends` و `skills` عندما
  لا تحتاج الإضافة إليها.
- إذا كانت إضافتك تعتمد على وحدات أصلية، فقم بتوثيق خطوات البناء وأي
  متطلبات لقائمة سماح مدير الحزم (مثل pnpm ‏`allow-build-scripts`
  - `pnpm rebuild <package>`).
