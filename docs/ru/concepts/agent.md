# Время выполнения агента

OpenClaw запускает единую встроенную среду выполнения агента.

## Рабочая область (обязательно)

OpenClaw использует единый каталог рабочей области агента (`agents.defaults.workspace`) в качестве **единственной** рабочей директории (`cwd`) для инструментов и контекста.

Рекомендация: используйте команду `openclaw setup` для создания файла `~/.openclaw/openclaw.json`, если он отсутствует, и инициализации файлов рабочей области.

Полное описание структуры рабочей области и руководство по резервному копированию: [Рабочая область агента](/concepts/agent-workspace).

Если параметр `agents.defaults.sandbox` включён, неглавные сессии могут переопределить это с помощью рабочих областей для каждой сессии в каталоге `agents.defaults.sandbox.workspaceRoot` (см. [Конфигурация шлюза](/gateway/configuration)).

## Файлы начальной загрузки (внедряются)

В каталоге `agents.defaults.workspace` OpenClaw ожидает следующие файлы, доступные для редактирования пользователем:

- `AGENTS.md` — инструкции по работе + "память";
- `SOUL.md` — личность, границы, тон;
- `TOOLS.md` — заметки пользователя о инструментах (например, `imsg`, `sag`, соглашения);
- `BOOTSTRAP.md` — ритуал однократного запуска при первом использовании (удаляется после выполнения);
- `IDENTITY.md` — имя агента / стиль / эмодзи;
- `USER.md` — профиль пользователя + предпочтительный способ обращения.

На первом шаге новой сессии OpenClaw внедряет содержимое этих файлов непосредственно в контекст агента.

Пустые файлы пропускаются. Большие файлы обрезаются и усекаются с маркером, чтобы подсказки оставались лаконичными (для полного содержимого нужно прочитать файл).

Если файл отсутствует, OpenClaw внедряет единственную строку-маркер "файл отсутствует" (а команда `openclaw setup` создаст безопасный шаблон по умолчанию).

Файл `BOOTSTRAP.md` создаётся только для **совершенно новой рабочей области** (если другие файлы начальной загрузки отсутствуют). Если вы удалите его после выполнения ритуала, он не должен воссоздаваться при последующих перезапусках.

Чтобы полностью отключить создание файлов начальной загрузки (для предварительно заполненных рабочих областей), установите:

```json5
{ agent: { skipBootstrap: true } }
```

## Встроенные инструменты

Основные инструменты (чтение/выполнение/редактирование/запись и связанные системные инструменты) всегда доступны в соответствии с политикой инструментов. Инструмент `apply_patch` является необязательным и регулируется параметром `tools.exec.applyPatch`. Файл `TOOLS.md` **не** определяет, какие инструменты существуют; он содержит указания о том, как _вы_ хотите их использовать.

## Навыки

OpenClaw загружает навыки из следующих местоположений (в порядке убывания приоритета):

- Рабочая область: `<workspace>/skills`;
- Навыки проектного агента: `<workspace>/.agents/skills`;
- Личные навыки агента: `~/.agents/skills`;
- Управляемые/локальные: `~/.openclaw/skills`;
- Встроенные (поставляются вместе с установкой);
- Дополнительные папки навыков: `skills.load.extraDirs`.

Доступ к навыкам может регулироваться конфигурацией/окружением (см. `skills` в [Конфигурация шлюза](/gateway/configuration)).

## Границы времени выполнения

Встроенная среда выполнения агента построена на ядре агента Pi (модели, инструменты и конвейер подсказок). Управление сессиями, обнаружение, подключение инструментов и доставка по каналам — это слои, принадлежащие OpenClaw и расположенные поверх этого ядра.

## Сессии

Транскрипты сессий хранятся в формате JSONL по адресу:

- `~/.openclaw/agents/<agentId>/sessions/<SessionId>.jsonl`

Идентификатор сессии стабилен и выбирается OpenClaw.
Папки сессий из других инструментов в устаревшем формате не считываются.

## Управление во время потоковой передачи

Когда режим очереди установлен в `steer`, входящие сообщения внедряются в текущий запуск.
Очередное управление доставляется **после завершения выполнения вызовов инструментов текущим помощником**, перед следующим вызовом LLM. Управление больше не пропускает оставшиеся вызовы инструментов из текущего сообщения помощника; вместо этого оно внедряет сообщение из очереди на следующей границе модели.

Когда режим очереди установлен в `followup` или `collect`, входящие сообщения удерживаются до окончания текущего шага, после чего начинается новый шаг агента с данными из очереди. См. [Очередь](/concepts/queue) для получения информации о режиме, а также о поведении при подавлении и ограничении.

Потоковая передача блоков отправляет завершённые блоки помощника сразу после их завершения; по умолчанию она **отключена** (`agents.defaults.blockStreamingDefault: "off"`).
Настройте границу с помощью параметра `agents.defaults.blockStreamingBreak` (`text_end` против `message_end`; по умолчанию — `text_end`).
Управляйте разбиением мягких блоков с помощью параметра `agents.defaults.blockStreamingChunk` (по умолчанию — 800–1200 символов; предпочтение отдаётся разрывам абзацев, затем переносам строк, предложениям — в последнюю очередь).
Объединяйте потоковые фрагменты с помощью параметра `agents.defaults.blockStreamingCoalesce`, чтобы уменьшить количество однострочных сообщений (объединение по бездействию перед отправкой). Для каналов, отличных от Telegram, требуется явное указание `*.blockStreaming: true` для включения ответов в виде блоков.
Подробные сводки по инструментам выводятся при запуске инструмента (без подавления); пользовательский интерфейс передаёт вывод инструмента через события агента, когда это возможно.
Подробнее: [Потоковая передача и разбиение на фрагменты](/concepts/streaming).

## Ссылки на модели

Ссылки на модели в конфигурации (например, `agents.defaults.model` и `agents.defaults.models`) разбираются путём разделения по **первому** символу `/`.

- Используйте формат `provider/model` при настройке моделей.
- Если идентификатор модели сам содержит `/` (в стиле OpenRouter), укажите префикс провайдера (например: `openrouter/moonshotai/kimi-k2`).
- Если вы опустите провайдера, OpenClaw сначала попытается использовать псевдоним, затем найдёт совпадение с точно указанным идентификатором модели для настроенного провайдера и только потом обратится к настроенному провайдеру по умолчанию. Если этот провайдер больше не предоставляет настроенную модель по умолчанию, OpenClaw обратится к первой настроенной паре провайдер/модель вместо того, чтобы использовать устаревшую модель удалённого провайдера.

## Конфигурация (минимальная)

Как минимум, установите:

- `agents.defaults.workspace`;
- `channels.whatsapp.allowFrom` (настоятельно рекомендуется).
