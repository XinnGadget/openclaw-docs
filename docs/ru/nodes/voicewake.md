# Голосовое пробуждение (глобальные слова пробуждения)

В OpenClaw **слова пробуждения рассматриваются как единый глобальный список**, принадлежащий **шлюзу (Gateway)**.

- **Нет пользовательских слов пробуждения для отдельных узлов.**
- **Любой узел или пользовательский интерфейс приложения может редактировать** список; изменения сохраняются шлюзом и рассылаются всем участникам.
- В macOS и iOS есть локальные переключатели **включения/отключения голосового пробуждения** (локальный пользовательский опыт и разрешения отличаются).
- В Android голосовое пробуждение в настоящее время отключено, используется ручной захват микрофона на вкладке "Голос".

## Хранение (хост шлюза)

Слова пробуждения хранятся на машине шлюза по пути:

- `~/.openclaw/settings/voicewake.json`

Структура:

```json
{ "triggers": ["openclaw", "claude", "computer"], "updatedAtMs": 1730000000000 }
```

## Протокол

### Методы

- `voicewake.get` → `{ triggers: string[] }`
- `voicewake.set` с параметрами `{ triggers: string[] }` → `{ triggers: string[] }`

Примечания:

- Триггеры нормализуются (обрезаются, пустые значения удаляются). Пустые списки возвращаются к значениям по умолчанию.
- Для обеспечения безопасности применяются ограничения (лимиты по количеству и длине).

### События

- `voicewake.changed` с полезной нагрузкой `{ triggers: string[] }`

Кто получает событие:

- Все клиенты WebSocket (приложение для macOS, WebChat и т. д.).
- Все подключённые узлы (iOS/Android), а также при подключении узла в качестве начальной отправки "текущего состояния".

## Поведение клиента

### Приложение для macOS

- Использует глобальный список для управления триггерами `VoiceWakeRuntime`.
- При редактировании "Слов-триггеров" в настройках голосового пробуждения вызывается `voicewake.set`, после чего синхронизация других клиентов обеспечивается за счёт рассылки.

### Узел iOS

- Использует глобальный список для обнаружения триггеров в `VoiceWakeManager`.
- При редактировании слов пробуждения в настройках вызывается `voicewake.set` (через WebSocket шлюза), при этом локальное обнаружение слов пробуждения остаётся отзывчивым.

### Узел Android

- Голосовое пробуждение в настоящее время отключено в среде выполнения/настройках Android.
- В Android для голоса используется ручной захват микрофона на вкладке "Голос" вместо триггеров слов пробуждения.
