# Механизм памяти QMD

[QMD](https://github.com/tobi/qmd) — это локальный поисковый сайдкар, который работает вместе с OpenClaw. Он объединяет BM25, векторный поиск и переранжирование в одном бинарном файле и может индексировать контент за пределами файлов памяти вашего рабочего пространства.

## Что добавлено по сравнению со встроенным решением

- **Переранжирование и расширение запросов** для улучшения полноты поиска.
- **Индексация дополнительных каталогов** — документации проекта, заметок команды, любых данных на диске.
- **Индексация транскриптов сессий** — возможность вспомнить предыдущие диалоги.
- **Полная локальность** — работает через Bun + node-llama-cpp, автоматически загружает модели GGUF.
- **Автоматический откат** — если QMD недоступен, OpenClaw беспроблемно переключается на встроенный движок.

## Начало работы

### Предварительные требования

- Установите QMD: `npm install -g @tobilu/qmd` или `bun install -g @tobilu/qmd`.
- Сборка SQLite с поддержкой расширений (`brew install sqlite` на macOS).
- QMD должен находиться в переменной `PATH` шлюза.
- macOS и Linux работают "из коробки". Для Windows рекомендуется использовать WSL2.

### Включение

```json5
{
  memory: {
    backend: "qmd",
  },
}
```

OpenClaw создаёт изолированную домашнюю директорию QMD по пути `~/.openclaw/agents/<agentId>/qmd/` и автоматически управляет жизненным циклом сайдкара — коллекции, обновления и встраивание выполняются за вас. Предпочтение отдаётся текущей коллекции QMD и форматам запросов MCP, но при необходимости происходит откат к устаревшим флагам коллекции `--mask` и старым названиям инструментов MCP.

## Как работает сайдкар

- OpenClaw создаёт коллекции из файлов памяти вашего рабочего пространства и любых настроенных `memory.qmd.paths`, затем запускает `qmd update` + `qmd embed` при загрузке и периодически (по умолчанию каждые 5 минут).
- Обновление при загрузке выполняется в фоновом режиме, поэтому запуск чата не блокируется.
- При поиске используется настроенное значение `searchMode` (по умолчанию: `search`; также поддерживаются `vsearch` и `query`). Если режим завершается ошибкой, OpenClaw повторяет попытку с `qmd query`.
- Если QMD полностью выходит из строя, OpenClaw переключается на встроенный движок SQLite.

<Info>
Первый поиск может быть медленным — QMD автоматически загружает модели GGUF (~2 ГБ) для переранжирования и расширения запросов при первом запуске `qmd query`.
</Info>

## Переопределение моделей

Переменные окружения модели QMD передаются без изменений из процесса шлюза, поэтому вы можете настроить QMD глобально, не добавляя новую конфигурацию в OpenClaw:

```bash
export QMD_EMBED_MODEL="hf:Qwen/Qwen3-Embedding-0.6B-GGUF/Qwen3-Embedding-0.6B-Q8_0.gguf"
export QMD_RERANK_MODEL="/absolute/path/to/reranker.gguf"
export QMD_GENERATE_MODEL="/absolute/path/to/generator.gguf"
```

После изменения модели встраивания перезапустите процесс встраивания, чтобы индекс соответствовал новому векторному пространству.

## Индексация дополнительных путей

Укажите QMD дополнительные каталоги, чтобы сделать их доступными для поиска:

```json5
{
  memory: {
    backend: "qmd",
    qmd: {
      paths: [{ name: "docs", path: "~/notes", pattern: "**/*.md" }],
    },
  },
}
```

Фрагменты из дополнительных путей отображаются в результатах поиска как `qmd/<collection>/<relative-path>`. `memory_get` распознаёт этот префикс и считывает данные из правильного корневого каталога коллекции.

## Индексация транскриптов сессий

Включите индексацию сессий, чтобы иметь возможность вспомнить предыдущие диалоги:

```json5
{
  memory: {
    backend: "qmd",
    qmd: {
      sessions: { enabled: true },
    },
  },
}
```

Транскрипты экспортируются в виде очищенных диалогов пользователя и ассистента в отдельную коллекцию QMD по пути `~/.openclaw/agents/<id>/qmd/sessions/`.

## Область поиска

По умолчанию результаты поиска QMD отображаются только в личных сообщениях (не в группах или каналах). Чтобы изменить это, настройте `memory.qmd.scope`:

```json5
{
  memory: {
    qmd: {
      scope: {
        default: "deny",
        rules: [{ action: "allow", match: { chatType: "direct" } }],
      },
    },
  },
}
```

Когда область поиска запрещает поиск, OpenClaw записывает предупреждение с указанием канала и типа чата, что упрощает отладку пустых результатов.

## Цитирования

Когда `memory.citations` имеет значение `auto` или `on`, фрагменты поиска включают нижний колонтитул `Source: <path#line>`. Установите `memory.citations = "off"`, чтобы исключить нижний колонтитул, при этом путь всё равно будет передаваться агенту внутри системы.

## Когда использовать

Выберите QMD, если вам нужно:

- Переранжирование для получения более качественных результатов.
- Поиск по документации проекта или заметкам за пределами рабочего пространства.
- Вспомнить предыдущие диалоги в сессиях.
- Полностью локальный поиск без API-ключей.

Для более простых настроек [встроенный движок](/concepts/memory-builtin) хорошо работает без дополнительных зависимостей.

## Устранение неполадок

**QMD не найден?** Убедитесь, что бинарный файл находится в переменной `PATH` шлюза. Если OpenClaw работает как служба, создайте симлинк: `sudo ln -s ~/.bun/bin/qmd /usr/local/bin/qmd`.

**Первый поиск очень медленный?** QMD загружает модели GGUF при первом использовании. Предварительно прогрейте систему с помощью `qmd query "test"`, используя те же директории XDG, что и OpenClaw.

**Поиск завершается по таймауту?** Увеличьте значение `memory.qmd.limits.timeoutMs` (по умолчанию: 4000 мс). Установите значение `120000` для более медленного оборудования.

**Пустые результаты в групповых чатах?** Проверьте `memory.qmd.scope` — по умолчанию разрешены только личные сообщения.

**Временные репозитории, видимые в рабочем пространстве, вызывают ошибку `ENAMETOOLONG` или нарушают индексацию?** В настоящее время обход QMD следует поведению сканера QMD, а не встроенным правилам символических ссылок OpenClaw. Храните временные копии монорепозиториев в скрытых директориях, таких как `.tmp/`, или за пределами индексируемых корней QMD, пока QMD не предоставит безопасный обход циклов или явные средства исключения.
