# Логика состояния панели меню

## Что отображается

- Текущее состояние работы агента отображается в значке панели меню и в первой строке состояния меню.
- Статус работоспособности скрыт во время активной работы; он возвращается, когда все сессии находятся в режиме ожидания.
- Блок "Nodes" в меню отображает только **устройства** (сопряжённые узлы через `node.list`), а не записи клиента/присутствия.
- Раздел "Usage" появляется в разделе Context, если доступны снимки использования провайдера.

## Модель состояния

- Сессии: события поступают с `runId` (для каждого запуска) плюс `sessionKey` в полезной нагрузке. "Основная" сессия — это ключ `main`; если он отсутствует, используется самая недавно обновлённая сессия.
- Приоритет: основная сессия всегда имеет приоритет. Если основная сессия активна, её состояние отображается немедленно. Если основная сессия находится в режиме ожидания, отображается самая недавно активная неосновная сессия. Мы не переключаемся во время активности; переключение происходит только тогда, когда текущая сессия переходит в режим ожидания или основная сессия становится активной.
- Виды активности:
  - `job`: выполнение команды высокого уровня (`state: started|streaming|done|error`).
  - `tool`: `phase: start|result` с `toolName` и `meta/args`.

## Перечисление IconState (Swift)

- `idle`
- `workingMain(ActivityKind)`
- `workingOther(ActivityKind)`
- `overridden(ActivityKind)` (отладка, переопределение)

### Соответствие ActivityKind и глифа

- `exec` → 💻
- `read` → 📄
- `write` → ✍️
- `edit` → 📝
- `attach` → 📎
- по умолчанию → 🛠️

### Визуальное сопоставление

- `idle`: обычный значок.
- `workingMain`: бейдж с глифом, насыщенный оттенок, анимация "работающей" лапки.
- `workingOther`: бейдж с глифом, приглушённый оттенок, без анимации движения.
- `overridden`: используется выбранный глиф/оттенок независимо от активности.

## Текст строки состояния (меню)

- Во время активной работы: `<Роль сессии> · <метка активности>`
  - Примеры: `Main · exec: pnpm test`, `Other · read: apps/macos/Sources/OpenClaw/AppState.swift`.
- В режиме ожидания: отображается сводка о состоянии работоспособности.

## Приём событий

- Источник: события канала управления `agent` (`ControlChannel.handleAgentEvent`).
- Разобранные поля:
  - `stream: "job"` с `data.state` для начала/завершения.
  - `stream: "tool"` с `data.phase`, `name`, необязательными `meta`/`args`.
- Метки:
  - `exec`: первая строка `args.command`.
  - `read`/`write`: сокращённый путь.
  - `edit`: путь плюс предполагаемый тип изменения из `meta`/количества различий.
  - по умолчанию: название инструмента.

## Переопределение для отладки

- Настройки ▸ Отладка ▸ Выпадающий список "Переопределение значка":
  - `System (auto)` (по умолчанию)
  - `Working: main` (для каждого вида инструмента)
  - `Working: other` (для каждого вида инструмента)
  - `Idle`
- Сохраняется через `@AppStorage("iconOverride")`; сопоставляется с `IconState.overridden`.

## Чек-лист тестирования

- Запустить задание основной сессии: убедиться, что значок переключается немедленно и в строке состояния отображается метка основной сессии.
- Запустить задание неосновной сессии, когда основная сессия находится в режиме ожидания: значок/состояние отображает неосновную сессию; остаётся стабильным до завершения.
- Запустить основную сессию, когда другая активна: значок мгновенно переключается на основную.
- Быстрые всплески инструментов: убедиться, что бейдж не мерцает (TTL-задержка для результатов инструментов).
- Строка состояния работоспособности появляется снова, когда все сессии переходят в режим ожидания.
