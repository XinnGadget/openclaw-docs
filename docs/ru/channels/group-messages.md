# Групповые сообщения (веб-канал WhatsApp)

Цель: позволить Clawd находиться в группах WhatsApp, активироваться только при упоминании и вести диалог в отдельной ветке, не смешивая его с личной перепиской в DM.

Примечание: параметр `agents.list[].groupChat.mentionPatterns` теперь используется также в Telegram, Discord, Slack и iMessage; в этом документе рассматривается поведение, специфичное для WhatsApp. Для конфигураций с несколькими агентами задайте `agents.list[].groupChat.mentionPatterns` для каждого агента (или используйте `messages.groupChat.mentionPatterns` в качестве глобального запасного варианта).

## Текущая реализация (2025-12-03)

- Режимы активации: `mention` (по умолчанию) или `always`. Режим `mention` требует упоминания (реальные упоминания в WhatsApp через `mentionedJids`, безопасные регулярные выражения или номер бота в формате E.164 в любом месте текста). Режим `always` активирует агента при каждом сообщении, но он должен отвечать только тогда, когда может добавить существенную ценность; в противном случае возвращается токен молчания `NO_REPLY` / `no_reply`. Значения по умолчанию можно задать в конфигурации (`channels.whatsapp.groups`) и переопределить для каждой группы с помощью команды `/activation`. Когда задан `channels.whatsapp.groups`, он также действует как белый список групп (используйте `"*"` для разрешения всех).
- Политика для групп: `channels.whatsapp.groupPolicy` определяет, принимаются ли групповые сообщения (`open|disabled|allowlist`). Режим `allowlist` использует `channels.whatsapp.groupAllowFrom` (запасной вариант: явный `channels.whatsapp.allowFrom`). По умолчанию установлен `allowlist` (сообщения блокируются, пока вы не добавите отправителей).
- Сессии для каждой группы: ключи сессий имеют вид `agent:<agentId>:whatsapp:group:<jid>`, поэтому команды вроде `/verbose on` или `/think high` (отправленные как отдельные сообщения) применяются только к этой группе; состояние личной переписки DM остаётся неизменным. Пульсации (heartbeats) для групповых веток пропускаются.
- Внедрение контекста: групповые сообщения в состоянии **pending-only** (по умолчанию — 50), которые _не_ привели к запуску, добавляются в начало с пометкой `[Chat messages since your last reply - for context]`, а строка, вызвавшая реакцию, — с пометкой `[Current message - respond to this]`. Сообщения, уже находящиеся в сессии, повторно не внедряются.
- Отображение отправителя: каждый пакет групповых сообщений теперь завершается строкой `[from: Sender Name (+E164)]`, чтобы Pi знал, кто говорит.
- Временные сообщения / сообщения с однократным просмотром: мы распаковываем их перед извлечением текста и упоминаний, поэтому упоминания внутри них по-прежнему вызывают реакцию.
- Системный промпт для группы: на первом шаге групповой сессии (и всякий раз, когда команда `/activation` меняет режим) в системный промпт вставляется короткий фрагмент вида `You are replying inside the WhatsApp group "<subject>". Group members: Alice (+44...), Bob (+43...), … Activation: trigger-only … Address the specific sender noted in the message context.` Если метаданные недоступны, агенту всё равно сообщается, что это групповой чат.

## Пример конфигурации (WhatsApp)

Добавьте блок `groupChat` в файл `~/.openclaw/openclaw.json`, чтобы упоминания по отображаемому имени работали, даже если WhatsApp удаляет визуальный символ `@` в теле сообщения:

```json5
{
  channels: {
    whatsapp: {
      groups: {
        "*": { requireMention: true },
      },
    },
  },
  agents: {
    list: [
      {
        id: "main",
        groupChat: {
          historyLimit: 50,
          mentionPatterns: ["@?openclaw", "\\+?15555550123"],
        },
      },
    ],
  },
}
```

Примечания:

- Регулярные выражения не чувствительны к регистру и используют те же механизмы защиты безопасных регулярных выражений, что и другие поверхности конфигурации с регулярными выражениями; недопустимые шаблоны и небезопасные вложенные повторения игнорируются.
- WhatsApp по-прежнему отправляет канонические упоминания через `mentionedJids`, когда кто-то выбирает контакт, поэтому запасной вариант с номером редко требуется, но может служить полезной страховкой.

### Команда активации (только для владельца)

Используйте команду группового чата:

- `/activation mention`
- `/activation always`

Изменить режим может только номер владельца (из `channels.whatsapp.allowFrom` или собственный номер бота в формате E.164, если он не задан). Отправьте `/status` как отдельное сообщение в группе, чтобы узнать текущий режим активации.

## Как использовать

1. Добавьте свой аккаунт WhatsApp (тот, который используется для OpenClaw) в группу.
2. Напишите `@openclaw …` (или укажите номер). Только отправители из белого списка могут активировать бота, если вы не установили `groupPolicy: "open"`.
3. В промпте агента будет включён недавний контекст группы, а также маркер `[from: …]`, чтобы агент мог обратиться к нужному человеку.
4. Директивы уровня сессии (`/verbose on`, `/think high`, `/new` или `/reset`, `/compact`) применяются только к сессии этой группы; отправьте их как отдельные сообщения, чтобы они были зарегистрированы. Ваша личная сессия DM остаётся независимой.

## Тестирование / проверка

- Ручное дымовое тестирование:
  - Отправьте упоминание `@openclaw` в группе и убедитесь, что получен ответ, ссылающийся на имя отправителя.
  - Отправьте второе упоминание и убедитесь, что блок истории включён, а затем очищен на следующем шаге.
- Проверьте логи шлюза (запустите с параметром `--verbose`), чтобы увидеть записи `inbound web message` с указанием `from: <groupJid>` и суффиксом `[from: …]`.

## Известные особенности

- Пульсации (heartbeats) намеренно пропускаются для групп, чтобы избежать шумных широковещательных сообщений.
- Подавление эхо использует объединённую строку пакета; если вы отправите идентичный текст дважды без упоминаний, ответ будет дан только на первое сообщение.
- Записи хранилища сессий будут отображаться как `agent:<agentId>:whatsapp:group:<jid>` в хранилище сессий (`~/.openclaw/agents/<agentId>/sessions/sessions.json` по умолчанию); отсутствие записи означает, что группа ещё не вызвала запуск.
- Индикаторы набора текста в группах следуют параметру `agents.defaults.typingMode` (по умолчанию: `message`, если бот не упомянут).
