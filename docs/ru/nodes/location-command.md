# Команда определения местоположения (узлы)

## Кратко

- `location.get` — команда для узла (вызывается через `node.invoke`).
- По умолчанию отключена.
- В настройках приложения на Android используется селектор: Отключено / Во время использования.
- Отдельный переключатель: Точное местоположение.

## Почему используется селектор (а не просто переключатель)

Разрешения ОС имеют многоуровневую структуру. Мы можем предоставить селектор в приложении, но окончательное решение о предоставлении разрешения принимает ОС.

- В iOS/macOS в системных подсказках/настройках могут быть варианты **Во время использования** или **Всегда**.
- Приложение на Android в настоящее время поддерживает только определение местоположения в фоновом режиме.
- Точное местоположение требует отдельного разрешения (в iOS 14+ — "Точное", в Android — "точное" (fine) vs "приблизительное" (coarse)).

Селектор в пользовательском интерфейсе определяет запрашиваемый режим; фактическое разрешение задаётся в настройках ОС.

## Модель настроек

Для каждого устройства узла:

- `location.enabledMode`: `off | whileUsing`
- `location.preciseEnabled`: bool

Поведение интерфейса:

- Выбор `whileUsing` запрашивает разрешение на использование местоположения в фоновом режиме.
- Если ОС отклоняет запрошенный уровень, возвращается к максимально возможному предоставленному уровню и отображает статус.

## Сопоставление разрешений (node.permissions)

Необязательно. Узел macOS передаёт `location` через карту разрешений; в iOS/Android это может быть опущено.

## Команда: `location.get`

Вызывается через `node.invoke`.

Параметры (рекомендуемые):

```json
{
  "timeoutMs": 10000,
  "maxAgeMs": 15000,
  "desiredAccuracy": "coarse|balanced|precise"
}
```

Ответ (полезная нагрузка):

```json
{
  "lat": 48.20849,
  "lon": 16.37208,
  "accuracyMeters": 12.5,
  "altitudeMeters": 182.0,
  "speedMps": 0.0,
  "headingDeg": 270.0,
  "timestamp": "2026-01-03T12:34:56.000Z",
  "isPrecise": true,
  "source": "gps|wifi|cell|unknown"
}
```

Ошибки (стабильные коды):

- `LOCATION_DISABLED`: селектор отключён.
- `LOCATION_PERMISSION_REQUIRED`: отсутствует разрешение для запрошенного режима.
- `LOCATION_BACKGROUND_UNAVAILABLE`: приложение находится в фоновом режиме, но разрешён только режим "Во время использования".
- `LOCATION_TIMEOUT`: местоположение не определено в установленное время.
- `LOCATION_UNAVAILABLE`: сбой системы / отсутствуют провайдеры.

## Поведение в фоновом режиме

- Приложение на Android отклоняет `location.get`, когда находится в фоновом режиме.
- Необходимо оставлять OpenClaw открытым при запросе местоположения на Android.
- На других платформах узлов поведение может отличаться.

## Интеграция с моделью/инструментами

- Интерфейс инструмента: инструмент `nodes` добавляет действие `location_get` (требуется узел).
- CLI: `openclaw nodes location get --node <id>`.
- Рекомендации для агентов: вызывать команду только тогда, когда пользователь включил определение местоположения и понимает область её применения.

## Тексты для пользовательского интерфейса (рекомендуемые)

- Отключено: "Обмен данными о местоположении отключён".
- Во время использования: "Только когда OpenClaw открыт".
- Точное: "Использовать точное GPS-местоположение. Отключите, чтобы делиться приблизительным местоположением".
