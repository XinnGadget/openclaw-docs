# IRC

Используйте IRC, если хотите подключить OpenClaw к классическим каналам (`#room`) и прямым сообщениям.
IRC поставляется в виде плагина-расширения, но настраивается в основном конфигурационном файле в разделе `channels.irc`.

## Быстрый старт

1. Включите конфигурацию IRC в файле `~/.openclaw/openclaw.json`.
2. Задайте как минимум следующие параметры:

```json5
{
  channels: {
    irc: {
      enabled: true,
      host: "irc.example.com",
      port: 6697,
      tls: true,
      nick: "openclaw-bot",
      channels: ["#openclaw"],
    },
  },
}
```

Для координации работы ботов предпочтительно использовать частный сервер IRC. Если вы намеренно используете публичную сеть IRC, можно выбрать одну из распространённых: Libera.Chat, OFTC или Snoonet. Избегайте предсказуемых публичных каналов для обмена данными между ботами или роем (backchannel traffic).

3. Запустите или перезапустите шлюз:

```bash
openclaw gateway run
```

## Настройки безопасности по умолчанию

- `channels.irc.dmPolicy` по умолчанию имеет значение `"pairing"`.
- `channels.irc.groupPolicy` по умолчанию имеет значение `"allowlist"`.
- При `groupPolicy="allowlist"` задайте `channels.irc.groups`, чтобы определить разрешённые каналы.
- Используйте TLS (`channels.irc.tls=true`), если вы намеренно не хотите использовать передачу данных в открытом тексте.

## Управление доступом

Для каналов IRC предусмотрены два отдельных "фильтра":

1. **Доступ к каналу** (`groupPolicy` + `groups`): определяет, принимает ли бот сообщения из канала вообще.
2. **Доступ отправителя** (`groupAllowFrom` / `groups["#channel"].allowFrom` для отдельного канала): определяет, кому разрешено активировать бота в этом канале.

Ключевые параметры конфигурации:

- Список разрешённых отправителей для DM (доступ отправителя для DM): `channels.irc.allowFrom`.
- Список разрешённых отправителей для группы (доступ отправителя для канала): `channels.irc.groupAllowFrom`.
- Настройки для отдельного канала (канал + отправитель + правила упоминаний): `channels.irc.groups["#channel"]`.
- `channels.irc.groupPolicy="open"` разрешает использование не настроенных каналов (**по умолчанию они всё равно ограничены упоминаниями**).

Записи в списке разрешённых должны использовать стабильные идентификаторы отправителей (`nick!user@host`).
Сопоставление только по нику (bare nick matching) может меняться и включается только при `channels.irc.dangerouslyAllowNameMatching: true`.

### Распространённая ошибка: `allowFrom` предназначен для DM, а не для каналов

Если вы видите в логах сообщения вроде:

- `irc: drop group sender alice!ident@host (policy=allowlist)`

…это означает, что отправитель не был разрешён для **сообщений группы/канала**. Чтобы исправить это, выполните одно из следующих действий:

- задайте `channels.irc.groupAllowFrom` (глобально для всех каналов);
- задайте списки разрешённых отправителей для отдельных каналов: `channels.irc.groups["#channel"].allowFrom`.

Пример (разрешить любому пользователю в `#tuirc-dev` общаться с ботом):

```json5
{
  channels: {
    irc: {
      groupPolicy: "allowlist",
      groups: {
        "#tuirc-dev": { allowFrom: ["*"] },
      },
    },
  },
}
```

## Активация ответов (упоминания)

Даже если канал разрешён (через `groupPolicy` + `groups`) и отправитель разрешён, OpenClaw по умолчанию использует **контроль упоминаний** в групповых контекстах.

Это означает, что вы можете увидеть в логах сообщения вроде `drop channel … (missing-mention)`, если сообщение не содержит шаблона упоминания, соответствующего боту.

Чтобы бот отвечал в канале IRC **без необходимости упоминания**, отключите контроль упоминаний для этого канала:

```json5
{
  channels: {
    irc: {
      groupPolicy: "allowlist",
      groups: {
        "#tuirc-dev": {
          requireMention: false,
          allowFrom: ["*"],
        },
      },
    },
  },
}
```

Или чтобы разрешить **все** каналы IRC (без списка разрешённых для отдельных каналов) и при этом отвечать без упоминаний:

```json5
{
  channels: {
    irc: {
      groupPolicy: "open",
      groups: {
        "*": { requireMention: false, allowFrom: ["*"] },
      },
    },
  },
}
```

## Примечание по безопасности (рекомендуется для публичных каналов)

Если вы разрешаете `allowFrom: ["*"]` в публичном канале, любой пользователь сможет отправлять запросы боту.
Чтобы снизить риски, ограничьте инструменты для этого канала.

### Одни и те же инструменты для всех в канале

```json5
{
  channels: {
    irc: {
      groups: {
        "#tuirc-dev": {
          allowFrom: ["*"],
          tools: {
            deny: ["group:runtime", "group:fs", "gateway", "nodes", "cron", "browser"],
          },
        },
      },
    },
  },
}
```

### Разные инструменты для разных отправителей (у владельца больше прав)

Используйте `toolsBySender`, чтобы применить более строгую политику для `"*"` и более мягкую — для вашего ника:

```json5
{
  channels: {
    irc: {
      groups: {
        "#tuirc-dev": {
          allowFrom: ["*"],
          toolsBySender: {
            "*": {
              deny: ["group:runtime", "group:fs", "gateway", "nodes", "cron", "browser"],
            },
            "id:eigen": {
              deny: ["gateway", "nodes", "cron"],
            },
          },
        },
      },
    },
  },
}
```

Примечания:

- Ключи `toolsBySender` должны использовать префикс `id:` для идентификаторов отправителей в IRC: `id:eigen` или `id:eigen!~eigen@174.127.248.171` для более точного сопоставления.
- Устаревшие ключи без префикса по-прежнему принимаются и сопоставляются только как `id:`.
- Применяется первая подходящая политика для отправителя; `"*"` — это запасной вариант с подстановочным знаком.

Подробнее о разнице между доступом к группам и контролем упоминаний (и о том, как они взаимодействуют), см.: [/channels/groups](/channels/groups).

## NickServ

Чтобы идентифицироваться с NickServ после подключения:

```json5
{
  channels: {
    irc: {
      nickserv: {
        enabled: true,
        service: "NickServ",
        password: "your-nickserv-password",
      },
    },
  },
}
```

Дополнительная одноразовая регистрация при подключении:

```json5
{
  channels: {
    irc: {
      nickserv: {
        register: true,
        registerEmail: "bot@example.com",
      },
    },
  },
}
```

Отключите `register` после регистрации ника, чтобы избежать повторных попыток регистрации.

## Переменные окружения

Учётная запись по умолчанию поддерживает следующие переменные:

- `IRC_HOST`
- `IRC_PORT`
- `IRC_TLS`
- `IRC_NICK`
- `IRC_USERNAME`
- `IRC_REALNAME`
- `IRC_PASSWORD`
- `IRC_CHANNELS` (через запятую)
- `IRC_NICKSERV_PASSWORD`
- `IRC_NICKSERV_REGISTER_EMAIL`

## Устранение неполадок

- Если бот подключается, но никогда не отвечает в каналах, проверьте `channels.irc.groups` **и** то, не отбрасывает ли контроль упоминаний сообщения (`missing-mention`). Если вы хотите, чтобы бот отвечал без упоминаний, задайте `requireMention:false` для канала.
- Если вход в систему не удаётся, проверьте доступность ника и пароль сервера.
- Если возникают проблемы с TLS в пользовательской сети, проверьте хост/порт и настройку сертификата.
