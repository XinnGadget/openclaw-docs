# SGLang

SGLang может предоставлять модели с открытым исходным кодом через **HTTP API, совместимый с OpenAI**.
OpenClaw может подключаться к SGLang с использованием API `openai-completions`.

OpenClaw также может **автоматически обнаруживать** доступные модели в SGLang, если вы активируете эту функцию с помощью `SGLANG_API_KEY` (подходит любое значение, если на вашем сервере не настроена аутентификация) и не задаёте явную запись `models.providers.sglang`.

## Быстрый старт

1. Запустите SGLang с сервером, совместимым с OpenAI.

Ваш базовый URL должен предоставлять endpoints `/v1` (например, `/v1/models`, `/v1/chat/completions`). Обычно SGLang работает на:

- `http://127.0.0.1:30000/v1`

2. Активируйте функцию (подходит любое значение, если аутентификация не настроена):

```bash
export SGLANG_API_KEY="sglang-local"
```

3. Запустите процесс онбординга и выберите `SGLang` или задайте модель напрямую:

```bash
openclaw onboard
```

```json5
{
  agents: {
    defaults: {
      model: { primary: "sglang/your-model-id" },
    },
  },
}
```

## Обнаружение моделей (неявный провайдер)

Если задано значение `SGLANG_API_KEY` (или существует профиль аутентификации) и вы **не** задаёте `models.providers.sglang`, OpenClaw выполнит запрос:

- `GET http://127.0.0.1:30000/v1/models`

и преобразует полученные идентификаторы в записи моделей.

Если вы явно задаёте `models.providers.sglang`, автоматическое обнаружение пропускается, и вам необходимо вручную определить модели.

## Явная конфигурация (ручное задание моделей)

Используйте явную конфигурацию в следующих случаях:

- SGLang работает на другом хосте/порту.
- Вы хотите зафиксировать значения `contextWindow`/`maxTokens`.
- Ваш сервер требует реальный API-ключ (или вы хотите управлять заголовками).

```json5
{
  models: {
    providers: {
      sglang: {
        baseUrl: "http://127.0.0.1:30000/v1",
        apiKey: "${SGLANG_API_KEY}",
        api: "openai-completions",
        models: [
          {
            id: "your-model-id",
            name: "Local SGLang Model",
            reasoning: false,
            input: ["text"],
            cost: { input: 0, output: 0, cacheRead: 0, cacheWrite: 0 },
            contextWindow: 128000,
            maxTokens: 8192,
          },
        ],
      },
    },
  },
}
```

## Устранение неполадок

- Проверьте доступность сервера:

```bash
curl http://127.0.0.1:30000/v1/models
```

- Если запросы завершаются с ошибками аутентификации, задайте реальный `SGLANG_API_KEY`, соответствующий конфигурации вашего сервера, или явно настройте провайдера в разделе `models.providers.sglang`.

## Поведение в стиле прокси

SGLang рассматривается как бэкенд в стиле прокси, совместимый с OpenAI `/v1`, а не как нативный endpoint OpenAI.

- формирование запросов, специфичное только для OpenAI, здесь не применяется;
- отсутствуют `service_tier`, Responses `store`, подсказки для кэширования промтов и формирование полезной нагрузки, совместимой с рассуждениями OpenAI;
- скрытые заголовки атрибуции OpenClaw (`originator`, `version`, `User-Agent`) не добавляются к пользовательским базовым URL SGLang.
