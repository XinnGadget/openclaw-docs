# Настройка (приложение для macOS)

В этом документе описан **текущий** процесс настройки при первом запуске. Цель — обеспечить комфортное начало работы ("день 0"): выбрать, где будет работать Gateway, настроить аутентификацию, запустить мастер настройки и позволить агенту выполнить начальную загрузку.

Для общего обзора путей настройки см. [Обзор настройки](/start/onboarding-overview).

Модель доверия в области безопасности:

- По умолчанию OpenClaw — это персональный агент: одна граница доверенного оператора.
- Для общих/многопользовательских настроек требуется ужесточение безопасности (разделение границ доверия, минимизация доступа к инструментам и соблюдение рекомендаций из раздела [Безопасность](/gateway/security)).
- При локальной настройке новые конфигурации по умолчанию устанавливаются как `tools.profile: "coding"`, поэтому при новых локальных настройках сохраняются инструменты работы с файловой системой и средой выполнения без принудительного использования неограниченного профиля `full`.
- Если включены хуки/вебхуки или другие каналы с недоверенным контентом, используйте современную модель высокого уровня и соблюдайте строгую политику использования инструментов/песочницы.

</Step>
<Step title="Локальный или удалённый режим">
<Frame>
<img src="/assets/macos-onboarding/04-choose-gateway.png" alt="" />
</Frame>

Где будет работать **Gateway**?

- **Этот Mac (только локально):** в процессе настройки можно настроить аутентификацию и записать учётные данные локально.
- **Удалённо (через SSH/Tailnet):** в процессе настройки локальная аутентификация **не** настраивается; учётные данные должны существовать на хосте Gateway.
- **Настроить позже:** пропустить настройку и оставить приложение без конфигурации.

<Tip>
**Совет по аутентификации Gateway:**

- Мастер настройки теперь генерирует **токен** даже для loopback, поэтому локальные WS-клиенты должны проходить аутентификацию.
- Если вы отключите аутентификацию, подключиться сможет любой локальный процесс; используйте это только на полностью доверенных машинах.
- Используйте **токен** для доступа с нескольких машин или для привязок, не использующих loopback.

</Tip>
</Step>

В процессе настройки запрашиваются разрешения TCC, необходимые для:

- Автоматизации (AppleScript)
- Уведомлений
- Специальных возможностей
- Записи экрана
- Микрофона
- Распознавания речи
- Камеры
- Определения местоположения

</Step>
<Step title="CLI">
  <Info>Этот шаг необязателен</Info>
  Приложение может установить глобальный CLI `openclaw` через npm, pnpm или bun.
  Сначала предпочтение отдаётся npm, затем pnpm, а затем bun — если это единственный обнаруженный менеджер пакетов. Для среды выполнения Gateway рекомендуется использовать Node.
</Step>
<Step title="Чат настройки (отдельная сессия)">
  После завершения настройки приложение открывает отдельную сессию чата настройки, чтобы агент мог представиться и подсказать следующие шаги. Это позволяет отделить руководство для первого запуска от обычных диалогов. См. раздел [Начальная загрузка](/start/bootstrapping), чтобы узнать, что происходит на хосте Gateway во время первого запуска агента.
</Step>
</Steps>
