# API Brave Search

OpenClaw поддерживает API Brave Search в качестве провайдера `web_search`.

## Получение API-ключа

1. Создайте учётную запись API Brave Search на [https://brave.com/search/api/](https://brave.com/search/api/).
2. В панели управления выберите тариф **Search** и сгенерируйте API-ключ.
3. Сохраните ключ в конфигурации или задайте `BRAVE_API_KEY` в среде Gateway.

## Пример конфигурации

```json5
{
  plugins: {
    entries: {
      brave: {
        config: {
          webSearch: {
            apiKey: "BRAVE_API_KEY_HERE",
            mode: "web", // или "llm-context"
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        provider: "brave",
        maxResults: 5,
        timeoutSeconds: 30,
      },
    },
  },
}
```

Специфические для провайдера Brave настройки поиска теперь находятся в `plugins.entries.brave.config.webSearch.*`.
Устаревший параметр `tools.web.search.apiKey` по-прежнему загружается через слой совместимости, но больше не является каноническим путём к конфигурации.

Параметр `webSearch.mode` управляет транспортом Brave:

- `web` (по умолчанию): обычный веб-поиск Brave с заголовками, URL и фрагментами;
- `llm-context`: API Brave LLM Context с предварительно извлечёнными фрагментами текста и источниками для обоснования.

## Параметры инструмента

| Параметр | Описание |
| --- | --- |
| `query` | Поисковый запрос (обязательно) |
| `count` | Количество возвращаемых результатов (1–10, по умолчанию: 5) |
| `country` | Двухбуквенный код страны по ISO (например, "US", "DE") |
| `language` | Код языка по ISO 639-1 для результатов поиска (например, "en", "de", "fr") |
| `search_lang` | Код языка поиска Brave (например, `en`, `en-gb`, `zh-hans`) |
| `ui_lang` | Код языка по ISO для элементов интерфейса |
| `freshness` | Фильтр по времени: `day` (24 ч), `week`, `month` или `year` |
| `date_after` | Только результаты, опубликованные после указанной даты (ГГГГ-ММ-ДД) |
| `date_before` | Только результаты, опубликованные до указанной даты (ГГГГ-ММ-ДД) |

**Примеры:**

```javascript
// Поиск с учётом страны и языка
await web_search({
  query: "renewable energy",
  country: "DE",
  language: "de",
});

// Недавние результаты (за последнюю неделю)
await web_search({
  query: "AI news",
  freshness: "week",
});

// Поиск по диапазону дат
await web_search({
  query: "AI developments",
  date_after: "2024-01-01",
  date_before: "2024-06-30",
});
```

## Примечания

- OpenClaw использует тариф Brave **Search**. Если у вас устаревшая подписка (например, первоначальный бесплатный тариф с 2 000 запросами в месяц), она остаётся действительной, но не включает новые функции, такие как LLM Context или повышенные лимиты запросов.
- Каждый тариф Brave включает **5 долларов в месяц в виде бесплатного кредита** (с продлением). Тариф Search стоит 5 долларов за 1 000 запросов, таким образом, кредит покрывает 1 000 запросов в месяц. Установите лимит использования в панели управления Brave, чтобы избежать непредвиденных расходов. Актуальные тарифы см. на [портале API Brave](https://brave.com/search/api/).
- Тариф Search включает конечную точку LLM Context и права на вывод ИИ. Для хранения результатов с целью обучения или тонкой настройки моделей требуется тариф с явными правами на хранение. См. [Условия предоставления услуг Brave](https://api-dashboard.search.brave.com/terms-of-service).
- В режиме `llm-context` возвращаются обоснованные записи источников вместо обычных фрагментов веб-поиска.
- В режиме `llm-context` не поддерживаются параметры `ui_lang`, `freshness`, `date_after` и `date_before`.
- Параметр `ui_lang` должен включать субтег региона, например `en-US`.
- По умолчанию результаты кэшируются в течение 15 минут (можно настроить через `cacheTtlMinutes`).

Полную конфигурацию web_search см. в разделе [Web tools](/tools/web).
