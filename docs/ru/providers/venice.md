# Venice AI (основные возможности Venice)

**Venice** — это наша конфигурация Venice для инференс-обработки с приоритетом конфиденциальности и опциональным анонимизированным доступом к проприетарным моделям.

Venice AI обеспечивает инференс-обработку ИИ с акцентом на конфиденциальность, поддерживает нецензурированные модели и предоставляет доступ к основным проприетарным моделям через анонимизированный прокси-сервер. Вся инференс-обработка по умолчанию конфиденциальна — данные не используются для обучения, логи не ведутся.

## Зачем использовать Venice в OpenClaw

- **Конфиденциальная инференс-обработка** для моделей с открытым исходным кодом (без ведения логов).
- **Нецензурированные модели**, когда они вам нужны.
- **Анонимизированный доступ** к проприетарным моделям (Opus/GPT/Gemini), когда качество имеет значение.
- Конечные точки, совместимые с OpenAI `/v1`.

## Режимы конфиденциальности

Venice предлагает два уровня конфиденциальности — понимание различий между ними важно для выбора модели:

| Режим | Описание | Модели |
| --- | --- | --- |
| **Конфиденциальный** | Полностью конфиденциальный режим. Запросы и ответы **никогда не сохраняются и не логируются**. Временный режим. | Llama, Qwen, DeepSeek, Kimi, MiniMax, Venice Uncensored и др. |
| **Анонимизированный** | Проходят через прокси-сервер Venice с удалением метаданных. Базовый провайдер (OpenAI, Anthropic, Google, xAI) видит анонимизированные запросы. | Claude, GPT, Gemini, Grok |

## Возможности

- **Ориентация на конфиденциальность**: выбор между режимами "конфиденциальный" (полностью конфиденциальный) и "анонимизированный" (через прокси).
- **Нецензурированные модели**: доступ к моделям без ограничений по контенту.
- **Доступ к основным моделям**: использование Claude, GPT, Gemini и Grok через анонимизированный прокси-сервер Venice.
- **API, совместимый с OpenAI**: стандартные конечные точки `/v1` для простой интеграции.
- **Потоковая передача**: ✅ Поддерживается для всех моделей.
- **Вызов функций**: ✅ Поддерживается для некоторых моделей (проверьте возможности модели).
- **Работа с изображениями**: ✅ Поддерживается для моделей с соответствующей функциональностью.
- **Отсутствие жёстких ограничений по частоте запросов**: может применяться регулирование использования при чрезмерной нагрузке.

## Настройка

### 1. Получите API-ключ

1. Зарегистрируйтесь на [venice.ai](https://venice.ai).
2. Перейдите в **Settings → API Keys → Create new key**.
3. Скопируйте свой API-ключ (формат: `vapi_xxxxxxxxxxxx`).

### 2. Настройте OpenClaw

**Вариант A: Переменная окружения**

```bash
export VENICE_API_KEY="vapi_xxxxxxxxxxxx"
```

**Вариант B: Интерактивная настройка (рекомендуется)**

```bash
openclaw onboard --auth-choice venice-api-key
```

Это позволит:

1. Запросить ваш API-ключ (или использовать существующий `VENICE_API_KEY`).
2. Показать все доступные модели Venice.
3. Выбрать модель по умолчанию.
4. Автоматически настроить провайдера.

**Вариант C: Неинтерактивная настройка**

```bash
openclaw onboard --non-interactive \
  --auth-choice venice-api-key \
  --venice-api-key "vapi_xxxxxxxxxxxx"
```

### 3. Проверьте настройку

```bash
openclaw agent --model venice/kimi-k2-5 --message "Hello, are you working?"
```

## Выбор модели

После настройки OpenClaw отображает все доступные модели Venice. Выберите модель в соответствии с вашими потребностями:

- **Модель по умолчанию**: `venice/kimi-k2-5` — мощная модель для конфиденциальной обработки с поддержкой работы с изображениями.
- **Модель с расширенными возможностями**: `venice/claude-opus-4-6` — наиболее мощная анонимизированная модель в Venice.
- **Конфиденциальность**: выбирайте "конфиденциальные" модели для полностью конфиденциальной инференс-обработки.
- **Возможности**: выбирайте "анонимизированные" модели для доступа к Claude, GPT, Gemini через прокси-сервер Venice.

Вы можете изменить модель по умолчанию в любое время:

```bash
openclaw models set venice/kimi-k2-5
openclaw models set venice/claude-opus-4-6
```

Вывести список всех доступных моделей:

```bash
openclaw models list | grep venice
```

## Настройка через `openclaw configure`

1. Запустите `openclaw configure`.
2. Выберите **Model/auth**.
3. Выберите **Venice AI**.
