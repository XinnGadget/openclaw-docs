# Переменные окружения

OpenClaw получает переменные окружения из нескольких источников. Правило: **никогда не перезаписывать существующие значения**.

## Приоритет (от высшего к низшему)

1. **Окружение процесса** (то, что процесс Gateway уже имеет от родительской оболочки/демона).
2. **`.env` в текущей рабочей директории** (значение по умолчанию для dotenv; не перезаписывает).
3. **Глобальный `.env`** по пути `~/.openclaw/.env` (он же `$OPENCLAW_STATE_DIR/.env`; не перезаписывает).
4. **Блок `env` в конфигурации** в `~/.openclaw/openclaw.json` (применяется только если значение отсутствует).
5. **Опциональный импорт окружения login-shell** (`env.shellEnv.enabled` или `OPENCLAW_LOAD_SHELL_ENV=1`), применяется только для отсутствующих ожидаемых ключей.

В свежих установках Ubuntu, использующих директорию состояния по умолчанию, OpenClaw также рассматривает `~/.config/openclaw/gateway.env` как запасной вариант совместимости после глобального `.env`. Если оба файла существуют и содержат противоречивые данные, OpenClaw сохраняет `~/.openclaw/.env` и выводит предупреждение.

Если конфигурационный файл полностью отсутствует, шаг 4 пропускается; импорт из оболочки всё равно выполняется, если он включён.

## Блок `env` в конфигурации

Есть два эквивалентных способа задать встроенные переменные окружения (оба не перезаписывают существующие значения):

```json5
{
  env: {
    OPENROUTER_API_KEY: "sk-or-...",
    vars: {
      GROQ_API_KEY: "gsk-...",
    },
  },
}
```

## Импорт окружения оболочки

`env.shellEnv` запускает вашу login-shell и импортирует только **отсутствующие** ожидаемые ключи:

```json5
{
  env: {
    shellEnv: {
      enabled: true,
      timeoutMs: 15000,
    },
  },
}
```

Эквиваленты переменных окружения:

- `OPENCLAW_LOAD_SHELL_ENV=1`
- `OPENCLAW_SHELL_ENV_TIMEOUT_MS=15000`

## Переменные окружения, внедряемые во время выполнения

OpenClaw также внедряет маркеры контекста в порождённые дочерние процессы:

- `OPENCLAW_SHELL=exec`: устанавливается для команд, выполняемых через инструмент `exec`.
- `OPENCLAW_SHELL=acp`: устанавливается для процессов бэкенда ACP (например, `acpx`).
- `OPENCLAW_SHELL=acp-client`: устанавливается для `openclaw acp client`, когда он порождает процесс моста ACP.
- `OPENCLAW_SHELL=tui-local`: устанавливается для локальных команд оболочки TUI `!`.

Это маркеры времени выполнения (не требуют настройки со стороны пользователя). Их можно использовать в логике оболочки/профиля для применения правил, специфичных для контекста.

## Переменные окружения для UI

- `OPENCLAW_THEME=light`: принудительно устанавливает светлую палитру TUI, если у вашего терминала светлый фон.
- `OPENCLAW_THEME=dark`: принудительно устанавливает тёмную палитру TUI.
- `COLORFGBG`: если ваш терминал экспортирует эту переменную, OpenClaw использует подсказку о цвете фона для автоматического выбора палитры TUI.

## Подстановка переменных окружения в конфигурации

Вы можете ссылаться на переменные окружения непосредственно в строковых значениях конфигурации, используя синтаксис `${VAR_NAME}`:

```json5
{
  models: {
    providers: {
      "vercel-gateway": {
        apiKey: "${VERCEL_GATEWAY_API_KEY}",
      },
    },
  },
}
```

Подробные сведения см. в разделе [Configuration: Env var substitution](/gateway/configuration-reference#env-var-substitution).

## Ссылки на секреты против строк `${ENV}`

OpenClaw поддерживает два шаблона, управляемых окружением:

- Подстановка строк `${VAR}` в значениях конфигурации.
- Объекты SecretRef (`{ source: "env", provider: "default", id: "VAR" }`) для полей, поддерживающих ссылки на секреты.

Оба способа разрешаются из окружения процесса во время активации. Подробности о SecretRef описаны в разделе [Secrets Management](/gateway/secrets).

## Переменные окружения, связанные с путями

| Переменная | Назначение |
| --- | --- |
| `OPENCLAW_HOME` | Переопределяет домашнюю директорию, используемую для разрешения всех внутренних путей (`~/.openclaw/`, директории агентов, сессии, учётные данные). Полезно при запуске OpenClaw от имени выделенного сервисного пользователя. |
| `OPENCLAW_STATE_DIR` | Переопределяет директорию состояния (по умолчанию `~/.openclaw`). |
| `OPENCLAW_CONFIG_PATH` | Переопределяет путь к конфигурационному файлу (по умолчанию `~/.openclaw/openclaw.json`). |

## Логирование

| Переменная | Назначение |
| --- | --- |
| `OPENCLAW_LOG_LEVEL` | Переопределяет уровень логирования для файла и консоли (например, `debug`, `trace`). Имеет приоритет над `logging.level` и `logging.consoleLevel` в конфигурации. Некорректные значения игнорируются с предупреждением. |

### `OPENCLAW_HOME`

При установке `OPENCLAW_HOME` заменяет системную домашнюю директорию (`$HOME` / `os.homedir()`) для разрешения всех внутренних путей. Это позволяет обеспечить полную изоляцию файловой системы для безголовых сервисных учётных записей.

**Приоритет:** `OPENCLAW_HOME` > `$HOME` > `USERPROFILE` > `os.homedir()`

**Пример** (macOS LaunchDaemon):

```xml
<key>EnvironmentVariables</key>
<dict>
  <key>OPENCLAW_HOME</key>
  <string>/Users/user</string>
</dict>
```

`OPENCLAW_HOME` также можно установить в путь с тильдой (например, `~/svc`), который будет развёрнут с использованием `$HOME` перед использованием.

## Пользователи nvm: ошибки TLS в web_fetch

Если Node.js был установлен через **nvm** (а не через системный менеджер пакетов), встроенный `fetch()` использует хранилище CA, поставляемое с nvm, в котором могут отсутствовать современные корневые CA (ISRG Root X1/X2 для Let's Encrypt, DigiCert Global Root G2 и т. д.). Это приводит к сбоям `web_fetch` с сообщением `"fetch failed"` на большинстве HTTPS-сайтов.

В Linux OpenClaw автоматически обнаруживает nvm и применяет исправление в реальной среде запуска:

- `openclaw gateway install` записывает `NODE_EXTRA_CA_CERTS` в окружение службы systemd;
- точка входа CLI `openclaw` перезапускает себя с установленным `NODE_EXTRA_CA_CERTS` перед запуском Node.

**Ручное исправление (для старых версий или прямого запуска `node ...`):**

Экспортируйте переменную перед запуском OpenClaw:

```bash
export NODE_EXTRA_CA_CERTS=/etc/ssl/certs/ca-certificates.crt
openclaw gateway run
```

Не полагайтесь на запись этой переменной только в `~/.openclaw/.env`; Node считывает `NODE_EXTRA_CA_CERTS` при запуске процесса.
