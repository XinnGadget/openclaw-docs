# Сновидения (экспериментальная функция)

"Сновидения" — это система фоновой консолидации памяти в `memory-core`.
Она помогает OpenClaw переносить сильные краткосрочные сигналы в долговременную память, сохраняя при этом прозрачность и возможность просмотра процесса.

Функция "Сновидения" **активируется по желанию пользователя** и по умолчанию отключена.

## Что записывает система "Сновидения"

Система "Сновидения" сохраняет два типа данных:

- **Состояние машины** в каталоге `memory/.dreams/` (хранилище воспоминаний, сигналы фаз, контрольные точки приёма данных, блокировки).
- **Данные, читаемые человеком**, в файле `DREAMS.md` (или существующем `dreams.md`) и в необязательных файлах отчётов по фазам в каталоге `memory/dreaming/<phase>/YYYY-MM-DD.md`.

Продвижение в долговременную память по-прежнему осуществляется только в файл `MEMORY.md`.

## Модель фаз

Система "Сновидения" использует три взаимосвязанные фазы:

| Фаза | Назначение | Запись в долговременную память |
| ----- | ----------------------------------------- | ----------------- |
| Лёгкая | Сортировка и подготовка недавнего краткосрочного материала | Нет |
| Глубокая | Оценка и продвижение кандидатов в долговременную память | Да (`MEMORY.md`) |
| REM | Анализ тем и повторяющихся идей | Нет |

Эти фазы являются деталями внутренней реализации, а не отдельными режимами, настраиваемыми пользователем.

### Лёгкая фаза

Лёгкая фаза обрабатывает недавние сигналы памяти за день и следы воспоминаний, удаляет дубликаты и подготавливает строки-кандидаты.

- Читает данные из краткосрочного состояния воспоминаний, недавних файлов памяти за день и отредактированных транскриптов сессий (если они доступны).
- Записывает управляемый блок `## Light Sleep`, если хранилище включает встроенный вывод.
- Регистрирует сигналы подкрепления для последующей глубокой ранжировки.
- Никогда не записывает данные в `MEMORY.md`.

### Глубокая фаза

Глубокая фаза определяет, какие данные попадут в долговременную память.

- Ранжирует кандидатов с использованием взвешенной оценки и пороговых значений.
- Для прохождения требуется соответствие параметрам `minScore`, `minRecallCount` и `minUniqueQueries`.
- Восстанавливает фрагменты из актуальных дневных файлов перед записью, поэтому устаревшие или удалённые фрагменты пропускаются.
- Добавляет продвинутые записи в `MEMORY.md`.
- Записывает сводку `## Deep Sleep` в `DREAMS.md` и при необходимости создаёт файл `memory/dreaming/deep/YYYY-MM-DD.md`.

### Фаза REM

Фаза REM извлекает шаблоны и рефлексивные сигналы.

- Формирует сводки по темам и рефлексиям на основе недавних краткосрочных следов.
- Записывает управляемый блок `## REM Sleep`, если хранилище включает встроенный вывод.
- Регистрирует сигналы подкрепления REM, используемые при глубокой ранжировке.
- Никогда не записывает данные в `MEMORY.md`.

## Приём транскриптов сессий

Система "Сновидения" может включать отредактированные транскрипты сессий в корпус сновидений. Когда транскрипты доступны, они передаются в лёгкую фазу вместе с ежедневными сигналами памяти и следами воспоминаний. Личный и конфиденциальный контент редактируется перед приёмом.

## Дневник сновидений

Система "Сновидения" также ведёт повествовательный **дневник сновидений** в файле `DREAMS.md`.
После того как у каждой фазы накапливается достаточно материала, `memory-core` запускает фоновый ход суб-агента (с использованием модели выполнения по умолчанию) и добавляет короткую запись в дневник.

Этот дневник предназначен для чтения человеком в интерфейсе Dreams UI, а не для продвижения данных.

Также существует канал исторического заполнения для проверки и восстановления:

- `memory rem-harness --path ... --grounded` — предварительный просмотр обоснованного вывода дневника из исторических заметок `YYYY-MM-DD.md`.
- `memory rem-backfill --path ...` — запись обратимых обоснованных записей дневника в `DREAMS.md`.
- `memory rem-backfill --path ... --stage-short-term` — подготовка обоснованных кандидатов в долговременную память в том же хранилище краткосрочных доказательств, которое уже использует обычная глубокая фаза.
- `memory rem-backfill --rollback` и `--rollback-short-term` — удаление подготовленных артефактов заполнения без затрагивания обычных записей дневника или краткосрочных воспоминаний в реальном времени.

Интерфейс управления (Control UI) предоставляет тот же поток заполнения/сброса дневника, чтобы вы могли проверить результаты в сцене Dreams перед тем, как решить, заслуживают ли обоснованные кандидаты продвижения. Сцена также отображает отдельный обоснованный канал, чтобы вы могли увидеть, какие подготовленные краткосрочные записи поступили из исторического воспроизведения, какие продвинутые элементы были обоснованы, и удалить только обоснованные подготовленные записи, не затрагивая обычное краткосрочное состояние в реальном времени.

## Сигналы глубокой ранжировки

Глубокая ранжировка использует шесть взвешенных базовых сигналов плюс сигналы подкрепления фаз:

| Сигнал | Вес | Описание |
| ------------------- | ------ | ------------------------------------------------- |
| Частота | 0,24 | Количество краткосрочных сигналов, накопленных записью |
| Релевантность | 0,30 | Среднее качество извлечения для записи |
| Разнообразие запросов | 0,15 | Различные контексты запросов в день, в которых запись была выявлена |
| Актуальность | 0,15 | Оценка свежести с учётом временного затухания |
| Консолидация | 0,10 | Сила повторения в течение нескольких дней |
| Богатство концепций | 0,06 | Плотность тегов концепций из фрагмента/пути |

Попадания в лёгкой фазе и фазе REM добавляют небольшой бонус с учётом затухания актуальности из файла `memory/.dreams/phase-signals.json`.

## Планирование

При активации `memory-core` автоматически управляет одним заданием cron для полного цикла "сновидений". Каждый цикл выполняет фазы в следующем порядке: лёгкая → REM → глубокая.

Поведение по умолчанию:

| Настройка | Значение по умолчанию |
| -------------------- | ----------- |
| `dreaming.frequency` | `0 3 * * *` |

## Быстрый старт

Включите "Сновидения":

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true
          }
        }
      }
    }
  }
}
```

Включите "Сновидения" с пользовательским циклом выполнения:

```json
{
  "plugins": {
    "entries": {
      "memory-core": {
        "config": {
          "dreaming": {
            "enabled": true,
            "timezone": "America/Los_Angeles",
            "frequency": "0 */6 * * *"
          }
        }
      }
    }
  }
}
```

## Командная строка (slash-команды)

```
/dreaming status
/dreaming on
/dreaming off
/dreaming help
```

## Рабочий процесс через CLI

Используйте CLI для предварительного просмотра или ручного применения:

```bash
openclaw memory promote
openclaw memory promote --apply
openclaw memory promote --limit 5
openclaw memory status --deep
```

Команда `memory promote` вручную использует пороговые значения глубокой фазы по умолчанию, если они не переопределены флагами CLI.

Объясните, почему конкретный кандидат будет или не будет продвинут:

```bash
openclaw memory promote-explain "router vlan"
openclaw memory promote-explain "router vlan" --json
```

Предварительный просмотр рефлексий REM, истин кандидатов и вывода глубокого продвижения без записи каких-либо данных:

```bash
openclaw memory rem-harness
openclaw memory rem-harness --json
```

## Основные значения по умолчанию

Все настройки находятся в разделе `plugins.entries.memory-core.config.dreaming`.

| Ключ | Значение по умолчанию |
| ----------- | ----------- |
| `enabled` | `false` |
| `frequency` | `0 3 * * *` |

Политика фаз, пороговые значения и поведение хранилища являются деталями внутренней реализации (не настройками для пользователя).

Полный список ключей см. в [справочнике по конфигурации памяти](/reference/memory-config#dreaming-experimental).

## Интерфейс Dreams UI

При активации вкладка **Dreams** в Gateway отображает:

- текущее состояние активации "Сновидений";
- статус на уровне фаз и наличие управляемого цикла;
- количество краткосрочных, обоснованных, сигнальных и продвинутых сегодня записей;
- время следующего запланированного запуска;
- отдельный обоснованный канал сцены для подготовленных записей исторического воспроизведения;
