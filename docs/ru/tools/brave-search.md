# API Brave Search

OpenClaw поддерживает API Brave Search в качестве провайдера `web_search`.

## Получение API-ключа

1. Создайте учётную запись API Brave Search на [https://brave.com/search/api/](https://brave.com/search/api/).
2. В панели управления выберите тариф **Search** и сгенерируйте API-ключ.
3. Сохраните ключ в конфигурации либо задайте `BRAVE_API_KEY` в окружении Gateway.

## Пример конфигурации

```json5
{
  plugins: {
    entries: {
      brave: {
        config: {
          webSearch: {
            apiKey: "BRAVE_API_KEY_HERE",
            mode: "web", // или "llm-context"
          },
        },
      },
    },
  },
  tools: {
    web: {
      search: {
        provider: "brave",
        maxResults: 5,
        timeoutSeconds: 30,
      },
    },
  },
}
```

Специфичные для провайдера настройки поиска Brave теперь располагаются в `plugins.entries.brave.config.webSearch.*`. Устаревший параметр `tools.web.search.apiKey` по-прежнему загружается через слой совместимости, но больше не является каноническим путём к конфигурации.

Параметр `webSearch.mode` управляет транспортом Brave:

- `web` (по умолчанию): обычный веб-поиск Brave с заголовками, URL и фрагментами;
- `llm-context`: API Brave LLM Context с предварительно извлечёнными фрагментами текста и источниками для обоснования (grounding).

## Параметры инструмента

| Параметр | Описание |
| --- | --- |
| `query` | Поисковый запрос (обязательный) |
| `count` | Количество возвращаемых результатов (1–10, по умолчанию: 5) |
| `country` | Двухбуквенный код страны по ISO (например, "US", "DE") |
| `language` | Код языка по ISO 639-1 для результатов поиска (например, "en", "de", "fr") |
| `search_lang` | Код языка поиска Brave (например, `en`, `en-gb`, `zh-hans`) |
| `ui_lang` | Код языка по ISO для элементов интерфейса |
| `freshness` | Фильтр по времени: `day` (24 часа), `week`, `month` или `year` |
| `date_after` | Только результаты, опубликованные после указанной даты (ГГГГ-ММ-ДД) |
| `date_before` | Только результаты, опубликованные до указанной даты (ГГГГ-ММ-ДД) |

**Примеры:**

```javascript
// Поиск с указанием страны и языка
await web_search({
  query: "renewable energy",
  country: "DE",
  language: "de",
});

// Недавние результаты (за последнюю неделю)
await web_search({
  query: "AI news",
  freshness: "week",
});

// Поиск в заданном диапазоне дат
await web_search({
  query: "AI developments",
  date_after: "2024-01-01",
  date_before: "2024-06-30",
});
```

## Примечания

- OpenClaw использует тариф Brave **Search**. Если у вас устаревшая подписка (например, исходный бесплатный тариф с 2 000 запросов в месяц), она остаётся действительной, но не включает новые функции, такие как LLM Context или повышенные лимиты частоты запросов.
- Каждый тариф Brave включает **5 долларов США в месяц в виде бесплатного кредита** (обновляется автоматически). Тариф Search стоит 5 долларов за 1 000 запросов, то есть кредит покрывает 1 000 запросов в месяц. Установите лимит использования в панели управления Brave, чтобы избежать непредвиденных расходов. Актуальные тарифы см. на [портале API Brave](https://brave.com/search/api/).
- Тариф Search включает эндпоинт LLM Context и права на вывод с использованием ИИ. Для хранения результатов с целью обучения или настройки моделей требуется тариф с явными правами на хранение. См. [Условия использования Brave](https://api-dashboard.search.brave.com/terms-of-service).
- В режиме `llm-context` возвращаются обоснованные записи-источники вместо обычных фрагментов из веб-поиска.
- Режим `llm-context` не поддерживает параметры `ui_lang`, `freshness`, `date_after` и `date_before`.
- Для `ui_lang` необходимо указать субтег региона, например `en-US`.
- По умолчанию результаты кэшируются в течение 15 минут (можно настроить через `cacheTtlMinutes`).
