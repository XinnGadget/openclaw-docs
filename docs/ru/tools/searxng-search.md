# Поиск SearXNG

OpenClaw поддерживает [SearXNG](https://docs.searxng.org/) в качестве поставщика `web_search` с **самостоятельным размещением, без ключей**. SearXNG — это метапоисковая система с открытым исходным кодом, которая агрегирует результаты из Google, Bing, DuckDuckGo и других источников.

Преимущества:

- **Бесплатно и без ограничений** — не требуется API-ключ или коммерческая подписка
- **Конфиденциальность / изолированная сеть (air-gap)** — запросы никогда не покидают вашу сеть
- **Работает где угодно** — нет региональных ограничений, характерных для коммерческих поисковых API

## Настройка

<Steps>
  <Step title="Запустите экземпляр SearXNG">
    ```bash
    docker run -d -p 8888:8080 searxng/searxng
    ```

    Или используйте любой существующий развёрнутый экземпляр SearXNG, к которому у вас есть доступ. См. [документацию SearXNG](https://docs.searxng.org/) для настройки в продакшн-среде.

  </Step>
  <Step title="Настройте">
    ```bash
    openclaw configure --section web
    # Выберите "searxng" в качестве поставщика
    ```

    Или задайте переменную окружения, чтобы система автоматически обнаружила экземпляр:

    ```bash
    export SEARXNG_BASE_URL="http://localhost:8888"
    ```

  </Step>
</Steps>

## Конфигурация

```json5
{
  tools: {
    web: {
      search: {
        provider: "searxng",
      },
    },
  },
}
```

Настройки на уровне плагина для экземпляра SearXNG:

```json5
{
  plugins: {
    entries: {
      searxng: {
        config: {
          webSearch: {
            baseUrl: "http://localhost:8888",
            categories: "general,news", // необязательно
            language: "en", // необязательно
          },
        },
      },
    },
  },
}
```

Поле `baseUrl` также поддерживает объекты SecretRef.

Правила передачи данных:

- `https://` работает для публичных и приватных хостов SearXNG
- `http://` принимается только для доверенных хостов в приватной сети или для loopback-хостов
- публичные хосты SearXNG должны использовать `https://`

## Переменная окружения

Задайте `SEARXNG_BASE_URL` в качестве альтернативы конфигурации:

```bash
export SEARXNG_BASE_URL="http://localhost:8888"
```

Если `SEARXNG_BASE_URL` задана, а поставщик явно не настроен, система автоматически выбирает SearXNG (с наименьшим приоритетом — любой поставщик с API и ключом будет иметь приоритет).

## Справочник по конфигурации плагина

| Поле | Описание |
| ------------ | ------------------------------------------------------------------ |
| `baseUrl` | Базовый URL вашего экземпляра SearXNG (обязательно) |
| `categories` | Категории через запятую, например `general`, `news` или `science` |
| `language` | Код языка для результатов, например `en`, `de` или `fr` |

## Примечания

- **JSON API** — используется собственный эндпоинт SearXNG `format=json`, а не парсинг HTML
- **Без API-ключа** — работает с любым экземпляром SearXNG "из коробки"
- **Проверка базового URL** — `baseUrl` должен быть корректным URL с протоколом `http://` или `https://`; для публичных хостов обязательно использование `https://`
- **Порядок автоматического обнаружения** — SearXNG проверяется последним (порядок 200) при автоматическом обнаружении. Сначала запускаются поставщики с API и настроенными ключами, затем DuckDuckGo (порядок 100), затем Ollama Web Search (порядок 110)
- **Самостоятельное размещение** — вы контролируете экземпляр, запросы и поисковые системы верхнего уровня
- **Категории** по умолчанию принимают значение `general`, если не настроены

<Tip>
  Чтобы JSON API SearXNG работал, убедитесь, что в вашем экземпляре SearXNG включён формат `json` в файле `settings.yml` в разделе `search.formats`.
</Tip>
