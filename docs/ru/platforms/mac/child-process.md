# Жизненный цикл шлюза на macOS

Приложение для macOS по умолчанию **управляет шлюзом через launchd** и не запускает шлюз в качестве дочернего процесса. Сначала оно пытается подключиться к уже работающему шлюзу на настроенном порту; если подключиться не удаётся, оно активирует службу launchd с помощью внешней CLI‑утилиты `openclaw` (без встроенного рантайма). Это обеспечивает надёжный автоматический запуск при входе в систему и перезапуск при сбоях.

Режим дочернего процесса (когда шлюз запускается непосредственно приложением) в настоящее время **не используется**. Если требуется более тесная интеграция с пользовательским интерфейсом, запустите шлюз вручную в терминале.

## Поведение по умолчанию (launchd)

- Приложение устанавливает LaunchAgent для конкретного пользователя с меткой `ai.openclaw.gateway` (или `ai.openclaw.<profile>` при использовании параметров `--profile`/`OPENCLAW_PROFILE`; поддерживается устаревший формат `com.openclaw.*`).
- Когда включён локальный режим, приложение гарантирует загрузку LaunchAgent и при необходимости запускает шлюз.
- Журналы записываются в путь к журналу шлюза launchd (можно увидеть в настройках отладки).

Распространённые команды:

```bash
launchctl kickstart -k gui/$UID/ai.openclaw.gateway
launchctl bootout gui/$UID/ai.openclaw.gateway
```

Замените метку на `ai.openclaw.<profile>`, если используете именованный профиль.

## Несигнатурные сборки для разработки

Скрипт `scripts/restart-mac.sh --no-sign` предназначен для быстрых локальных сборок, когда у вас нет ключей подписи. Чтобы launchd не ссылался на несигнатурный бинарный файл ретранслятора, скрипт:

- Записывает файл `~/.openclaw/disable-launchagent`.

Сигнатурные запуски `scripts/restart-mac.sh` удаляют это переопределение, если маркер присутствует. Чтобы сбросить настройки вручную:

```bash
rm ~/.openclaw/disable-launchagent
```

## Режим только подключения (attach-only)

Чтобы заставить приложение для macOS **никогда не устанавливать и не управлять launchd**, запустите его с параметром `--attach-only` (или `--no-launchd`). Это установит файл `~/.openclaw/disable-launchagent`, и приложение будет только подключаться к уже работающему шлюзу. Вы можете включить такое же поведение в настройках отладки.

## Удаленный режим

В удалённом режиме локальный шлюз никогда не запускается. Приложение использует SSH‑туннель к удалённому хосту и подключается через этот туннель.

## Почему мы предпочитаем launchd

- Автоматический запуск при входе в систему.
- Встроенная семантика перезапуска/KeepAlive.
- Предсказуемые журналы и мониторинг.

Если когда‑либо снова потребуется настоящий режим дочернего процесса, его следует задокументировать как отдельный, явно обозначенный режим только для разработчиков.
