# Bun (Экспериментальный)

<Warning>
Bun **не рекомендуется для использования в качестве среды выполнения шлюза** (известны проблемы с WhatsApp и Telegram). Для продакшена используйте Node.
</Warning>

Bun — это необязательная локальная среда выполнения для прямого запуска TypeScript (`bun run ...`, `bun --watch ...`). Менеджером пакетов по умолчанию остаётся `pnpm` — он полностью поддерживается и используется в инструментах документации. Bun не может использовать `pnpm-lock.yaml` и будет игнорировать этот файл.

## Установка

<Steps>
  <Step title="Установить зависимости">
    ```sh
    bun install
    ```

    Файлы `bun.lock` / `bun.lockb` игнорируются git, поэтому изменения в репозитории не будут возникать. Чтобы полностью избежать записи в файл блокировки, используйте:

    ```sh
    bun install --no-save
    ```

  </Step>
  <Step title="Собрать и протестировать">
    ```sh
    bun run build
    bun run vitest run
    ```
  </Step>
</Steps>

## Скрипты жизненного цикла

Bun блокирует скрипты жизненного цикла зависимостей, если они явно не помечены как доверенные. Для данного репозитория обычно блокируемые скрипты не требуются:

- `@whiskeysockets/baileys` `preinstall` — проверяет, что основная версия Node ≥ 20 (OpenClaw по умолчанию использует Node 24 и по-прежнему поддерживает Node 22 LTS, в настоящее время `22.14+`)
- `protobufjs` `postinstall` — выводит предупреждения о несовместимых схемах версий (без артефактов сборки)

Если вы столкнулись с проблемой во время выполнения, требующей этих скриптов, явно пометьте их как доверенные:

```sh
bun pm trust @whiskeysockets/baileys protobufjs
```

## Предупреждения

Некоторые скрипты по-прежнему жёстко закодированы под pnpm (например, `docs:build`, `ui:*`, `protocol:check`). Пока что запускайте их через pnpm.
