# Конфигурация

OpenClaw считывает необязательную конфигурацию в формате **JSON5** из файла `~/.openclaw/openclaw.json`.

Если файл отсутствует, OpenClaw использует безопасные значения по умолчанию. Типичные причины для добавления конфигурации:

- Подключение каналов и управление тем, кто может отправлять сообщения боту.
- Настройка моделей, инструментов, песочниц или автоматизации (cron, хуки).
- Настройка сессий, медиа, сети или пользовательского интерфейса.

См. [полную справочную информацию](/gateway/configuration-reference) обо всех доступных полях.

<Tip>
**Новичкам в настройке конфигурации?** Начните с команды `openclaw onboard` для интерактивной настройки или ознакомьтесь с руководством [Примеры конфигурации](/gateway/configuration-examples), где представлены готовые к использованию конфигурации.
</Tip>

## Минимальная конфигурация

```json5
// ~/.openclaw/openclaw.json
{
  agents: { defaults: { workspace: "~/.openclaw/workspace" } },
  channels: { whatsapp: { allowFrom: ["+15555550123"] } },
}
```

## Редактирование конфигурации

<Tabs>
  <Tab title="Интерактивный мастер">
    ```bash
    openclaw onboard       # полный процесс настройки
    openclaw configure     # мастер настройки конфигурации
    ```
  </Tab>
  <Tab title="CLI (однострочники)">
    ```bash
    openclaw config get agents.defaults.workspace
    openclaw config set agents.defaults.heartbeat.every "2h"
    openclaw config unset plugins.entries.brave.config.webSearch.apiKey
    ```
  </Tab>
  <Tab title="Пользовательский интерфейс управления">
    Откройте [http://127.0.0.1:18789](http://127.0.0.1:18789) и используйте вкладку **Config**.
    Пользовательский интерфейс управления отображает форму на основе текущей схемы конфигурации, включая метаданные полей `title` / `description`, а также схемы плагинов и каналов (если доступны). Также имеется редактор **Raw JSON** для прямого редактирования. Для детализированных пользовательских интерфейсов и других инструментов шлюз предоставляет метод `config.schema.lookup` для получения узла схемы, ограниченного путём, вместе с краткими сведениями о непосредственных дочерних элементах.
  </Tab>
  <Tab title="Прямое редактирование">
    Отредактируйте файл `~/.openclaw/openclaw.json` напрямую. Шлюз отслеживает файл и применяет изменения автоматически (см. [горячую перезагрузку](#config-hot-reload)).
  </Tab>
</Tabs>

## Строгая проверка

<Warning>
OpenClaw принимает только конфигурации, полностью соответствующие схеме. Неизвестные ключи, неверно сформированные типы или недопустимые значения приводят к тому, что шлюз **отказывается запускаться**. Единственное исключение на корневом уровне — `$schema` (строка), чтобы редакторы могли прикреплять метаданные JSON Schema.
</Warning>

Примечания по инструментам работы со схемой:

- `openclaw config schema` выводит ту же семью схем JSON Schema, которая используется в пользовательском интерфейсе управления и при проверке конфигурации.
- Рассматривайте вывод схемы как канонический машиночитаемый контракт для `openclaw.json`; данный обзор и справочная информация по конфигурации представляют собой его краткое изложение.
- Значения полей `title` и `description` переносятся в вывод схемы для использования в редакторах и формах.
- Вложенные объекты, подстановочные знаки (`*`) и элементы массива (`[]`) наследуют те же метаданные документации, если существует соответствующая документация по полям.
- Ветви композиции `anyOf` / `oneOf` / `allOf` также наследуют те же метаданные документации, поэтому варианты объединения/пересечения сохраняют ту же справку по полям.
- `config.schema.lookup` возвращает один нормализованный путь конфигурации с узлом схемы (поля `title`, `description`, `type`, `enum`, `const`, общие ограничения и аналогичные поля проверки), метаданными подсказок для пользовательского интерфейса и краткими сведениями о непосредственных дочерних элементах для детализированных инструментов.
- Схемы плагинов/каналов во время выполнения объединяются, когда шлюз может загрузить текущий реестр манифестов.
- `pnpm config:docs:check` обнаруживает расхождения между артефактами базовой конфигурации, ориентированными на документацию, и текущей поверхностью схемы.

При сбое проверки:

- Шлюз не запускается.
- Работают только диагностические команды (`openclaw doctor`, `openclaw logs`, `openclaw health`, `openclaw status`).
- Запустите `openclaw doctor`, чтобы увидеть точные проблемы.
- Запустите `openclaw doctor --fix` (или `--yes`), чтобы применить исправления.

## Типичные задачи

<AccordionGroup>
  <Accordion title="Настройка канала (WhatsApp, Telegram, Discord и т. д.)">
    Каждый канал имеет свой раздел конфигурации в `channels.<provider>`. См. специальную страницу канала для шагов по настройке:

    - [WhatsApp](/channels/whatsapp) — `channels.whatsapp`
    - [Telegram](/channels/telegram) — `channels.telegram`
    - [Discord](/channels/discord) — `channels.discord`
    - [Feishu](/channels/feishu) — `channels.feishu`
    - [Google Chat](/channels/googlechat) — `channels.googlechat`
    - [Microsoft Teams](/channels/msteams) — `channels.msteams`
    - [Slack](/channels/slack) — `channels.slack`
    - [Signal](/channels/signal) — `channels.signal`
    - [iMessage](/channels/imessage) — `channels.imessage`
    - [Mattermost](/channels/mattermost) — `channels.mattermost`

    Все каналы используют один и тот же шаблон политики прямых сообщений:

    ```json5
    {
      channels: {
        telegram: {
          enabled: true,
          botToken: "123:abc",
          dmPolicy: "pairing",   // pairing | allowlist | open | disabled
          allowFrom: ["tg:123"], // только для allowlist/open
        },
      },
    }
    ```

  </Accordion>

  <Accordion title="Выбор и настройка моделей">
    Задайте основную модель и дополнительные резервные модели:

    ```json5
    {
      agents: {
        defaults: {
          model: {
            primary: "anthropic/claude-sonnet-4-6",
            fallbacks: ["openai/gpt-5.4"],
          },
          models: {
            "anthropic/claude-sonnet-4-6": { alias: "Sonnet" },
            "openai/gpt-5.4": { alias: "GPT" },
          },
        },
      },
    }
    ```

    - `agents.defaults.models` определяет каталог моделей и действует как белый список для `/model`.
    - Ссылки на модели используют формат `provider/model` (например, `anthropic/claude-opus-4-6`).
    - `agents.defaults.imageMaxDimensionPx` управляет уменьшением размера изображений в транскриптах/инструментах (по умолчанию `1200`); меньшие значения обычно снижают использование токенов зрения при работе с большим количеством скриншотов.
    - См. [CLI моделей](/concepts/models) для переключения моделей в чате и [Отказоустойчивость моделей](/concepts/model-failover) для ротации аутентификации и поведения при отказоустойчивости.
    - Для пользовательских/самостоятельных провайдеров см. [Пользовательские провайдеры](/gateway/configuration-reference#custom-providers-and-base-urls) в справочной информации.

  </Accordion>

  <Accordion title="Управление тем, кто может отправлять сообщения боту">
    Доступ к прямым сообщениям контролируется для каждого канала через `dmPolicy`:

    - `"pairing"` (по умолчанию): неизвестные отправители получают одноразовый код для подтверждения
    - `"allowlist"`: только отправители из `allowFrom` (или из хранилища парных разрешений)
    - `"open"`: разрешены все входящие прямые сообщения (требуется `allowFrom: ["*"]`)
    - `"disabled"`: игнорируются все прямые сообщения

    Для групп используйте `groupPolicy` + `groupAllowFrom` или списки разрешений для конкретных каналов.

    См. [полную справочную информацию](/gateway/configuration-reference#dm-and-group-access) для деталей по каждому каналу.

  </Accordion>
