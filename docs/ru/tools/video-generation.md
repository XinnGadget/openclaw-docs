# Генерация видео

Агенты OpenClaw могут создавать видео на основе текстовых подсказок, эталонных изображений или существующих видео. Поддерживается 12 бэкенд-провайдеров, каждый из которых предлагает различные варианты моделей, режимы ввода и наборы функций. Агент автоматически выбирает подходящего провайдера на основе вашей конфигурации и доступных API-ключей.

<Note>
Инструмент `video_generate` отображается только при наличии хотя бы одного провайдера генерации видео. Если вы не видите его среди инструментов агента, установите API-ключ провайдера или настройте `agents.defaults.videoGenerationModel`.
</Note>

OpenClaw рассматривает генерацию видео как три режима выполнения:

- `generate` — для запросов "текст-в-видео" без эталонных медиафайлов;
- `imageToVideo` — если запрос включает одно или несколько эталонных изображений;
- `videoToVideo` — если запрос включает одно или несколько эталонных видео.

Провайдеры могут поддерживать любое подмножество этих режимов. Инструмент проверяет активный режим перед отправкой и сообщает о поддерживаемых режимах в `action=list`.

## Быстрый старт

1. Установите API-ключ для любого поддерживаемого провайдера:

```bash
export GEMINI_API_KEY="your-key"
```

2. При необходимости закрепите модель по умолчанию:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "google/veo-3.1-fast-generate-preview"
```

3. Задайте вопрос агенту:

> Создайте 5-секундное кинематографическое видео, на котором дружелюбный лобстер занимается сёрфингом на закате.

Агент автоматически вызывает `video_generate`. Разрешения на использование инструмента не требуются.

## Что происходит при генерации видео

Генерация видео выполняется асинхронно. Когда агент вызывает `video_generate` в сессии:

1. OpenClaw отправляет запрос провайдеру и сразу возвращает идентификатор задачи (task ID).
2. Провайдер обрабатывает задание в фоновом режиме (обычно от 30 секунд до 5 минут — в зависимости от провайдера и разрешения).
3. Когда видео готово, OpenClaw возобновляет ту же сессию с помощью внутреннего события завершения.
4. Агент отправляет готовое видео обратно в исходный диалог.

Пока задание выполняется, повторные вызовы `video_generate` в той же сессии возвращают текущий статус задачи вместо запуска новой генерации. Используйте `openclaw tasks list` или `openclaw tasks show <taskId>`, чтобы проверить прогресс из интерфейса командной строки (CLI).

Вне сессионных запусков агента (например, при прямом вызове инструмента) инструмент переключается на встроенную генерацию и возвращает путь к итоговому медиафайлу в том же шаге.

### Жизненный цикл задачи

Каждый запрос `video_generate` проходит через четыре состояния:

1. **queued** — задача создана, ожидает принятия провайдером.
2. **running** — провайдер обрабатывает задачу (обычно от 30 секунд до 5 минут — в зависимости от провайдера и разрешения).
3. **succeeded** — видео готово; агент возобновляет работу и отправляет его в диалог.
4. **failed** — ошибка провайдера или тайм-аут; агент возобновляет работу с деталями ошибки.

Проверьте статус из CLI:

```bash
openclaw tasks list
openclaw tasks show <taskId>
openclaw tasks cancel <taskId>
```

Предотвращение дублирования: если видеозадача уже находится в состоянии `queued` или `running` для текущей сессии, `video_generate` возвращает статус существующей задачи вместо запуска новой. Используйте `action: "status"`, чтобы явно проверить статус без запуска новой генерации.

## Параметры инструмента

### Обязательные

| Параметр | Тип | Описание |
| --------- | ------ | ----------------------------------------------------------------------------- |
| `prompt` | string | Текстовое описание видео для генерации (обязательно для `action: "generate"`) |

### Входные данные контента

| Параметр | Тип | Описание |
| --------- | -------- | ------------------------------------ |
| `image` | string | Одно эталонное изображение (путь или URL) |
| `images` | string[] | Несколько эталонных изображений (до 5) |
| `video` | string | Одно эталонное видео (путь или URL) |
| `videos` | string[] | Несколько эталонных видео (до 4) |

### Управление стилем

| Параметр | Тип | Описание |
| ----------------- | ------- | ----------------------------------------------------------------- |
| `aspectRatio` | string | `1:1`, `2:3`, `3:2`, `3:4`, `4:3`, `4:5`, `5:4`, `9:16`, `16:9`, `21:9` |
| `resolution` | string | `480P`, `720P`, `768P` или `1080P` |
| `durationSeconds` | number | Целевая длительность в секундах (округляется до ближайшего поддерживаемого провайдером значения) |
| `size` | string | Подсказка по размеру, если провайдер её поддерживает |
| `audio` | boolean | Включить сгенерированный звук, если поддерживается |
| `watermark` | boolean | Включить водяные знаки провайдера, если поддерживается |

### Расширенные

| Параметр | Тип | Описание |
| ---------- | ------ | ----------------------------------------------- |
| `action` | string | `"generate"` (по умолчанию), `"status"` или `"list"` |
| `model` | string | Переопределение провайдера/модели (например, `runway/gen4.5`) |
| `filename` | string | Подсказка по имени выходного файла |

Не все провайдеры поддерживают все параметры. OpenClaw уже нормализует длительность до ближайшего поддерживаемого провайдером значения, а также переназначает переведённые подсказки по геометрии (например, размер в соотношение сторон), когда запасной провайдер предоставляет другой интерфейс управления. Действительно неподдерживаемые переопределения игнорируются по принципу "best-effort" и сообщаются как предупреждения в результате работы инструмента. Жёсткие ограничения возможностей (например, слишком много эталонных входных данных) приводят к ошибке до отправки запроса.

Результаты работы инструмента сообщают о применённых настройках. Когда OpenClaw переназначает длительность или геометрию во время переключения на запасной провайдер, возвращаемые значения `durationSeconds`, `size`, `aspectRatio` и `resolution` отражают то, что было отправлено, а `details.normalization` фиксирует преобразование от запрошенного к применённому.

Эталонные входные данные также выбирают режим выполнения:

- Нет эталонных медиафайлов: `generate`.
- Любое эталонное изображение: `imageToVideo`.
- Любое эталонное видео: `videoToVideo`.

Смешанные эталонные изображения и видео не являются стабильной общей возможностью. Рекомендуется использовать один тип эталонных данных на запрос.

## Действия

- **generate** (по умолчанию) — создать видео на основе заданного промпта и необязательных эталонных входных данных.
- **status** — проверить состояние выполняемой видеозадачи для текущей сессии без запуска новой генерации.
- **list** — показать доступных провайдеров, модели и их возможности.

## Выбор модели

При генерации видео OpenClaw определяет модель в следующем порядке:

1. **Параметр инструмента `model`** — если агент указывает его в вызове.
2. **`videoGenerationModel.primary`** — из конфигурации.
3. **`videoGenerationModel.fallbacks`** — проверяются по порядку.
4. **Автообнаружение** — используются провайдеры с действительной аутентификацией, начиная с текущего провайдера по умолчанию, затем остальные провайдеры в алфавитном порядке.

Если провайдер выдаёт ошибку, автоматически пробуется следующий кандидат. Если все кандидаты выдают ошибку, в сообщении об ошибке будут приведены детали каждой попытки.

Установите `agents.defaults.mediaGenerationAutoProviderFallback: false`, если хотите, чтобы генерация видео использовала только явные записи `model`, `primary` и `fallbacks`.

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "google/veo-3.1-fast-generate-preview",
        fallbacks: ["runway/gen4.5", "qwen/wan2.6-t2v"],
      },
    },
  },
}
```

## Примечания к провайдерам

| Провайдер | Примечания |
| -------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Alibaba | Использует асинхронный эндпоинт DashScope/Model Studio. Эталонные изображения и видео должны быть доступны по удалённым URL-адресам `http(s)`. |
| BytePlus | Поддерживается только одно эталонное изображение. |
| ComfyUI | Локальное или облачное выполнение на основе рабочего процесса. Поддерживает преобразование "текст-в-видео" и "изображение-в-видео" через настроенный граф. |
| fal | Использует очередь для длительных заданий. Поддерживается только одно эталонное изображение. |
| Google | Использует Gemini/Veo. Поддерживает одно эталонное изображение или одно эталонное видео. |
| MiniMax | Поддерживается только одно эталонное изображение. |
| OpenAI | Передаётся только переопределение `size`. Другие переопределения стиля (`aspectRatio`, `resolution`, `audio`, `watermark`) игнорируются с предупреждением. |
| Qwen | Тот же бэкенд DashScope, что и у Alibaba. Эталонные входные данные должны быть доступны по удалённым URL-адресам `http(s)`; локальные файлы отклоняются заранее. |
| Runway | Поддерживает локальные файлы через data URI. Для преобразования "видео-в-видео" требуется `runway/gen4_aleph`. При работе только с текстом доступны соотношения сторон `16:9` и `9:16`. |
| Together | Поддерживается только одно эталонное изображение. |
| Vydra | Использует `https://www.vydra.ai/api/v1` напрямую, чтобы избежать перенаправлений с потерей аутентификации. `veo3` поставляется только для преобразования "текст-в-видео"; `kling` требует удалённого URL-адреса изображения. |
| xAI | Поддерживает преобразования "текст-в-видео", "изображение-в-видео" и редактирование/расширение удалённого видео. |

## Режимы возможностей провайдеров

Общий контракт генерации видео теперь позволяет провайдерам объявлять возможности для конкретных режимов вместо только общих совокупных ограничений. Новым реализациям провайдеров следует отдавать предпочтение явным блокам режимов:

```typescript
capabilities: {
  generate: {
    maxVideos: 1,
    maxDurationSeconds: 10,
    supportsResolution: true,
  },
  imageToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputImages: 1,
    maxDurationSeconds: 5,
  },
  videoToVideo: {
    enabled: true,
    maxVideos: 1,
    maxInputVideos: 1,
    maxDurationSeconds: 5,
  },
}
```

Плоских совокупных полей, таких как `maxInputImages` и `maxInputVideos`, недостаточно для объявления поддержки режимов преобразования. Провайдеры должны явно объявлять `generate`, `imageToVideo` и `videoToVideo`, чтобы живые тесты, контрактные тесты и общий инструмент `video_generate` могли детерминированно проверять поддержку режимов.

## Живые тесты

Участие в живом покрытии для общих встроенных провайдеров:

```bash
OPENCLAW_LIVE_TEST=1 pnpm test:live -- extensions/video-generation-providers.live.test.ts
```

Обёртка репозитория:

```bash
pnpm test:live:media video
```

Этот живой файл загружает отсутствующие переменные окружения провайдера из `~/.profile`, по умолчанию отдаёт предпочтение живым/переменным окружения API-ключей перед сохранёнными профилями аутентификации и запускает объявленные режимы, которые можно безопасно протестировать с локальными медиаданными:

- `generate` для каждого провайдера в проходе;
- `imageToVideo`, если `capabilities.imageToVideo.enabled`;
- `videoToVideo`, если `capabilities.videoToVideo.enabled` и провайдер/модель принимают локальные видеоданные в буфере в рамках общего прохода.

На текущий момент общий канал `videoToVideo` в живом режиме охватывает:

- `runway` — только при выборе `runway/gen4_aleph`.

## Конфигурация

Установите модель генерации видео по умолчанию в конфигурации OpenClaw:

```json5
{
  agents: {
    defaults: {
      videoGenerationModel: {
        primary: "qwen/wan2.6-t2v",
        fallbacks: ["qwen/wan2.6-r2v-flash"],
      },
    },
  },
}
```

Или через CLI:

```bash
openclaw config set agents.defaults.videoGenerationModel.primary "qwen/wan2.6-t2v"
```
