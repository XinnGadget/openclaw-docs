## Быстрая настройка

Вам нужно создать новое приложение с ботом, добавить бота на свой сервер и сопрячь его с OpenClaw. Мы рекомендуем добавить бота на собственный приватный сервер. Если у вас его ещё нет, [создайте его сначала](https://support.discord.com/hc/en-us/articles/204849977-How-do-I-create-a-server) (выберите **Create My Own > For me and my friends**).

<Steps>
  <Step title="Создайте приложение и бота в Discord">
    Перейдите на [портал разработчика Discord](https://discord.com/developers/applications) и нажмите **New Application**. Назовите его, например, "OpenClaw".

    Нажмите **Bot** в боковой панели. Задайте **Username** — имя, под которым будет выступать ваш агент OpenClaw.
  </Step>

  <Step title="Включите привилегированные намерения (intents)">
    На странице **Bot** прокрутите вниз до **Privileged Gateway Intents** и включите:

    - **Message Content Intent** (обязательно);
    - **Server Members Intent** (рекомендуется; необходимо для списков разрешённых ролей и сопоставления имени с ID);
    - **Presence Intent** (необязательно; требуется только для обновлений присутствия).
  </Step>

  <Step title="Скопируйте токен бота">
    Прокрутите вверх на странице **Bot** и нажмите **Reset Token**.

    <Note>
    Несмотря на название, это действие сгенерирует ваш первый токен — ничего не "сбрасывается".
    </Note>

    Скопируйте токен и сохраните его. Это ваш **Bot Token**, он понадобится вам в ближайшее время.
  </Step>

  <Step title="Сгенерируйте URL-адрес для приглашения и добавьте бота на сервер">
    Нажмите **OAuth2** в боковой панели. Вы сгенерируете URL-адрес для приглашения с необходимыми разрешениями для добавления бота на сервер.

    Прокрутите вниз до **OAuth2 URL Generator** и включите:

    - `bot`;
    - `applications.commands`.

    Ниже появится раздел **Bot Permissions**. Включите:

    - View Channels;
    - Send Messages;
    - Read Message History;
    - Embed Links;
    - Attach Files;
    - Add Reactions (необязательно).

    Скопируйте сгенерированный URL внизу, вставьте его в браузер, выберите свой сервер и нажмите **Continue**, чтобы подключиться. Теперь вы должны увидеть бота на сервере Discord.
  </Step>

  <Step title="Включите режим разработчика и соберите ID">
    Вернитесь в приложение Discord и включите режим разработчика, чтобы скопировать внутренние ID.

    1. Нажмите **User Settings** (значок шестерёнки рядом с вашим аватаром) → **Advanced** → включите **Developer Mode**.
    2. Щёлкните правой кнопкой мыши по **значку сервера** в боковой панели → **Copy Server ID**.
    3. Щёлкните правой кнопкой мыши по **своему аватару** → **Copy User ID**.

    Сохраните **Server ID** и **User ID** вместе с Bot Token — вы отправите все три значения в OpenClaw на следующем шаге.
  </Step>

  <Step title="Разрешите личные сообщения от участников сервера">
    Чтобы сопряжение работало, Discord должен разрешать боту отправлять вам личные сообщения. Щёлкните правой кнопкой мыши по **значку сервера** → **Privacy Settings** → включите **Direct Messages**.

    Это позволит участникам сервера (включая ботов) отправлять вам личные сообщения. Оставьте эту настройку включённой, если планируете использовать личные сообщения Discord с OpenClaw. Если вы планируете использовать только каналы гильдии, вы можете отключить личные сообщения после сопряжения.
  </Step>

  <Step title="Безопасно задайте токен бота (не отправляйте его в чате)">
    Токен вашего бота Discord — это секретная информация (как пароль). Задайте его на машине, на которой работает OpenClaw, до того, как будете общаться с агентом.

```bash
export DISCORD_BOT_TOKEN="YOUR_BOT_TOKEN"
openclaw config set channels.discord.token --ref-provider default --ref-source env --ref-id DISCORD_BOT_TOKEN --dry-run
openclaw config set channels.discord.token --ref-provider default --ref-source env --ref-id DISCORD_BOT_TOKEN
openclaw config set channels.discord.enabled true --strict-json
openclaw gateway
```

    Если OpenClaw уже работает как фоновая служба, перезапустите его через приложение OpenClaw для Mac или остановив и перезапустив процесс `openclaw gateway run`.
  </Step>

  <Step title="Настройте OpenClaw и выполните сопряжение">

    <Tabs>
      <Tab title="Спросите своего агента">
        Пообщайтесь со своим агентом OpenClaw в любом существующем канале (например, в Telegram) и скажите ему. Если Discord — ваш первый канал, используйте CLI или вкладку конфигурации.

        > "Я уже задал токен бота Discord в конфигурации. Пожалуйста, завершите настройку Discord с User ID `<user_id>` и Server ID `<server_id>`".
      </Tab>
      <Tab title="CLI / конфигурация">
        Если вы предпочитаете конфигурацию на основе файлов, задайте:

```json5
{
  channels: {
    discord: {
      enabled: true,
      token: {
        source: "env",
        provider: "default",
        id: "DISCORD_BOT_TOKEN",
      },
    },
  },
}
```

        Резервное значение из окружения для учётной записи по умолчанию:

```bash
DISCORD_BOT_TOKEN=...
```

        Поддерживаются значения токена в виде открытого текста. Также поддерживаются значения SecretRef для `channels.discord.token` в провайдерах env/file/exec. См. [Управление секретами](/gateway/secrets).
      </Tab>
    </Tabs>
  </Step>

  <Step title="Одобрите первое сопряжение через личные сообщения">
    Дождитесь запуска шлюза, затем отправьте личное сообщение боту в Discord. Он ответит кодом сопряжения.

    <Tabs>
      <Tab title="Спросите своего агента">
        Отправьте код сопряжения своему агенту в существующем канале:

        > "Одобрите этот код сопряжения Discord: `<CODE>`".
      </Tab>
      <Tab title="CLI">

```bash
openclaw pairing list discord
openclaw pairing approve discord <CODE>
```
      </Tab>
    </Tabs>

    Коды сопряжения истекают через 1 час.

    Теперь вы сможете общаться со своим агентом в Discord через личные сообщения.
  </Step>
</Steps>

<Note>
Разрешение токена зависит от учётной записи. Значения токена в конфигурации имеют приоритет над резервными значениями из окружения. `DISCORD_BOT_TOKEN` используется только для учётной записи по умолчанию.
Для расширенных исходящих вызовов (инструменты сообщений/действия каналов) для каждого вызова используется явный токен. Это относится к действиям по отправке и чтению/пробному доступу (например, read/search/fetch/thread/pins/permissions). Настройки политики учётной записи/повторных попыток по-прежнему берутся из выбранной учётной записи в активном снимке состояния.
</Note>

## Рекомендуется: настройте рабочее пространство гильдии

После того как личные сообщения будут работать, вы можете настроить свой сервер Discord как полноценное рабочее пространство, где для каждого канала будет своя сессия агента со своим контекстом. Это рекомендуется для приватных серверов, где находитесь только вы и ваш бот.

<Steps>
  <Step title="Добавьте свой сервер в список разрешённых гильдий">
    Это позволит вашему агенту отвечать в любом канале на вашем сервере, а не только в личных сообщениях.

    <Tabs>
      <Tab title="Спросите своего агента">
        > "Добавьте мой ID сервера Discord `<server_id>` в список разрешённых гильдий".
      </Tab>
      <Tab title="Конфигурация">

```json5
{
  channels: {
    discord: {
      groupPolicy: "allowlist",
      guilds: {
        YOUR_SERVER_ID: {
          requireMention: true,
          users: ["YOUR_USER_ID"],
        },
      },
    },
  },
}
```
      </Tab>
