# Tlon

Tlon — децентрализованный мессенджер, построенный на платформе Urbit. OpenClaw подключается к вашему кораблю Urbit и может отвечать на личные сообщения (DM) и сообщения в групповых чатах. По умолчанию для ответов в группах требуется упоминание через @, также доступ можно дополнительно ограничить с помощью белых списков.

Статус: плагин, включённый в комплект поставки. Поддерживаются личные сообщения, упоминания в группах, ответы в темах, форматирование текста с расширенными возможностями и загрузка изображений. Реакции и опросы пока не поддерживаются.

## Плагин, включённый в комплект поставки

В текущих версиях OpenClaw плагин Tlon включён в комплект поставки, поэтому для стандартных сборок отдельная установка не требуется.

Если вы используете более старую сборку или пользовательскую установку, в которой отсутствует Tlon, установите его вручную:

Установка через CLI (реестр npm):

```bash
openclaw plugins install @openclaw/tlon
```

Локальная проверка (при запуске из репозитория git):

```bash
openclaw plugins install ./path/to/local/tlon-plugin
```

Подробности: [Плагины](/tools/plugin)

## Настройка

1. Убедитесь, что плагин Tlon доступен:
   - В текущих упакованных версиях OpenClaw он уже включён.
   - В более старых или пользовательских установках его можно добавить вручную с помощью команд, приведённых выше.
2. Соберите URL вашего корабля и код для входа.
3. Настройте `channels.tlon`.
4. Перезапустите шлюз.
5. Отправьте боту личное сообщение или упомяните его в групповом канале.

Минимальная конфигурация (один аккаунт):

```json5
{
  channels: {
    tlon: {
      enabled: true,
      ship: "~sampel-palnet",
      url: "https://your-ship-host",
      code: "lidlut-tabwed-pillex-ridrup",
      ownerShip: "~your-main-ship", // рекомендуется: ваш корабль, всегда разрешён
    },
  },
}
```

## Частные/LAN-корабли

По умолчанию OpenClaw блокирует частные/внутренние имена хостов и диапазоны IP-адресов для защиты от SSRF. Если ваш корабль работает в частной сети (localhost, LAN IP или внутреннее имя хоста), вам необходимо явно разрешить это:

```json5
{
  channels: {
    tlon: {
      url: "http://localhost:8080",
      allowPrivateNetwork: true,
    },
  },
}
```

Это относится к URL-адресам типа:

- `http://localhost:8080`
- `http://192.168.x.x:8080`
- `http://my-ship.local:8080`

⚠️ Включайте эту настройку только в том случае, если вы доверяете своей локальной сети. Эта настройка отключает защиту от SSRF для запросов к URL вашего корабля.

## Групповые каналы

Автообнаружение включено по умолчанию. Вы также можете закрепить каналы вручную:

```json5
{
  channels: {
    tlon: {
      groupChannels: ["chat/~host-ship/general", "chat/~host-ship/support"],
    },
  },
}
```

Отключение автообнаружения:

```json5
{
  channels: {
    tlon: {
      autoDiscoverChannels: false,
    },
  },
}
```

## Контроль доступа

Белый список для личных сообщений (пустой = личные сообщения запрещены, используйте `ownerShip` для потока утверждений):

```json5
{
  channels: {
    tlon: {
      dmAllowlist: ["~zod", "~nec"],
    },
  },
}
```

Авторизация в группах (по умолчанию ограничена):

```json5
{
  channels: {
    tlon: {
      defaultAuthorizedShips: ["~zod"],
      authorization: {
        channelRules: {
          "chat/~host-ship/general": {
            mode: "restricted",
            allowedShips: ["~zod", "~nec"],
          },
          "chat/~host-ship/announcements": {
            mode: "open",
          },
        },
      },
    },
  },
}
```

## Владелец и система утверждений

Задайте корабль-владелец, чтобы получать запросы на утверждение, когда неавторизованные пользователи пытаются взаимодействовать:

```json5
{
  channels: {
    tlon: {
      ownerShip: "~your-main-ship",
    },
  },
}
```

Корабль-владелец **автоматически авторизован везде** — приглашения в личные сообщения принимаются автоматически, а сообщения в каналах всегда разрешены. Вам не нужно добавлять владельца в `dmAllowlist` или `defaultAuthorizedShips`.

При настройке владелец получает уведомления в личных сообщениях о:

- запросах в личные сообщения от кораблей, не включённых в белый список;
- упоминаниях в каналах без авторизации;
- запросах на приглашение в группу.

## Настройки автоматического принятия

Автоматическое принятие приглашений в личные сообщения (для кораблей, включённых в `dmAllowlist`):

```json5
{
  channels: {
    tlon: {
      autoAcceptDmInvites: true,
    },
  },
}
```

Автоматическое принятие приглашений в группы:

```json5
{
  channels: {
    tlon: {
      autoAcceptGroupInvites: true,
    },
  },
}
```

## Цели доставки (CLI/cron)

Используйте их с `openclaw message send` или доставкой через cron:

- Личные сообщения: `~sampel-palnet` или `dm/~sampel-palnet`
- Группы: `chat/~host-ship/channel` или `group:~host-ship/channel`

## Встроенное умение

Плагин Tlon включает встроенное умение ([`@tloncorp/tlon-skill`](https://github.com/tloncorp/tlon-skill)), которое предоставляет доступ через CLI к операциям Tlon:

- **Контакты**: получение/обновление профилей, список контактов;
- **Каналы**: список, создание, отправка сообщений, получение истории;
- **Группы**: список, создание, управление участниками;
- **Личные сообщения**: отправка сообщений, реакция на сообщения;
- **Реакции**: добавление/удаление эмодзи-реакций на посты и личные сообщения;
- **Настройки**: управление разрешениями плагина с помощью команд через слэш.

Умение автоматически становится доступным после установки плагина.

## Возможности

| Функция | Статус |
| --------------- | --------------------------------------- |
| Личные сообщения | ✅ Поддерживаются |
| Группы/каналы | ✅ Поддерживаются (по умолчанию требуют упоминания) |
| Темы | ✅ Поддерживаются (автоответы в теме) |
| Форматированный текст | ✅ Markdown преобразуется в формат Tlon |
| Изображения | ✅ Загружаются в хранилище Tlon |
| Реакции | ✅ Через [встроенное умение](#встроенное-умение) |
| Опросы | ❌ Пока не поддерживаются |
| Нативные команды | ✅ Поддерживаются (по умолчанию только для владельца) |

## Устранение неполадок

Сначала выполните следующую последовательность команд:

```bash
openclaw status
openclaw gateway status
openclaw logs --follow
openclaw doctor
```

Распространённые проблемы:

- **Игнорирование личных сообщений**: отправитель не включён в `dmAllowlist`, и не настроен `ownerShip` для потока утверждений.
- **Игнорирование сообщений в группах**: канал не обнаружен или отправитель не авторизован.
- **Ошибки подключения**: проверьте доступность URL корабля; включите `allowPrivateNetwork` для локальных кораблей.
- **Ошибки аутентификации**: убедитесь, что код для входа актуален (коды обновляются).

## Справочная информация по конфигурации

Полная конфигурация: [Конфигурация](/gateway/configuration)

Параметры провайдера:

- `channels.tlon.enabled`: включение/отключение запуска канала.
- `channels.tlon.ship`: имя корабля Urbit бота (например, `~sampel-palnet`).
- `channels.tlon.url`: URL корабля (например, `https://sampel-palnet.tlon.network`).
- `channels.tlon.code`: код для входа на корабль.
- `channels.tlon.allowPrivateNetwork`: разрешение URL localhost/LAN (обход SSRF).
- `channels.tlon.ownerShip`: корабль-владелец для системы утверждений (всегда авторизован).
- `channels.tlon.dmAllowlist`: корабли, которым разрешено отправлять личные сообщения (пусто = никто).
- `channels.tlon.autoAcceptDmInvites`: автоматическое принятие личных сообщений от кораблей из белого списка.
- `channels.tlon.autoAcceptGroupInvites`: автоматическое принятие всех приглашений в группы
