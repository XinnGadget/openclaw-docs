# Ollama Web Search

OpenClaw поддерживает **Ollama Web Search** в качестве встроенного поставщика `web_search`.
Он использует экспериментальный API веб-поиска Ollama и возвращает структурированные результаты
с заголовками, URL-адресами и фрагментами текста.

В отличие от поставщика моделей Ollama, для этой настройки по умолчанию не требуется ключ API. Необходимо следующее:

- хост Ollama, доступный из OpenClaw;
- выполнение команды `ollama signin`.

## Настройка

<Steps>
  <Step title="Запустите Ollama">
    Убедитесь, что Ollama установлен и запущен.
  </Step>
  <Step title="Выполните вход">
    Выполните команду:

    ```bash
    ollama signin
    ```

  </Step>
  <Step title="Выберите Ollama Web Search">
    Выполните команду:

    ```bash
    openclaw configure --section web
    ```

    Затем выберите **Ollama Web Search** в качестве поставщика.

  </Step>
</Steps>

Если вы уже используете Ollama для моделей, Ollama Web Search будет использовать тот же
настроенный хост.

## Конфигурация

```json5
{
  tools: {
    web: {
      search: {
        provider: "ollama",
      },
    },
  },
}
```

Опциональное переопределение хоста Ollama:

```json5
{
  models: {
    providers: {
      ollama: {
        baseUrl: "http://ollama-host:11434",
      },
    },
  },
}
```

Если явный базовый URL для Ollama не задан, OpenClaw использует `http://127.0.0.1:11434`.

Если ваш хост Ollama требует аутентификации по токену (bearer auth), OpenClaw повторно использует
значение `models.providers.ollama.apiKey` (или соответствующие данные аутентификации поставщика, заданные через переменные окружения)
также для запросов веб-поиска.

## Примечания

- Для этого поставщика не требуется отдельное поле с ключом API для веб-поиска.
- Если хост Ollama защищён аутентификацией, OpenClaw повторно использует обычный ключ API поставщика Ollama, если он задан.
- OpenClaw выдаёт предупреждение во время настройки, если Ollama недоступен или не выполнен вход, но это не блокирует выбор поставщика.
- При выполнении автоматически может быть задействован Ollama Web Search, если не настроен поставщик с более высоким приоритетом и учётными данными.
- Поставщик использует экспериментальную конечную точку Ollama `/api/experimental/web_search`.
